5.1 两个回调各自的落库职责

streamWriter
Consumer<PredictStreamChunkVO>
每产生一个业务事件，通过 accept(chunk) 推给 MTOP 层写出。
职责：实时推送数据给前端（不直接落库）
onScoreCompleteCallback
Runnable
</score> 闭合时由 parser 调用 run()。
职责：触发阶段一落库（状态更新 + 通知前端完成）
流进行中
parser 逐 token 解析
streamWriter 每解析出一个 chunk → streamWriter.accept(chunk) → MTOP 写出 → 前端打字机效果
</score> 标签闭合 → 阶段一落库
onScoreCompleteCallback run() → persistScoreAndComplete()
① updateTaskStatus(FINISHED)
② 借助 streamWriter 推送 COMPLETED chunk 通知前端
继续接收 report 内容
streamWriter report 的 chunk 继续通过 streamWriter.accept() 推送（但前端此时已收到 COMPLETED，可选择忽略）
流结束后
doStreamModelCall 返回 → 阶段二落库
请求线程 不经过回调，请求线程串行执行：
assembleFullResult() → persistReportResult()
用 task_id + execution_version 条件更新写入完整 report
两个回调 + 阶段二落库都在同一个调用线程上同步执行，不存在并发竞争
