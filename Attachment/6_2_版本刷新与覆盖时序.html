6.2 executionVersion 版本刷新与覆盖时序

用户
visa_llm_task
旧流 S1
新流 S2
T1
第一次提交
UPSERT
execution_version = V1
启动流，携带 V1
T2
重新提交（刷新版本）
UPDATE SET
execution_version = V2
（V1 被覆盖，旧流不知情）
启动新流，携带 V2
T3
旧流 report 落库
WHERE task_id = ? AND execution_version = V1
当前库中 version = V2
→ 匹配失败，0 行更新，静默跳过
T4
新流 report 落库
WHERE task_id = ? AND execution_version = V2
→ 匹配成功，写入 report ✓
