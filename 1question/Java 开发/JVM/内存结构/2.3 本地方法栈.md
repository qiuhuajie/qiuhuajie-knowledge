# 一、本地方法栈

![[IMG-20260620003734990.png|541]]

1. **本地方法**：`native` 方法
    1. 指的是==使用 Java 以外的其他语言编写的代码==
    2. 因为有些时候 Java 无法直接操作一些底层资源，只能通过 C 或汇编操作，因此需要通过本地方法来实现
2. **本地方法栈**（Native Method Stack）
    1. JVM ==使用本地方法栈来支持== `native` ==方法的执行==
    2. 当 Java 虚拟机使用其他语言（例如 C 语言）来实现指令集解释器时，也会使用到本地方法栈
3. 本地方法栈同样会发生 `StackOverFlowError` 和 `OutOfMemoryError` 异常
4. 🙋‍♂️**本地方法栈包括以下几个部分：**
    1. 参数和返回值部分：用于存储调用本地方法时传递的参数和本地方法返回的返回值
    2. 本地方法栈帧：用于存储本地方法执行过程中的局部变量、操作数栈、异常处理信息等数据
    3. 额外信息：用于存储本地方法调用时所需的其他信息，比如方法名、方法所在的类名、方法的访问标志等数据
5. 如果 Java 虚拟机不支持 `natvie` 方法，并且自己也不依赖传统栈的话，可以无需支持本地方法栈
