1. Program Counter Register 程序计数器（寄存器）
1. 作用：
    
    1. 任意时刻，一条 Java 虚拟机线程只会执行一个方法的代码，而这个被线程执行的方法称为该**线程的当前方法**，其地址被存在 PC 寄存器中
    
    1. 代码在执行过程中，拿到**一条 jvm 指令**会交给**解释器，**由解释器将指令转为**机器码**，再由**CPU**去执行机器码
    
    1. ⭐==而PC寄存器的作用就是：==**==在代码执行过程中，记住下一条 jvm 指令的执行地址，多线程下，线程切换后能恢复到正确的执行位置==**
    
1. 存储的内容：
    
    1. 如果这个方法不是 native 方法，那么 PC 寄存器就保存 Java 虚拟机正在执行的字节码指令地址，即记住下一条 jvm 指令的执行地址
    
    1. 如果是 native 方法，那么 PC 寄存器保存的值是 undefined
    
1. 特点：
    
    1. PC寄存器就是==**线程私有**==**的**，每个线程都有一个PC寄存器
    
    1. ==JVM中唯一一个====**不会存在内存溢出**====的区域==
    
    1. 物理上使用寄存器来实现，因为读取下条jvm指令的操作十分频繁
    
1. 作用演示 ：
    
    ```Plain
          jvm 指令（二进制字节码）                                     Java源代码
    
     0: getstatic      #20                                    // PrintStream out = System.out;
     3: astore_1                                              // --
     4: aload_1                                               // out.println(1);
     5: iconst_1                                              // --
     6: invokevirtual  #26                                    // --
     9: aload_1                                               // out.println(2);
     10: iconst_2                                             // --
     11: invokevirtual #26                                    // --
     14: aload_1                                              // out.println(3);
     15: iconst_3                                             // --
     16: invokevirtual #26                                    // --
     19: aload_1                                              // out.println(4);
     20: iconst_4                                             // --
     21: invokevirtual #26                                    // --
     24: aload_1                                              // out.println(5);
     25: iconst_5                                             // --
     26: invokevirtual #26                                    // --
     29: return
    ```
    
    - 其中虚拟机指令后的 `#26` 为常量池的符号地址，详见 2.5
