# 1. **Direct Memory**
1. 常见于 NIO 操作时，用于数据缓冲区，理解上可以参考零拷贝[[6.3 零拷贝]]
1. 性能：分配回收成本较高，但==读写性能高==
1. 因为是一块系统内存，所以==不受 JVM 内存回收管理==
# 2. 为什么直接内存读写性能高❓
1. 使用 Java 缓冲区时：
    
    ![[IMG-20260404031752197.png|Untitled 286.png]]
    
    - 磁盘文件需要先读取到系统内存中的系统缓冲区，再从内存缓存区读到用户态的 Java 缓冲区
    
    - 涉及到**两次用户态和内核态的切换**，以及**两次数据的拷贝**
    
1. ⭐**==使用直接内存时：==**
    
    ![[IMG-20260404031752263.png|Untitled 1 215.png]]
    
    - 直接内存是一块用户态和内核态程序共享的内存，故==用户态的 Java 程序可以直接访问==
    
    - 因此相比上面，**减少了一次数据在系统缓冲区和 Java 缓冲区之间的拷贝，效率也得到了提升**
    
# 3. **性能演示示例**
- 演示代码：演示 ByteBuffer 作用，读取一个视频文件，比较使用Java的缓冲区和使用直接内存的性能差异
    
    ```Java
     public class directDemo1 {
         static final String FROM = "Boot-sbPSjI4tt10.mp4";
         static final String TO = "E:\\a.mp4";
         static final int _1Mb = 1024 * 1024;
     
         public static void main(String[] args) {
             io(); // ⭐io 用时：1535.586957 1766.963399 1359.240226
             directBuffer(); // ⭐directBuffer 用时：479.295165 702.291454 562.56592
         }
     
         private static void directBuffer() {
             long start = System.nanoTime();
             try (FileChannel from = new FileInputStream(FROM).getChannel();
                  FileChannel to = new FileOutputStream(TO).getChannel();
             ) {
                 ByteBuffer bb = ByteBuffer.allocateDirect(_1Mb);
                 while (true) {
                     int len = from.read(bb);
                     if (len == -1) {
                         break;
                     }
                     bb.flip();
                     to.write(bb);
                     bb.clear();
                 }
             } catch (IOException e) {
                 e.printStackTrace();
             }
             long end = System.nanoTime();
             System.out.println("directBuffer 用时：" + (end - start) / 1000_000.0);
         }
     
         private static void io() {
             long start = System.nanoTime();
             try (FileInputStream from = new FileInputStream(FROM);
                  FileOutputStream to = new FileOutputStream(TO);
             ) {
                 byte[] buf = new byte[_1Mb];    //Java 开辟的用户态中的缓冲区
                 while (true) {
                     int len = from.read(buf);
                     if (len == -1) {
                         break;
                     }
                     to.write(buf, 0, len);
                 }
             } catch (IOException e) {
                 e.printStackTrace();
             }
             long end = System.nanoTime();
             System.out.println("io 用时：" + (end - start) / 1000_000.0);
         }
     }
    ```

# 5. **内存分配和回收**
1. **演示直接内存的分配和回收**
    
    1. 代码
        
        ```Java
         public class directDemo3 {
             static int _1Gb = 1024 * 1024 * 1024;
         
             /*
              * -XX:+DisableExplicitGC 显式的
              */
             public static void main(String[] args) throws IOException {
                 ByteBuffer byteBuffer = ByteBuffer.allocateDirect(_1Gb);
                 System.out.println("分配完毕...");
                 System.in.read();
                 System.out.println("开始释放...");
                 byteBuffer = null;
                 System.gc(); // 显式的垃圾回收，Full GC，对性能影响较大
                 System.in.read();
             }
         }
        ```
        
    
    1. 由于直接内存是系统内存，需要在任务管理器中查看
        
        1. `ByteBuffer byteBuffer = ByteBuffer.allocateDirect(_1Gb);` 执行完成后，可以看到系统分配了 1G 的内存空间
            
            ![[IMG-20260404031752376.png|Untitled 3 137.png]]
            
        
        1. `byteBuffer = null; System.gc();` 执行完成后，可以看到之前分配的 1G 内存没有了
            
            ![[IMG-20260404031752465.png|Untitled 4 113.png]]
            
        
    
1. **直接内存分配回收的原理：**
    
    1. **由于直接内存是系统内存，并不受 Java 回收机制的控制，那么是怎么做到回收的？**
    
    1. 使用了 `Unsafe` 对象完成直接内存的分配回收，并且回收需要主动调用 `freeMemory` 方法
        
        ![[IMG-20260404031752529.png|Untitled 5 92.png]]
        
    
    1. ByteBuffer 的实现类内部，使用了 Cleaner （虚引用）来**监测 ByteBuffer 对象（这个对象是Java对象，自然受Java的回收控制）**，一旦 ByteBuffer 对象被垃圾回收（`byteBuffer = null; System.gc();`），那么就会由 `ReferenceHandler` 线程通过 Cleaner 的 `clean` 方法调用 freeMemory 来释放直接内存[[3.2 Java 中的 4 种引用]]
        
        ![[IMG-20260404031752615.png|Untitled 6 79.png]]
