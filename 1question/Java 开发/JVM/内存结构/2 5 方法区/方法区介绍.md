# 一、简介

![[IMG-20260620004036490.png|571]]

1. 方法区（Method Area）
2. 存储内容：
    - 存储了==每一个类的结构信息==：如**运行时常量池、字段、方法以及构造函数的字节码内容**
    - 还包括一些在类、实例、接口初始化时用到的特殊方法

    ![[IMG-20260620004036640.png|279]]

3. 与 Java 堆一样，方法区也是==线程共享==的
# 二、方法区的实现方式
1. 在 **JDK 1.7 之前**的版本的 HotSpot 虚拟机中，**使用** ==**永久代**== **作为方法区的实现**
    - 此时方法区是==堆内存上的一个逻辑部分==
    - ==永久代的垃圾回收和老年代的垃圾回收是绑定的==，因此一旦其中一个区域被占满，这两个区都要进行垃圾回收，增大了 OOM 发生的概率，导致 `Full GC` [[分代回收算法]]

        ![[IMG-20260620004036748.png|680]]

    - StringTable 是永久代中常量池的一部分，例如 `String` 的 `intern()` 方法会因永久代的原因而导致不同虚拟机下有不同的表现，不利于代码迁移
2. 而在 **JDK 1.8 中**，永久代的实现被废弃，**使用** ==**`MetaSpace`**== **元空间 作为方法区的实现**
    - ==不再占用堆内存了，而是移到了本地内存==（系统内存），意味着只要本地内存足够，它就不会 OOM
    - 但是，其中 StringTable 仍在堆中（因为永久代的内存回收效率很低，而堆里的垃圾回收效率较高）[[StringTable（实现在堆中）]]

        ![[IMG-20260620004036893.png|807]]

# 三、方法区内存诊断
1. 演示 **元空间**（当前环境 jdk 1.8）内存溢出的代码：

    ```Java
     /**
      * 演示元空间内存溢出 java.lang.OutOfMemoryError: Metaspace
      * -XX:MaxMetaspaceSize=8m
      */
     public class metaSpaceDemo extends ClassLoader { // 可以用来加载类的二进制字节码
         public static void main(String[] args) {
             int j = 0;
             try {
                 metaSpaceDemo test = new metaSpaceDemo();
                 for (int i = 0; i < 100000; i++, j++) {
                     //1. 生成二进制字节码文件
                     // ClassWriter 作用是生成类的二进制字节码
                     ClassWriter cw = new ClassWriter(0);
                     // 版本号，访问修饰符 public，类名，包名，父类，接口
                     cw.visit(Opcodes.V1_8, Opcodes.ACC_PUBLIC, "Class" + i, null, "java/lang/Object", null);
                     // 返回 byte[]
                     byte[] code = cw.toByteArray();
                     //2. 将生成的二进制字节码加载为类
                     // 由于继承了ClassLoader，可以执行父类中的 类加载方法
                     test.defineClass("Class" + i, code, 0, code.length); // Class 对象
                 }
             } finally {
                 System.out.println(j);
             }
         }
     }
    ```
    1. 1.8 以前会导致**永久代**内存溢出
        - 错误： `java.lang.OutOfMemoryError: PermGen space`
        - 方法区大小参数设置：`XX:MaxPermSize=8m`
    2. 1.8 之后会导致**元空间**内存溢出
        - 错误： `java.lang.OutOfMemoryError: Metaspace`
        - 方法区大小参数设置：`XX:MaxMetaspaceSize=8m`
2. 在实际情况下，真的有上述的场景吗？
    1. 在Spring 和 MyBatis 等框架中，存在大量 **在运行时动态产生 Class，并加载这些类**的场景
    2. 如Spring AOP 和 MyBatis Mapper 接口实现类对象的获取等都是上述场景
