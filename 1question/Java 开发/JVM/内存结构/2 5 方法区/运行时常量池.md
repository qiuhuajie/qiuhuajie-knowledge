# 一、常量池表
1. 方法区就是一张表，用于==存放编译期生成的各种==**==字面量==**
2. 常量池表可以==帮助虚拟机指令找到要执行的各种信息==
    - 代码演示
        1. Java代码：

            ```Java
             public class constantPoolDemo {
                 public static void main(String[] args) {
                     System.out.println("hello world");
                 }
             }
            ```
        2. 找到编译后的 `.class` 文件，并将其反编译，显示文件中内容

            ![[IMG-20260620004159805.png|748]]

        3. 可以看到反编译后的信息中包含
            - **类的版本、字段、方法、接口等描述信息**

                ![[IMG-20260620004159925.png|738]]

                ![[IMG-20260620004159953.png|736]]

            - **常量池，用于存放编译期间产生的各种字面量和符号引用**

                ![[IMG-20260620004200069.png|735]]

        4. **那么常量池是==如何实现帮助虚拟机指令找到各种信息的？==**
            1. 在方法信息中可以看到由下面这行代码变成的 4 条虚拟机指令：

                ```Java
                  System.out.println("hello world");
                ```

                ![[IMG-20260620004200187.png|253]]

            2. 其中 `getstatic` 表示获得一个静态对象
                1. 那么获取什么类型的对象，需要使用后面的符号地址（逻辑上的）： `#2`
                2. 去常量表中找到

                    ```Plain
                     #2 = Fieldref           #21.#22        // java/lang/System.out:Ljava/io/PrintStream;
                    ```
                3. 再根据 `#21.#22` 找到

                    ```Plain
                     #21 = Class              #28            // java/lang/System
                     #22 = NameAndType        #29:#30        // out:Ljava/io/PrintStream;
                    ```

                    再根据 `#29:#30` 找到

                    ```Plain
                     #29 = Utf8               out
                     #30 = Utf8               Ljava/io/PrintStream;
                    ```
                4. **这样就将要执行的 ：获取 一个PrintStream 类型的，名为 out 的静态对象的 代码给描述了出来**

# 二、运行时常量池
1. 常量池是 `.class` 文件中的，==当该==**==类被加载后==**==，它的====**常量池**====信息就会放入====**运行时常量池**====中存放==
2. **并把常量池中的逻辑地址（`#2`、`#3`）变为真实的内存地址**
