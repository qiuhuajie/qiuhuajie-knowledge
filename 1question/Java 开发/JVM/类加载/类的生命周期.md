# 一、类的生命周期

> **加** ➡️ **链（验准解）** ➡️ **初** ➡️ **使** ➡️ **卸**
> **类加载过程就是一个类被加载到 JVM 中的过程**
> 1. ⭐这个过程也即类的==字节码生成框架==：JVM 读取 `.class` 字节码文件，取出二进制数据，加载到内存中，解析字节码文件内的信息，生成对应的 `Class` 类对象
> ![[aa810d35881df2312c012b6d1e6a563c_MD5.png|630]]
> 2. 注：[[代理模式 ⭐]]就是使用了字节码生成框架实现动态代理的
> 3. 下面详解这一过程
- 一个类从被加载到虚拟机内存中开始，到卸载出内存，它的整个生命周期包括 7 个阶段：
![[IMG-20260620012154636.png|538]]
- 注意：加载阶段和连接阶段可能是交叉进行的：加载阶段尚未结束，连接阶段可能就已经开始了
# 二、详解
- 在Java虚拟机中类的全过程，包括==**加载、验证、准备、解析和初始化**==这5个阶段所执行的具体动作，这些==**都是由类加载器来实现的**==
## 1. 加载 `.class` 文件
1. **🙋‍♂️类加载有三种方式：**
    1. 命令行启动应用时候由JVM初始化加载
    2. 通过 **`Class.forName()`** 方法动态加载
        - `Class.forName()` 将类的.class文件加载到jvm中之外，还会对类进行解释，执行类中的static块
    3. 通过 **`ClassLoader.loadClass()`** 方法动态加载
        - `ClassLoader.loadClass()` 只干一件事情，就是将.class文件加载到jvm中，不会执行static中的内容，只有在newInstance才会去执行static块
2. **🙋‍♂️类加载过程的第一步，主要完成下面 3 件事情：**
    1. 1️⃣ ==**类加载器**==**通过**==**类的全限定名**==**来获取此类的二进制字节流**
        - 这个二进制字节流来源有：class文件、zip、jar、war包中读取、网络中读取，比如Applet、运行时计算生成，比如动态代理技术、由其他文件生成，比如 JSP
    2. 2️⃣ **将字节流所代表的静态存储结构转换为** ==**类的信息**== **（==**`instanceKlass`**==），并加载在**==方法区==**中**
        - 内部采用 C++ 的一个数据结构： `instanceKlass` 来描述一个 java 类的各种信息
        - 它的重要 field 有：
            - 展开
                - `_java_mirror` 即 java 的类镜像，例如对 String 来说，就是 String.class，作用是把 klass 暴露给 java 使用
                - `super` 即父类
                - `_fields` 即成员变量
                - `methods` 即方法
                - `_constants` 即常量池
                - `class_loader` 即类加载器
                - `_vtable` 虚方法表
                - `itable` 接口方法表
                - `Static Field Table` 静态变量表
    3. 3️⃣ **同时在** ==**堆**== **中会生成一个** ==**Class 对象**== **（==**`_java_mirror`**==），作为方法区中 `instanceKlass` 的访问入口**
        1. Class 对象中会存储 `instanceKlass` 的地址，而 `instanceKlass` 中也会存有类对象的地址
        2. 也即， Class 对象包含了完整的类的结构信息，==可以通过这个 Class 对象看到类的结构==[[Export-e8249ac1-4ae1-42cd-83d5-041a266841fe/1question/Java 开发/Java/反射|反射]]
        3. 每当 new 一个实例对象出来时，会被放在堆中，==在其对象头中会存放这个 Class 对象的地址，但不论产生多少实例，Class 对象是唯一的==
            ![[IMG-20260620012154774.png|713]]

3. 类加载代码示例：
    1. 源代码

        ![[IMG-20260620012154828.png|727]]

    2. 首先 JVM 会将你的代码编译成一个 `.class` 字节码文件，然后被类加载器（Class Loader）加载进 JVM 的内存中（注意：如果这个类还有父类没有加载，先加载父类）
    3. **同时会创建一个** **`Date`** 类的 Class 对象（==**`_java_mirror`**==）存到堆中（注意这个不是 new 出来的对象，而是类的类型对象）
    4. JVM 在 `new` 创建 `Date` 对象前，会先检查其类对应的 Class 对象是否加载好，若加载好，则为其分配内存，然后再进行初始化

        ![[IMG-20260620012154945.png|842]]

    5. **每个类只有一个 Class 对象**，也就是说如果我们有第二条 `new Date()` 语句，JVM 不会再生成一个 `Date` 的 Class 对象，因为已经存在一个了

        ![[IMG-20260620012155028.png|690]]

## 2. 验证
1. **验证是为了**==**确保 Class 文件的字节流中包含的信息符合当前虚拟机的要求**==**，并且**==**不会危害虚拟机自身的安全**==
    1. 当加载的class文件不符合虚拟机的要求，java 虚拟机是无法执行这个字节码的
    2. 所以要先看看有没有符合，符合了才给虚拟机执行后续操作
2. 例如：用 UE 等支持二进制的编辑器修改 HelloWorld.class 的魔数，在控制台运行，会报下面的错误
![[IMG-20260620012155082.png|743]]
## 3. 准备
1. **准备阶段的主要目的是确保类的静态变量在使用前已经被分配内存，并设置了默认值。在此阶段，不会执行任何 Java 代码**
2. **在准备阶段会为 `static` 变量**==**分配空间**==**，并设置**==**默认值**==
    1. 为类的静态变量分配内存空间，静态变量包括类变量和静态成员变量
    2. 设置类的静态变量的默认值，对于基本数据类型，如 int、float 等，其默认值是 `0`；对于引用类型，其默认值是 `null`
    3. 如果类中存在常量（使用 `final` 关键字修饰的静态变量），则在准备阶段中已经将常量的值设置为声明时的值，而不是默认值

## 4. 解析
1. 解析是虚拟机==将常量池内的==**==符号引用==**==替换为==**==直接引用==**的过程
2. 类加载器**编译**的时候，不知道某个类 `com.xxx.xxx` 的实际内存地址，此时用符号引用
3. 但是此时**经过加载阶段**，**`com.xxx.xxx` 类信息已经加载到内存了**，此时就==可以====**改为直接引用**====，指向实际的内存位置==
## 5. 初始化
1. 🙋‍♂️**调用** `<clinit>()` **进行类的初始化，设置显示**==**初始值**==
    1. `<clinit>()`方法由 **静态变量显示赋值代码** 和 **静态代码块**[**（静态代码块）**](https://www.notion.so/8f9c2e8c195c45f1ab16febf29cb5226?pvs=21)组成，变量显示赋值代码和静态代码块代码从上到下顺序执行
    2. `<clinit>()`方法**只执行一次**
    3. 举例：

        ```Java
        public class Test {
            static int a; // 在准备阶段将 a 分配为 0
            static String s = "Hello"; // 在准备阶段将 s 分配为 null；在后面的初始化阶段会执行显示赋值语句，因此被覆盖为初始值 "Hello"
            static {
                a = 10; // 静态代码块中的初始化，将 a 赋为 10
            }
        }
        ```
2. `static` 变量在 JDK 7 之前存储于 `instanceKlass` 末尾，从 JDK 7 开始，存储于 `_java_mirror` 末尾
3. **类初始化是【懒惰的】**
    1. **会触发初始化的情况**
        1. **main 方法所在的类**，总会被**首先初始化**
        2. **首次访问**这个类的**静态变量**或**静态方法**时[[单例模式 ⭐]]
            - 相关案例：[[问题记录：编译通过但运行时报 NoClassDefFoundError]]
        3. 子类初始化，如果父类还没初始化，会引发父类的初始化，但子类访问父类的静态变量，只会触发父类的初始化
        4. **`new`** 关键字也会导致初始化
    2. **不会触发类初始化的情况**
        1. 访问**类的 static final 静态常量，且为基本类型或字符串**，不会触发初始化，因为在准备阶段就已经完成赋值了
        2. 类对象 .class 不会触发初始化
        3. 创建该类的数组不会触发初始化
        4. 类加载器的 loadClass 方法
        5. Class.forName 的参数 2 为 false 时
> **至此，一个类的加载过程结束**
