# 一、启动类加载器
1. 用 `-Xbootclasspath/a:.` 来模拟 Bootstrap 类加载器加载类
2. 加载的类：

    ```Java
    public class F {
    	static {
    		System.out.println("bootstrap F init");
    	}
    }
    ```
3. 执行的代码：

    ```Java
    public class Load5_1 {
    	public static void main(String[] args) throws ClassNotFoundException {
    		Class<?> aClass = Class.forName("cn.itcast.jvm.t3.load.F");
    		System.out.println(aClass.getClassLoader());
    	}
    }
    ```
4. 执行命令

    ```Bash
    E:\git\jvm\out\production\jvm>java -Xbootclasspath/a:. cn.itcast.jvm.t3.load.Load5
    ```
5. 输出

    ```Bash
    bootstrap F init
    null //如果类加载器打印为 null 则表示是启动类加载器
    ```
    > 1.
    > `Xbootclasspath` 表示设置 bootclasspath
    > 1. 其中 `/a:.` 表示将当前目录追加至 bootclasspath 之后
    > 2. 三种替换方式：
    >     3. `java -Xbootclasspath:<new bootclasspath>`
    >     4. `java -Xbootclasspath/a:`<后追加路径>
    >     5. `java -Xbootclasspath/p:`<前追加路径>
    > 6. ==用作底层的 JVM 核心类的替换，开发 JVM 的人才会去用==
