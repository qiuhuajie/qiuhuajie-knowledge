# 一、类加载器的层次
1. JVM 中内置了三个重要的 ClassLoader
    1. **BootstrapClassLoader（启动类加载器）**：
        1. 最顶层的加载类，由 C++实现，无法直接访问
        2. 负责加载 `%JAVA_HOME%/lib`目录下的 jar 包和类或者被 `Xbootclasspath`参数指定的路径中的所有类
    2. **ExtensionClassLoader（扩展类加载器）**：
        1. 主要负责加载 `%JRE_HOME%/lib/ext` 目录下的 jar 包和类
        2. 或被 `java.ext.dirs` 系统变量所指定的路径下的 jar 包
    3. **AppClassLoader（应用程序类加载器）**
        1. 负责加载我们写的代码
        2. 即当前应用 classpath 下的所有 jar 包和类

    ![[IMG-20260620011831627.png|576]]

2. **每个类加载器都有一个父类加载器**
    1. 通过下面的程序来验证

        ```Java
        public class ClassLoaderDemo {
            public static void main(String[] args) {
                System.out.println("ClassLodarDemo's ClassLoader is " + ClassLoaderDemo.class.getClassLoader());
                System.out.println("The Parent of ClassLodarDemo's ClassLoader is " + ClassLoaderDemo.class.getClassLoader().getParent());
                System.out.println("The GrandParent of ClassLodarDemo's ClassLoader is " + ClassLoaderDemo.class.getClassLoader().getParent().getParent());
            }
        }
        ```
    2. Output

        ```Java
        ClassLodarDemo's ClassLoader is sun.misc.Launcher$AppClassLoader@18b4aac2
        The Parent of ClassLodarDemo's ClassLoader is sun.misc.Launcher$ExtClassLoader@1b6d3586
        The GrandParent of ClassLodarDemo's ClassLoader is null
        ```
    3. `AppClassLoader`的父类加载器为`ExtClassLoader`
    4. `ExtClassLoader`的父类加载器为 null
        1. **null 并不代表** **`ExtClassLoader`** **没有父类加载器**
        2. **而是** **`BootstrapClassLoader`**（引导类加载器）是用 C 语言实现的，找不到一个确定的返回父 Loader 的方式，于是就返回 `null`
