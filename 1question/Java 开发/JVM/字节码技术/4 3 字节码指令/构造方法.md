# **类构造方法 <cinit>()V**
1. Java代码
    
    ```Java
     public class Demo {
         static int i = 10;          //静态变量 i
         static {
             i = 20;
         }
         static {
             i = 30;
         }
     }
    ```
    
1. 字节码
    
    - 编译器会按从上至下的顺序，收集所有 `static {}` **静态代码块**和**静态成员变量**赋值的代码，合并为一个特殊的方法 `<cinit>()V`
    
    - `<cinit>()V` 方法会在类加载的初始化阶段被调用
    
    ```Plain
     static {};
         descriptor: ()V
         flags: ACC_STATIC
         Code:
           stack=1, locals=0, args_size=0

              5: bipush        20
              7: putstatic     #2                  // Field i:I
    ```

# **实例对象构造方法 <init>()V**
1. Java代码
    
    ```Plain
     public class Demo {
         private String a = "s1";
         {
             b = 20;
         }
         private int b = 10;
         {
             a = "s2";
         }
         public constantPoolDemo(String a, int b) {
             this.a = a;
             this.b = b;
         }
         public static void main(String[] args) {
             constantPoolDemo d = new constantPoolDemo("s3", 30);
             System.out.println(d.a);
             System.out.println(d.b);
         }
     }
    ```
    
1. 字节码
    
    - 编译器会按从上至下的顺序，收集所有 `{}` **代码块**和**普通成员变量**赋值的代码，形成新的构造方法
    
    - 原始构造方法（`constantPoolDemo()`）内的代码**总是会被放在最后**
