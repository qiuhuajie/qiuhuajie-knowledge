# **try-catch**
1. Java源代码
    
    ```Java
     public class Demo {
     
         public static void main(String[] args) {
             int i = 0;
             try {
                 i = 10;
             } catch (Exception e) {
                 i = 20;
             }
         }
     }
    ```
    
1. 字节码指令
    
    - 可以看到多出来一个`Exception table`的结构，`[from, to)` 是前闭后开的检测范围
    
    - **一旦这个范围内的字节码执行出现异常，则通过** `**type**` **匹配异常类型**
        
        - 如果一致，进入 `target` 所指示行号的指令
        
        - `target ：8` 行的字节码指令 `astore_2` 是将异常对象引用存入局部变量表的 `slot 2` 位置

# **多个 single-catch 块**
1. Java源代码
    
    ```Java
     public class Demo {
     
         public static void main(String[] args) {
             int i = 0;
             try {
                 i = 10;
             } catch (ArithmeticException e) {
                 i = 30;
             } catch (NullPointerException e) {
                 i = 40;
             } catch (Exception e) {
                 i = 50;
             }
         }
     }
    ```
    
1. 字节码指令
    
    - 因为异常出现时，**只能进入** `**Exception table**` **中一个分支**，所以局部变量表 `slot 2` 位置被共用

# **finally**
1. Java源代码
    
    ```Java
     public class Demo {
     
         public static void main(String[] args) {
             int i = 0;
             try {
                 i = 10;
             } catch (Exception e) {
                 i = 20;
             } finally {
                 i = 30;
             }
         }
     }
    ```
    
1. 字节码指令
    
    - 可以看到 `finally` 中的代码被复制了 3 份，分别放入 `try` 流程，`catch` 流程以及 `catch` 剩余的异常类型流程
    
    - 因为**要保证** `**finally**` **中的代码不论是在出现异常还是不出现异常的情况下都执行**
        
        - 而 `Exception` 并不能将所有异常捕获，如`Exception`平级的异常 `Error` 等
        
        - 因此就需要第三个分支： `catch` 剩余的异常类型流程
        
        - 这样就可以保证 `finally` 中的代码总会执行
        
        ![[IMG-20260405035422040.png]]
        
    
1. 面试题一：
    
    1. 下面代码会输出什么？
        
        ```Java

                 System.out.println(result);
             }

        ```
        
    
    1. 字节码指令
        
        - 由于 finally 中的 return 被插入了所有可能的流程，因此返回结果肯定以 finally 的为准

            - 发现没有 `athrow` 了，因此如果在 finally 中出现了 return，会吞掉异常😱😱😱
            
            - ==因此 finally 中不能写 return 语句，否则会将 try-cath 块中的异常吞掉==
