# **Java源代码**
```Java
 public class Demo {
 
     public static void test(Animal animal) {
         animal.eat();
         System.out.println(animal.toString());
     }
 
     public static void main(String[] args) throws IOException {
         test(new Cat());
         test(new Dog());
         System.in.read();
     }
 }
 
 abstract class Animal {
 
     public abstract void eat();
 
     @Override
     public String toString() {
         return "我是" + this.getClass().getSimpleName();
     }
 }
 
 class Dog extends Animal {
     @Override
     public void eat() {
         System.out.println("啃骨头");
     }
 }
```

> test(new Dog()); 是如何在调用时知道应该调用 Dog 的 eat() 方法的 ?
# **多态原理**
1. 虚表（**vtable**）
    
    1. ⭐它在**类加载的链接阶段**就已经根据方法的重写规则生成好了
    
    1. ⭐在方法执行时，会去 vtable 里找方法的地址，因此**具体执行哪个方法是在 vtable 中确定好的**
    
    1. 示例：
        
        1. dog 的 **vtable** 中有 6 个虚方法的入口地址
            
            ```Plain
             0x000000001b7d41e0: 0x000000001b3d1b10
             0x000000001b7d41e8: 0x000000001b3d15e8
             0x000000001b7d41f0: 0x000000001b7d35e8
             0x000000001b7d41f8: 0x000000001b3d1540
             0x000000001b7d4200: 0x000000001b3d1678
             0x000000001b7d4208: 0x000000001b7d3fa8
            ```
            
        
        1. 通过 Tools -> Class Browser 查看每个类的方法定义，比较可知
            
            ```Plain
             Dog - public void eat() @0x000000001b7d3fa8
             Animal - public java.lang.String toString() @0x000000001b7d35e8;
             Object - protected void finalize() @0x000000001b3d1b10;
             Object - public boolean equals(java.lang.Object) @0x000000001b3d15e8;
             Object - public native int hashCode() @0x000000001b3d1540;
             Object - protected native java.lang.Object clone() @0x000000001b3d1678;
            ```
            
            **对号入座，发现**
            
            - eat() 方法是 Dog 类自己的
            
            - toString() 方法是继承 String 类的
            
            - finalize() ，equals()，hashCode()，clone() 都是继承 Object 类的
            
        
    
1. 当执行 invokevirtual 指令时
    
    1. 先通过栈帧中的对象引用找到对象
    
    1. 分析对象头，找到对象的实际 Class
    
    1. Class 结构中有 **vtable**
    
    1. 查表得到方法的具体地址
    
    1. 执行方法的字节码
