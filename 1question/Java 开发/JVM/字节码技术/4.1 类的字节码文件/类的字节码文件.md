# 一、类的字节码文件
1. 使用以下代码编译后的字节码文件来说明 类文件.class 的组成部分：

    ```Java
     public class constantPoolDemo {
         public static void main(String[] args) {
             System.out.println("hello world");
         }
     }
    ```
2. 编译：

    ```Java
     javac -parameters -d . constantPoolDemo.java
    ```
3. 查看字节码编译后的结果：

    ![[IMG-20260620012711373.png|457]]

4. 这种字节码文件完全看不懂，需要按一定规则来解读：
5. 参考文档：
    > Chapter 4. The class File Format
    > [https://docs.oracle.com/javase/specs/jvms/se8/html/jvms-4.html](https://docs.oracle.com/javase/specs/jvms/se8/html/jvms-4.html)
    1. 说明1：这些字节码由下面要介绍的结构紧密组成

        ![[IMG-20260620012711468.png|456]]

    2. 说明2：通常前几位介绍了是什么信息，会跟着一个表示长度的字节码，再往后截取这个长度的字节码就是这个信息的内容，紧接着的就是下一个信息的描述
    3. 说明3：这里 `12` 就代表一个字节，即 `8 bit` 位， `12` 是一个十六进制表示，换成二进制为 `0001 0010` ，换成十进制为 `17`
