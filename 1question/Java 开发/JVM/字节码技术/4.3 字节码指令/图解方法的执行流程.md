# 四、常量池载入运行时常量池
1. ==在加载时，所有常量池的内容都要加载到运行时常量池==
2. 但这里可以看到 main 方法中的字节码指令中只使用了 `#3,#4,#5` 三个常量池中的内容，故图中只画了这三个
3. 可以看到 `int a = 10` 并没有存在常量池中，而是和方法的字节码指令存在一起
4. 而一旦变量的值，超过了整数的最大值，那么这个变量就会存在常量池中，故 `int b = Short.MAX_VALUE + 1` 是存在常量池中

# 六、Main 线程开始运行，分配栈帧内存
1. 每个栈帧中都会有一个**局部变量表**（绿色）和一个**操作数栈**（蓝色）[[2.2 虚拟机栈]]
2. 这两个的大小由字节码文件中的 `stack=2, locals=4` 来规定

# 七、执行引擎开始执行字节码
## 1. bipush 10

![[IMG-20260620013033463.png|898]]

1. 将一个 `byte` （一个字节）压入操作数栈（其长度会补齐 `4` 个字节），类似的指令还有
    - `sipush` 将一个 `short` 压入操作数栈（其长度会补齐 4 个字节）
    - `ldc` 将一个 `int` 压入操作数栈
    - `ldc2_w` 将一个 `long` 压入操作数栈（分两次压入，因为 `long` 是 8 个字节）
2. 这里小的数字都是和字节码指令存在一起，超过 `short` 范围的数字存入了常量池
## 2. istore_1

将操作数栈顶数据弹出，存入局部变量表的 `slot 1`（一号槽位）

![[IMG-20260620013033552.png|887]]

![[IMG-20260620013033637.png|885]]

## 3. ldc #3

![[IMG-20260620013033723.png|887]]

2. 注意： `Short.MAX_VALUE` 是 `32767`，所以 `32768 = Short.MAX_VALUE + 1` 实际是在编译期间计算好的

## 11. invokevirtual #5

![[IMG-20260620013035134.png|800]]

1. 到常量池找那个找到一个方法的引用

3. **生成新的栈帧**（分配 `locals、stack` 等）
4. 传递参数，执行新栈帧中的字节码
