# 一、Java源代码
```Java
 public class Demo {
     //构造方法
     public Demo() { }
     //私有方法
     private void test1() { }
     //私有final方法
     private final void test2() { }
     //公共方法
     public void test3() { }
     //公有静态方法
     public static void test4() { }
     public static void main(String[] args) {
         Demo d = new Demo();
         d.test1();
         d.test2();
         d.test3();
         d.test4();
         Demo.test4();
     }
 }
```
# 二、字节码指令
- new 是创建对象，给对象分配**堆内存**，执行成功后，会将**对象引用**压入操作数栈
- dup 是赋值操作数栈栈顶的内容，本例即为【对象引用】，为什么需要两份引用呢
    - 一个是要配合 invokespecial 调用该对象的构造方法 `<init>":()V`，该方法调用完会消耗掉栈顶一个引用
    - 另一个要配合 astore_1 赋值给局部变量
- **⭐ invokespecial 指令与 invokevirtual 指令**
    - 最终方法（final），私有方法（private），构造方法都是由 invokespecial 指令来调用，属于**静态绑定**
    - 普通成员方法是由 invokevirtual 调用，属于**动态绑定**，即支持多态
- **⭐ 静态方法的调用是不需要对象引用来参与的**
    - 例如：`d.test4();` 的字节码：是通过【对象引用】调用一个静态方法

        ```Plain
         20: aload_1
         21: pop
         22: invokestatic  #7                  // Method test4:()V
        ```
    - `Demo.test4();`的字节码：是直接通过类调用一个静态方法

        ```Plain
         25: invokestatic  #7                  // Method test4:()V
        ```
    - 可以看到在调用 `invokestatic` 之前执行了 `pop` 指令，把【对象引用】从操作数栈弹掉了
    - 对象引用实际没有参与静态方法的调用，所以在调用静态方法时，直接使用下面的代码写法就好
