# 1. 标记清除算法
1. 标记清除算法分为两个阶段：标记阶段和清除阶段
    
    ![[IMG-20260404031756397.png|Untitled 289.png]]
    
    - **标记阶段：**==标记所有由 GC Root 触发的可达对象==。此时，所有未被标记的对象就是垃圾对象
    
    - **清除阶段：**==清除所有未被标记的对象==，这里只需要记录要清除的对象内存空间的起始地址即可，新对象来了直接覆盖
    
1. 存在的问题：**空间碎片**问题
    
    1. 如果空间碎片过多，则会导致内存空间的不连续
    
    1. 虽说大对象也可以分配在不连续的空间中，但是效率要低于连续的内存空间
    
# 2. 标记压缩算法
1. 标记压缩算法可以说是标记清除算法的优化版，不会产生内存碎片
1. 其同样需要经历两个阶段，分别是：标记结算、压缩阶段
    
    ![[Attachment/1question/大数据/Java 开发/JVM/垃圾回收/IMG-20260405035414822.png|Untitled 1 218.png]]
    
    - **标记阶段：**==标记所有由 GC Root 触发的可达对象==
    
    - **压缩阶段：**其则是==将所有存活的对象压缩在内存的一边，之后清理边界外的所有空间==
    
1. 存在的问题：**整理压缩内存存活对象需要耗费很大的代价**
# 3. 复制算法
1. 核心思想：是将原有的内存空间分为两块 `from` 和 `to`，每次只使用一块
    
    ![[IMG-20260405035420129.png|Untitled 2 183.png]]
    
    - 在垃圾回收时，==将 from 中的存活对象复制到 to 中==
    
    - 之后==清除 from 中的所有对象==
    
    - 之后交换两个内存块的角色， ==from 变为 to， to 变为 from==，最后完成垃圾回收
    
1. 存在的问题：**要将内存空间折半，极大地浪费了内存空间**
# 4. 三种算法对比
1. 对比一下上述**三种算法各自的优点和缺点：**
    
    - 标记清除算法：虽然会产生内存碎片，但是不需要移动太多对象，比较适合在存活对象比较多的情况
    
    - 标记整理算法：则是标记清除算法的优化版，减少了空间碎片
    
    - 复制算法：虽然需要将内存空间折半，并且需要移动存活对象，但是其清理后不会有空间碎片，比较适合存活对象比较少的情况
    
1. **如果我们**==**单独采用任何一种算法，那么最终的垃圾回收效率都不会很好**==**，其实 JVM 虚拟机的建造者们也是这么想的，由此**==**产生了 分代算法**==**⬇**
