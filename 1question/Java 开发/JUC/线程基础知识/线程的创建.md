# 1. 创建**方式一：继承 Thread 类**
1. 创建流程
    
    1. 创建一个继承于 `Thread` 类的子类
    
    1. 重写 `Thread` 类的 `run()`
    
    1. 创建 `Thread` 类的子类对象
    
    1. 通过此对象调用 `start()`
    
1. ⭐`**start()**`
    
    1. **作用：启动当前线程（新建 ➡️ 就绪），自动调用当前线程的 `run()`**
    
    1. 这里 `**myThread.start()**` **可以**换成直接调用 `**myThread.run()**`吗❓
        
        1. new 一个 Thread ，线程进入了新建状态
        
        1. 接着，调用 `start()` 方法
            
            1. ==会启动一个新建状态的线程，使线程进入了就绪状态，当分配到时间片后就可以开始运行了==
            
            1. `start()` 会自动执行 `run()` 方法的内容，这是真正的多线程工作
            
        
        1. 但是，**如果直接执行 `run()` 方法，会把 `run()` 方法当成一个 main 线程下的普通方法去执行，并不会在某个线程中执行它，所以这并不是多线程工作**
        
        > [!important] **`start()` 与 `run()` 的区别**
        > 
        > 1. ==调用 start 方法是真正开启一个线程==来执行任务，而调用 run 方法相当于执行普通方法 run，并不会开启新线程
        > 
        > 1. run 方法也叫做线程体，它里面包含了具体要执行的业务代码，当调用 run 方法时，会立即执行 run 方法中的代码（如果当前线程时间片未用完）；而==调用 start 方法时，是启动一个线程并将线程的状态设置为就绪状态，并不会立即执行==
        > 
        > 1. run 方法可以被调用多次，而 ==start 方法只能被调用一次（如果线程状态不是新建状态，会直接抛出异常）==
        > 
        > ![[IMG-20260404031739701.png|Untitled 295.png]]
        >  
        
    
1. 代码示例
    
    ```Java
     class MyThread extends Thread{ //1.创建一个继承于Thread类的子类
     
         //2.重写 Thread 类的 run( )
         @Override
         public void run() {
             for (int i = 0; i < 100; i++) {
                 if(i % 2 == 0){
                     System.out.println(i+"run()");
                 }
             }
         }
     }
     public class ThreadTest {
         public static void main(String[] args) {
    
             //3.创建Thread类的子类的对象
             MyThread myThread = new MyThread();
    
             //4.通过此对象调用 start()，start() 会去调 MyThread 的 run()
             myThread.start();
    
             //5.如果想再启动一个线程，需要再 new 一个MyThread对象
             MyThread myThread2 = new MyThread();   
             myThread2.start();
         }
     }
    ```
    
- **Thread 类中的常用方法**
    
    1. `**start()**`：启动当前线程、调用当前线程的run( )
    
    1. `**run()**`：通常需要重写Thread类中的此方法，将创建的线程要执行的操作声明在此方法中
    
    1. `**currentThread()**`：静态方法，返回当前代码执行的线程
    
    1. `**getName()**`：获取当前线程的名字
    
    1. `**setName()**`：设置当前线程的名字
        
        ```Java
         public class ThreadTest {
             public static void main(String[] args) {
                 MyThread myThread1 = new MyThread();
                 myThread1.start();
                 //给线程1命名
                 myThread1.setName("线程一");
        
                 //给主线程main命名
                 Thread.currentThread().setName("线程二");
             }
         }
        ```
        
    
    1. `**yield()**`：线程让步，暂停当前执行的线程，将执行机会让给其他线程
        
        ```Java
         class MyThread extends Thread{
             @Override
             public void run() {
                 for (int i = 0; i < 100; i++) {
                     if(i % 2 == 0){
                         System.out.println(i+Thread.currentThread().getName());
                     }
                     if(i % 20 == 0){
                       this.yield();         //让步
                     }
                 }
             }
         }
         public class ThreadTest {
             public static void main(String[] args) {
                 MyThread myThread1 = new MyThread();
                 myThread1.start();
                 myThread1.setName("线程一");
         
                 MyThread myThread2 = new MyThread();
                 myThread2.start();
                 myThread2.setName("线程二");
             }
         }
         
         //40线程一
         //30线程二
         
         //40线程二
         //42线程一
        ```
        
    
    1. `**join()**` ：在线程a中调用b的join()，此时将当前线程a阻塞，直到join()方法调入的线程b执行完成后，再执行被阻塞的线程a
        
        ```Java
         class MyThread extends Thread{
             @Override
             public void run() {
                 for (int i = 0; i < 100; i++) {
                     if(i % 2 == 0){              System.out.println(i+Thread.currentThread().getName());
                     }
                 }
             }
         }
         public class ThreadTest {
             public static void main(String[] args) {
                 MyThread myThread1 = new MyThread();
                 myThread1.start();
                 myThread1.setName("线程一");
         
                 for (int i = 0; i < 100; i++) {
                     if(i % 2 == 0){
                         System.out.println(i+Thread.currentThread().getName());
                     }
                     if(i == 20){
                         try {
                             Thread.currentThread().join();//插入
                         } catch (InterruptedException e) {
                             e.printStackTrace();
                         }
                     }
                 }
             }
         }
         
         //16main
         //18main
         //20main
         //0线程一
         //2线程一
         //4线程一
        ```
        
    
    1. `**sleep()**`：
        
        ```Java
         class MyThread extends Thread{
             @Override
             public void run() {
                 for (int i = 0; i < 100; i++) {
                     if(i % 2 == 0){
                         try {//只能用try-catch，不能throws因为被重写的run()并没有throws
                             sleep(1000);//延迟
                         } catch (InterruptedException e) {
                             e.printStackTrace();
                         }
                         System.out.println(i+Thread.currentThread().getName());
                     }
                 }
             }
         }
        ```
        
    
    1. `**suspend()**`：挂起进程
    
    1. `**notify()**`：唤醒等待的线程
        
        1. 如果监视器种只有一个等待线程，使用notify()可以唤醒
        
        1. 但是如果有多条线程notify()是随机唤醒其中一条线程
        
        1. 与之对应的就是notifyAll()就是唤醒所有等待的线程
        

        唤醒在此对象监规器上等待的单个线程。如果所有线程都在此对象上等待，则会选择唤醒其中一个线程。选择是任意性的，并在对实现做出决定时发生。线程通过调用其中一个  
        wait 方法，在对象的监视器上等待。  
        直到当前线程放弃此对象上的锁定，才能继续执行被唤醒的线程。被唤醒的线程将以常规方式与在该对象上主动同步的其他所有线程进行竞争，例如，唤醒的线程在作为锁定此对  
        象的下一个线程方面没有可靠的特权或劣势。

        
    
    1. `**resume()**`：结束挂起得状态
    
    1. `**stop()**`：强制线程生命期结束
    
    1. `**boolean isAlive()**`：返回boolean，判断线程是否还活着
    
# 3. 创建**方式二：实现 Runnable 接口**
1. **继承 Thread 类的实现方式存在的问题**
    
    - Java **不支持多重继承**，因此继承了 Thread 类就无法继承其它类，但是**可以实现多个接口**
    
    - 类可能只要求可执行就行，继承整个 Thread 类**开销过大**
    
1. ==**实现 Runnable 接口**==
    
    1. 步骤：
        
        1. 创建一个实现 `Runnable` 接口的类
        
        1. 实现类去实现 `Runnable` 中的抽象方法 `run()`
        
        1. 创建实现类的对象
        
        1. 将此对象作为参数传入 `Thread` 类的构造器中，创建 `Thread` 类的对象
            

            ![[IMG-20260404031739754.png|Untitled 1 221.png]]

            
        
        1. 通过 `Thread` 类的对象调用 `start()` 方法
        
    
    1. 其实 Thread 也是 Runnable 的实现类，方式二只是绕过了Thread
    
1. ==**特点：无返回值**==
1. 代码示例：
    

    ![[IMG-20260404031739782.png|Untitled 2 186.png]]

    
    ```Java
     class MyThread implements Runnable{
         @Override
         public void run() {
             for (int i = 0; i < 100; i++) {
                 if(i % 2 == 0){
                     System.out.println(i+Thread.currentThread().getName());
                 }
             }
         }
     }
     public class ThreadTest {
         public static void main(String[] args) {
             MyThread r = new MyThread();
             Thread t1= new Thread(r);
             t1.start();
             t1.setName("线程1");
     
    				 // ⭐ 当需要一个新的线程时，无需再 new 一个线程实现类对象。而是直接将之前 new 出来的实现类对象作为参数传入 Thread 类的构造器中，即可创建一个新的 Thread 类的对象
             Thread t2= new Thread(r);   
             t2.start();
             t2.setName("线程2");
         }
     }
    ```
    
1. 回到卖票的场景：
    
    ```Java
     class MyThread implements Runnable {
     
         private int ticket = 100;
         @Override
         public void run() {
             while(true){
                 if(ticket > 0){
                     System.out.println(Thread.currentThread().getName()+":卖票，票号为:"+ticket);
                     ticket--;
                 }else {
                     break;
                 }
             }
         }
     }
     public class ThreadTest {
         public static void main(String[] args) {
             MyThread myThread = new MyThread();
     
             //⭐只new了一个MyThread对象，故ticket之有一个
             //不用加static了
             Thread t1 = new Thread(myThread);
             t1.start();
    
             Thread t2 = new Thread(myThread);
             t2.start();
     
             Thread t3 = new Thread(myThread);
             t3.start();
         }
     }
    ```
    
# 3. **JDK5.0** 新增方式一：实现 **Callable** 接口
1. 与使用 `Runnable` 相比， `**Callable**` **功能更强大些**
    
    1. **`run()` 没有返回值，而 `call()` 可以**==**有返回值**==**，且支持泛型的返回值**
    
    1. **`call()`** ==**可以抛出异常**==
    
    1. 需要借助 `FutureTask` 实现类，比如获取返回结果
        

        ![[IMG-20260405035421944.png|Untitled 3 141.png]]

        
    
1. 步骤
    
    1. 创建一个实现 Callable 的实现类
    
    1. 实现 `**call()**`，将此线程需要执行的操作声明在方法中
    
    1. 创建 Callable 接口实现类的对象
    
    1. **将此 Callable 接口实现类 的对象作为参数传递到 FutureTask 构造器中，创建 FutureTask 对象**
    
    1. **将 FutureTask 对象作为参数传递到 Thread 类的构造器中，创建 Thread 对象，并调用 start( )**
    
    1. 获取 Callable 中 call() 的返回值
    
1. 示例：
    
    ```Java
     import java.util.concurrent.Callable;
     import java.util.concurrent.ExecutionException;
     import java.util.concurrent.FutureTask;
     
     class MyThread  implements Callable {
     
         @Override
         public Object call() throws Exception {
             int sum = 0;
             for (int i = 1; i <= 100 ; i++) {
                 if(i % 2 == 0){
                     System.out.println(i);
                     sum += i;
                 }
             }
             return sum;
         }
     }
     public class ThreadTest {
         public static void main(String[] args) throws ExecutionException, InterruptedException {
             MyThread myThread = new MyThread();
     
             FutureTask future = new FutureTask(myThread);
     
             new Thread(future).start();
     
             //get()的返回值即为FutureTask构造器参数Callable实现类重写的call()的返回值
             Object sum = future.get();
             System.out.println("总和为:"+sum);
     
         }
     }
    ```
