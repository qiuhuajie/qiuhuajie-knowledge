# 1. 介绍
1. Java 提供了一种用于停止线程的机制——中断🙋‍♂️
1. 提供中断机制的原因
    
    1. 首先，一个线程不应该由其他线程来强制中断或停止，而是应该由线程自己自行停止，所以，Thread.stop, Thread.suspend, Thread.resume 都已经被废弃了
        
        > [!important]
        > 
        > `stop()` 和 `interrupt()` 是 Java 中用于终止线程的方法，但它们之间有一些重要的区别🙋‍♂️
        > 
        > 1. `stop`方法：
        > 
        >     - ==会导致线程立即停止，并且不会进行任何清理工作==。这可能导致线程所持有的锁无法被释放，资源无法正确释放，从而引发潜在的问题，比如数据不一致等
        > 
        >     - 已被废弃，不推荐使用
        > 
        > 
        > 1. `interrupt`方法：
        > 
        >     - 是==一种更加安全和合理的方式来终止线程==。它不会立即终止线程，而是设置线程的中断标志位为 true，再轮询去判断这个标志，进而做中断操作
        > 
        >     - 在使用 interrupt 方法终止线程时，线程有机会做一些清理工作、释放资源，并且可以更好地处理中断信号，以保证线程的正确终止
        >  
        
    
    1. 其次，在 Java 中没有办法立即停止一条线程，然而停止线程却显得尤为重要，如取消一个耗时操作

1. 如何使用 API 实现中断
    
    1. 中断只是一种协作机制，Java 没有给中断增加任何语法，中断的过程完全需要程序员自己实现
    
    1. **每个线程对象中都有一个标识，用于表示线程是否被中断**，该标识位为true表示中断，为false表示未中断
    
    1. ⭐**若要中断一个线程，你需要手动调用该线程的**`**interrupt**`**方法，但是，该方法也仅仅是将线程对象的中断标识设成**`**true**`
    
    1. 接着你**需要自己写代码** `**while(true)**` **不断地检测当前线程的标识位是否为** `**true**`，如果为true，表示别的线程要求这条线程中断， 此时究竟该做什么需要你自己写代码实现

# 2. 中断 API 代码演示
## 2.1 isInterrupted()
1. **==判断此线程是否已被中断==**，与 `interrupted()` 的区别在于不会清除中断标志位
    

    ![[IMG-20260404031739435.png|Untitled 2 188.png]]

    

    （会将`是否清除中断标志 = false` 作为参数传入）

    
1. 线程的中断状态不受此方法的影响
## 2.2 interrupted()
1. **==判断当前线程是否已被中断==。同时，此方法**==**会清除线程的中断状态，重置为**== `**false**`
    

    ![[IMG-20260404031739472.png|Untitled 3 143.png]]

    

    （会将`是否清除中断标志 = true` 作为参数传入）

    
1. 示例
    
    ```Java
    public class InterruptDemo3 {
        public static void main(String[] args) throws InterruptedException {
            System.out.println(Thread.currentThread().getName() + "---" + Thread.interrupted());
    
            System.out.println("-------------");
            Thread.currentThread().interrupt();
    
            System.out.println(Thread.currentThread().getName() + "---" + Thread.interrupted());
            System.out.println(Thread.currentThread().getName() + "---" + Thread.interrupted());
        }
    }
    ```
    

    运行结果：第二次调用 interrupted()，会将中断状态重置为 `false`

## 2.3 interrupt()
- ==**用于中断一个线程，实际是将设置该线程的中断状态设置为**== **`true`**
- 中断不活动的线程不会产生任何影响
### 打断一个正常运行的线程
```Java
public class InterruptDemo1 {
    public static void main(String[] args) {
        Thread t1 = new Thread(() -> {
            while (true) {
                if (Thread.currentThread().isInterrupted()) {
                    System.out.println("-----t1 线程被中断了，break，程序结束");
                    break;
                }
                System.out.println("-----check");
            }
        }, "t1");
        t1.start();
        //暂停5毫秒
        try { TimeUnit.MILLISECONDS.sleep(5); } catch (InterruptedException e) { e.printStackTrace(); }
        t1.interrupt();
    }
}
```

### 打断一个 Waiting 状态的线程
- **如果该线程正在调用 [`wait()`](https://www.runoob.com/manual/jdk11api/java.base/java/lang/Object.html#wait\(\)) ， [`sleep(long)`](https://www.runoob.com/manual/jdk11api/java.base/java/lang/Thread.html#sleep\(long\)) 等方法，那么它的中断状态将被清除，并且将收到 [`InterruptedException`](https://www.runoob.com/manual/jdk11api/java.base/java/lang/InterruptedException.html)**
```Java
public class Demo {
    public static void main(String[] args) {
        Thread t1 = new Thread(() -> {
            while (true) {
                if (Thread.currentThread().isInterrupted()) {
                    System.out.println("-----t1 线程被中断了，break，程序结束");
                    break;
                }
                try {
                    Thread.sleep(200);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                System.out.println("-----check");
            }
        }, "t1");
        t1.start();
        //暂停5毫秒
        try { TimeUnit.MILLISECONDS.sleep(5); } catch (InterruptedException e) { e.printStackTrace(); }
        t1.interrupt();
    }
}
```
- 运行结果：抛出了 `InterruptedException` ，并且线程 t1 的中断状态被清除，变为了 `false`，继续不断循环检查
![[IMG-20260404031739549.png|Untitled 5 95.png]]
- **解决方法：在 `sleep()` 的异常处理代码中，重新将线程中断**

### 打断一个 Park 中的线程
1. `interrupt()` 可以正常打断一个 `park` 中的线程
1. **但是，此时若再次想 `park` 这个线程，则无法生效**
    

    ![[IMG-20260404031739671.png|Untitled 7 68.png]]

    
1. 需要将线程的打断标记位再次置为 `false` 从才能生效
