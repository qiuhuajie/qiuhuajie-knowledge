# 一、基础线程机制
## 1. sleep()
1. **`Thread.sleep(millisec)`** 可以让线程等待指定的时间，到达时间后自动唤醒；`millisec` 单位为毫秒
2. `sleep()` 可能会抛出 InterruptedException
    1. ==**因为异常不能跨线程传播回**== **`main()`** ==**中**==，因此必须在本地进行处理
    2. 线程中抛出的其它异常也同样需要在本地进行处理
3. 代码示例：

    ```Java
    public void run() {
        try {
            Thread.sleep(3000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
    ```
## 2. yield()
1. 调用 `Thread.yield()` 后，**线程会**==**将 CPU 时间片主动让出**==**，会让当前线程从 Running 进入 Runnable 就绪状态，然后 CPU 调度执行其它线程**
2. 具体的实现依赖于操作系统的任务调度器
3. 该方法只是对线程调度器的一个**建议**，而且也只是建议具有相同优先级的其它线程可以运行
4. 代码示例：

    ```Java
    public void run() {
        Thread.yield();
    }
    ```

## 3. 比较
- ==**ReentrantLock 和 Synchronized 的对比**====❓==
    1. **获取锁和释放锁方式不同**
        1. synchronized 会自动加锁和释放锁，修饰普通方法、静态方法和代码块
        2. 而 ReentrantLock 需要手动加锁和释放锁，使用之前需要先创建 ReentrantLock 对象，然后使用 lock 方法进行加锁，使用完之后再调用 unlock 方法释放锁
    2. **底层实现不同**
        1. synchronized 是 JVM 层面通过监视器（Monitor）实现的，两个字节码指令
        2. 而 ReentrantLock 是通过 AQS（AbstractQueuedSynchronizer）程序级别的 API 实现
    3. **公平锁/非公平锁**
        1. synchronized 属于非公平锁
        2. 而 ReentrantLock 既可以是公平锁也可以是非公平锁。默认情况下 ReentrantLock 为非公平锁
    4. **响应中断不同**🟡
        1. synchronized 不能响应中断，也就是如果发生了死锁，使用 synchronized 会一直等待下去
        2. 而 ReentrantLock 可以使用 `lock.lockInterruptibly()` 获取锁，这种方式可以响应中断并释放锁，从而解决死锁的问题
    5. **锁绑定多个条件**🟡
        1. 一个 ReentrantLock 可以同时绑定多个 Condition 对象，让不同的线程去不同的 waitSet 中等待，实现根据不同条件，唤醒不同的线程
        2. 而 synchronized 只有一个 waitSet
- **除非需要使用 ReentrantLock 的高级功能，否则优先使用 synchronized**
    1. 这是因为 synchronized 是 JVM 实现的一种锁机制，JVM 原生地支持它，而 ReentrantLock 不是所有的 JDK 版本都支持
    2. 并且使用 synchronized 不用担心没有释放锁而导致死锁问题，因为 JVM 会确保锁的释放

# 三、线程之间的协作
## 1. join()
1. 在线程中调用另一个线程的 `join()` 方法，会将当前线程挂起，而不是忙等待，直到目标线程结束
2. **如果**==**一个线程需要使用另一个线程的计算结果**==
    1. 如果使用 `sleep()` 的话，也可以，但是并不能确定另一个线程需要多长时间才能计算完成
    2. 而**使用 `join()` 的话更合理**，在一个线程中调用另一个线程的 `join()`
3. 也可以给 `join()` 传入一个等待时间：`t1.join(1500)`
4. 代码示例：

    ```Java
    public class JoinDemo {
        private static Thread t1, t2, t3;
        public static void main(String[] args) {
            Thread t2 = new Thread(() -> {
                System.out.println("B");
            });
            Thread t1 = new Thread(() -> {
                System.out.println("A1");
                try {
                    // ⭐在 t1 线程中调用了 t2 线程的 join() 方法，t1 线程会等待 t2 线程结束才继续执行
                    t2.join();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    System.out.println("A2");
                }
            });
            t1.start();
            t2.start();
        }
    }
    ```

    运行结果

    ```Java
    A1
    B
    A2
    ```
## 2. wait()、notify()、notifyAll()
1. 这些方法**操作的是对象关联的 Monitor 对象，所以在是 Object 中定义的方法，且只有获取了这个 Object 锁才能调用**
2. **工作机制：**
    1. 调用 `wait()` 使得线程等待某个条件满足，线程在等待时会被挂起
    2. 当其他线程的运行使得这个条件满足时，其它线程会调用 `notify()` 或者 `notifyAll()` 来唤醒挂起的线程
3. ⭐**使用** `wait()` **挂起期间，线程会释放锁**
    > 如果没有释放锁，那么其它线程就无法进入对象的同步方法或者同步控制块中，那么就无法执行 notify() 或者 notifyAll() 来唤醒挂起的线程，造成死锁
4. ⚠️**使用时的注意事项：**
    1. 只能用在同步方法或者同步控制块中使用（**只有先能成为 owner，才有可能进 waitSet**），否则会在运行时抛出 `IllegalMonitorStateExeception` 异常
        1. 如果 wait 和 notify/notifyAll 不强制和 synchronized 一起使用
        2. 那么在多线程执行时，就会出现线程 a 判断当前不满足执行条件，准备执行 wait 时，线程b修改导致满足了执行条件，进而先执行了 notify

            ![[IMG-20260619224544612.png|703]]

        3. 而线程 a 已经完成判断了，继续进入 wait，从而导致程序执行混乱
    2. 必须**先执行** `wait()` **，后执行** `notify()` 才能正常等待唤醒，否则会抛出 `IllegalMonitorStateExeception` 异常
5. **原理：**

    ![[IMG-20260619224544777.png|641]]

    ![[IMG-20260619224544853.png|751]]

    1. `obj.wait()` 让进入 object 监视器的线程到 waitSet 等待
    2. `obj.notify()` 在 object 上正在 waitSet 等待的线程中**挑一个**唤醒
    3. `obj.notifyAll()` 让 object 上正在 waitSet 等待的线程全部唤醒
6. 💁‍♂️ **`Thread.sleep()`** **和** **`Object.wait()`** **的区别**
    - 两者都可以暂停线程的执行
    - sleep 不会释放占有的锁，wait 会释放占有的锁
    - sleep 不需要在同步代码块中使用，而 wait 需要
    - 唤醒方式不同
        - wait()方法被调用后，线程不会自动苏醒，需要别的线程调用同一个对象上的notify()或者 notifyAll() 方法
        - sleep()方法执行完成后，线程会自动苏醒，或者也可以使用 wait(long timeout) 超时后线程会自动苏醒
    - wait() 通常被用于线程间交互/通信，sleep() 通常被用于暂停执行
    - sleep()是 Thread 类的静态本地方法wait() 则是 Object 类的本地方法
7. 代码示例

    ```Java
    public class LockSupportDemo {
        public static void main(String[] args) {
            Object objectLock = new Object();
            new Thread(() -> {
                synchronized (objectLock) {
                    System.out.println(Thread.currentThread().getName() + "等待通知");
                    try {
                        objectLock.wait();
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                }
                System.out.println(Thread.currentThread().getName() + "被唤醒了");
            }, "t1").start();
            //暂停几秒钟线程
            try { TimeUnit.SECONDS.sleep(3L); } catch (InterruptedException e) { e.printStackTrace(); }
            new Thread(() -> {
                synchronized (objectLock) {
                    objectLock.notify();
                    System.out.println(Thread.currentThread().getName() + "发出通知");
                }
            }, "t2").start();
        }
    }
    ```
    - 运行结果

        ```Java
        t1等待通知
        t2发出通知
        t1被唤醒了
        ```
## 3. await()、signal()、signalAll()
1. java.util.concurrent 类库中提供了 **`Condition`** 类来实现线程之间的协调
    1. 可以在 `Condition` 上调用 `await()` 方法使线程等待
    2. 其它线程调用 `signal()` 或 `signalAll()` 方法唤醒等待的线程
2. ⭐`Condition` **需要使用一个** **`Lock`** **来获得**
3. Object.wait() 和 Condition.await() 的区别
    - **相比于 `wait()` 这种等待方式，`await()` 可以指定**==**多个等待的条件**==**，因此更加灵活**
    - Object.wait() 和 Condition.await() 的原理是基本一致
    - 不同的是 **`Condition.await()`** **底层是调用** **`LockSupport.park()`** **来实现阻塞当前线程的**
        - 实际上，它在阻塞当前线程之前还干了两件事，一是把当前线程添加到条件队列中，二是“完全”释放锁，也就是让state状态变量变为0
        - 然后才是调用LockSupport.park()阻塞当前线程。
4. ==**限制条件：**==
    1. `await()` 和 `signal()` 方法必须要在获取锁 `lock.lock()` 和释放锁 `lock.unlock()` 的代码之间
    2. 且必须先执行 `await()` ，后执行 `signal()` 才能正常等待唤醒
5. 代码示例：

    ```Java
    public class LockSupportDemo2 {
        public static void main(String[] args) {
            Lock lock = new ReentrantLock();
    				// 使用 Lock 来获取一个 Condition 对象
            Condition condition = lock.newCondition();
            new Thread(() -> {
                lock.lock();
                System.out.println(Thread.currentThread().getName() + "等待通知");
                try {
                    condition.await();
                    System.out.println(Thread.currentThread().getName() + "被唤醒了");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    lock.unlock();
                }
            }, "t1").start();
            //暂停几秒钟线程
            try { TimeUnit.SECONDS.sleep(3L); } catch (InterruptedException e) { e.printStackTrace(); }
            new Thread(() -> {
                lock.lock();
                try {
                    condition.signal();
                    System.out.println(Thread.currentThread().getName() + "发出通知");
                } catch (Exception e) {
                    e.printStackTrace();
                } finally {
                    lock.unlock();
                }
            }, "t2").start();
        }
    }
    ```
    - 运行结果

        ```Java
        t1等待通知
        t2发出通知
        t1被唤醒了
        ```

        static int flag = 1;

                next.signal();
                lock.unlock();
            }
        }

## 2. 生产者和消费者🙋‍♂
1. 实现方式：
    > 生产者消费者问题解决了==多个进程间共享有限资源的同步问题==。它确保了多个生产者和消费者在访问共享有限缓冲区时不会发生冲突。常用的同步方法是采用信号或加锁机制，保证资源在任意时刻至多被一个线程访问。

2. **方案一：使用** **`BlockingQueue`** **实现**

    ```Java
    public class ProducerConsumerExample {
        public static void main(String[] args) {
            // 创建一个容量为10张缓存图片的的阻塞队列
            BlockingQueue<ImageData> queue = new ArrayBlockingQueue<>(10);
            Thread bitmapLoadThread = new Thread(new BitmapLoader(queue));
            Thread bitmapPlayerThread = new Thread(new BitmapPlayer(queue));
            bitmapLoadThread.start(); // 启动生产者线程
            bitmapPlayerThread.start(); // 启动消费者线程
        }
        // 创建一个生产者线程, 负责预加载图片数据
        private static class BitmapLoader implements Runnable {
            private final BlockingQueue<ImageData> queue;
            public BitmapLoader(BlockingQueue<ImageData> queue) {
                this.queue = queue;
            }
            public void run() {
                try {
                    long startTime = System.currentTimeMillis();
                    //假设一共有 20 张图片
                    for (int i = 0; i < 20; i++) {
                        //生成一张图片
                        ImageData imageData = generateImageData(i);
                        // 往队列中添加元素，如果队列已满则会阻塞等待
                        queue.put(imageData);
                    }
                    long duration = System.currentTimeMillis() - startTime;
                    System.out.println("---生产者生产耗时：" + duration);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
            private ImageData generateImageData(int index) {
                ImageData imageData = new ImageData();
                try {
                    //假设从IO中加载一张图片耗时10ms
                    Thread.sleep(10);
                } catch (InterruptedException e) {
                    throw new RuntimeException(e);
                }
                imageData.bitmap = "图片_" + index;
                imageData.nextFrameTime = 30;
                System.out.println("生产者生产：" + imageData.bitmap);
                return imageData;
            }
        }
        // 创建一个图片播放线程, 作为消费者
        private static class BitmapPlayer implements Runnable {
            private BlockingQueue<ImageData> queue;
            public BitmapPlayer(BlockingQueue<ImageData> queue) {
                this.queue = queue;
            }
            public void run() {
                try {
                    long startTime = System.currentTimeMillis();
                    // 播放20张图片后结束
                    for (int i = 0; i < 20; i++) {
                        // 从队列中取出元素，如果队列为空则会阻塞等待
                        ImageData imageData = queue.take();
                        System.out.println("消费者消费：" + imageData.bitmap);
                        //每张图片播放30ms
                        Thread.sleep(30);
                    }
                    long duration = System.currentTimeMillis() - startTime;
                    System.out.println("---消费者耗时：" + duration);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }
        public static class ImageData {
            String bitmap;
            float shootTime;
            long nextFrameTime;
        }
    }
    ```

3. **方案二：使用 wait() 和 notify() 实现**

    ```Java
    public class pc {
        static List<Integer> buffer = null;
        static Integer capacity = 10;
        static Integer pointer = 0;
        public static void main(String[] args) {
            Object obj = new Object();
            buffer = new ArrayList<>();
            Thread producer = new Thread(() -> {
                while (true) {
                    synchronized (obj) {
    										// 当容器满时要阻塞生产者
                        if (buffer.size() == capacity) {
                            try {
                                obj.wait();
                            } catch (InterruptedException e) {
                                e.printStackTrace();
                            }
                        }
                        ++pointer;
                        System.out.println(Thread.currentThread().getName() + ": put " + pointer);
                        buffer.add(pointer);
                        System.out.println(buffer.size());
    										// 每生产一个产品，要唤醒所有消费者
                        obj.notifyAll();
                    }
                }
            }, "producer");
            Thread consumer = new Thread(() -> {
                while (true) {
                    synchronized (obj) {
    										// 当容器空时要阻塞消费者
                        if (buffer.isEmpty()) {
                            try {
                                obj.wait();
                            } catch (InterruptedException e) {
                                e.printStackTrace();
                            }
                        }
                        Integer remove = buffer.remove(buffer.size() - 1);
                        System.out.println(Thread.currentThread().getName() + ": get " + remove);
                        System.out.println(buffer.size());
    		                // 每消费一个产品，要唤醒所有生产者
                        obj.notifyAll();
                    }
                }
            }, "consumer");
            producer.start();
            consumer.start();
        }
    }
    ```

    面向对象版：

    - 展开全部

        ```Java
        public class ProducerConsumerExample {
            public static void main(String[] args) {
                LinkedList<Integer> buffer = new LinkedList<>();
                int capacity = 5;
                Thread producerThread = new Thread(new Producer(buffer, capacity), "Producer");
                Thread consumerThread = new Thread(new Consumer(buffer), "Consumer");
                producerThread.start();
                consumerThread.start();
            }
        }
        class Producer implements Runnable {
            private final LinkedList<Integer> buffer;
            private final int capacity;
            public Producer(LinkedList<Integer> buffer, int capacity) {
                this.buffer = buffer;
                this.capacity = capacity;
            }
            public void run() {
        				// 生产者生产 10 个产品
                for (int i = 0; i < 10; i++) {
                    try {
                        produce(i);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                }
            }
            private void produce(int i) throws InterruptedException {
        				// 在同步代码块中使用 wait()
                synchronized (buffer) {
        					  // 当容器满时要阻塞生产者
                    while (buffer.size() == capacity) {
                        System.out.println("Buffer is full, waiting for consumer to consume...");
                        buffer.wait();
                    }
                    buffer.add(i);
                    System.out.println("Produced " + i);
        						// 每生产一个产品，要唤醒所有消费者
                    buffer.notifyAll();
                }
            }
        }
        class Consumer implements Runnable {
            private final LinkedList<Integer> buffer;
            public Consumer(LinkedList<Integer> buffer) {
                this.buffer = buffer;
            }
            public void run() {
        				// 消费者循环等待消费产品
                while (true) {
                    try {
                        consume();
                        Thread.sleep(1000);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                }
            }
            private void consume() throws InterruptedException {
        				// 在同步代码块中使用 wait()
                synchronized (buffer) {
        						// 当容器空时要阻塞消费者
                    while (buffer.isEmpty()) {
                        System.out.println("Buffer is empty, waiting for producer to produce...");
                        buffer.wait();
                    }
                    int value = buffer.remove();
                    System.out.println("Consumed " + value);
        						// 每消费一个产品，要唤醒所有生产者
                    buffer.notifyAll();
                }
            }
        ```
