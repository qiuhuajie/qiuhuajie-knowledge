# 一、介绍
- 线程安全集合类可以分为三大类：
    1. **遗留的线程安全集合**如 Hashtable ， Vector，底层都是使用 `synchronized` 来实现线程安全，性能不高

        ![[IMG-20260619223628873.png|528]]

    2. Collections 工具类中有**一系列** **`synchronized`** **打头的集合类**，是在原来不安全的集合上装饰为线程安全的集合

        ![[IMG-20260619223628945.png|314]]

    3. `java.util.concurrent` ==**包下提供的线程安全集合类**==
        1. **Blocking 类的容器：**大部分实现基于锁，并提供用来阻塞的方法，里面很多方法实现了：让队列在不满足条件时，阻塞住
        2. **CopyOnWrite 类容器：**修改开销相对较重
        3. **Concurrent 类容器：**
            1. 内部很多操作使用 CAS 优化，一般可以提供较高吞吐量
            2. 缺点：弱一致性，体现在三个方面
                1. 遍历时弱一致性，例如，当利用迭代器遍历时，如果容器发生修改，迭代器仍然可以继续进行遍历，这时内容是旧的
                2. 求大小弱一致性，size 操作未必是 100% 准确
                3. 读取弱一致性
