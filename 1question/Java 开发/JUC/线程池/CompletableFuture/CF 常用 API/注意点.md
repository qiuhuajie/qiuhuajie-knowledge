# 注意点
> [!important] **任务之间的顺序执行**
> 
> 1. `thenRun(Runnable runnable)` ：任务 A 执行完执行 B，并且 B 不需要 A 的结果
> 
> 1. `thenAccept(Consumer action)` ：任务 A 执行完执行 B，B 需要 A 的结果，但是任务 B 无返回值
> 
> 1. `thenApply(Function fn)` ：任务 A 执行完执行 B，B 需要 A 的结果，同时任务 B 有返回值
> 
>     ```Java
>     public class CompletableFutureAPIDemo2 {
>         public static void main(String[] args) throws ExecutionException, InterruptedException, TimeoutException {
>     
>             System.out.println(CompletableFuture.supplyAsync(() -> "resultA").thenRun(() -> {}).join());
>     
>             System.out.println(CompletableFuture.supplyAsync(() -> "resultA").thenAccept(resultA -> {}).join());
>     
>             System.out.println(CompletableFuture.supplyAsync(() -> "resultA").thenApply(resultA -> resultA + " resultB").join());
>             
>         }
>     }
>     
>     执行结果：
>     null
>     null
>     resultA resultB
>     ```
>

> [!important] **异步回调要传线程池**
> 
> 1. [[CompletableFuture（CF） 介绍]]，这里我们建议==**强制传线程池，且根据实际情况做线程池隔离**==
> 
> 1. ==**❗当不传递线程池时，会使用**== ==`**ForkJoinPool**`== ==**中的公共线程池**== ==`**CommonPool**`==
> 
>     1. 这里所有调用将共用该线程池，**核心线程数=处理器数量-1（单核核心线程数为1）**
> 
>     1. 所有异步回调都会共用该 `CommonPool`，**核心与非核心业务都竞争同一个池中的线程，很容易成为系统瓶颈**
> 
> 
> 1. ==手动传递线程池参数==
> 
>     1. 可以更方便的调节参数
> 
>     1. 并且可以给不同的业务分配不同的线程池，以求资源隔离，减少不同业务之间的相互干扰
>  

> [!important] **线程池隔离**
> 
> 1. 示例代码：**线程池循环引用会导致死锁**
> 
>     ```Java
>     public class CompletableFutureAPIDemo4 {
>         public static void main(String[] args) throws ExecutionException, InterruptedException, TimeoutException {
>             ExecutorService threadPool = Executors.newFixedThreadPool(10);
>             CompletableFuture<String> parentTask = CompletableFuture.supplyAsync(() -> {
>                 try { TimeUnit.SECONDS.sleep(1); } catch (InterruptedException e) { e.printStackTrace(); }
>                 // 子任务
>                 return CompletableFuture.supplyAsync(() -> {
>                     System.out.println("childTask started");
>                     return "childTask finish";
>                 }, threadPool).join();
>             }, threadPool);
>             System.out.println(parentTask.join());
>         }
>     }
>     ```
> 
> 
> 1. 如上代码块所示， `main()` 方法第三行通过 `supplyAsync` 向 `threadPool`请求线程，并且内部子任务又向 `threadPool`请求线程 `threadPool`大小为 `10`
> 
>     1. 当**同一时刻有 `10` 个请求到达，则 `threadPool` 被打满**，子任务请求线程时进入阻塞队列排队
> 
>     1. 但是父任务的完成又依赖于子任务，这时由于子任务得不到线程，父任务无法完成
> 
>     1. **主线程执行 `parentTask.join()` 进入阻塞状态，并且永远无法恢复**
> 
> 
> 1. 为了修复该问题，需要将**父任务与子任务**==**做线程池隔离：**== ==**两个任务请求不同的线程池**==**，避免循环依赖导致的阻塞**
