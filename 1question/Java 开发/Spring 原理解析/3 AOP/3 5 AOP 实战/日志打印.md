# 1. 需求描述
1. 在保存订单和更新订单的同时，保存一个日志流水
1. 流水的写入对一致性的要求不是很高，所以没必要去保证事务
1. 希望对主链路的侵入越少越好
1. 解决方案：在每一个操作的方法之前，去加一个注解，通过切面处理这个方法，在处理逻辑中异步地去写入流水表
1. **⭐难点：**
    
    1. **虽然写入流水是一个标准化的模型，但是入参并不标准**
    
    1. **例如入参是不同的类，但是不同的类里都有商家ID，但是有的类中定义为 `orderId`，有的类中定义为 `id`**
        
        ```Java
        @Service
        public class OrderService {
            public Boolean saveOrder(SaveOrder saveOrder){
                System.out.println("save order , orderId : " + saveOrder.getId());
                return true;
            }
        
            public Boolean updateOrder(UpdateOrder updateOrder){
                System.out.println("update order , orderId : " + updateOrder.getOrderId());
                return true;
            }
        }
        ```
        
    
    1. **那么如何在统一的一个切面中，去根据不同的入参，取到想要的参数值**
    
    1. 难道要在切面的处理逻辑中逐个判断入参的类型，然后进行转换吗，显然不太优雅

# 3. 定义注解
```Java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface RecordOperate {
    String desc() default "";
		// 在注解中添加一个 Class 变量，可与帮助我们指定参数中某个字段该如何取值
		// 这里的变量类型都是 Convert 接口的实现类，他们中根据各自的类的不同，实现了不同的字段取值逻辑
    Class<? extends Convert> convert();
}
```
# 4. 定义切面
```Java
@Aspect
@Component
public class OperateAspect {
    /**
     * 1. 定义切入点
     * 2. 横切逻辑
     * 3. 织入
     */
    @Pointcut("@annotation(com.atguigu.admin.aopDemo.RecordOperate)")
    public void pointcut(){}
		// 并不希望记录流水这个操作对主链路有任何性能上的影响，这里可以异步去做，这里定义一个线程池
    private ThreadPoolExecutor poolExecutor = new ThreadPoolExecutor(
            1, 1, 1, TimeUnit.SECONDS,new LinkedBlockingDeque<>(100)
    );
    @Around("pointcut()")
    public Object around(ProceedingJoinPoint joinPoint) throws Throwable {
        // 切点执行
				Object result = joinPoint.proceed();
				// 日志记录
        poolExecutor.execute(() ->{
            try {
                MethodSignature methodSignature = (MethodSignature) joinPoint.getSignature();
                RecordOperate annotation = methodSignature.getMethod().getAnnotation(RecordOperate.class);
								
								// 获取到注解中的 convert Class变量，并生成一个该类的对象：logConvert
                Class<? extends Convert> convert = annotation.convert();
                Convert logConvert = convert.newInstance();
								// logConvert 就是不同的 Convert 接口实现类的实例对象，调用其各自实现的 convert() 方法逻辑，即可根据不同的需求做出对应的取值操作
                OperateLogDO operateLogDO = logConvert.convert(joinPoint.getArgs()[0]);
								// 如果不用上面的处理方法，则需要在切面的处理逻辑中，用 if()else() 语句来判断当前入参是什么类型的，再将正确的对应字段取出来
								// 对其他值进行赋值
                operateLogDO.setDesc(annotation.desc());
                operateLogDO.setResult(result.toString());
                System.out.println("insert operateLog" + JSON.toJSONString(operateLogDO));
            } catch (InstantiationException e) {
                e.printStackTrace();
            } catch (IllegalAccessException e) {
                e.printStackTrace();
            }
        });
        return result;
    }
}
```
# 5. **Convert 接口及其实现类**
1. `Convert`
    
    ```Java
    public interface Convert<PARM> {
        OperateLogDO convert(PARM parm);
    }
    ```
    
1. `SaveOrderConvert` ：**在这个实现类中，`orderId` 取得是 `id` 字段**
    
    ```Java
    public class SaveOrderConvert implements Convert<SaveOrder>{
        @Override
        public OperateLogDO convert(SaveOrder saveOrder) {
            OperateLogDO operateLogDO = new OperateLogDO();
            operateLogDO.setOrderId(saveOrder.getId());
            return operateLogDO;
        }
    }
    ```
    
1. `UpdateOrderConvert` ：**在这个实现类中，`orderId` 取得是 `orderId` 字段**
    
    ```Java
    public class UpdateOrderConvert implements Convert<UpdateOrder>{
        @Override
        public OperateLogDO convert(UpdateOrder updateOrder) {
            OperateLogDO operateLogDO = new OperateLogDO();
            operateLogDO.setOrderId(updateOrder.getOrderId());
            return operateLogDO;
        }
    }
    ```
    
# 6. 业务流程代码
1. `OrderController`
    
    ```Java
    @RestController
    @RequestMapping(value = "/aop")
    public class OrderController {
        @Autowired
        OrderService orderService;
    
        @RequestMapping(value = "/test", method = RequestMethod.POST)
        public void test() {
            SaveOrder saveOrder = new SaveOrder();
            saveOrder.setId(1L);
            orderService.saveOrder(saveOrder);
    
            UpdateOrder updateOrder = new UpdateOrder();
            updateOrder.setOrderId(2L);
            orderService.updateOrder(updateOrder);
        }
    }
    ```
    
1. `OrderService`
    
    ```Java
    @Service
    public class OrderService {
        @RecordOperate(desc = "保存订单", convert = SaveOrderConvert.class)
        public Boolean saveOrder(SaveOrder saveOrder){
            System.out.println("save order , orderId : " + saveOrder.getId());
            return true;
        }
    
        @RecordOperate(desc = "更新订单", convert = UpdateOrderConvert.class)
        public Boolean updateOrder(UpdateOrder updateOrder){
            System.out.println("update order , orderId : " + updateOrder.getOrderId());
            return true;
        }
    }
    ```
