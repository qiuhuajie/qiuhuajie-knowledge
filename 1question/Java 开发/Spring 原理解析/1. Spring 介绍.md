# 1. **Spring框架**

> [!info] Home  
> Level up your Java code and explore what Spring can do for you.  
> [https://spring.io/](https://spring.io/)  
- Spring是轻量级的、开源的 JavaEE 框架
    
    - 轻量级：Spring所含的jar包比较少、比较小，需要大量引入
    
    - 框架：解决企业应用开发的复杂性
    
- ==Spring有两个核心部分：====**IOC**====和====**AOP**==
    
    1. IOC：控制反转，之前是new个类创建对象，现在是把创建对象的过程交给spring进行管理
    
    1. AOP：面向切面，不修改代码的情况下，进行功能增强
    
    ![[IMG-20260404031819144.png|Untitled 78.png]]
    
- Spring框架的**特点**
    
    1. 方便解耦，简化开发
    
    1. 支持AOP编程
    
    1. 方便程序的测试
    
    1. 方便集成各种常见框架，如mybatis
    
    1. 方便进行事务的操作
    
    1. 降低API的开发难度，对其中的功能进行封装
    
# 2. 设计模式在 Spring 中的应用⭐
1. **工厂设计模式：**Spring使用工厂模式通过 `BeanFactory` 和 `ApplicationContext` 对象 创建 bean[[简单工厂模式 ⭐]]
1. **代理设计模式：**Spring AOP 功能的实现[[代理模式 ⭐]]
1. **单例设计模式：**Spring 中的 Bean 默认都是单例的[[单例模式 ⭐]]
1. **观察者模式：**Spring 事件驱动模型就是观察者模式很经典的一个应用[[观察者模式⭐]]
1. **适配器模式：**
    
    - Spring AOP 的增强或通知（Advice）使用到了适配器模式[[适配器模式 ⭐]]
    
    - spring MVC 中也是用到了适配器模式适配 Controller[[适配器模式 ⭐]]
    
1. **模板模式：**Spring IOC 容器在初始化工程中，使用的 `AbstractApplicationContext` 的 `refresh()` 方法是一个模板方法[[模板模式 ⭐]]
1. **职责链模式：**
    
    - Spring AOP 中增强方法与被增强方法则组织[[责任链模式 ⭐]]
    
    - Spring MVC 中拦截器与处理器中的业务逻辑方法组织成了一个执行链[[责任链模式 ⭐]]
    
# 3. 快速开始
1. 下载Spring5所依赖的jar包
    
    1. 选取Spring5.x版本
    
    1. 下载地址：
        
        > [!info] Index of release/org/springframework/spring  
        > 1.  
        > [https://repo.spring.io/release/org/springframework/spring/](https://repo.spring.io/release/org/springframework/spring/)  
        
    
1. 导入Spring基本包
    
    ```Plain
     //4个核心基本包
     spring-beans-5.2.6.RELEASE
     spring-context-5.2.6.RELEASE
     spring-core-5.2.6.RELEASE
     spring-expression-5.2.6.RELEASE
     
     //日志包
     commons-logging-1.1.1
    ```
    
1. 创建User类
    
    ```Java
     package com.at.spring5;
     
     public class User {
         public void add(){
             System.out.println("add....");
         }
     }
    ```
    
1. 使用配置文件创建一个User对象：配置bean.xml（不再使用new的方式）
    
    ```XML
     <?xml version="1.0" encoding="UTF-8"?>
     <beans xmlns="http://www.springframework.org/schema/beans"
            xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
            xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">
     
            <!--配置User对象创建-->
            <bean id="user" class="com.at.spring5.User"></bean>
     </beans>
    ```
    
1. 创建测试类
    
    ```Java
     package com.test;
     
     import com.at.spring5.User;
     import org.junit.Test;
     import org.springframework.context.ApplicationContext;
     import org.springframework.context.support.ClassPathXmlApplicationContext;
     
     public class testdemo {
         @Test
         public void testadd(){
             //1.加载spring配置文件（ApplicationContext在加载配置文件时，就将对象创建好了）
             ApplicationContext context = new ClassPathXmlApplicationContext("bean1.xml");
     
             //2. 通过id值 获取 配置中已经创建好的对象
             User user = context.getBean("user", User.class);
     
             System.out.println(user);
             user.add();
         }
     }
    ```
    
1. 测试结果
    
    ```Java
     com.at.spring5.User@4466af20    //打印user对象的地址
     add....                         //执行user对象的方法
    ```
