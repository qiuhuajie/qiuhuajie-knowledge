# 1. IOC 接口

1. 由上篇可知：**IoC Container 可以直接隐藏具体的创建实例的细节，在我们来看它就像一个工厂**

    ![IMG-20260620230826310.webp](IMG-20260620230826310.webp)

2. IOC思想基于IOC容器完成，IOC容器底层就是对象工厂，在里面可以读取配置文件、用反射创建对象
3. Spring提供了IOC 容器实现的两种方式：（两个接口）作用相似
    - **BeanFactory**
    - **ApplicationContext**
4. BeanFactory 和 ApplicationContext 都是用于**管理 Spring Bean 的容器**，它们的区别和联系如下

## 1.1 **BeanFactory** 🙋‍♂️

1. 介绍
    1. BeanFactory 是 Spring 框架的**基础设施**，**用于管理 Bean 的生命周期和依赖关系**，提供了 **IoC** 和 **DI** 功能
    2. Spring内部使用的接口，不提供给开发人员使用
    3. BeanFactory 是**延迟加载**的，即只有在**获取 Bean 时才会进行实例化**，可以减少系统资源的占用
    4. BeanFactory 是**单例模式**，即在整个应用中只有一个 BeanFactory 实例
2. 它有多种实现，比如 AnnotationConfigApplicationContext、XmlWebApplicationContext 等
3. BeanFactory 中定义了 Spring 容器基本方法，如使用 `getBean(beanName,Class)` 获取 Bean 对象

    ![640 (7).png](IMG-20260620230826432.png)

## 1.2 **ApplicationContext** 🙋‍♂️

1. 介绍
    1. **ApplicationContext 是 BeanFactory 的扩展（继承了 BeanFactory）**，除了拥有 BeanFactory 支持的所有功能外，还提供了例如**国际化支持**、**AOP 支持**等
    2. 一般面向开发人员
    3. ApplicationContext 在**启动时会立即加载所有的 Bean**，导致启动时间较长
    4. ApplicationContext 可以有**多个实例**，并且可以通过**父子容器**的方式组织起来，方便模块化开发
2. 下图是 ApplicationContext 接口的继承关系

    ![640 (4).png](IMG-20260620230826597.png)

3. ApplicationContext 接口的**实现类**
    - `FileSystemXmlApplicationContext`：从文件系统加载bean定义，以及相关资源的applicationContext实现，构造参数：盘路径，磁盘符全路径
    - `ClassPathXmlApplicationContext`：从classpath加载bean定义，以及相关资源的applicationContext实现，构造参数：类路径，src中的
    - `XmlWebApplicatonContext`：用于web程序的applicationcontext实现

# 5. **FactoryBean**

1. FactoryBean 是属于 Spring 的一个 Bean

    ![Untitled](IMG-20260620230826904.png)

2. Spring 有两种Bean，一种是普通bean，另一种是工厂bean（FactoryBean）
    1. 普通bean：配置文件中，bean标签中定义的对象类型，就是该bean返回对象的类型
    2. 工厂bean： 配置文件中，bean标签中定义的对象类型和该bean返回对象的类型不一样
3. 在 IOC 容器的基础上给 Bean 的实现加上了一个简单工厂模式和装饰模式
    1. 是一个可以生产对象和装饰对象的工厂 Bean ，由 Spring 管理
    2. 生产的对象是由 getObject() 方法决定的
4. 注意：它是泛型的，只能固定生产某一类对象，而不像 BeanFactory 那样可以生产多种类型的 Bean
5. **使用场景：**
    1. 在对于某些特殊的 Bean 的处理中，在其进行单独的实例化操作逻辑中，可能我们并不想走 Spring 的那一套逻辑，此时就可以实现 FactoryBean 接口自己控制逻辑
    2. 例如 Netty 实战中 RPC 的实现案例就用到了 FactoryBean[（详见 Netty RPC 实现 方案 2：HTTP + JDK 动态代理 + SpringBoot）](../../Netty/Netty%20%E5%AE%9E%E6%88%98/9%206%20RPC%20%E7%AE%80%E5%8D%95%E5%AE%9E%E7%8E%B0/%E6%96%B9%E6%A1%88%202%EF%BC%9AHTTP%20+%20JDK%20%E5%8A%A8%E6%80%81%E4%BB%A3%E7%90%86%20+%20SpringBoot%2032bde00d5bbb4f8082a6385234a394de.md)
6. 示例：

    ```java
     public class Book {
     
         private String bname;
     
         public void setBname(String bname) {
             this.bname = bname;
         }
     }
    ```

    ```java
     //MyBean是一个FactoryBean的实现类
     public class MyBean implements FactoryBean<Book> {
     
         //getObject()可以定义bean返回的对象类型
         @Override
         public Book getObject() throws Exception {
             Book book = new Book();
             book.setBname("abc");
             return book;
         }
     
         @Override
         public Class<?> getObjectType() {
             return null;
         }
     
         @Override
         public boolean isSingleton() {
             return FactoryBean.super.isSingleton();
         }
     }
    ```

    ```xml
     <bean id="myBean" class="com.factoryBean.MyBean"></bean>
    ```

    ```java
     @Test
     public void testfactorybean(){
         ApplicationContext context = new ClassPathXmlApplicationContext("bean5.xml");
    
         //虽然标签里class定义的是MyBean，应该返回一个MyBean的类型的对象
         //但由于MyBean是一个factoryBean，且重写了MyBean的getObject方法，便可返回一个book类
         Book book = context.getBean("myBean", Book.class);
    
         System.out.println(book);
     }
    ```

    ```java
     //结果：返回了一个book对象
     com.at.spring5.Book@262b2c86
    ```
