### 1️⃣ **根据 request 找到对应处理请求的 mappedHandler**
1. 对应代码：**`mappedHandler = getHandler(processedRequest)`**
1. 返回的 `**mappedHandler**` 是一个 `**HandlerExecutionChain**` 对象[[责任链模式 ⭐]]，其包含三个值：
    
    1. **handler**：浏览器发送的请求所匹配的控制器方法
    
    1. **interceptorList**：处理控制器方法的所有拦截器集合
    
    1. **interceptorIndex**：拦截器索引，控制拦截器的执行
        
        ![[IMG-20260405035420364.png|Untitled 1 350.png]]
        
    
1. 可以看到返回的 `mappedHandler` 精确的定位到了我们的 controller 的方法上
    
    ![[IMG-20260405035422074.png|Untitled 2 286.png]]
    
1. ⭐**那么，是怎样做到 请求信息 与 handler之间的对应的** ❓
    
    1. 首先我们自己写的所有能处理请求的 handler 与 对应请求 的**映射关系**，在加载时就写到了 `**handlerMappings**` 中（在 DispatcherServlet 初始化的过程中会将**映射关系**初始化到`**handlerMappings**` 中[[初始化的过程]][[初始化的过程]][[初始化的过程]][[初始化的过程]]）
    
    1. `getHandler()`中会的到 `handlerMappings` 中遍历所有 `handlerMapping` ，找到包含符合本次请求的 `handler` 的 `handlerMapping` ，并返回这个 `handler` （**即找到能处理当前请求的**`**controller**` **的方法**）
        
        ```Java
         protected HandlerExecutionChain getHandler(HttpServletRequest request) throws Exception {
             if (this.handlerMappings != null) {
        
                 //迭代器遍历handlerMappings找handler
                 Iterator var2 = this.handlerMappings.iterator();
                 while(var2.hasNext()) {
                     HandlerMapping mapping = (HandlerMapping)var2.next();
                     HandlerExecutionChain handler = mapping.getHandler(request);
                     if (handler != null) {
                         return handler;
                     }
                 }
             }
             return null;
         }
        ```
        
    
1. `handlerMappings` 中有以下几种 `handlerMapping`
    
    ![[IMG-20260405035434153.png|Untitled 3 215.png]]
    
1. ⭐ **`handlerMapping` 匹配的规则：**
    
    1. 依次匹配：谁能匹配，就去哪个 handlerMappings 去找 handler
    
    1. 请求进来，挨个尝试所有的 `HandlerMapping` 看是否有请求信息
        
        1. 如果有就找到这个请求对应的 `handler`
        
        1. 如果没有就是下一个 `HandlerMapping`
        
    
    1. **==例如：如果请求是：====`/`==**
        
        1. 首先按顺序去匹配**`RequestMappingHandlerMapping` ，**`RequestMappingHandlerMapping` 中保存了所有`@RequestMapping` 和 `handler`的映射规则
            
            ![[IMG-20260405035445515.png|Untitled 4 165.png]]
            
        
        1. 由于请求是 `/` 访问欢迎页，**不是我们自定义的**`@RequestMapping`
        
        1. 则`RequestMappingHandlerMapping`没法处理，交由第二个 `**WelcomePageHandlerMapping**` 来寻找对应的 `handler`
        
    
    1. 我们需要一些自定义的映射处理，我们也可以自己给容器中放**HandlerMapping**。自定义 **HandlerMapping**
    
### 2️⃣ 找到对应的处理器适配器 **HandlerAdapter**
首先会找到能执行当前处理器的适配器，由适配器来调用正确的处理方法
1. 对应代码：**`HandlerAdapter ha = getHandlerAdapter(mappedHandler.getHandler())`**
1. [[适配器模式 ⭐]]
### 3️⃣ 调用拦截器的 preHandle()
1. 在执行目标业务方法前，首先要对加在当前处理器上的拦截器进行处理（预处理）
1. [[责任链模式 ⭐]]
### 4️⃣ 由处理器适配器调用具体的 handle() 处理器方法
1. 对应代码：**`mv = ha.handle(processedRequest, response, mappedHandler.getHandler());`**
1. 执行完后最终获得了一个 ModelAndView 对象
### 5️⃣ 调用拦截器的 postHandle()
1. 在执行目标业务方法后，还要对加在当前处理器上的拦截器进行处理（后处理）
1. [[责任链模式 ⭐]]
### 6️⃣ 处理模型数据和渲染视图
`**processDispatchResult()**`
```Java
 private void processDispatchResult(HttpServletRequest request, HttpServletResponse response,
                                    @Nullable HandlerExecutionChain mappedHandler, @Nullable ModelAndView mv,
                                    @Nullable Exception exception) throws Exception {
 
     boolean errorView = false;
 
     if (exception != null) {
         if (exception instanceof ModelAndViewDefiningException) {
             logger.debug("ModelAndViewDefiningException encountered", exception);
             mv = ((ModelAndViewDefiningException) exception).getModelAndView();
         }
         else {
             Object handler = (mappedHandler != null ? mappedHandler.getHandler() : null);
             mv = processHandlerException(request, response, handler, exception);
             errorView = (mv != null);
         }
     }
 
     if (mv != null && !mv.wasCleared()) {
         // ⭐处理模型数据和渲染视图
         render(mv, request, response);
         if (errorView) {
             WebUtils.clearErrorRequestAttributes(request);
         }
     
     else {
         if (logger.isTraceEnabled()) {
             logger.trace("No view rendering, null ModelAndView returned.");
         }
     }
 
     if (WebAsyncUtils.getAsyncManager(request).isConcurrentHandlingStarted()) {
         return;
     }
 
     if (mappedHandler != null) {
         // ⭐调用拦截器的 afterCompletion()
         mappedHandler.triggerAfterCompletion(request, response, null);
     }
 }
```
