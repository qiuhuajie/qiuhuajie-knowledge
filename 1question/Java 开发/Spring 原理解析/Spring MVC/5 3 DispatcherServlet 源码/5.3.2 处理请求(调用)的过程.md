3. `service(HttpServletRequest req, HttpServletResponse resp)`

    ```java
     protected void service(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
    
         //从请求中获取 请求方法
         String method = req.getMethod();
         long lastModified;
    
         //判断请求方法，执行相应的方法
         if (method.equals("GET")) {
             lastModified = this.getLastModified(req);
             if (lastModified == -1L) {
                 //执行 doGet
                 this.doGet(req, resp);
             } else {
                ...
             }
         } else if (method.equals("HEAD")) {
             lastModified = this.getLastModified(req);
             this.maybeSetLastModified(resp, lastModified);
             this.doHead(req, resp);
         } else if (method.equals("POST")) {
             this.doPost(req, resp);
         } else if (method.equals("PUT")) {
             this.doPut(req, resp);
         } else if (method.equals("DELETE")) {
             this.doDelete(req, resp);
         } else if (method.equals("OPTIONS")) {
             this.doOptions(req, resp);
         } else if (method.equals("TRACE")) {
             this.doTrace(req, resp);
         } else {
             String errMsg = lStrings.getString("http.method_not_implemented");
             Object[] errArgs = new Object[]{method};
             errMsg = MessageFormat.format(errMsg, errArgs);
             resp.sendError(501, errMsg);
         }
     }
    ```


# 2. **FrameworkServlet**

1. `service()`
    1. `service()`是在 FrameworkServlet.class 中开始调用的，但实际调用的是 HttpServlet.class 的（见下面源码注释）
    2. 因此，上面 HttpServlet.class 的 `service()` 方法代码中，哪些 `this` 关键字调用的 `this.doPost(req, resp)`  、 `doGet`、`doPost`方法实际是 FrameworkServlet.class的

    ```java
     protected void service(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
         HttpMethod httpMethod = HttpMethod.resolve(request.getMethod());
         if (httpMethod != HttpMethod.PATCH && httpMethod != null) {
    
             //**调用父类的 service()，而 其父类 HttpServletBean 并没有对该方法重写，故调用的是在往上的父类 HttpServlet.class 的service()**
             super.service(request, response);
         } else {
             this.processRequest(request, response);
         }
     
     }
    ```

2. 而 **FrameworkServlet.class** 对这些 `doGet()、doPost() ...` 都做了重写

    ```java
     protected final void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
         this.processRequest(request, response);
     }
     
     protected final void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
         this.processRequest(request, response);
     }
     
     protected final void doPut(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
         this.processRequest(request, response);
     }
     
     protected final void doDelete(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
         this.processRequest(request, response);
     }
    // **可以看到都是调用了 processRequest() 方法**
    ```

 protected void doDispatch(HttpServletRequest request, HttpServletResponse response) throws Exception {
     HttpServletRequest processedRequest = request;
     HandlerExecutionChain mappedHandler = null;
     boolean multipartRequestParsed = false;
 
     WebAsyncManager asyncManager = WebAsyncUtils.getAsyncManager(request);
 
     try {
         ModelAndView mv = null;
         Exception dispatchException = null;
 
         try {
             processedRequest = checkMultipart(request);
             multipartRequestParsed = (processedRequest != request);
 
             // ⭐**根据 request 找到对应处理请求的 mappedHandler**
             mappedHandler = getHandler(processedRequest);
             if (mappedHandler == null) {
                 noHandlerFound(processedRequest, response);
                 return;
             }
 
             // ⭐通过控制器方法创建相应的处理器适配器，调用所对应的控制器方法
             HandlerAdapter ha = getHandlerAdapter(mappedHandler.getHandler());
 
             ...

             // ⭐调用拦截器的 preHandle()
             if (!mappedHandler.applyPreHandle(processedRequest, response)) {
                 return;
             }
 
             // ⭐由处理器适配器调用具体的handle()控制器方法，最终获得ModelAndView对象
             mv = ha.handle(processedRequest, response, mappedHandler.getHandler());
 
             if (asyncManager.isConcurrentHandlingStarted()) {
                 return;
             }
 
             applyDefaultViewName(processedRequest, mv);

             // ⭐调用拦截器的postHandle()
             mappedHandler.applyPostHandle(processedRequest, response, mv);
         }
         catch (Exception ex) {
             dispatchException = ex;
         }
         catch (Throwable err) {

             dispatchException = new NestedServletException("Handler dispatch failed", err);
         }

         //⭐ 后续处理：处理模型数据和渲染视图

### 1️⃣ **根据 Request 找到对应处理请求的 mappedHandler**

1. 对应代码：**`mappedHandler = getHandler(processedRequest)`**
2. 返回的 **`mappedHandler`** 是一个 **`HandlerExecutionChain`** 对象[（详见 设计模式 职责链模式）](../../../%E8%AE%BE%E8%AE%A1%E6%A8%A1%E5%BC%8F/%E8%A1%8C%E4%B8%BA%E5%9E%8B%E6%A8%A1%E5%BC%8F/%E8%B4%A3%E4%BB%BB%E9%93%BE%E6%A8%A1%E5%BC%8F%20%E2%AD%90%203512b3bd8cfe4a49a9935baee9352faf.md)，其包含三个值：
    1. **handler**：浏览器发送的请求所匹配的控制器方法
    2. **interceptorList**：处理控制器方法的所有拦截器集合
    3. **interceptorIndex**：拦截器索引，控制拦截器的执行

        ![Untitled](IMG-20260620231111591.png)

3. 可以看到返回的 `mappedHandler` 精确的定位到了我们的 controller 的方法上

    ![Untitled](IMG-20260620231111624.png)

4. ⭐**那么，是怎样做到 请求信息 与 handler之间的对应的** ❓
    1. 首先我们自己写的所有能处理请求的 handler 与 对应请求 的**映射关系**，在加载时就写到了 **`handlerMappings`** 中（在 DispatcherServlet 初始化的过程中会将**映射关系**初始化到 **`handlerMappings`** 中[（详见 SpringBoot—初始化过程 **`DispatcherServlet.class:initStrategies`**）](%E5%88%9D%E5%A7%8B%E5%8C%96%E7%9A%84%E8%BF%87%E7%A8%8B%201bfe90a2ee914006affdc50111a0caff.md)）
    2. `getHandler()`中会的到 `handlerMappings` 中遍历所有 `handlerMapping` ，找到包含符合本次请求的 `handler` 的 `handlerMapping` ，并返回这个 `handler`  （**即找到能处理当前请求的`controller` 的方法**）

        ```java
         protected HandlerExecutionChain getHandler(HttpServletRequest request) throws Exception {
             if (this.handlerMappings != null) {
        
                 //迭代器遍历handlerMappings找handler
                 Iterator var2 = this.handlerMappings.iterator();
                 while(var2.hasNext()) {
                     HandlerMapping mapping = (HandlerMapping)var2.next();
                     HandlerExecutionChain handler = mapping.getHandler(request);
                     if (handler != null) {
                         return handler;
                     }
                 }
             }
             return null;
         }
        ```

5. `handlerMappings` 中有以下几种 `handlerMapping`

    ![Untitled](IMG-20260620231111656.png)

6. ⭐ **`handlerMapping`  匹配的规则：**
    1. 依次匹配：谁能匹配，就去哪个 handlerMappings 去找 handler
    2. 请求进来，挨个尝试所有的 `HandlerMapping` 看是否有请求信息
        1. 如果有就找到这个请求对应的 `handler`
        2. 如果没有就是下一个 `HandlerMapping`
    3. **例如：如果请求是：`/`**
        1. 首先按顺序去匹配**`RequestMappingHandlerMapping`  ，**`RequestMappingHandlerMapping` ****中保存了所有`@RequestMapping` 和 `handler`的映射规则

            ![Untitled](IMG-20260620231111682.png)

        2. 由于请求是 `/` 访问欢迎页，**不是我们自定义的**`@RequestMapping`
        3. 则`RequestMappingHandlerMapping`没法处理，交由第二个 **`WelcomePageHandlerMapping`** 来寻找对应的 `handler`
    4. 我们需要一些自定义的映射处理，我们也可以自己给容器中放**HandlerMapping**。自定义 **HandlerMapping**

1. 对应代码：**`HandlerAdapter ha = getHandlerAdapter(mappedHandler.getHandler())`**
2. [**详见：设计模式 适配器模式在 SpringMVC 框架应用的源码分析**](../../../%E8%AE%BE%E8%AE%A1%E6%A8%A1%E5%BC%8F/%E7%BB%93%E6%9E%84%E5%9E%8B%E6%A8%A1%E5%BC%8F/%E9%80%82%E9%85%8D%E5%99%A8%E6%A8%A1%E5%BC%8F%20%E2%AD%90%202b037a9aa2cd47c59d02cdc4054ad619.md)

### 3️⃣ 调用拦截器的 preHandle()

1. 在执行目标业务方法前，首先要对加在当前处理器上的拦截器进行处理（预处理）
2. [详见 职责链模式在 SpringMVC 框架应用的源码分析](../../../%E8%AE%BE%E8%AE%A1%E6%A8%A1%E5%BC%8F/%E8%A1%8C%E4%B8%BA%E5%9E%8B%E6%A8%A1%E5%BC%8F/%E8%B4%A3%E4%BB%BB%E9%93%BE%E6%A8%A1%E5%BC%8F%20%E2%AD%90%203512b3bd8cfe4a49a9935baee9352faf.md)

### 4️⃣ 由处理器适配器调用具体的 handle() 处理器方法

### 6️⃣ 处理模型数据和渲染视图

**`processDispatchResult()`**

```java
 private void processDispatchResult(HttpServletRequest request, HttpServletResponse response,
                                    @Nullable HandlerExecutionChain mappedHandler, @Nullable ModelAndView mv,
                                    @Nullable Exception exception) throws Exception {
 
     boolean errorView = false;
 
     if (exception != null) {
         if (exception instanceof ModelAndViewDefiningException) {
             logger.debug("ModelAndViewDefiningException encountered", exception);
             mv = ((ModelAndViewDefiningException) exception).getModelAndView();
         }
         else {
             Object handler = (mappedHandler != null ? mappedHandler.getHandler() : null);
             mv = processHandlerException(request, response, handler, exception);
             errorView = (mv != null);
         }
     }
 
     if (mv != null && !mv.wasCleared()) {

         // ⭐处理模型数据和渲染视图
         render(mv, request, response);
         if (errorView) {
             WebUtils.clearErrorRequestAttributes(request);
         }
     
     else {
         if (logger.isTraceEnabled()) {
             logger.trace("No view rendering, null ModelAndView returned.");
         }
     }
 
     if (WebAsyncUtils.getAsyncManager(request).isConcurrentHandlingStarted()) {
         return;
     }
 
     if (mappedHandler != null) {

         // ⭐**调用拦截器的 afterCompletion()**
         mappedHandler.triggerAfterCompletion(request, response, null);
     }
 }
```
