# 1. MVC

1. MVC是模型(Model)、视图(View)、控制器(Controller)的简写，是一种软件设计规范
2. MVC 的目的：
    1. 将业务逻辑、数据、显示分离的方法来组织代码
    2. 降低视图与业务逻辑间的双向偶合
3. MVC不是一种设计模式，MVC是一种架构模式
    1. **Model（模型）：**
        1. 数据模型，提供要展示的数据，因此包含数据和行为
        2. 可以认为是领域模型或JavaBean组件（包含数据和行为）
        3. JavaBean分为两类：
            1. 一类称为**实体类Bean**：专门存储业务数据的，如 Student、User 等
            2. 一类称为**业务处理 Bean**：指 Service 或 Dao 对象，专门用于处理业务逻辑和数据访问
    2. **View（视图）：**
        1. 负责进行模型的展示，一般就是我们见到的用户界面，客户想看到的东西
        2. 作用是与用户进行交互，展示数据
    3. **Controller（控制器）：**
        1. 接收用户请求，委托给模型进行处理，处理完毕后把返回的模型数据返回给视图，由视图负责展示
        2. 也就是说控制器做了个调度员的工作，作用就是是接收请求和响应浏览器
4. **MVC 的工作流程：**

    ![Untitled](IMG-20260620224902243.png)

    1. 用户通过视图层发送请求到服务器，在服务器中请求被Controller接收
    2. Controller调用相应的Model层处理请求，处理完毕将结果返回到Controller
    3. Controller再根据请求处理的结果找到相应的View视图，渲染数据后最终响应给浏览器

# 2. **SpringMVC**

1. 官方文档：

    [Web on Servlet Stack](https://docs.spring.io/spring-framework/docs/current/reference/html/web.html#spring-web)

2. SpringMVC是Spring的一个后续产品，是Spring的一个子项目
3. SpringMVC 是 Spring 为表述层开发提供的一整套完备的解决方案。在表述层框架历经 Strust、WebWork、Strust2 等诸多产品的历代更迭之后，目前业界普遍选择了 SpringMVC 作为 Java EE 项目表述层开发的**首选方案**
4. **SpringMVC 的特点：**
    - **Spring 家族原生产品**，与 IOC 容器等基础设施无缝对接
    - **基于原生的Servlet**，通过了功能强大的**前端控制器 DispatcherServlet**，对请求和响应进行统一处理
    - 表述层各细分领域需要解决的问题**全方位覆盖**，提供**全面解决方案**
    - **代码清新简洁**，大幅度提升开发效率
    - 内部组件化程度高，可插拔式组件**即插即用**，想要什么功能配置相应组件即可
    - **性能卓著**，尤其适合现代大型、超大型互联网项目要求
