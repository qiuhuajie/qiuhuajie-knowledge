# 1. 传播类型介绍

1. 事务传播类型（PROPAGATION 传播）
    1. 指的是事务与事务之间的交互策略
    2. 举例说明：**在事务方法 A 中调用事务方法 B，当事务方法 B 失败回滚时，事务方法 A 应该如何操作？这就是事务传播类型**
2. Spring 事务中定义了 7 种事务传播类型，分别是
    - **REQUIRED（必须的）**
    - SUPPORTS
    - MANDATORY
    - **REQUIRES_NEW**
    - NOT_SUPPORTED
    - NEVER
    - **NESTED（嵌套的）**
3. **其中最常用的只有 3 种，即：REQUIRED、REQUIRES_NEW、NESTED**
4. 针对事务传播类型，我们要弄明白的是 4 个点：
    1. **子事务与父事务的关系，是否会启动一个新的事务？**
    2. **子事务异常时，父事务是否会回滚？**
    3. **父事务异常时，子事务是否会回滚？**
    4. **父事务捕捉异常后，父事务是否还会回滚？**

# 2. 常用的 3 种传播类型

## 2.1 PROPAGATION_REQUIRED

1. REQUIRED 是 Spring 默认的事务传播类型
2. 该传播类型的特点是：
    1. **当前方法存在事务时，子方法加入该事务**
        1. **此时父子方法共用一个事务，无论父子方法哪个发生异常回滚，整个事务都回滚**
        2. **即使父方法捕捉了异常，也是会回滚**
    2. **而当前方法不存在事务时，子方法新建一个事务**

## 2.2 PROPAGATION_REQUIRES_NEW

1. REQUIRES_NEW 也是常用的一个传播类型

1. 该传播类型的特点是：
    1. **无论当前方法是否存在事务，子方法都新建一个事务**
    2. **此时父子方法的事务时独立的，它们都不会相互影响**
    3. **但父方法需要注意子方法抛出的异常，避免因子方法抛出异常，而导致父方法回滚**

## 2.3 PROPAGATION_**NESTED**

1. NESTED 也是常用的一个传播类型
2. 该方法的特性与 REQUIRED 非常相似，其特性是：
    1. **当前方法存在事务时，子方法加入在嵌套事务执行**
    2. **当父方法事务回滚时，子方法事务也跟着回滚**
    3. **当子方法事务发生回滚时，父事务是否回滚取决于是否捕捉了异常。如果捕捉了异常，那么就不回滚，否则回滚**
3. 可以看到 NESTED 与 REQUIRED 的区别在于：父方法与子方法对于共用事务的描述是不一样的
    - REQUIRED 说的是**共用同一个事务**：无论父子方法哪个发生异常，全都会回滚
    - 而 NESTED 说的是在**嵌套事务执行**：父方法发生异常回滚时，子方法事务会回滚。而子方法事务发送回滚时，父事务是否回滚取决于是否捕捉了异常
