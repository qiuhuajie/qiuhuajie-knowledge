# 1. 业务背景

1. 业务代码中有需要在事务中发送 MQ 的情况

    ![Untitled](IMG-20260620231106020.png)

2. 但是，在业务上是本地事务提交成功了，才希望去发这个 MQ
3. 所以这个代码是有问题的：比如说本地事务回滚了，但是中间件的消息已经发出去了，但这个 MQ 消息没法撤回，这里本地事务和发送消息就没有保证原子性
4. 处理方案：把发送 MQ 消息放在本地事务执行之后

    ![Untitled](IMG-20260620231106035.png)

5. 那么，**如何保证在本地事务执行成功之后，再做回调操作（发送 MQ）**❓

# 2. TransactionSynchronization 接口

1. Spring 提供了扩展：`TransactionSynchronization` 接口

    ![Untitled](IMG-20260620231106060.png)

2. 这是一个事务同步回调的接口，同时带有 order 能力，可以对多个回调操作的顺序进行排序
3. 也提供了事务执行状态的几个常量
4. 因此，**可以对 `TransactionSynchronization` 接口做扩展，完成在事务结束时，调用设置的回调方法，执行 MQ 发送**

# 3. 代码实现

1. `DoTransactionCompletion` 先编写事务扩展点

    ```java
    public class DoTransactionCompletion implements TransactionSynchronization {
    
        private final Runnable runnable;
    
        public DoTransactionCompletion(Runnable runnable) {
            // 回调操作由构造方法传入
            this.runnable = runnable;
        }
        @Override
        public void afterCompletion(int status) {
            // ⭐**只有当当前事务成功提交之后采取做处理**
            if (status == TransactionSynchronization.STATUS_COMMITTED) {
                // **定义回调函数，即本地事务完成后要做什么，这里⭐️用线程 Runnable 完成回调操作**
                this.runnable.run();
            }
        }
    }
    ```

2. `TransactionUtils`

    ```java
    public class TransactionUtils {
        public static void doAfterTransaction(DoTransactionCompletion doTransactionCompletion) {
    
            // **判断当前的上下文中是否有事务被激活**
            // TransactionSynchronizationManager 的静态方法 isActualTransactionActive() 可以判断当前是否有事务被激活
            if (TransactionSynchronizationManager.isActualTransactionActive()) {
                // 如果有事务激活，就把这个回调函数的接口的实现，注册到当前事务的同步器中
                TransactionSynchronizationManager.registerSynchronization(doTransactionCompletion);
            }
        }
    }
    ```

3. `AccountServiceImpl` 使用扩展点完成需求

    ```java
    @Service
    public class AccountServiceImpl implements AccountService {
    
        @Autowired
        AccountMapper accountMapper;
    
        @Override
        @Transactional
        public Account getAccount() {
            // **auto start tx**
    
            TransactionUtils.doAfterTransaction(new DoTransactionCompletion(new Runnable() {
                @Override
                public void run() {
                    // **send MQ or RPC，等事务结束后，执行该回调函数**
                }
            }));
    
            return accountMapper.getAccount();
    
            // **auto stop tx**
        }
    }
    ```


> 💡 ⚠️ 注意：这个并不是分布式事务的解决方案
> ![Untitled](IMG-20260620231106077.png)
