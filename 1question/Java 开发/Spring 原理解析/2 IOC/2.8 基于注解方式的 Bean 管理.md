# 1. **创建对象**
1. spring 针对 bean 管理中 创建对象提供的注解：
    
    1. `@Component`
    
    1. `@Service`：Service层
    
    1. `@Controller`：Web层
    
    1. `@Repository`：Dao层
    
    > [!important] 四个注解功能是一样的，只是为了开发清晰规范，在不同层使用不同的注解（混用也行）
    
1. 示例：
    
    1. 除了基本jar包外，还需要引入**aop依赖**
        
        ```Java
         spring-aop-5.2.6.RELEASE.jar
        ```
        
    
    1. 开启**组件扫描**
        
        ```XML
         <?xml version="1.0" encoding="UTF-8"?>
         
         <!--需要添加 context 名称空间-->
         <beans xmlns="http://www.springframework.org/schema/beans"xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
                xmlns:p="http://www.springframework.org/schema/p"
                xmlns:util="http://www.springframework.org/schema/util"
                xmlns:context="http://www.springframework.org/schema/context"
                xsi:schemaLocation="http://www.springframework.org/schema/beans
                                    http://www.springframework.org/schema/beans/spring-beans.xsd
                                    http://www.springframework.org/schema/util
                                    http://www.springframework.org/schema/util/spring-util.xsd
                                    http://www.springframework.org/schema/context
                                    http://www.springframework.org/schema/context/spring-context.xsd">
             <!--若有多个包的类需要扫描，则用逗号隔开，或者直接使用他们的公共上层目录-->
             <context:component-scan base-package="com.Service,com.Dao"></context:component-scan>
         
         </beans>
        ```
        
        > [!important] 开启组件扫描细节配置
        > 
        > 1. **`context:component-scan`**
        >     
        >     - 属性
        >         
        >         - `use-default-filters` ：是否使用默认的 fliter
        >         
        >     
        >     - 子标签
        >         
        >         - `context:include-filter`子标签：设置扫描哪些内容
        >         
        >         - `context:exclude-filter` 子标签：设置不扫描哪些内容
        >         
        >     
        > 
        > 1. 示例：
        >     
        >     ```XML
        >      <context:component-scan base-package="com.atguigu" use-defaultfilters="false">
        >              <context:include-filter type="annotation" expression="org.springframework.stereotype.Controller"/>
        >      </context:component-scan>
        >     ```
        >     
        >     ```XML
        >      <context:component-scan base-package="com.atguigu">
        >              <context:exclude-filter type="annotation" expression="org.springframework.stereotype.Controller"/>
        >      </context:component-scan>
        >     ```
        >     
        
    
    1. 创建类UserService，在类上面添加创建对象注解
        
        ```Java
         @Service(value = "userService")
         public class UserService {
             public void add(){
                 System.out.println("add...");
             }
         }
         //⭐
         //@Service(value = "userService")
         //     等价于之前xml配置文件中的
         //<bean id="userService" class="com.Service.UserService"></bean>
         
         //🟡注解所在的位置替代了class=""
         //🟡注解的value值替代了id=""  (value值也可以省略，会默认使用类名的首字母小写)
        ```
        
        ```Java
         //测试
         @Test
         public void test2(){
             ApplicationContext context = new ClassPathXmlApplicationContext("bean5.xml");
             UserService userService = context.getBean("userService",UserService.class);
             userService.add();//add...
         }
        ```
        
    
# 2. **DI 依赖注入**
## 2.1 **注入普通属性 @Value**
1. `**@Value**`：注入普通类型属性
1. 示例：
    
    ```Java
     @Service
     public class UserService {
     
         @Value(value = "abc")
         private String name;
     
         public void printName(){
             System.out.println(name);
         }
     }
    ```
    
    ```Java
     @Test
     public void test2(){
         ApplicationContext context = new ClassPathXmlApplicationContext("bean5.xml");
         UserService userService = context.getBean("userService",UserService.class);
         userService.printName();
         //abc
     }
    ```

## 2.2 **注入对象属性**
### 1️⃣**@Autowired**
1. 介绍
    
    1. **`@Autowird` 默认的注入方式为 byType，也就是根据类型匹配，当有多个实现时，则通过byName注入**
    
    1. 也可以通过配合 `@Qualifier` 注解来显式指定 name 值，指明要使用哪个具体的实现类
    
    1. 注意：`@Autowired`注解注入的对象需要在IOC容器中存在，否则需要加上属性`required=false`，表示忽略当前要注入的bean，如果有直接注入，没有跳过，不会报错
    
1. 示例
    
    1. 首先有一个接口UserService和两个实现类 `UserServiceImpl1` 和 `UserServiceImpl2`，并且这两个实现类已经加入到Spring的IOC容器中了
        
        ```Java
        @Service
        public class UserServiceImpl1 implements UserService
        
        @Service
        public class UserServiceImpl2 implements UserService
        ```
        
    
    1. 通过 `@Autowired` 注入使用
        
        ```Java
        @Autowired
        private UserService userService;
        ```
        
    
    1. 直接这么使用是会报错的，原因：
        
        1. 首先通过byType注入，判断`UserService`类型有两个实现，无法确定具体是哪一个
        
        1. 于是通过byName方式，这里的变量名userService也无法匹配IOC容器中id（此处指的`userServiceImpl1`和`userServiceImpl2`），于是报错
            
            ```Java
            // 报错信息
            Consider marking one of the beans as @Primary, updating the consumer to accept multiple beans, or using @Qualifier to identify the bean that should be consumed
            ```
            
            > [!important]
            > 
            > 注意：通过注解注入到IOC容器的id值默认是其类名首字母小写
            
        
    
    1. **解决方案：**
        
        1. 方式一：改变变量名
            
            ```Java
            @Autowired
            private UserService userServiceImpl1;
            ```
            
        
        1. 方式二：配合`@Qualifier`注解来显式指定name值
            
            ```Java
            @Autowired
            @Qualifier(value = "userServiceImpl1")
            private UserService userService;
            ```
            
        
    
### 2️⃣**@Resource**
1. 介绍：
    
    1. `@Resource` 默认通过byName注入，如果没有匹配则通过byType注入
    
    1. `@Resource` **并不是 spring 提供的，而是 Java 的扩展包（**==**spring官方并不推荐使用@Resource**==**）**
        
        ```Java
         import org.springframework.beans.factory.annotation.Autowired;
         import org.springframework.beans.factory.annotation.Qualifier;
         
         import javax.annotation.Resource;
        ```
        
    
      
    
1. 示例
    
    1. 首先有一个接口UserService和两个实现类 `UserServiceImpl1` 和 `UserServiceImpl2`，并且这两个实现类已经加入到Spring的IOC容器中了
        
        ```Java
        @Service
        public class UserServiceImpl1 implements UserService
        
        @Service
        public class UserServiceImpl2 implements UserService
        ```
        
    
    1. 通过 `@Resource` 注入使用
        
        ```Java
        @Resourc
        private UserService userService;
        ```
        
    
    1. 直接这么使用是会报错的，原因：
        
        1. 首先通过byName匹配，变量名userService无法匹配IOC容器中任何一个id（这里指的`userServiceImpl1`和`userServiceImpl2`）
        
        1. 于是通过byType匹配，发现类型UserService的实现类有两个，仍然无法确定，于是报错
            
            ```Java
            // 报错信息
            No qualifying bean of type 'com.example.ormdemo.service.UserService' available: expected single matching bean but found 2: userServiceImpl1,userServiceImpl2
            ```
            
        
    
1. `@Resource` 还有两个重要的属性：`**name**`**和**`**type**`**，用来显式指定byName和byType方式注入**
    
    ```Java
    // 1. 既没指定name属性，也没指定type属性：默认通过byName方式注入，如果byName匹配失败，则使用byType方式注入（也就是上面的那个例子）
    @Resource  
    private UserService userDao; 
    
    // 2. 指定name属性：通过byName方式注入，把变量名和IOC容器中的id去匹配，匹配失败则报错
    @Resource(name="userService")  
    private UserService userService; 
    
    // 3. 指定type属性：通过byType方式注入，在IOC容器中匹配对应的类型，如果匹配不到或者匹配到多个则报错
    @Resource(type=UserService.class)  
    private UserService userService; 
    
    // 4. 同时指定name属性和type属性： 在IOC容器中匹配，名字和类型同时匹配则成功，否则失败
    @Resource(name="userService",type=UserService.class)  
    private UserService userService;
    ```
    
# 3. **完全注解开发**
1. 上边的基于注解的Bean管理，仍需要配置xml文件
    
    ```XML
     <!--开启组件扫描-->
     <context:component-scan base-package="com.Service,com.Dao"></context:component-scan>
    ```
    
1. 然而还能实现完全不需要配置文件的注解开发
    
    **创建配置类，使用 @Configuration @ComponentScan 来替代 xml 配置文件**
    
    ```Java
     @Configuration
     @ComponentScan(basePackages = "com")
     public class SpringConfig {
     }
    ```
    
    ```Java
     @Test
     public void test3(){
         //这里创建context的方式会有不同
         ApplicationContext context = new AnnotationConfigApplicationContext(SpringConfig.class);
         UserService userService = context.getBean("userService",UserService.class);
         userService.add();
         //Service add...
         //Dao update...
     }
    ```
