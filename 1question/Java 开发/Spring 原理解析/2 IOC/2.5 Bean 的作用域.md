# 1. **Bean 的作用域**
1. Spring 框架作为一个管理 Bean 的 IoC 容器，那么 Bean 自然是 Spring 中的重要资源了
1. Bean 的作用域是指 **Bean 在 Spring 整个框架中的某种行为模式**
1. 比如 singleton 单例作用域，（默认作用域为单例）就表示 Bean 在整个 Spring 中只有一份，它是全局共享的，当有人修改了这个值之后，那么另一个人读取到的就是被修改后的值
# 2. **作用域的分类**
在 Spring 中，Bean 的常见作用域有以下 5 种：
- singleton：单例作用域
- prototype：原型作用域（多例作用域）
- request：请求作用域
- session：会话作用域
- application：全局作用域

> [!important] 注意：后 3 种作用域，只适用于 Spring MVC 框架
## **2.1 singleton**
1. 官方说明：(Default) Scopes a single bean definition to a single object instance for each Spring IoC container.
1. 描述：
    
    1. 该作用域下的 Bean **在 IoC 容器中只存在一个实例**
    
    1. 获取 Bean（即通过 applicationContext.getBean等方法获取）及装配 Bean（即通过 @Autowired 注入）都是同一个对象
    
    1. 加载 spring 配置文件时候就会创建单实例对象
    
1. 场景：
    
    1. **通常无状态的 Bean 使用该作用域**
    
    1. 无状态表示 Bean 对象的属性状态不需要更新
    
1. 备注：**Spring 默认选择该作用域**
## **2.2 prototype**
1. 官方说明：Scopes a single bean definition to any number of object instances.
1. 描述：
    
    1. **每次对该作用域下的 Bean 的请求都会创建新的实例**
    
    1. 获取 Bean（即通过 applicationContext.getBean 等方法获取）及装配 Bean（即通过 @Autowired 注入）都是新的对象实例
    
    1. 不是在加载 spring 配置文件时候创建对象，而是在调用 getBean 方法时候创建多实例对象
    
1. 场景：**通常有状态的 Bean 使用该作用域**
## **2.3 request**
- 官方说明：Scopes a single bean definition to the lifecycle of a single HTTP request. That is, each HTTP request has its own instance of a bean created off the back of a single bean definition. Only valid in the context of a web-aware Spring ApplicationContext.
- 描述：**每次 Http 请求会创建新的 Bean 实例**，类似于 prototype
- 场景：**一次 Http 的请求和响应的共享 Bean**
- 备注：限定 Spring MVC 框架中使用
## **2.4 session**
- 官方说明：Scopes a single bean definition to the lifecycle of an HTTP Session. Only valid in the context of a web-aware Spring ApplicationContext.
- 描述：**在一个 Http Session 中，定义一个 Bean 实例**
- 场景：**用户会话的共享 Bean，比如：记录一个用户的登陆信息**
- 备注：限定 Spring MVC 框架中使用
## **2.5 application**
- 官方说明：Scopes a single bean definition to the lifecycle of a ServletContext. Only valid in the context of a web-aware Spring ApplicationContext.
- 描述：**在一个 Http Servlet Context 中，定义一个 Bean 实例**
- 场景：**Web 应用的上下文信息，比如：记录一个应用的共享信息**
- 备注：限定 Spring MVC 框架中使用
# 3. **设置作用域**
1. 可以通过 `@Scope`  注解来设置 Bean 的作用域，它的设置方式有以下两种：
    
    1. 方式一：直接设置作用域的具体值，如：`@Scope("prototype")`
    
    1. 方式二：设置 ConfigurableBeanFactory 和 WebApplicationContext 提供的 `SCOPE_XXX` 变量，如`@Scope(ConfigurableBeanFactory.SCOPE_PROTOTYPE)`
