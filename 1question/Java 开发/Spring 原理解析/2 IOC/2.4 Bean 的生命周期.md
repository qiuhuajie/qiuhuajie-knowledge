## 创建 Bean（实例化）
1. **通过 `loadBeanDefinitions()` 扫描所有xml配置、注解，将代码中**==**定义的所有 Bean 类一个个都找到**==**，并且放在容器内的一个 `beanDefinitionMap` 中**
1. **接着遍历这个 map 集合，通过 `createBean()` 来创建一个个 Bean 对象**
    
    1. 先用反射机制从 `beanDefinitionMap` 中的 BeanClass 拿到这个类的==构造方法==
        
        - 在拿取构造方法时，是有一些规则的
            
            1. 如果这个 Bean 只有一个构造方法，无论这个构造方法有没有入参，我们只能拿它
            
            1. 当 Bean 中有多个构造方法时，会优先拿带有 @Autowired 注解的构造方法
                
                1. 如果有多个构造方法都加了 @Autowired 注解，则会报错
                
                1. 如果都没有 @Autowired 注解，则会优先拿没有入参的构造方法
                
                1. 如果都没有 @Autowired 注解，且没有无入参的构造方法，则会报错
                
                > [!important] 不建议在 Bean 中添加多个构造方法，这样可读性低，理解和维护会很困难
                
            
        
    
    1. 选择了确定的构造方法后，就要准备==构造方法需要的参数==了
        
        1. 首先会在容器内的单例池中，根据参数的 Class 类进行查找
        
        1. 如果这个类在容器体内有多个实例，则会根据参数名再一步进行匹配
        
        1. 如果没有找到，则会任务构造信息导致不完整，而直接报错
        
    
    1. 在参数准备好后，就可以==通过反射进行 Bean 的构造==了
    
    1. 如果开发人员提供了 `Supplier` 或者 `工厂方法`，Spring 也会直接使用开发者提供的创建方式
    
## **merged BeanDefinition**
1. 本阶段是 Spring 提供的一个拓展点，通过 `MergedBeanDefinitionPostProcessor` 类型的后置处理器，可以**对bean对应的 `BeanDefinition` 进行修改**
1. Spring自身也充分利用该拓展点，做了很多初始化操作（并没有修改BeanDefinition）
    
    1. 比如查找标注了`@Autowired`、 `@Resource`、`@PostConstruct`、`@PreDestory` 的属性和方法，方便后续进行属性注入和初始化回调
    
    1. 当然，我们也可以自定义实现，用来修改BeanDefinition信息或者我们需要的初始化操作
    
## **暴露工厂对象**
1. 本阶段主要是**将早期 bean 对象提前放入到三级缓存 `singletonFactories` 中，为循环依赖做支持**
1. 在后续进行属性填充时，如果发生循环依赖，可以从三级缓存中通过`getObject()`获取该bean，完成循环依赖场景下的自动注入
## 属性填充
1. 在 Bean 对象实例化完成后，为 Bean 设置相关属性和依赖
1. 通过 `populateBean()` 方法为 Bean 内部所需的属性进行赋值填充，通常是 @Autowired、@Resource 注解的变量
1. 容器体内会通过三级缓存机制[[2.7 Spring 解决循环依赖]]进行填充，也就是所谓的依赖注入
## 初始化

> [!important] 注意：
> 
> **实例化 和 初始化的区别**
> 
> 1. 实例化是在 jvm 的堆中创建了这个对象实例，此时它只是一个空的对象，所有的属性为 null
> 
> 1. 初始化的过程是将对象依赖的一些属性进行赋值之后，调用某些方法来开启一些默认加载
> 
> 1. 比如 Spring 中配置的数据库属性Bean，在初始化的时候就会将这些属性填充，比如 driver、jdbcurl 等，然后初始化连接
1. **在对 Bean 的属性填充后，紧接着要通过 `initializeBean()` ，来对这个 Bean 实例进行初始化了**
    
    1. **Step 1️⃣：初始化容器的相关信息**
        
        1. 通过 `invokeAwareMethods()` 方法，==为实现了各种 Aware 接口（信息感知接口）的 Bean 设置诸如 beanName、beanFactory 等容器信息==
        
        1. Aware 接口代表信息感知接口，一旦实现了这些接口，就可以在 Bean 实例中感知并获取到对应的信息，非常方便
        
    
    1. **Step 2️⃣：初始化方法的执行**
        
        1. 通过 `invokeInitMethods()` 方法执行 ==Bean 的初始化方法== **==`afterPropertiesSet`==**==`()`== ，这个方法通过实现 InitializingBean 接口而实现[[yuwell 数据库表同步]]
        
        1. 接着，还会继续执行代码中在 ==Bean 上自定义的== **==`InitMethods`==**==`()`== ==方法==
        
    
    1. **Step 3️⃣：处理后置处理器 BeanPostProcessors**
        
        1. 在执行 `afterPropertiesSet()` 方法和 `InitMethods()` 方法==前后==，还需要==对 Bean 的后置处理器 BeanPostProcessors 进行处理==
        
        1. 通过 applyBeanPostProcessors**BeforeInitialization** 和 applyBeanPostProcessors**AfterInitialization**，分别在 `InitMethods()` 方法之前和之后处理 Bean 的各种后置处理器
        
        1. 这些处理器包括：
            
            1. 容器自身提供的负责 ==AOP== 处理的 AnnotationAwareAspectJAutoProxyCreator
            
            1. 负责构造后 ==@PostConstruct== 和 销毁前 ==@PreDestroy== 处理的 InitDestoryAnnotationBeanPostProcessor 等系统级处理器
            
            1. 以及程序员通过实现 BeanPostProcessors 接口的==自定义==处理器
            
        
        1. 可以通过 PriorityOrdered 接口来指定这些处理器的执行顺序，进行逐一处理
        
    
    1. Step 4️⃣：注册销毁
        
        1. 为了让 Bean 实例优雅地销毁，容器通过 `reigsterDisposableBean()` 方法，将实现了 DisposableBean 接口的 Bean 进行注册
        
        1. 是一个回调接口，==当需要销毁时，就会回来执行这个销毁方法== ==`destroy()`==
        
    
1. **上面步骤 完成时，Bean 实例就以及可以使用了**
## 使用
1. 在 Bean 实例创建、初始化完成后，会将这些完整的 Bean 实例对象通过 **`addSingleton()`** 方法放入**单例池 `singleObjects`** 中，就可以被获取和使用
1. ==🙎‍♂️====**问什么 Spring 要使用单例模式**====❓==
    
    1. 源码：可以看到在加载 Bean 之前要先去单例池中找，如果没有才使用 `singletonFactory` 创建一个新的 Bean
        
        ![[IMG-20260405035420221.png|Untitled 2 199.png]]
        
    
    1. ==使用单例模式的好处：==
        
        1. 单例模式**避免了频繁创建和销毁对象的开销**，从而提高了应用程序的性能
        
        1. 可以**快速获取到 bean**，因为单例的获取 bean 操作除了第一次生成之外其余的都是从缓存里获取的所以很快
        
        1. **对象状态的一致性**：如果一个 Bean 在不同的地方使用，那么使用单例模式可以保证它们使用的是同一个实例，从而避免了状态不一致的问题
        
        1. **减少 jvm GC 垃圾回收**，由于不会给每个请求都新生成bean实例，所以自然回收的对象少了
        
    
    1. ==单例的 bean 的劣势==
        
        1. **线程不安全**！由于所有请求都共享一个 bean 实例，所以这个 bean 要是有状态的一个 bean 的话可能在并发场景下出现问题
        
        1. 而原型的bean则不会有这样问题，因为给每个请求都新创建实例
        
    
1. 接下来程序员即可在容器中任意地获取使用这些 Bean 实例对象了
## 销毁
1. 当容器执行 `close()` 方法时，会将容器内的 Bean 实例销毁
1. 和产生 Bean 实例类似，要先执行销毁前处理器 `postProcessBeforeDestruction`，这里就会执行 Bean 实例中 @PreDestory 注解的方法
1. 然后通过 **`destoryBeans()`** 方法逐一销毁容器体内所有的 Bean 实例，销毁时会执行 **`destroy()`** 方法
1. 在 `destroy()` 方法执行之后，会通过 **`invokeCustomDestoryMethod()`** 来执行程序员在 Bean 实例上**自定义的 `destoryMethod()`** 方法
# 2. 代码示例
1. `TestBean`
    
    ```Java
    public class TestBean implements InitializingBean, BeanFactoryAware, BeanNameAware, DisposableBean {
    
        private String name;
    
        public TestBean() {
            System.out.println("1.调用构造方法：我出生了！");
        }
    
        public String getName() {
            return name;
        }
    
        public void setName(String name) {
            this.name = name;
            System.out.println("2.设置属性：我的名字叫" +name);
        }
    
        @Override
        public void setBeanName(String s) {
            System.out.println("3.调用BeanNameAware\#setBeanName方法：我要上学了，起了个学名" + s);
        }
    
        @Override
        public void setBeanFactory(BeanFactory beanFactory) throws BeansException {
            System.out.println("4.调用BeanFactoryAware\#setBeanFactory方法：选好学校了");
        }
    
        @Override
        public void afterPropertiesSet() throws Exception {
            System.out.println("6.InitializingBean\#afterPropertiesSet方法：入学登记");
        }
    
        public void init() {
            System.out.println("7.自定义init方法：努力上学ing");
        }
    
        @Override
        public void destroy() throws Exception {
            System.out.println("9.DisposableBean\#destroy方法：平淡的一生落幕了");
        }
    
        public void destroyMethod() {
            System.out.println("10.自定义destroy方法:睡了，别想叫醒我");
        }
    
        public void work(){
            System.out.println("Bean使用中：工作，只有对社会没有用的人才放假。。");
        }
    }
    ```
    
1. `MyBeanPostProcessor`
    
    ```Java
    public class MyBeanPostProcessor implements BeanPostProcessor {
    
        @Override
        public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException {
            System.out.println("5.BeanPostProcessor.postProcessBeforeInitialization方法：到学校报名啦");
            return bean;
        }
    
        @Override
        public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {
            System.out.println("8.BeanPostProcessor\#postProcessAfterInitialization方法：终于毕业，拿到毕业证啦！");
            return bean;
        }
    }
    ```

    ```XML
    <?xml version="1.0" encoding="UTF-8"?>

           xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"

        <bean name="myBeanPostProcessor" class="com.atguigu.admin.bean.MyBeanPostProcessor" />
        <bean name="testBean" class="com.atguigu.admin.bean.TestBean"
              init-method="init" destroy-method="destroyMeod">
    ```

    ```Java
    public class testbean {
    ```
