# 一、如何优化文件传输的性能
1. ==**如何减少「用户态与内核态的上下文切换」的次数呢**== ==❓==
    1. 读取磁盘数据的时候，之所以要发生上下文切换，这是因为用户空间没有权限操作磁盘或网卡，内核的权限最高，这些操作设备的过程都需要交由操作系统内核来完成，所以一般要通过内核去完成某些任务的时候，就需要使用操作系统提供的系统调用函数
    2. 而一次系统调用必然会发生 2 次上下文切换：首先从用户态切换到内核态，当内核执行完任务后，再切换回用户态交由进程代码执行
    3. 所以，**要想减少上下文切换到次数，就要减少系统调用的次数**
2. ==**如何减少「数据拷贝」的次数**== ==❓==
    1. 在前面我们知道了，传统的文件传输方式会历经 4 次数据拷贝，而且这里面，「从内核的读缓冲区拷贝到用户的缓冲区里，再从用户的缓冲区里拷贝到 socket 的缓冲区里」，这个过程是没有必要的
    2. 因为文件传输的应用场景中，在用户空间我们并不会对数据「再加工」，所以数据实际上可以不用搬运到用户空间，因此**用户的缓冲区是没有必要存在的**
3. ==**下面介绍零拷贝技术是如何减少「上下文切换」和「数据拷贝」的次数的**==
# 二、零拷贝实现方式 1：系统调用函数 mmap() + write()
1. 具体过程如下：
    - 应用进程调用了 `mmap()` 后，DMA 会把磁盘的数据拷贝到内核的缓冲区里
    - 接着，会==直接把内核缓冲区里的数据「====**映射**====」到用户空间==，也即**==应用进程跟操作系统内核「共享」这个内核态的缓存区==**
    - 应用进程再调用 `write()`，操作系统直接将内核缓冲区的数据拷贝到 socket 缓冲区中，这一切都发生在内核态，**由 CPU 来搬运数据**
    - 最后，把内核的 socket 缓冲区里的数据，拷贝到网卡的缓冲区里，这个过程是由 DMA 搬运的

    ![[IMG-20260621000538260.png|575]]

2. 优化的地方：
    1. ⭐**通过使用 `mmap()` 来代替 `read()`，这样**==**不再需要将内核缓冲区中的数据拷贝到用户态的缓冲区，减少了一次数据拷贝的过程**==，减少到了 3 次
    2. 但这还不是最理想的零拷贝，因为仍然需要通过 CPU 把内核缓冲区的数据拷贝到 socket 缓冲区里，而且仍然需要 4 次上下文切换，因为系统调用还是 2 次

# 三、零拷贝实现方式 2：系统调用函数 sendfile()
## 1. Linux 内核版本 2.1
1. 在 Linux 内核版本 2.1 中，提供了一个专门发送文件的系统调用函数 `sendfile()`
2. ⭐优化的地方：
    - **将前面的 `read()` 和 `write()` 这两个系统调用**==**整合成了一个系统调用**==**，这样就**==**可以减少一次系统调用**==**，也就减少了 2 次上下文切换的开销**
    - **其次 `sendfile()`** ==**可以直接把内核缓冲区里的数据拷贝到 socket 缓冲区里，不需要再拷贝到用户态**==**，这样就只有 2 次上下文切换，和 3 次数据拷贝**

    ![[IMG-20260621000538365.png|560]]

3. 但是这还不是真正的零拷贝技术
## 2. Linux 内核版本 2.4 起
1. 从 Linux 内核 `2.4` 版本开始起，如果 **网卡支持 SG-DMA（_The Scatter-Gather Direct Memory Access_）技术**（和普通的 DMA 有所不同）
    - ==**网卡的 SG-DMA 控制器就可以直接将内核缓存中的数据拷贝到网卡的缓冲区里**==，数据都不再需要经过socket 缓冲区中了，这样就又减少了一次数据拷贝

    ![[IMG-20260621000538458.png|578]]

2. 这就是所谓的**零拷贝技术**
# 四、零拷贝（_Zero-copy_）技术
1. 零拷贝技术的文件传输方式
    1. ⭐**只需要** ==**2 次上下文切换和 2 次数据拷贝次数**==**，就可以完成文件的传输，而且 2 次的数据拷贝过程，**==**都不需要通过 CPU**==**，2 次都是由 DMA 来搬运**
    2. 相比传统文件传输的方式，减少了 2 次上下文切换和 2 次数据拷贝次数
2. 零拷贝技术可以把文件传输的性能提高至少一倍以上，使用了零拷贝能够缩短 `65%` 的文件传输时间，大幅度提升了机器传输数据的吞吐量

    ![[IMG-20260621000538563.png|770]]

# 五、使用零拷贝技术的项目
## 1. Kafka
1. 事实上，Kafka 这个开源项目，就利用了「零拷贝」技术，从而大幅提升了 I/O 的吞吐率，这也是 Kafka 在处理海量数据为什么这么快的原因之一
2. 如果你追溯 Kafka 文件传输的代码，你会发现，最终它调用了 Java NIO 库里的 `transferTo` 方法：

    ```Java
    @Overridepublic 
    long transferFrom(FileChannel fileChannel, long position, long count) throws IOException { 
        return fileChannel.transferTo(position, count, socketChannel);
    }
    ```
3. ⭐**如果 Linux 系统支持 `sendfile()`系统调用，那么 `transferTo()`实际上最后就会使用到 `sendfile()`系统调用函数** [[6.4 NIO 零拷贝案例]]
## 2. Nginx
1. Nginx 也支持零拷贝技术，一般默认是开启零拷贝技术，这样有利于提高文件传输的效率，是否开启零拷贝技术的配置如下：

    ```Java
    http {
    ...
        sendfile on
    ...
    }
    ```
2. p`sendfile` 配置的具体意思：[[Nginx 配置]]
    - 设置为 `on` 表示，使用零拷贝技术来传输文件：`sendfile` ，这样只需要 2 次上下文切换，和 2 次数据拷贝。
    - 设置为 `off` 表示，使用传统的文件传输技术：`read + write`，这时就需要 4 次上下文切换，和 4 次数据拷贝。
3. 当然，要使用 sendfile，Linux 内核版本必须要 2.1 以上的版本
