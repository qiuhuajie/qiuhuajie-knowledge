# 1. 介绍
1. **如果Handler处理器有一些长时间的业务处理，可以交给**`**taskQueue**`**异步处理**
1. 任务队列中的 Task 有 3 种典型使用场景
    
    1. 用户程序自定义的普通任务 [举例说明]
    
    1. 用户自定义定时任务
    
    1. 非当前 Reactor 线程调用 Channel 的各种方法
        
        1. 例如在推送系统的业务线程里面，根据用户的标识， 找到对应的 Channel 引用，然后调用 Write 类方法向该用户推送消息，就会进入到这种场景
        
        1. 最终的 Write 会提交到任务队列中后被异步消费
        
    
# 2. TaskQueue 的使用
## 2.1 普通任务
1. 解决的问题：有一个非常耗时的任务，可能会造成其它简单逻辑的业务的阻塞
1. 解决方案：
    
    1. 异步执行，将这个异步任务提交到`channel`对应的`NioEventLoop`的**`taskQueue`**中去
    
    1. **`ctx.channel().eventLoop()`**：从`ChannelHandlerContext`中获取当前需要使用的`channel`的对应`NioEventLoop`
    
    1. 同一个`taskQueue`中的任务是**串行执行**的，**一个**`taskQueue`**只有一个线程**，因此这个耗时任务的打印语句会在 $5s + 5s = 10s$ 后输出
    
1. 关键代码：`ctx.channel().eventLoop().execut(``_new_` `Runnable() {...}`
1. 代码：`NettyServerHandler`
    
    ```Java
    public class NettyServerHandler extends ChannelInboundHandlerAdapter {
        @Override
        public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {
            System.out.println("服务器读取数据使用的线程：" + Thread.currentThread().getName() + "，使用的通道：channel = " + ctx.channel());
            System.out.println("客户端的地址：" + ctx.channel().remoteAddress());
            ChannelPipeline pipeline = ctx.pipeline(); // ChannelPipeline本质是一个双向链接, 出站入站
    
            // 获取客户端发来的消息，注意：ByteBuf 是 Netty 提供的，不是 NIO 的 ByteBuffer
            ByteBuf buf = (ByteBuf) msg;
            System.out.println("客户端发送消息是：" + buf.toString(CharsetUtil.UTF_8));
    
            // 模拟一个非常耗时的业务，解决方案：异步执行，将这个异步任务提交到channel对应的NioEventLoop的taskQueue中去
            // ctx.channel().eventLoop()：从ChannelHandlerContext中获取当前需要使用的channel的对应NioEventLoop
            ctx.channel().eventLoop().execute(new Runnable() {
                @Override
                public void run() {
                    try {
                        Thread.sleep(5 * 1000);
                        ctx.writeAndFlush(Unpooled.copiedBuffer("hello2，客户端", CharsetUtil.UTF_8));
    										System.out.println("hello2 channel code：" + ctx.channel().hashCode());
                        System.out.println("hello2 thread id：" + Thread.currentThread().getId());
                    } catch (Exception e) {
                        e.printStackTrace();
                    }
                }
            });
    
            // 将第二个任务添加到taskQueue中去
            // 注意同一个taskQueue中的任务是串行执行的，一个taskQueue只有一个线程，因此这个耗时任务的打印语句会在 5s + 5s = 10s 后输出
            ctx.channel().eventLoop().execute(new Runnable() {
                @Override
                public void run() {
                    try {
                        Thread.sleep(5 * 1000);
                        ctx.writeAndFlush(Unpooled.copiedBuffer("hello3，客户端", CharsetUtil.UTF_8));
    										System.out.println("hello3 channel code：" + ctx.channel().hashCode());
                        System.out.println("hello3 thread id：" + Thread.currentThread().getId());
                    } catch (Exception e) {
                        e.printStackTrace();
                    }
                }
            });
        }
    
        @Override
        public void channelReadComplete(ChannelHandlerContext ctx) throws Exception {
            ctx.writeAndFlush(Unpooled.copiedBuffer("hello1, 客户端", CharsetUtil.UTF_8));
    				System.out.println("hello1 channel code：" + ctx.channel().hashCode());
            System.out.println("hello1 thread id：" + Thread.currentThread().getId());
        }
    
        @Override
        public void exceptionCaught(ChannelHandlerContext ctx, Throwable cause) throws Exception {
            ctx.close();
        }
    }
    ```

## 2.2 定时任务
1. **延时任务队列和上面介绍的任务队列非常相似，只是多了一个可延迟一定时间再执行的设置**
1. 用户自定义定时任务，将这个定时异步任务提交到`channel`对应的`NioEventLoop`的**`scheduledTaskQueue`**中去
1. 关键代码：`ctx.channel().eventLoop().schedule(``_new_` `Runnable() {...}`
1. `NettyServerHandler`
    
    ```Java
    public class NettyServerHandler extends ChannelInboundHandlerAdapter {
        @Override
        public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {
            System.out.println("服务器读取数据使用的线程：" + Thread.currentThread().getName() + "，使用的通道：channel = " + ctx.channel());
            System.out.println("客户端的地址：" + ctx.channel().remoteAddress());
            ChannelPipeline pipeline = ctx.pipeline(); // ChannelPipeline本质是一个双向链接, 出站入站
    
            // 获取客户端发来的消息，注意：ByteBuf 是 Netty 提供的，不是 NIO 的 ByteBuffer
            ByteBuf buf = (ByteBuf) msg;
            System.out.println("客户端发送消息是：" + buf.toString(CharsetUtil.UTF_8));
    
            // 模拟一个非常耗时的业务，解决方案：异步执行，将这个异步任务提交到channel对应的NioEventLoop的taskQueue中去
            // ctx.channel().eventLoop()：从ChannelHandlerContext中获取当前需要使用的channel的对应NioEventLoop
            ctx.channel().eventLoop().execute(new Runnable() {
                @Override
                public void run() {
                    try {
                        Thread.sleep(5 * 1000);
                        ctx.writeAndFlush(Unpooled.copiedBuffer("hello2，客户端", CharsetUtil.UTF_8));
                        System.out.println("hello2 channel code：" + ctx.channel().hashCode());
                        System.out.println("hello2 thread id：" + Thread.currentThread().getId());
                    } catch (Exception e) {
                        e.printStackTrace();
                    }
                }
            });
    
            // 将这个定时异步任务提交到channel对应的NioEventLoop的scheduledTaskQueue中去
            ctx.channel().eventLoop().schedule(new Runnable() {
                @Override
                public void run() {
                    try {
                        Thread.sleep(5 * 1000);
                        ctx.writeAndFlush(Unpooled.copiedBuffer("hello4，客户端", CharsetUtil.UTF_8));
                        System.out.println("hello4 channel code：" + ctx.channel().hashCode());
                        System.out.println("hello4 thread id：" + Thread.currentThread().getId());
                    } catch (Exception ex) {
                        ex.printStackTrace();
                    }
                }
            }, 2, TimeUnit.SECONDS);
    
            // 将第二个任务添加到taskQueue中去
            // 注意同一个taskQueue中的任务是串行执行的，一个taskQueue只有一个线程，因此这个耗时任务的打印语句会在 5s + 5s = 10s 后输出
            ctx.channel().eventLoop().execute(new Runnable() {
                @Override
                public void run() {
                    try {
                        Thread.sleep(5 * 1000);
                        ctx.writeAndFlush(Unpooled.copiedBuffer("hello3，客户端", CharsetUtil.UTF_8));
                        System.out.println("hello3 channel code：" + ctx.channel().hashCode());
                        System.out.println("hello3 thread id：" + Thread.currentThread().getId());
                    } catch (Exception e) {
                        e.printStackTrace();
                    }
                }
            });
    
            System.out.println("go on ...");
        }
    
        @Override
        public void channelReadComplete(ChannelHandlerContext ctx) throws Exception {
    
            // 发送消息给客户端，writeAndFlush 是 write + flush：将数据写入到缓存，并刷新
            ctx.writeAndFlush(Unpooled.copiedBuffer("hello1, 客户端", CharsetUtil.UTF_8));
            System.out.println("hello1 channel code：" + ctx.channel().hashCode());
            System.out.println("hello1 thread id：" + Thread.currentThread().getId());
        }
    
        @Override
        public void exceptionCaught(ChannelHandlerContext ctx, Throwable cause) throws Exception {
            // 发生异常，关闭通道
            ctx.close();
        }
    }
    ```
