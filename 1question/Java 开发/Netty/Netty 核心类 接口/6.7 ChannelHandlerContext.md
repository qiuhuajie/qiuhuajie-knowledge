# 一、ChannelHandlerContext
1. 产生的原因：
    1. 在Netty中，Handler处理器是自定义的，如果想在Handler中获取pipeline对象，或者channel对象，怎么获取呢？
    2. 于是Netty设计了这个 ChannelHandlerContext 上下文对象，就**可以拿到** **`channel`**、**`pipeline`** **等对象，就可以进行读写等操作**
2. 介绍：
    1. ChannelHandlerContext 中包含一个具体的事件处理器 ChannelHandler
    2. 同时ChannelHandlerContext 中也绑定了对应的 pipeline 和 Channel 的信息， 方便对 ChannelHandler进行调用
    3. ChannelHandlerContext是一个接口，下面有三个实现类

    ![[IMG-20260621000600584.png|588]]

3. 常用方法
    - `ChannelFuture close()`：关闭通道
    - `ChannelOutboundInvoker flush()`：刷新
    - `ChannelFuture writeAndFlush(Object msg)`：将数据写到 ChannelPipeline 中当前 ChannelHandler 的下一个 ChannelHandler 开始处理（出站）
