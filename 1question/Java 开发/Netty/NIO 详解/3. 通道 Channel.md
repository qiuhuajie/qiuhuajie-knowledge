# 1. 介绍
1. 所有的 NIO 操作始于通道，通道是**数据来源**或**数据写入的目的地**
1. Channel 在NIO中是一个接口 `public interface Channel extends Closeable`
1. 常用的 Channel 类有：
    
    ![[IMG-20260404031810714.png|Untitled 362.png]]
    
    - **`FileChannel`**：用于**文件**的数据读写
    
    - **`DatagramChannel`**：用于 **UDP** 连接的接收和发送
    
    - **`ServerSocketChannel`**：把它理解为 TCP 连接通道，简单理解就是 **TCP 客户端**
    
    - **`SocketChannel`**：**TCP 服务端**，用于监听某个端口进来的请求
    
1. **Channel 经常翻译为通道，类似 IO 中的流，用于读取和写入。它与前面介绍的 `Buffer` 打交道**
    
    - 读操作的时候将 `Channel` 中的数据填充到 `Buffer` 中：`_public abstract int_` `read(ByteBuffer dst)` `_throws_` `IOException;`
    
    - 而写操作时将 `Buffer` 中的数据写入到 `Channel` 中：`_public abstract int_` `write(ByteBuffer src)` `_throws_` `IOException;`
    
1. NIO的通道**类似于流**，但有些区别如下：
    
    1. 通道可以同时进行读写， 而流只能读或者只能写
    
    1. 通道可以实现异步读写数据
    
    1. 通道可以从缓冲读数据， 也可以写数据到缓冲
    
1. 可以将一个普通Buffer 转成**只读 Buffer**
    
    ```Java
    IntBuffer readOnlyBuffer = intBuffer.asReadOnlyBuffer();
    ```
    
    源码实现方式：重新 `new` 了一个`HeapIntBufferR` 对象，`HeapIntBufferR` 中重写了 `put()` ，方法内直接抛了一个 `ReadOnlyBufferException`
    
    ```Java
    public IntBuffer asReadOnlyBuffer() {
            return new HeapIntBufferR(hb, this.markValue(), this.position(), this.limit(), this.capacity(), offset);
    }
    ```
    
    ```Java
    public IntBuffer put(int x) {
            throw new ReadOnlyBufferException();
    }
    ```
    
# 2. FileChannel
## 2.1 介绍
FileChannel主要用来对本地文件进行 IO 操作， 常见的方法有
- `public int read(ByteBuffer dst)` ：从通道读取数据并放到缓冲区中
- `public int write(ByteBuffer src)` ：把缓冲区的数据写到通道中
- `public long transferFrom(ReadableByteChannel src, long position, long count)` ：从目标通道中复制数据到当前通道
- `public long transferTo(long position, long count, WritableByteChannel target)` ：数据从当前通道复制给目标通道
## 2.2 代码示例：文件拷贝
1. 使用 `FileChannel` 和方法 `read()` ，`write()` 完成文件的拷贝
    
    ![[IMG-20260404031810840.png|Untitled 1 272.png]]
    
    ```Java
    public class NIOChannelDemo01 {
        public static void main(String[] args) {
            File src = new File("E:/1.txt");
            File dst = new File("E:/2.txt");
            FileInputStream fis = null;
            FileOutputStream fos = null;
            try {
                fis = new FileInputStream(src);
                fos = new FileOutputStream(dst);
                // 1.通过输入、输出流获取channel
                FileChannel fisChannel = fis.getChannel();
                FileChannel fosChannel = fos.getChannel();
                // 2.初始化buffer，分配内存
                ByteBuffer byteBuffer = ByteBuffer.allocate((int) src.length());
                // 3.将channel中的数据读入到buffer
                fisChannel.read(byteBuffer);
                // 4.对buffer进行flip
                byteBuffer.flip();
                // 5.将buffer中的数据写出到channel
                fosChannel.write(byteBuffer);
            } catch (IOException e) {
                e.printStackTrace();
            } finally {
                try {
                    fis.close();
                    fos.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
        }
    }
    ```
    
1. 使用 `transferFrom` 方法实现文件拷贝
    
    ![[IMG-20260405035420172.png|Untitled 2 227.png]]
    
    ```Java
    public class NIOChannelDemo02 {
        public static void main(String[] args) {
            File src = new File("E:/1.txt");
            File dst = new File("E:/2.txt");
            FileInputStream fis = null;
            FileOutputStream fos = null;
            try {
                fis = new FileInputStream(src);
                fos = new FileOutputStream(dst);
                // 1.通过输入、输出流获取channel
                FileChannel fisChannel = fis.getChannel();
                FileChannel fosChannel = fos.getChannel();
                // 2.使用transferFrom，将一个通道的数据复制到一个通道
                // Transfers bytes into this channel's file from the given readable byte channel.
                fosChannel.transferFrom(fisChannel, 0, src.length());
            } catch (IOException e) {
                e.printStackTrace();
            } finally {
                try {
                    fis.close();
                    fos.close();
                } catch (IOException e) {
                    e.printStackTrace();
                }
            }
        }
    }
    ```
    
# 3. **SocketChannel**
1. 可以将 SocketChannel 理解成一个 **TCP 客户端**。虽然这么理解有点狭隘，因为我们在介绍 ServerSocketChannel 的时候会看到另一种使用方式
1. 代码示例：
    
    1. 打开一个 TCP 连接
        
        ```Java
        SocketChannel socketChannel = SocketChannel.open(new InetSocketAddress("https://www.javadoop.com", 80));
        ```
        
    
    1. SocketChannel 的读写和 FileChannel 没什么区别，就是操作缓冲区
        
        ```Java
        // 从连接中读取数据
        socketChannel.read(buffer);
        ```
        
        ```Java
        // 写入数据到网络连接中
        while(buffer.hasRemaining()) {
            socketChannel.write(buffer);   
        }
        ```
        
    
1. API
    
    ```Java
    public abstract Selectorselector)//得到与之关联的Selector 对象

    public finalObjectattachment()//得到与之关联的共享数

    public final boolean isAcceptable()//是否可以accept
    public finalboolean isReadable()//是否可以读
    public final boolean isWritable()//是否可以写
    ```
    
# 4. **ServerSocketChannel**
1. ServerSocketChannel 就是对应的 TCP 服务端
1. ServerSocketChannel 用于监听机器端口，管理从这个端口进来的 TCP 连接
    
    ```Java
    // 实例化
    ServerSocketChannel serverSocketChannel = ServerSocketChannel.open();
    // 监听 8080 端口
    serverSocketChannel.socket().bind(new InetSocketAddress(8080));
    
    while (true) {
        // 一旦有一个 TCP 连接进来，就对应创建一个 SocketChannel 进行处理
        SocketChannel socketChannel = serverSocketChannel.accept();
    }
    ```
    
1. `serverSocketChannel.accept()` 也是另一种 `SocketChannel` 的实例化方式
1. ⭐小总结：
    
    - `SocketChannel` 了它不仅仅是 TCP 客户端，它代表的是一个网络通道，可读可写
    
    - `ServerSocketChannel` 不和 Buffer 打交道了，因为它并**不实际处理数据**
    
    - `ServerSocketChannel` 一旦**接收到请求后，会实例化** `SocketChannel`，之后在这个连接通道上的数据传递它就不管了，因为它需要继续监听端口，等待下一个连接
    
1. API
    
    ```Java
    public static ServerSocketChannel open()//得到一个 ServerSocketChannel 通道
    public final ServerSocketChannel bind(SocketAddress local)//设置服务器端端号
    public final SelectableChannel configureBlocking(boolean block)//设置阻塞或阻塞模式，取值false表示采用非阻塞模式
    public SocketChannel accept()//接受一个连接，返回代表这个连接的通道对家
    public finalSelectionKeyregister(Selectorsel, int ops)//注册一个选择器并设置监听事件
    ```
    
# 5. **DatagramChannel**
1. UDP 和 TCP 不一样，UDP 是==面向==**==无连接==**==的==，不需要和对方握手，不需要通知对方，就可以直接将数据包投出去，至于能不能送达，它是不知道的
1. 因此，**相比上面的TCP服务端：没有`serverSocketChannel.accept()` 这和操作**
1. 故 UDP 只用 DatagramChannel 一个类就可以处理服务端和客户端之间的连接
1. **监听端口：**
    
    ```Java
    DatagramChannel channel = DatagramChannel.open();
    channel.socket().bind(new InetSocketAddress(9090));
    
    ByteBuffer buf = ByteBuffer.allocate(48);
    channel.receive(buf);
    ```
    
1. **发送数据：**
    
    ```Java
    String newData = "New String to write to file..." + System.currentTimeMillis();
    
    ByteBuffer buf = ByteBuffer.allocate(48);
    buf.put(newData.getBytes());
    buf.flip();
    
    int bytesSent = channel.send(buf, new InetSocketAddress("jenkov.com", 80));
    ```
