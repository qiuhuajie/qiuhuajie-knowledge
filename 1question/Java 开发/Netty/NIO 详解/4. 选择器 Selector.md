# 一、介绍
1. Selector 建立在非阻塞的基础之上，大家经常听到的 **多路复用器** 在 Java 世界中指的就是它，用于实现一个线程管理多个 Channel

    ![[IMG-20260621000439129.png|656]]

2. Selector 能够检测多个注册的通道上**是否有事件发生**
    1. 如果有事件发生，便获取事件然后针对每个事件进行相应的处理
    2. 线程通常将非阻塞 IO 的空闲时间用于在其他通道上执行 IO 操作，这样就可以只用一个单线程去管理多个通道，也就是管理多个连接和请求
    > 💡 注意：多个 Channel 以事件的方式可以注册到同一个 Selector
3. 选择器 Selector 带来的好处
    1. 只有在 连接/通道 真正有读写事件发生时，才会进行读写，就大大地减少了系统开销，从根本上解决了传统同步阻塞 I/O 一连接一线程模型， 架构的性能、 弹性伸缩能力和可靠性都得到了极大的提升
    2. 避免了多线程之间的上下文切换导致的开销

# 二、Selector 工作原理分析
## 1. 组件关系图

![[IMG-20260621000419017.png|828]]

1. **`SelectableChannel`**
    1. 是一类可以与 Selector 进行配合的通道，例如 **Socket 相关通道（`ServerSocketChannel`、`SocketChannel`）以及 Pipe 产生的通道都属于 `SelectableChannel`**
    2. 这类通道可以将自己感兴趣的操作（例如`read`、`write`、`accept`和`connect`）注册到一个Selector上，并在Selector的控制下进行IO相关操作
2. **`Selector`**
    1. 是一个控制器，它负责管理已注册的多个SelectableChannel
    2. 当这些通道的某些状态改变时，Selector会被唤醒（从`select()`方法的阻塞中），并对所有就绪的通道进行轮询操作
3. **`SelectionKey`**
    1. 是一个用来记录 SelectableChannel 和 Selector 之间关系的对象，它由 SelectableChannel 的`register()`方法返回，并存储在 Selector 的多个集合中
    2. 它不仅记录了两个对象的引用，还包含了 SelectableChannel 感兴趣的操作，即`OP_READ`，`OP_WRITE`，`OP_ACCEPT`和`OP_CONNECT`

## 2. Selector 的三个集合

![[IMG-20260621000419336.png|955]]

1. Selector对象会维护**三个存储了SelectionKey的集合**，分别是
    1. **`keys`集合：**存储了所有与Selector关联的SelectionKey对象
    2. **`selectedKeys`集合⭐：**存储了在一次`select()`方法调用后，所有状态改变的通道关联的SelectionKey对象`Set<SelectionKey> selectedKeys = selector.selectedKeys();`
    3. **`cancelledKeys`集合：**存储了一轮`select()`方法调用过程中，所有被取消但还未从keys中删除的SelectionKey对象
2. 其中**最值得关注的是selectedKeys集合**，它使用Selector对象的selectedKeys()方法获得，并通常会进行轮询处理
## 3. register() 方法
1. 第一个参数指明要注册的 Selector，第二个参数指明本通道感兴趣的操作，此参数的取值可以是以下四个，以及它们的逻辑值，例如
    - `SelectionKey.OP_READ`：对应 00000001，通道中有数据可以进行读取
    - `SelectionKey.OP_WRITE`：对应 00000100，可以往通道中写入数据
    - `SelectionKey.OP_CONNECT`：对应 00001000，成功建立 TCP 连接
    - `SelectionKey.OP_ACCEPT`：对应 00010000，接受 TCP 连接
    > 💡 可以同时监听一个 Channel 中的发生的多个事件：
    > `SelectionKey.OP_READ & SelectionKey.OP_WRITE`
2. 方法的返回值是一个 SelectionKey，**这个对象会被自动加入 Selector 的 keys 集合**，因此不必特意保留这个 SelectionKey 的对象引用，需要时可以使用Selector的`keys()`方法得到所有的SelectionKey对象引用。
3. **注册完成后，该通道就与Selector保持关联了**。当通道的状态改变时，其改变会自动被Selector感知，并在Selector的三个集合中反应出来
## 4. select() 方法

* Selector 类的`select()`方法是一个**阻塞方法**，它有两种形式：

- `int select()` ：不带参数的方法会一直阻塞，直到至少有一个注册的通道状态改变，才会被唤醒
- `int select(long timeout)` ：带有`timeout`参数的方法也会一直阻塞，直到时间耗尽，或者有通道的状态改变
## 5. Selector 工作流程

![[IMG-20260621000419478.png|800]]

1. 首先`ServerSocketChannel` 自己会注册在 `Selector` 上，如果有新的连接到来时，`ServerSocketChannel` 会创建 `SocketChannel`
2. `Selector` 会处理通道的状态改变，将发生状态改变的通道放在 `selectedKeys`集合中
3. `Selector` 进行监听 `select()` 方法， `select()` 方法会一直阻塞，直到时间耗尽或有通道状态改变，则会被唤醒
4. 接着会**通过`Selector.selectedKeys()` 获取`selectedKeys`集合**，然后遍历这个集合
    1. 如果是`SelectionKey.OP_ACCEPT` 事件： 会通过 `ServerSocketChannel` 得到 `SocketChannel` 实例，并将这个新的通道注册在 `Selector` 上，注册后返回一个 SelectionKey，会和该 Selector 关联
    2. 如果是`SelectionKey.OP_READ` 事件：可以**通过 `SelectionKey.channel()` 反向获取 与之关联的 `Channel`**，得到`Channel` 后就可以从里面读数据了

# 三、代码示例
1. 服务端

    ```Java
    public class NIOServer {
        public static void main(String[] args) {
            try {
                // 1.获取Selector
                Selector selector = Selector.open();
                // 2.获取ServerSocketChannel，绑定监听端口后，将其注册到Selector中
                ServerSocketChannel serverSocketChannel = ServerSocketChannel.open();
                serverSocketChannel.socket().bind(new InetSocketAddress(6666));
                serverSocketChannel.configureBlocking(false);
                serverSocketChannel.register(selector, SelectionKey.OP_ACCEPT);
                // 3.轮询执行select()，监测已注册的通道的状态变化
                while (true) {
                    if (selector.select(1000) == 0) {
                        System.out.println("服务器等待了1秒，未检测到有任何SocketChannel通道发生状态改变");
                        continue;
                    }
                    // 执行到这里代表select()返回 !0 ，表示有通道状态发生了改变
                    // 使用selectedKeys()获取当前已注册的channel中那些状态改变了的channel所关联的SelectionKey
                    Set<SelectionKey> selectedKeys = selector.selectedKeys();
                    // 遍历所有改变状态的channel所关联的SelectionKey
                    Iterator<SelectionKey> iterator = selectedKeys.iterator();
                    while (iterator.hasNext()) {
                        SelectionKey key = iterator.next();
                        // 根据当前遍历到的SelectionKey是发生了什么类型的状态改变，而做出不同的操作
                        // a connection was accepted by a ServerSocketChannel.
                        if (key.isAcceptable()) {
                            ServerSocketChannel sscTemp = (ServerSocketChannel) key.channel();
                            SocketChannel socketChannel = sscTemp.accept();
                            System.out.println("检测到ServerSocketChannel接受了新连接，生成一个SocketChannel实例：" + socketChannel.hashCode());
                            socketChannel.configureBlocking(false);
                            socketChannel.register(selector, SelectionKey.OP_READ, ByteBuffer.allocate(1024));
                        }
                        // a channel is ready for reading.
                        if (key.isReadable()) {
                            SocketChannel channel = (SocketChannel) key.channel();
                            ByteBuffer buffer = (ByteBuffer) key.attachment();
                            channel.read(buffer);
                            System.out.println("监测到有SocketChannel中有需要读取的数据，读取到的数据：" + new String(buffer.array()));
                        }
                        // a connection was established with a remote server.
                        if (key.isConnectable()) {
                        }
                        // a channel is ready for writing
                        if (key.isWritable()) {
                        }
                        iterator.remove();
                    }
                }
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
    ```
    > 💡
    > - 在一次`select()`方法返回后，应对 selectedKeys 集合中的所有 SelectionKey 对象进行轮询操作
    > - 并需要在操作完成后手动将 SelectionKey 对象从selectedKeys集合中删除（该通道的事件已经被处理完了）
    > - `ServerSocketChannel sscTemp = (ServerSocketChannel) key.channel();` 、`SocketChannel channel = (SocketChannel) key.channel();` 这里可以强转的原因是：能够进入到当分支的 `SelectionKey` 是可以确定是什么类型的
2. 客户端

    ```Java
    public class NIOClient {
        public static void main(String[] args) {
            try {
                SocketChannel socketChannel = SocketChannel.open();
                socketChannel.configureBlocking(false);
                if (!socketChannel.connect(new InetSocketAddress("127.0.0.1", 6666))) {
                    while (!socketChannel.finishConnect()) {
                        System.out.println("正在连接，连接需要时间，客户端不会阻塞，可以做其他工作");
                    }
                }
                // 连接完成，发送数据
                String str = "hello";
                /**
                 *  ByteBuffer.wrap(str.getBytes())等效于下面三行代码：
                 * ByteBuffer byteBuffer = ByteBuffer.allocate(str.length());
                 * byteBuffer.put(str.getBytes());
                 *  byteBuffer.flip();
                 */
                ByteBuffer byteBuffer = ByteBuffer.wrap(str.getBytes());
                socketChannel.write(byteBuffer);
                Thread.sleep(30000);
            } catch (IOException | InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
    ```
