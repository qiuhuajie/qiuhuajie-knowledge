# 一、需求说明
- 编写一个 Netty 心跳检测机制案例
    - 当服务器超过3秒没有读时，就提示读空闲
    - 当服务器超过5秒没有写操作时，就提示写空闲
    - 实现当服务器超过7秒没有读或者写操作时，就提示读写空闲

# 二、代码示例
> 💡 在群聊系统的代码基础上，只需添加心跳检测 handler 即可实现
1. **`GroupChatServer`**

    ```Java
    public GroupChatServer(int port) {
            this.port = port;
        }
        // 编写run方法，处理客户端的请求
        public void run() throws  Exception{
            // 创建两个线程组
            EventLoopGroup bossGroup = new NioEventLoopGroup(1);
            EventLoopGroup workerGroup = new NioEventLoopGroup(); // 8个 NioEventLoop
            try {
                ServerBootstrap bootstrap = new ServerBootstrap();
                bootstrap.group(bossGroup, workerGroup)
                        .channel(NioServerSocketChannel.class)
                        .option(ChannelOption.SO_BACKLOG, 128)
                        .childOption(ChannelOption.SO_KEEPALIVE, true)
                        .childHandler(new ChannelInitializer<SocketChannel>() {
                            @Override
                            protected void initChannel(SocketChannel ch) throws Exception {
                                ChannelPipeline pipeline = ch.pipeline();
                                // 向pipeline加入解码器
                                pipeline.addLast("decoder", new StringDecoder());
                                // 向pipeline加入编码器
                                pipeline.addLast("encoder", new StringEncoder());
                                // 加入自己的业务处理handler
                                pipeline.addLast(new GroupChatServerHandler());
    														// 加入 IdleStateHandler
                                pipeline.addLast(new IdleStateHandler(3,5,7, TimeUnit.SECONDS));
                                // 加入一个对空闲检测进一步处理的handler(自定义)
                                pipeline.addLast(new HeartBeatHandler());
                            }
                        });
                System.out.println("netty 服务器启动");
                ChannelFuture channelFuture = bootstrap.bind(port).sync();
                // 监听关闭
                channelFuture.channel().closeFuture().sync();
            }finally {
                bossGroup.shutdownGracefully();
                workerGroup.shutdownGracefully();
            }
        }
        public static void main(String[] args) throws Exception {
            new GroupChatServer(7000).run();
        }
    ```
    > 💡 **`IdleStateHandler`**
    > 1. 先加入一个netty 提供 IdleStateHandler
    > 2. IdleStateHandler 是netty 提供的处理空闲状态的处理器
    > 3. 构造函数的三个参数：
    >     4. `long readerIdleTime` : 表示多长时间没有读, 就会发送一个心跳检测包检测是否连接
    >     5. `long writerIdleTime` : 表示多长时间没有写, 就会发送一个心跳检测包检测是否连接
    >     6. `long allIdleTime` : 表示多长时间没有读写, 就会发送一个心跳检测包检测是否连接
    > 7. ==**在 client 与 server 之间在一定时间内没有数据交互时，即处于 idle 状态时。IdleStateHandler 就会被触发**==
2. **`HeartBeatHandler`**
    > 💡 **自定义的心跳检测 Handler**
    > 1. ==当 IdleStateEvent 触发后 , 就会==**==传递给管道 的下一个 handler 去处理==**
    > 2. 这里紧跟着的就是自定义**的心跳检测 Handler：**`HeartBeatHandler`
    > 3. 通过调用（触发）下一个 handler 的 `userEventTiggered()` 方法 , 在该方法中去处理 IdleStateEvent（读空闲，写空闲，读写空闲）

```Java
    public class HeartBeatHandler extends ChannelInboundHandlerAdapter {
        SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
        @Override
        public void userEventTriggered(ChannelHandlerContext ctx, Object evt) throws Exception {
            if (evt instanceof IdleStateEvent) {
                //将  evt 向下转型 IdleStateEvent
                IdleStateEvent event = (IdleStateEvent) evt;
                String eventType = null;
                switch (event.state()) {
                    case READER_IDLE:
                        eventType = "读空闲";
                        break;
                    case WRITER_IDLE:
                        eventType = "写空闲";
                        break;
                    case ALL_IDLE:
                        eventType = "读写空闲";
                        break;
                }
                System.out.println(sdf.format(new Date()) + "--" + ctx.channel().remoteAddress() + " 发生超时事件：" + eventType);
                System.out.println("服务器做相应处理..");
            }
        }
    }
```
# 三、运行结果
