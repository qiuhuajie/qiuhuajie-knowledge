# 一、需求描述
1. Netty 服务器在 6668 端口监听，浏览器发出请求 "http://localhost:6668/"
2. 服务器可以回复消息给客户端 "Hello! 我是服务器"
3. 并对特定请求资源进行过滤
# 二、代码示例

![[IMG-20260621000627925.png|745]]

1. `HttpServerInitializer`

    ```Java
    public class HttpServerInitializer extends ChannelInitializer<SocketChannel> {
        @Override
        protected void initChannel(SocketChannel socketChannel) throws Exception {
            //向管道加入处理器
            // 1.得到管道
            ChannelPipeline pipeline = socketChannel.pipeline();
            // 2.加入一个 netty 提供的httpServerCodec codec => [coder - decoder]
            // HttpServerCodec 是 netty 提供的处理http的 编-解码器
            pipeline.addLast("MyHttpServerCodec", new HttpServerCodec());
            // 3.增加一个自定义的 handler
            pipeline.addLast("MyTestHttpServerHandler", new HttpServerHandler());
            System.out.println("ok~~~~");
        }
    }
    ```
2. `Server`

    ```Java
    public class Server {
        public static void main(String[] args) throws Exception {
            // 1.创建两个线程组
            EventLoopGroup bossGroup = new NioEventLoopGroup(1);
            EventLoopGroup workerGroup = new NioEventLoopGroup();
            try {
                // 2.创建服务端的启动器，并配置参数
                ServerBootstrap serverBootstrap = new ServerBootstrap();
                // 设置两个线程组
                serverBootstrap.group(bossGroup, workerGroup)
                        .channel(NioServerSocketChannel.class)
                        .childHandler(new HttpServerInitializer());
                // 3.绑定端口号，启动服务端
                ChannelFuture channelFuture = serverBootstrap.bind(8001).sync();
                // 4.对关闭通道进行监听
                channelFuture.channel().closeFuture().sync();
            }finally {
                bossGroup.shutdownGracefully();
                workerGroup.shutdownGracefully();
            }
        }
    }
    ```
3. `HttpServerHandler`

    ```Java
    public class HttpServerHandler extends SimpleChannelInboundHandler<HttpObject> {
        @Override
        protected void channelRead0(ChannelHandlerContext ctx, HttpObject msg) throws Exception {
            System.out.println("当前 channel hashcode = " + ctx.channel().hashCode() +
                    "，当前 pipeline hashcode = " + ctx.pipeline().hashCode());
            // 1.判断 msg 是不是 http request请求
            if(msg instanceof HttpRequest) {
                System.out.println("msg 类型 = " + msg.getClass());
                System.out.println("客户端地址 = " + ctx.channel().remoteAddress());
                // 2.将获取到读取到的浏览器发来的请求信息，转为 HttpRequest
                HttpRequest httpRequest = (HttpRequest) msg;
                // 3.获取uri, 过滤掉访问指定资源的浏览器请求
                URI uri = new URI(httpRequest.uri());
                if("/favicon.ico".equals(uri.getPath())) {
                    System.out.println("请求了 favicon.ico, 不做响应");
                    return;
                }
                // 4.回复信息给浏览器，之前TCP案例中直接返回字符串即可，但是 http 返回的信息必须满足 http 协议规定的信息格式
                ByteBuf content = Unpooled.copiedBuffer("hello, 我是服务器", CharsetUtil.UTF_8);
                FullHttpResponse response = new DefaultFullHttpResponse(HttpVersion.HTTP_1_1, HttpResponseStatus.OK, content);
                response.headers().set(HttpHeaderNames.CONTENT_TYPE, "application/json");
                response.headers().set(HttpHeaderNames.CONTENT_LENGTH, content.readableBytes());
                // 5.将构建好 response返回
                ctx.writeAndFlush(response);
                System.out.println("==========================================================");
            }
        }
    }
    ```

# 三、说明
1. **客户端和`SocketChannel`、`ChannelPipeline` 的关系**
    1. 由上面运行结果发现，每个**不同浏览器客户端**都会在server端有一个自己独立的`SocketChannel`、`ChannelPipeline`
    2. 而同一个浏览器客户端使用的`SocketChannel`、`ChannelPipeline` 都是同一个
