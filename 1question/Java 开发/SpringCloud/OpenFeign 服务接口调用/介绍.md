# 1. **概述**
## 1.1 Feign
1. Feign旨在使得Java Http客户端变得更容易。
1. Feign集成了Ribbon、RestTemplate实现了负载均衡的执行Http调用
    
    1. 只不过对原有的方式（Ribbon+RestTemplate）进行了封装，开发者不必手动使用RestTemplate调服务，而是定义一个接口，在这个接口中标注一个注解即可完成服务调用
    
    1. 这样更加符合面向接口编程的宗旨，简化了开发
    
1. 但遗憾的是Feign现在停止迭代了，当然现在也是有不少企业在用
1. **Feign 的作用：**==**比 RestTemplate + Ribbon 更方便地实现：服务调用 + 负载均衡**==
    
    1. **RestTemplate + Ribbon**
        
        1. **服务调用：**前面在使用 Ribbon+RestTemplate 时，利用 RestTemplate 对 http 请求的封装处理，形成了一套模版化的调用方法
            
            ```Java
             public static final String PaymentSrv_URL = "http://CLOUD-PAYMENT-SERVICE"; //支付服务提供者的请求地址
             
             @Autowired
             private RestTemplate restTemplate;
             
             @GetMapping("/consumer/payment/create")
             public CommonResult create(Payment payment) {
                 //使用 restTemplate 通过 服务名字地址来调用远程的服务
                 return restTemplate.postForObject(PaymentSrv_URL + "/payment/create",payment,CommonResult.class);
             }
            ```
            
        
        1. **负载均衡：**由于 Eureka 自带了 Rinbbon ，所以只需要在 RestTemplate 的配置类的组件加载方法上添加注解：`@LoadBalanced` 即可实现负载均衡
            
            ```Java
             @Configuration
             public class ApplicationContextConfig {
                 @Bean
                 @LoadBalanced  // 使用 @LoadBalanced 注解赋予 RestTemplate 负载均衡的能力
                 public RestTemplate restTemplate(){
                     return new RestTemplate();
                 }
             }
            ```
            
        
    
    1. **Feign**
        
        1. 但是在实际开发中，由于对服务依赖的调用可能不止一处，往往一个接口会被多处调用，上面 RestTemplate + Ribbon 的方式代码量太大了
        
        1. 所以通常都会针对每个微服务自行封装一些客户端类来包装这些依赖服务的调用
        
        1. **Feign 就是**==**在 RestTemplate + Ribbon 基础上做了进一步封装**==，由他来**帮助我们定义和实现依赖服务接口**
            
            1. 在 Feign的实现下，只需创建一个接口并使用注解的方式来配置它，开发量大大减少
            
            1. 会绑定 **Feign 接口里的方法** 与 **服务提供者接口里的方法**
            
        
        1. ==⭐====**Feign 是如何实现简化微服务调用的**====❓==
            
            1. **通过 Eureka 等服务中心来**==**发现服务**==
                
                1. 具体地说，Feign通过Eureka Server来注册和发现微服务，当调用某个微服务时，Feign会向Eureka Server发送服务请求，获取可用服务列表
                
                1. 在获取到可用服务列表后，Feign会根据负载均衡算法来选择其中一个服务进行调用
                
            
            1. **并使用 Ribbon 等负载均衡器来**==**选择服务**==
                
                1. Feign支持多种负载均衡算法，如轮询、随机等。可以通过在Feign客户端配置文件中指定相应的负载均衡策略来实现
                
                1. 例如，在使用Ribbon作为负载均衡器时，可以使用@LoadBalanced注解来标记RestTemplate或FeignClient，从而实现负载均衡
                
            
        
        1. **基于 Feign 的调用属于**==**同步调用**==**的方式，是存在一些问题的**[[1.1 同步调用与异步调用]]
        
    
## 1.2 OpenFeign
1. **OpenFeign 是 springcloud** ==**在 Feign 的基础上支持了 SpringMVC 的注解**==
    
    1. 如 **`@RequestMapping`** 等等
    
    1. **OpenFeign 的 `@FeignClient` 可以**==**解析 SpringMVC 的**== **`@RequestMapping`** ==**注解下的接口**==**，并**==**通过动态代理的方式产生实现类**==**，**==**实现类中做负载均衡**==**并**==**调用**==**其他服务**
