# 2. **解决方法：超时控制**
1. openFeign其实是有默认的超时时间的，默认分别是连接超时时间 **`10s`**、读超时时间 **`60`**s，源码在
1. 那**为什么只设置了睡眠3秒就报超时呢**❓
    
    - 其实 openFeign 集成了 Ribbon，Ribbon 的默认超时连接时间、读超时时间都是是 `1s`
    
    - Ribbon **源码大致意思是**如果 openFeign 没有设置对应得超时时间，那么将会采用 Ribbon 的默认超时时间
    
    - **开发中，更推荐**==**直接设置 openFeign 的超时时间**==
        
        - OpenFeign 的超时时间应该根据具体情况来设置
            
            - 如果请求的服务实例很少，而且服务的响应速度比较快，可以将超时时间设置为 1 秒钟左右
            
            - 如果请求的服务实例比较多，或者服务的响应速度比较慢，可以适当增加超时时间，但是也不能太长，否则会影响客户端的性能和用户体验
            
        
        - 一般来说，==**超时时间**==**应该在 `3~5s` 之间**。当然，具体的超时时间还需要根据实际情况进行调整
        
    
1. 解决方法：
    
    - 只需要在配置文件中配置 openFeign 的超时时间，**在 feign 80 的yaml 配置文件中 添加超时控制**
    
    ```YAML
    feign:
      client:
        config:
          ## default 设置的全局超时时间，指定服务名称可以设置单个服务的超时时间
          default:
    				# 指的是建立连接所用的时间，适用于网络状况正常的情况下,两端连接所用的时间
            connectTimeout: 5000
            # 指的是建立连接后从服务器读取到可用资源所用的时间
            readTimeout: 5000
    ```
    
1. **`default`** 设置的是**全局超时时间，对所有的openFeign接口服务都生效**
    
    1. 也可以给指定服务单独设置超时时间
    
    1. 配置如下：
        
        ```YAML
        feign:
          client:
            config:
              ## default 设置的全局超时时间，指定服务名称可以设置单个服务的超时时间
              default:
                connectTimeout: 5000
                readTimeout: 5000
              ## 为 serviceC 这个服务单独配置超时时间，且单个配置的超时时间将会覆盖全局配置
              serviceC:
                connectTimeout: 30000
                readTimeout: 30000
        ```
