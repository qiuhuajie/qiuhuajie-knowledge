9. **但** **`@SentinelResource`** **注解中有专门给流量控制设置兜底方法的属性：****`blockHandler`**
    1. 给原方法添加 `blockHandler` 属性，指定当触发流量限制规则后，该执行哪个方法（保留之前的 `fallback` 属性和其兜底方法）

    ```Java
    @RestController
    @Slf4j
    public class CircleBreakerController {
        public static final String SERVICE_URL = "http://nacos-payment-provider";
        @Resource
        private RestTemplate restTemplate;
        @RequestMapping("/consumer/fallback/{id}")
        @SentinelResource(value = "fallback",fallback = "handlerFallback",blockHandler = "blockHandler") //⭐
        public CommonResult<Payment> fallback(@PathVariable Long id) {
            //调用 9003、9004的服务
            CommonResult<Payment> result = restTemplate.getForObject(SERVICE_URL + "/paymentSQL/"+id, CommonResult.class,id);
            //模拟抛出两种异常
                //1. id = 4 ：抛出IllegalArgumentException
                //2. 使用id在9003、9004上查出的数据 = null ：抛出NullPointerException
            if (id == 4) {
                throw new IllegalArgumentException ("IllegalArgumentException,非法参数异常....");
            }else if (result.getData() == null) {
                throw new NullPointerException ("NullPointerException,该ID没有对应记录,空指针异常");
            }
            return result;
        }
        //⭐业务异常兜底方法
        public CommonResult handlerFallback(@PathVariable  Long id,Throwable e) {
            return new CommonResult<>(444,"当前出现的异常："+e.getMessage()+"，故执行了兜底处理方法");
        }
        //⭐流量控制规则超出阈值后的兜底方法
        public CommonResult blockHandler(@PathVariable  Long id, BlockException blockException) {
            return new CommonResult<>(445,"当前访问QPS超过阈值，sentinel进行限流: blockException："+blockException.getMessage());
        }
    }
    ```
    2. 配置：同样给资源名 fallback 设置，限流规则
    3. 测试：
        1. 高并发访问：[http://localhost:84/consumer/fallback/1](http://localhost:84/consumer/fallback/1)

    ![[Attachment/1question/Java 开发/SpringCloud/Sentinel 熔断降级 + 流量控制/Sentinel 使用/IMG-20260405035422119.png|790]]
        2. 可以看到，不再执行 fallback 指定的兜底方法了，而是执行了 blockHandler 指定的兜底方法
    4. **结论：**
        1. **代码业务异常，由** **`fallback`** **属性指定的方法做兜底处理**
        2. **请求访问流量控制触发，由** **`blockHandler`** **指定的方法做兜底处理**

8. **⭐额外的处理类：写 服务降级 解耦出的兜底方法，以应对异常、请求超时 或 服务器宕机** 兜底方法必须写出 静态方法 static

    ```Java
    public class myFallbackHandler {
        public static CommonResult<Payment> fallbackHandler(Long id) {
    ```

9. **Controller**

    ```Java
    @RestController
    @Slf4j
    public class CircleBreakerController {
        @Resource
        private PaymentFeignService paymentFeignService;
        @GetMapping(value = "/consumer/openfeign/{id}")
        @SentinelResource(
                value = "openfeignTest",
                fallbackClass = myFallbackHandler.class,      //指定服务降级兜底
                fallback = "fallbackHandler",
                blockHandlerClass = myBlockHandler.class,     //指定流量控制兜底
                blockHandler = "blockHandler")
        public CommonResult<Payment> paymentSQL(@PathVariable("id") Long id) {
            if(id == 4) {
                throw new IllegalArgumentException("出现异常");
            }
            return paymentFeignService.paymentSQL(id);
        }
    }
    ```
