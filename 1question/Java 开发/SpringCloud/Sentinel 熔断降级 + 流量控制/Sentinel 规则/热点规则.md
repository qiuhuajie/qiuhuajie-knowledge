# 1. **基本介绍**
1. 热点参数限流会统计传入参数中的热点参数，并根据配置的限流阈值与模式，对包含热点参数的资源调用进行限流
1. 比如：
    
    - 商品 ID 为参数，统计一段时间内最常购买的商品 ID 并进行限制
    
    - 用户 ID 为参数，针对一段时间内频繁访问的用户 ID 进行限制
    
1. 热点参数限流可以看做是一种特殊的流量控制，**仅对包含热点参数的资源调用生效**
    
    ![[IMG-20260405035414064.png|Untitled 510.png]]
    
1. Sentinel 利用 LRU 策略统计最近最常访问的热点参数，结合==**令牌桶算法**==[[流量控制（限流）算法]]来进行参数级别的流控
1. 热点参数限流支持集群模式
# 2. **测试案例**
1. **添加 controller 方法**
    
    ```Java
    @RestController
    public class FlowLimitController {
    
        ...
    
        //⭐ value要与下面指定仪表盘新增规则时设置的资源名一致！
        //而 blockHandler 是指定的兜底方法
        @SentinelResource(value = "testHotKey",blockHandler = "dealHandler_testHotKey")
    
        @GetMapping("/testHotKey")
        public String testHotKey(@RequestParam(value = "p1",required = false) String p1,
                                 @RequestParam(value = "p2",required = false) String p2){
            return "------testHotKey，执行正常";
        }
        //兜底方法
        public String dealHandler_testHotKey(String p1, String p2, BlockException exception){
            return "-----dealHandler_testHotKey，执行了Sentinel的兜底方法";
        }
    
    }
    ```
    
1. **新增热点规则：**根据请求参数列表的索引来对参数设置先流规则

    1. 例如当 p1 的值等于 5 时，它的阈值可以达到200
