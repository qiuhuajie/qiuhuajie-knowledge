# 一、服务降级 fallback
1. 两种降级办法：
    1. **处理1：一个商品信息，当服务压力大时，优先加载价格和商品名字，商品图片和商品描述信息可以延迟加载**
    2. **处理2：服务器忙，请稍后再试，不让客户端等待并立刻返回一个友好提示**

    ```Java
    if(友好提示){
        备选响应1
    }else if(友好提示){
        备选响应2
    }else{
         兜底的解决方式
    }
    ```
2. **触发服务降级的时机**
    1. 程序运行异常
    2. 超时
    3. 服务熔断 也会触发服务降级
    4. 线程池、信号量打满时

# 二、服务熔断
1. 类比保险丝
2. 达到最大服务访问后，直接拒绝访问（拉闸断电），然后**调用服务降级的兜底方法**并返回友好提示
3. 服务熔断的流程：3步
    1. 服务的降级
    2. 熔断
    3. 恢复调用链路

# 三、服务限流
1. 秒杀高并发等操作，严禁一窝蜂的过来拥挤，大家排队，一秒钟N个，有序进行
