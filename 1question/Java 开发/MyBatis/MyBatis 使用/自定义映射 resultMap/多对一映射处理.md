# 1. 问题描述
**查询员工信息以及员工所对应的部门信息** 多对一关联查询返回的是一个包含部门信息的员工对象
员工 emp ：部门 dept
n ： 1
# 2. 解决方案
## 2.1 方案1：**级联方式处理映射**
```Java
 public class Emp {
     private int eid;
     private String empName;
     private int age;
     private Dept dept;
     ...
 }
```
```Java
 public interface EmpMapper {
     Emp getEmpAndDeptByEid(@Param("eid") int eid);
 }
```
```XML
 <select id="getEmpAndDeptByEid" resultMap="empDeptMap">
     select emp.*,dept.* from t_emp emp left join t_dept dept on emp.dept = dept.dept_name where emp.eid = #{eid}
 </select>
 
 <resultMap id="empDeptMap" type="com.qhj.pojo.Emp">
     <id property="eid" column="eid"></id>
     <result property="empName" column="emp_name" ></result>
     <result property="age" column="age" ></result>
     <!--⭐使用级联映射给 emp 的 dept 属性的各个属性 做映射-->
     <result property="dept.did" column="did" ></result>
     <result property="dept.deptName" column="dept_name" ></result>
 </resultMap>
```
```Java
 EmpMapper empMapper = sqlSession.getMapper(EmpMapper.class);
 Emp emp = empMapper.getEmpAndDeptByEid(2);
 System.out.println(emp);
 //Emp{eid=2, empName='lisi', age=34, dept=Dept{did=3, name='c'}}
```
## 2.2 方案2：**association 处理映射**
```Java
 public interface EmpMapper {
     Emp getEmpAndDeptByEid(@Param("eid") int eid);
 }
```
```XML
 <select id="getEmpAndDeptByEid" resultMap="empDeptMap">
     select emp.*,dept.* from t_emp emp left join t_dept dept on emp.dept = dept.dept_name where emp.eid = #{eid}
 </select>
 <resultMap id="empDeptMap" type="com.qhj.pojo.Emp">
     <id property="eid" column="eid"></id>
     <result property="empName" column="emp_name" ></result>
     <result property="age" column="age" ></result>
 
     <!--⭐先将部门属性 与 部门实体类做映射；再在 association 标签里面 给部门的各个属性做映射-->
     <association property="dept" javaType="com.qhj.pojo.Dept">
         <id property="did" column="did"></id>
         <result property="deptName" column="dept_name"></result>
     </association>
 </resultMap>
```
```Java
 EmpMapper empMapper = sqlSession.getMapper(EmpMapper.class);
 Emp emp = empMapper.getEmpAndDeptByEid(2);
 System.out.println(emp);
 //Emp{eid=2, empName='lisi', age=34, dept=Dept{did=3, name='c'}}
```
## 2.3 方案3：**分步查询** ☑️

> 先查员工，再根据查出的员工的 did 去查部门
### **step1：查询员工信息：**
1. EmpMapper
    
    ```Java
     public interface EmpMapper {
         Emp getEmpByStep(@Param("eid") int eid);
     }
    ```
    
1. EmpMapper.xml
    
    ```XML
     <mapper namespace="com.qhj.mapper.EmpMapper">
    
         <select id="getEmpByStep" resultMap="empDeptStepMap">
             select * from t_emp where eid = #{eid}
         </select>
         <resultMap id="empDeptStepMap" type="com.qhj.pojo.Emp">
             <id property="eid" column="eid"></id>
             <result property="empName" column="emp_name" ></result>
             <result property="age" column="age" ></result>
    
             <!--⭐给 emp 的 dept 属性做映射-->
             <!--将其和一个 sql select 语句做映射：使用的是下一步要执行的 sql 语句的唯一标识（绑定 mapper接口中的方法，它和sql映射文件中的一个sql语句绑定） -->
             <!--这里的 column 和上面的不一样，是第一步查询结果的某个字段，使用它作为下一个 mapper 接口中，查询方法的参数-->
             <association property="dept" select="com.qhj.mapper.DeptMapper.getEmpDeptByStep" column="dept"></association>
         </resultMap>
     
     </mapper>
    ```
    
### **step2：根据员工所对应的部门名 dept 查询部门信息**
1. DeptMapper
    
    ```Java
     public interface DeptMapper {
         Dept getEmpDeptByStep(@Param("dept_name") String dept_name);
     }
    ```
    
1. DeptMapper.xml
    
    ```XML
     <mapper namespace="com.qhj.mapper.DeptMapper">
         <select id="getEmpDeptByStep" resultType="com.qhj.pojo.Dept">
             select * from t_dept where dept_name = #{dept_name}
         </select>
     </mapper>
    ```
    
### **⭐分步查询的优点：**
可以实现延迟加载，但是必须在核心配置文件中设置全局配置信息：
1. `lazyLoadingEnabled`：延迟加载的全局开关，当开启时，所有关联对象都会延迟加载
    
    ```XML
     <setting name="lazyLoadingEnabled" value="true"/>
    ```
    
1. `aggressiveLazyLoading`：当开启时，任何方法的调用都会加载该对象的所有属性；否则，每个属性会按需加载
    
    ```XML
     <setting name="aggressiveLazyLoading" value="false"/>
    ```
    
1. 示例：当完成上面的设置后：
    
    ```Java
     EmpMapper empMapper = sqlSession.getMapper(EmpMapper.class);
     Emp empByStep = empMapper.getEmpByStep(2);
     System.out.println(empByStep.getEmpName());
     System.out.println("--------------------");
     System.out.println(empByStep);
     
     //结果：
     DEBUG 02-28 21:59:50,098 ==>  Preparing: select * from t_emp where eid = ? (BaseJdbcLogger.java:137)
     DEBUG 02-28 21:59:50,131 ==> Parameters: 2(Integer) (BaseJdbcLogger.java:137)
     DEBUG 02-28 21:59:50,210 <==      Total: 1 (BaseJdbcLogger.java:137)
     lisi
     --------------------
     DEBUG 02-28 21:59:50,211 ==>  Preparing: select * from t_dept where dept_name = ? (BaseJdbcLogger.java:137)
     DEBUG 02-28 21:59:50,211 ==> Parameters: c(String) (BaseJdbcLogger.java:137)
     DEBUG 02-28 21:59:50,213 <==      Total: 1 (BaseJdbcLogger.java:137)
     Emp{eid=2, empName='lisi', age=34, dept=Dept{did=3, name='c'}}
    ```
    
    分析：
    
    1. 分割线上面的代码只有 EmpName 属性被使用到了，且 aggressiveLazyLoading 设置为了 false，故其他属性不会加载
    
    1. lazyLoadingEnabled 设置为了 true，emp 与其关联对象 dept 会延迟加载，只有当 empt 的 dept 属性 在分割线下面那句代码中使用到时，dept 的 sql 语句才会被加载
    
    对比关闭延迟加载的情况：
    
    ```Java
     //两个 sql 是一起被加载的
     DEBUG 02-28 22:07:43,145 ==>  Preparing: select * from t_emp where eid = ? (BaseJdbcLogger.java:137)
     DEBUG 02-28 22:07:43,182 ==> Parameters: 2(Integer) (BaseJdbcLogger.java:137)
     DEBUG 02-28 22:07:43,212 ====>  Preparing: select * from t_dept where dept_name = ? (BaseJdbcLogger.java:137)
     DEBUG 02-28 22:07:43,212 ====> Parameters: c(String) (BaseJdbcLogger.java:137)
     DEBUG 02-28 22:07:43,215 <====      Total: 1 (BaseJdbcLogger.java:137)
     DEBUG 02-28 22:07:43,216 <==      Total: 1 (BaseJdbcLogger.java:137)
     lisi
     --------------------
     Emp{eid=2, empName='lisi', age=34, dept=Dept{did=3, name='c'}}
    ```
    
1. 而且在全局配置 延迟加载 开启后，可以在指定的 association 标签中设置，本管理是否需要开启延时加载（更加灵活）
    
    ```XML
     <association fetchType="eager" property="dept" select="com.qhj.mapper.DeptMapper.getEmpDeptByStep" column="dept"></association>
     
     <!--fetchType="lazy(延迟加载)|eager(立即加载)-->
    ```
