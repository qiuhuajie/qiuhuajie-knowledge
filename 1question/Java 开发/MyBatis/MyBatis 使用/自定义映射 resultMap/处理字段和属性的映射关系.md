# 处理字段和属性的映射关系

# 1. 存在的问题

1. 若字段名和实体类中的属性名不一致，则可以通过resultMap设置自定义映射
2. 例如：
    1. 在数据库字段的命名规则中使用下划线拼接两个单词：**emp_name**
    2. 而在 java 实体类中，属性的命名规则使用驼峰拼接两个单词：**empName**
3. 这样会使mybatis 无法自动将 **属性** 和 **字段** 做映射，当查出时，会查不出这个字段：显示为 null

    ```java
     Emp{eid=1, name='null', age=12}
    ```


# 2. 解决方案

1. **解决方法一：⭐在 mybatis 全局配置文件中开启 驼峰命名规则，会将 _a 自动变为 A**

    ```xml
     <settings>
         <!--将表中字段的下划线自动转换为驼峰-->
         <setting name="mapUnderscoreToCamelCase" value="true"/>
     </settings>
    ```

2. **解决方法二：使用 resultMap 将两个不同名的 属性手动做映射**

    EmpMapper

    ```java
     public interface EmpMapper {
         Emp selectEmp(@Param("empname") String empname);
     }
    ```

    EmpMapper.xml

    ```xml
     <mapper namespace="com.qhj.mapper.EmpMapper">
    
         <select id="selectEmp" resultMap="empMap">
             select * from t_emp where emp_name = #{empname}
         </select>
    
         <!--将实体类属性 和 数据库表的字段名 手动做映射-->
         <!--
             ⭐resultMap：设置自定义映射
                 属性：
                     id：表示自定义映射的唯一标识
                     type：查询的数据要映射的实体类的类型
                 子标签：
                     id：设置主键的映射关系
                     result：设置普通字段的映射关系
         -->
         <resultMap id="empMap" type="com.qhj.pojo.Emp">
             <id property="eid" column="eid"></id>
             <result property="empName" column="emp_name"></result>
             <result property="age" column="age"></result>
         </resultMap>
     </mapper>
    ```

    ```java
     EmpMapper empMapper = sqlSession.getMapper(EmpMapper.class);
     Emp zhangsan = empMapper.selectEmp("zhangsan");
     System.out.println(zhangsan);
     //Emp{eid=1, name='zhangsan', age=12}
    ```

    > 虽然当前问题 使用驼峰配置更简单，但方式二适用于更多的 不同的命名的属性和字段之间做映射
