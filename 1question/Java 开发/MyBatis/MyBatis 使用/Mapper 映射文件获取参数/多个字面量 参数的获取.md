# 1. 介绍
1. 若mapper接口中的方法参数为多个时
1. 此时MyBatis会自动将这些参数放在一个map集合中
    
    1. 以 `**arg0,arg1...**`**为键，以参数为值**
    
    1. **或以**`**param1,param2...**`**为键，以参数为值**
    
    > [!important]
    
    > [!important]
    > 
    > arg 的索引从0开始；param 的索引从1开始
    
1. 因此只需要通过`${}`和`#{}`访问`map`集合的键就可以获取相对应的值，注意`${}`需要手动加单引号
# 2. 示例
1. mapper接口：
    
    ```Java
     public interface UserMapper {
         User selectUser(String name, int age);
     }
    ```
    
1. sql 映射文件：
    
    ```Java
     <mapper namespace="com.qhj.mapper.UserMapper">
         <!--User selectUser(String name,int age)-->
         <select id="selectUser" resultType="com.qhj.pojo.User">
             select * from user where name = #{arg0} and age = #{arg1}
             <!--或 select * from user where name = #{param1} and age = #{arg1}-->
         </select>
     </mapper>
    ```
    

> [!important] 如果还是像但字面量那样获取
> 
> ```SQL
>  select * from user where name = #{name} and age = #{age}
> ```
> 
> 则会报错：`BindingException: Parameter 'name' not found. Available parameters are [arg1, arg0, param1, param2]`
