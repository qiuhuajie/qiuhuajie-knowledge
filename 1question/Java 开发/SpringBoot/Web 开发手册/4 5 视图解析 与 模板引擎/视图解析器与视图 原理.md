1. 目标方法处理的过程中，所有数据都会被放在 **ModelAndViewContainer** （里面包括数据和视图地址）
2. 如果 方法的参数是一个**自定义类型对象**（他的值是从请求参数中确定的），也会把它放在 **ModelAndViewContainer** 中

    ```Java
     @PostMapping("/login")
     public String main(User user, HttpSession session, Model model){
         ...
     }
    ```
3. 最终，任何目标方法执行完成以后都会返回一个 **ModelAndView** 对象（里面存放着数据和视图地址）
4. ==**执行结果的处理**==
    1. 由 **`processDispatchResult`** 处理派发结果（页面该如何响应）
    2. `processDispatchResult` 中调用 **`render(mv, request, response)`** 进行页面渲染逻辑
        1. mv 就是 **ModelAndView** 对象（里面存放着数据和视图地址）
        2. `render()` 中：根据方法的String返回值得到 **View** 对象（view 中定义了页面的渲染逻辑）
5. **是如何得到 view 对象的**❓
    1. 所有的视图解析器尝试是否能根据当前返回值得到 View 对象

    ![[IMG-20260405035414005.png|800]]
    2. 会使用 `ContentNegotiatingViewResolver` 这个解析器
    3. `ContentNegotiatingViewResolver` 继续遍历尝试使用所有视图解析器，根据返回值 **`redirect:/main.html`** ，得到了一个 **`RedirectView`** （thymeleaf new出来的）

    ![[IMG-20260404031833615.png|800]]
    > 💡 不同返回值：
    > 1. 返回值以 forward: 开始： `new InternalResourceView(forwardUrl);`
    >     ```Java
    >     //底层渲染就是转发
    >     request.getRequestDispatcher(path).forward(request, response);
    >     ```
    > 2. 返回值以 redirect: 开始： `new RedirectView()`
    >     ```Java
    >     //底层render就是重定向（上面例子的渲染过程）
    >     response.sendRedirect(encodedURL)
    >     ```
    > 3. 返回值是 普通字符串： `new ThymeleafView()`
6. 接着使用得到的视图对象，调用自定义的render进行页面渲染工作

    ```Java
     view.render(mv.getModelInternal(), request, response);
    ```
    页面渲染过程：（例如：重定向）
    1. 获取目标url地址
    2. response.sendRedirect(encodedURL)
