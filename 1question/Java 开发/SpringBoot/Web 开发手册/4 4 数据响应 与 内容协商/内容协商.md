# 一、内容协商原理
> 2.4.2 中 利用消息转换器 writeWithMessageConverters() 将返回的数据转换为JSON类型的过程中
>
> 内容协商的详细过程：
1. 判断当前**响应头**中是否已经有确定的媒体类型（MediaType）
2. 如果没有：⭐**getAcceptableMediaTypes()** 获取客户端（PostMan、浏览器）支持接收的内容类型（客户端Accept请求头字段）如 ：application/xml

    ```Java
     private List<MediaType> getAcceptableMediaTypes(HttpServletRequest request) throws HttpMediaTypeNotAcceptableException {
         return this.contentNegotiationManager.resolveMediaTypes(new ServletWebRequest(request));
     }
    ```
    1. 要用到：**contentNegotiationManager** 内容协商管理器 **默认使用基于请求头的策略**（客户端能接受的数据类型由请求头里的内容决定）

# 二、开启浏览器参数方式内容协商功能
1. postman 可以修改请求头内容来决定接受哪种数据类型
2. 但是浏览器没有办法改接受哪种数据类型，故需要开启浏览器参数方式内容协商功能
3. 开启参数方式的内容协商

    ```Java
     spring.mvc.contentnegotiation.favor-parameter=true
    ```
4. 开启后，可以将希望接受的数据类型，以参数的方式传入

    ```Plain
     http://localhost:8080/test/person?format=JSON
     http://localhost:8080/test/person?format=xml
    ```

    ![[IMG-20260405035434414.png|681]]

5. 原理：
    1. 之前 **contentNegotiationManager** 内容协商管理器 **默认使用基于请求头的策略**
    2. 而现在开启参数方式的内容协商后，使用的策略变成了两个，多了一个以参数确定的策略（参数名 就是 format）

    ![[IMG-20260405035440465.png|667]]

    3. 当 ?format=JSON：Parameter 策略优先确定是要返回JSON数据
    4. 最终进行内容协商返回给客户端 JSON 即可

# 三、自定义 MessageConverter
> 2.3.5 中的是 自定义 converter，不是 MessageConverter ，converter 使用来将传入的 自定义的请求参数 做类型转换
1. 目的：**实现多协议数据兼容**

| 需求 | 可接受数据格式 | 使用的 converter |
| --- | --- | --- |
| 浏览器发请求返回 xml | application/xml | jacksonXmlConverter |
| ajax 发请求返回 JSON | application/json | jacksonJsonConverter |
| qhj APP 发请求返回自定义协议数据 | application/x-qhj | xxxxxxConverter |
2. Step1：添加自定义的 MassageConverter

    ```Java
     //添加自定义 MessageConverter ：MyMessageConverter.class
     public class MyMessageConverter implements HttpMessageConverter<Person> {
         //通过调用每个converter 的**canRead()，canWrite()**方法，看是否支持写入写出，从请求头或参数数传来的需要的数据类型
         @Override
         public boolean canRead(Class<?> aClass, MediaType mediaType) {
             return false;
         }
         //不再返回false，判断如果可以支持 写出Person类的数据，则返回true
         @Override
         public boolean canWrite(Class<?> aClass, MediaType mediaType) {
             return aClass.isAssignableFrom(Person.class);
         }
         //服务器要统计所有MessageConverter都能写出哪些内容类型
         //写出 application/x-qhj 类型
         @Override
         public List<MediaType> getSupportedMediaTypes() {
             return MediaType.parseMediaTypes("application/x-qhj");
         }
         @Override
         public Person read(Class<? extends Person> aClass, HttpInputMessage httpInputMessage) throws IOException, HttpMessageNotReadableException {
             return null;
         }
         @Override
         public void write(Person person, MediaType mediaType, HttpOutputMessage httpOutputMessage) throws IOException, HttpMessageNotWritableException {
             //自定义协议数据的写出
             String data = person.getUserName() + ";" + person.getAge() + ";" + person.getBirth();
             //写出去
             OutputStream body = httpOutputMessage.getBody();
             body.write(data.getBytes(StandardCharsets.UTF_8));
         }
     }
    ```
    ```Plain
     //将 MyMessageConverter add到 webMvcConfigurer组件中
     @Configuration(proxyBeanMethods = false)
     public class WebConfig{
         @Bean
         public WebMvcConfigurer webMvcConfigurer(){
             return new WebMvcConfigurer() {
                 @Override
                 public void extendMessageConverters(List<HttpMessageConverter<?>> converters) {
                     converters.add(new MyMessageConverter());
                 }
             };
         }
     }
    ```
3. Step2：系统底层每次在内容协商时，就会**统计出所有 MassageConverter 能产出哪些数据类型**（会统计出自定义的 MassageConverter 能处理 **x-qhj** 类型的数据）

5. 上面只能基于请求头指定返回的数据类型，如何实现基于参数的返回类型指定（浏览器上）

    ![[IMG-20260405035510295.png|655]]

    1. 在webMvcConfigurer组件中，重写 configureContentNegotiation() ，添加新的策略

    ```Java
     @Configuration(proxyBeanMethods = false)
     public class WebConfig{
         @Bean
         public WebMvcConfigurer webMvcConfigurer(){
             return new WebMvcConfigurer() {
                 @Override
                 public void extendMessageConverters(List<HttpMessageConverter<?>> converters) {
                     converters.add(new MyMessageConverter());
                 }
                 @Override
                 public void configureContentNegotiation(ContentNegotiationConfigurer configurer) {
                     //指定支持解析 ：哪些参数 对应 哪些数据类型
                     Map<String, MediaType> mediaTypeMap = new HashMap<>();
                     mediaTypeMap.put("json",MediaType.APPLICATION_JSON);
                     mediaTypeMap.put("xml",MediaType.APPLICATION_XML);
                     mediaTypeMap.put("x-qhj",MediaType.parseMediaType("application/x-qhj"));
                     ParameterContentNegotiationStrategy parameterStrategy = new ParameterContentNegotiationStrategy(mediaTypeMap);
                     configurer.strategies(Arrays.asList(parameterStrategy));
                 }
             };
         }
     }
    ```
    2. 此时浏览器添加参数也可以响应了

    ![[IMG-20260405035512186.png|584]]

6. **如果此时，使用请求头的方式，由于重新自定义了内容协商管理器，里面只由基于参数的策略，没有基于请求头的策略了。故当使用请求头方式时，则认为客户端可以接受任何数据类型，进而按权重来最终确定返回什么类型的数据**

    ```Java
     //可以在 strategies 里再添加一个 请求头的策略
     HeaderContentNegotiationStrategy headerStrategy = new HeaderContentNegotiationStrategy();
     //这样两个策略都能使用了
     configurer.strategies(Arrays.asList(parameterStrategy,headerStrategy));
    ```
