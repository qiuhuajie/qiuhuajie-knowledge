# 一、静态资源访问
1. **静态资源目录**
    1. 只要静态资源放在类路径下： called **`/static`** (or **`/public`** or **`/resources`** or **`/META-INF/resources`**

    ![[IMG-20260404031830423.png|447]]

    2. 访问 ： ==**当前项目根路径/ + 静态资源名**==（静态资源只要放在上面的目录里就可以访问到了）

    ![[IMG-20260404031830479.jpeg|800]]

2. 原理： 静态映射 /**
3. 如果 有一个 **动态请求** 也是**`/1.jpg`** 那么会优先访问动态请求

    ```Java
     @RestController
     public class HelloController {
         @RequestMapping("/1.jpg")
         public String staticTest(){
             return "aaaa";
         }
     }
    ```

    ![[IMG-20260405035420372.jpeg|442]]

4. **静态资源的访问流程**
    1. 请求进来，先去找 Controller 看能不能处理
    2. 不能处理的所有请求又都交给静态资源处理器
    3. 静态资源也找不到则响应404页面
5. **为了避免和动态请求的同名冲突：静态资源访问前缀**
    1. 在 yaml 配置中添加：

    ```YAML
     spring:
       mvc:
         # 这里的 /res/ 只是访问的时候需要加，但静态资源实际还是存放在默认的资源目录下的
          # 且 /res/ 可以自己命名
             static-path-pattern: /res/**
    ```
    2. 访问：localhost:8080/res/5.jpg
6. **也可改变默认的静态资源的目录**
    1. 在 yaml 配置中添加：

    ```YAML
     spring:
       mvc:
         static-path-pattern: /res/**
       web:
         resources:
             # 这里好像是添加一个静态资源目录？
             # 原来 默认的静态资源的目录 也能使用
             # 但讲的是替换 有疑问...
           static-locations: [classpath:/abc/]
    ```
    2. 访问：5.jpg 当前在 /abc/ 目录下存放

    ![[IMG-20260405035422097.png|551]]

    3. 官方文档里明确说了是 替换 默认静态资源目录

        You can also customize the static resource locations by using the spring.web.resources.static-locations property (**replacing the default values with a list of directory locations**). The root servlet context path, "/", is automatically added as a location as well.
