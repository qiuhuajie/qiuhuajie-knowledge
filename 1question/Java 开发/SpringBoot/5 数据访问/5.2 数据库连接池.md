1. spring-boot-starter-data-jdbc **默认导入的就是** ==**HikariDataSource 连接池**==
1. **HikariDataSource 连接池自动配置分析**
    
    1. springBoot 自动配置的一些和 JDBC 相关的类：
        
        ![[IMG-20260405035413874.png|Untitled 334.png]]
        
        ![[IMG-20260404031835681.png|Untitled 1 252.png]]
        
    
    1. DataSourceAutoConfiguration：数据连接池的自动配置
        
        1. 修改数据源相关的配置：spring.datasource
            
            ```Java
             @EnableConfigurationProperties({DataSourceProperties.class})
             
             
             //DataSourceProperties.class
             @ConfigurationProperties(prefix = "spring.datasource")
            ```
            
        
        1. 数据库连接池的配置，是自己容器中没有DataSource才自动配置的
            
            ```Java
             ...
             //容器中没有DataSource才自动配置
             @ConditionalOnMissingBean({DataSource.class, XADataSource.class})
             ...
             protected static class PooledDataSourceConfiguration {
                 protected PooledDataSourceConfiguration() {
                 }
             }
            ```
            
        
        1. 底层配置好的连接池是：**HikariDataSource**
            
            ```Java
             @Bean
             @ConfigurationProperties(
                 prefix = "spring.datasource.hikari"
             )
             HikariDataSource dataSource(DataSourceProperties properties) {
                 //创建数据源
                 HikariDataSource dataSource = (HikariDataSource)DataSourceConfiguration.createDataSource(properties, HikariDataSource.class);
                 if (StringUtils.hasText(properties.getName())) {
                     dataSource.setPoolName(properties.getName());
                 }
             
                 return dataSource;
             }
            ```
            
        
    
    1. DataSourceTransactionManagerAutoConfiguration：事务管理器的自动配置
        
        1. 可以修改这个配置项 "spring.jdbc" 来修改JdbcTemplate
            
            ```Java
             //JdbcProperties.class
             @ConfigurationProperties(prefix = "spring.jdbc")
            ```
            
        
        1. 已经自动配置了 一个 jdbcTemplate 组件了，如果想要 操作数据库，**直接将容器中的 jdbcTemplate 组件注入即可**
            
            ```Java
             //JdbcTemplateConfiguration.class
             @Bean
             @Primary
             JdbcTemplate jdbcTemplate(DataSource dataSource, JdbcProperties properties) {
                 JdbcTemplate jdbcTemplate = new JdbcTemplate(dataSource);
                 Template template = properties.getTemplate();
                 jdbcTemplate.setFetchSize(template.getFetchSize());
                 jdbcTemplate.setMaxRows(template.getMaxRows());
                 if (template.getQueryTimeout() != null) {
                     jdbcTemplate.setQueryTimeout((int)template.getQueryTimeout().getSeconds());
                 }
             
                 return jdbcTemplate;
             }
            ```
            
        
    
    1. JdbcTemplateAutoConfiguration：JdbcTemplate的自动配置，可以来对数据库进行crud
    
    1. JndiDataSourceAutoConfiguration： jndi的自动配置
    
    1. XADataSourceAutoConfiguration：分布式事务相关的
    
# 3. **Druid 数据连接池**
https://github.com/alibaba/druid
## 3.1 **使用自定义方式整合 Druid数据连接池**
1. 引入依赖：
    
    ```XML
     <dependency>
         <groupId>com.alibaba</groupId>
         <artifactId>druid</artifactId>
         <version>1.1.17</version>
     </dependency>
    ```
    
1. 向容器中 配置一个 DruidDataSource
    
    1. 自动配置的HikariDataSource连接池在底层配置写的条件注入如下：
        
        ```Java
         //容器中没有DataSource才自动配置
         @ConditionalOnMissingBean({DataSource.class, XADataSource.class})
        ```
        
    
    1. **故我们 向容器中 配置一个 DruidDataSource 后，系统检测到容器中已经有一个 DataSource了，就不会再配置 默认的 HikariDataSource 连接池了**
    
    1. 写一个配置类
        
        ```Java
         @Configuration
         public class myDataSorceConfig {
         
            //向容器中 配置一个 DruidDataSource
             @Bean
             @ConfigurationProperties("spring.datasource")
             public DataSource dataSource() throws SQLException {
        
                 DruidDataSource druidDataSource = new DruidDataSource();
                 return druidDataSource;
             }
         }
        ```
        
    
1. 单元测试看当前使用的是哪个 数据库连接池
    
    ```Java
     @Slf4j
     @SpringBootTest
     class SpringBootAdminApplicationTests {
     
         //容器中此时已经有了 dataSource 组件了，直接注入即可
         @Autowired
         DataSource dataSource;
    
         @Test
         void contextLoads() {
             log.info("当前使用的数据库连接池类型：{}",dataSource.getClass());
    
             //控制台日志输出结果：
             //当前使用的数据库连接池类型：class com.alibaba.druid.pool.DruidDataSource
         }
     }
    ```
    
1. **使用 druid 数据库连接池 提供的监控信息展示 html 页面**
    
    1. 在数据库连接池配置类中修改：配置_StatFilter 并 配置一个 StatViewServlet
        
        ```Java
         @Configuration
         public class myDataSorceConfig {
         
             @ConfigurationProperties("spring.datasource")
             @Bean
             public DataSource dataSource() throws SQLException {
                 DruidDataSource druidDataSource = new DruidDataSource();
        
                 //配置_StatFilter（监控功能开启）
                 druidDataSource.setFilters("stat");
                 return druidDataSource;
             }
         
             //向容器中配置一个 StatViewServlet（返回页面）
             // 2.9.2 RegistrationBean方式注入原生servlet
             @Bean
             public ServletRegistrationBean statViewServlet(){
                 StatViewServlet statViewServlet = new StatViewServlet();
                 ServletRegistrationBean<StatViewServlet> registrationBean = new ServletRegistrationBean<>(statViewServlet,"/druid/*");
                 return registrationBean;
             }
         }
        ```
        
    
    1. 写一个 handler 用于测试 监控页面是否生效
        
        ```Java
         @Slf4j
         @Controller
         public class IndexController {
        
             @Autowired
             JdbcTemplate jdbcTemplate;
         
             @ResponseBody
             @GetMapping("/sqlTest")
             public String queryFromDb(){
                 String sql = "select count(*) from t_book";
                 Long aLong = jdbcTemplate.queryForObject(sql, Long.class);
                 return aLong.toString();
             }
         }
        ```

## 3.2 **使用官方 starter 的方式**
1. 引入druid-starter（把上面的 druid 依赖注释掉）
    
    ```XML
     <dependency>
         <groupId>com.alibaba</groupId>
         <artifactId>druid-spring-boot-starter</artifactId>
         <version>1.1.17</version>
     </dependency>
    ```
    
    ![[IMG-20260405035520031.png|Untitled 8 62.png]]
    
1. **starter 会自动配置 druid 连接池（不需要我们再写了）**
    
    ![[IMG-20260405035527231.png|Untitled 9 54.png]]
    
1. 自动配置分析
    
    1. 扩展配置项 **spring.datasource.druid**
    
    1. **DruidSpringAopConfiguration.class**：监控SpringBean的，配置项：spring.datasource.druid.aop-patterns
    
    1. **DruidStatViewServletConfiguration.class**：监控页的配置，配置项：spring.datasource.druid.stat-view-servlet（默认开启）
    
    1. **DruidWebStatFilterConfiguration.class** ：web监控配置，配置项：spring.datasource.druid.web-stat-filter（默认开启）
    
    1. **DruidFilterConfiguration.class})** 所有Druid自己filter的配置
    
1. 在配置文件中配置，实现第一个方式的所有功能
    
    ```YAML
     spring:

             # druid 连接池相关 配置

             # 监控页

             enabled: true
             login-username: admin
    ```
