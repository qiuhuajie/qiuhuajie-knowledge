# 一、组件添加
## 1. @Configuration
1. 原始springMVC配置文件中的组件添加

    ```XML
     <?xml version="1.0" encoding="UTF-8"?>
     <beans xmlns="http://www.springframework.org/schema/beans"
            xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
            xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">
         <bean id="user01" class="com.atguigu.boot.bean.User">
             <property name="name" value="zhangsan"></property>
             <property name="age" value="18"></property>
         </bean>
         <bean id="pet01" class="com.atguigu.boot.bean.Pet">
             <property name="name" value="cat"></property>
         </bean>
     </beans>
    ```
2. **`@Configuration`** **注解的功能：**==**使用配配置类来代替配置文件**==

    ```Java
     @Configuration // 告诉spring这是一个配置类（完全注解开发，使用配置类 代替配置文件beans.xml）
     public class MyConfig {
         //给容器中添加组件
         @Bean
         public User user01(){
             return new User("zhangsan",18);
         }
         @Bean
         public Pet pet01(){
             return new Pet("cat");
         }
     }
    ```
3. 在主程序中测试 组件的创建

    ```Java
     @SpringBootApplication() //告诉SpringBoot这是一个SpringBoot应用
     public class MainApplication {
         public static void main(String[] args) {
             //1.主程序入口（固定写法），run 即 spring5中的context容器
             ConfigurableApplicationContext run = SpringApplication.run(MainApplication.class, args);
             //2.查看容器里的组件
             String[] names = run.getBeanDefinitionNames();
             for (String name : names) {
                 System.out.println(name);
             }
             //3.从容器中获取组件
             User user01 = run.getBean("user01", User.class);
             User user02 = run.getBean("user01", User.class);
             System.out.println(user01);
             //User{name='zhangsan', age=18}
             System.out.println("组件：" + (user01 == user02));
             //单实例：获取多次也是同一个。组件：true
             MyConfig bean = run.getBean(MyConfig.class);
             System.out.println(bean);
             //com.atguigu.boot.config.MyConfig$$EnhancerBySpringCGLIB$$681174ea@8dfe921
             //🟡配置类本身也是一个组件
         }
     }
    ```
4. `@Configuration` 有一个属性：**`proxyBeanMethods`**
    1. Full 模式：`proxyBeanMethods = true`，保证每个`@Bean`方法被调用多少次返回的组件都是**单实例的**
    2. Lite 模式：`proxyBeanMethods = false`，每个`@Bean`方法被调用多少次返回的组件都是**新创建的**
    3. 结论：
        1. **组件依赖（当某个组件被其他组件中使用到时）必须使用Full模式默认**
        2. **其他默认是否Lite模式**
    4. 示例：

    ```Java
     public class Pet {
         private String name;
         //get、set...
    ```
    ```Java
     public class User {
         private String name;
         private Integer age;
         private Pet pet;
         //在user中添加pet属性
         //get、set...
    ```
    ```Java
     //配置类
     @Configuration(proxyBeanMethods = true)
     public class MyConfig {
         @Bean
         public User user01(){
             User zhangsan = new User("zhangsan", 18);
             //⭐给zhangsan注入cat的对象依赖（组件依赖）
                 //如果时full模式下，调用pet01()后，得到的cat就是下面已经在容器中创建好的组件（这样就实现了组件依赖）
                 //如果时lite模式下，pet01()的调用会报错
             zhangsan.setPet(pet01());
             return zhangsan;
         }
         @Bean
         public Pet pet01(){
             return new Pet("cat");
         }
     }
    ```
    ```Java
     //测试
     @SpringBootApplication()
     public class MainApplication {
         public static void main(String[] args) {
             ConfigurableApplicationContext run = SpringApplication.run(MainApplication.class, args);
             User user011 = run.getBean("user01", User.class);
             Pet pet01 = run.getBean("pet01", Pet.class);
             System.out.println(user011.getPet() == pet01);
             //true
             //表明user011.getPet()得到的就是容器中pet01（⭐注入组件依赖成功）
         }
     }
    ```
## 2. @Bean
1. `@Bean` 作用在方法上
2. 被标注的方法里：定义产生这个 Bean，给容器注册组件，默认也是单实例的
    1. 组件的 id：方法名
    2. 组件类型：方法返回类型
    3. 返回值：就是组件在容器中的实例
3. 使用示例：

    ```Java
    @Configuration
    public class AppConfig {
    		@Bean
    		public TransferService transferService() {
    				return new TransferServiceImpl();
    		}
    }
    ```
4. 上面的代码相当于：

    ```XML
    <beans>
    		<bean id="transferService" class="com.acme.TransferServiceImpl"/>
    </beans>
    ```
## 3. @Component
1. 通用的注解，可标注任意类为 Spring 组件
2. 可以使用 **`@ComponentScan`** 注解，**定义要扫描的路径，进而从中找出标识了需要装配的类，最后自动装配到 IOC 容器中**
3. 如果一个 Bean 不知道属于哪个层，可以使用 `@Component` 注解标注
4. ==**@Bean**== ==好像和== ==**@Component 差不多**====❓==
    1. 作用对象不同：
        1. @Component 注解作用于类
        2. 而 @Bean 注解作用于方法
    2. 装配方式不同：
        1. @Component 通常是通过**类路径扫描**来自动侦测以及自动装配到 Spring 容器中
        2. @Bean 注解在**被标注的方法**里：定义产生这个 Bean
    3. @Bean 注解比 @Component 注解的自定义性更强，下面这个例子就是通过 **@Component 无法实现的**

    ```Java
    @Bean
    public OneService getService(status) {
    		case (status) {
    				when 1:
    						return new serviceImpl1();
    				when 2:
    						return new serviceImpl2();
    				when 3:
    						return new serviceImpl3();
    		}
    }
    ```
    4. 而且很多地方我们只能通过 @Bean ，比如当我们引用第三方库中的类需要装配到 Spring 容器时，则只能通过 @Bean 来实现

## 4. @Controller
1. `@Controller` ：对应 Spring MVC 控制层，主要用户接受用户请求并调用 Service 层返回数据给前端页面
2. 被 Controller 注解标注的类中的所有方法，如果返回值是 string 并且有具体的页面可以跳转，那么就会被视图解析器解析
## 5. @Service

`@Service` ：对应服务层，主要涉及一些复杂的逻辑，需要用到 Dao层

## 6. @Repository

`@Repository`：对应持久层即 Dao 层，主要用于数据库相关操作

> 💡 `@Component` 、`@Service`、`@Controller`、`@Repository` **四个注解功能是一样的，只是为了开发清晰规范，在不同层使用不同的注解（混用也行）**
## 7. @Import
1. 它的作用是**提供了一种显式地从其它地方加载配置类的方式**，这样可以避免使用性能较差的组件扫描(Component Scan)
    1.  给容器中自动创建出参数列表中传入的类，默认组件的名字就是全类名
    2. 传入的类可以是自己写的：`User.class`，也可以是引过来的第三方的 jar 包：`DBHelper.class`
2. 示例：

    ```Java
     @Import({User.class, DBHelper.class})
     @Configuration(proxyBeanMethods = true)
     public class MyConfig {
         @Bean
         public User user01(){
             User zhangsan = new User("zhangsan", 18);
             return zhangsan;
         }
     }
    ```
    ```Java
     //测试
     @SpringBootApplication()
     public class MainApplication {
         public static void main(String[] args) {
             ConfigurableApplicationContext run = SpringApplication.run(MainApplication.class, args);
             //查看容器里的组件
             String[] BeanNamesForType1 = run.getBeanNamesForType(User.class);
             for (String name : BeanNamesForType1) {
                 System.out.println(name);
                 //com.atguigu.boot.bean.User ----》一个是@Bean获取的组件
                 //user01    ----》另一个是@Import({User.class})获取的组件（认组件的名字就是全类名）
             }
     }
    ```
## 8. @ImportResource
1. 如果之前写的代码是使用xml配置文件的，如何使用注解将这些配置文件导入
2. 可以使用 **`@ImportResource`** 注解
3. 示例：

    ```XML
     <!--beans.xml-->
     <?xml version="1.0" encoding="UTF-8"?>
     <beans xmlns="http://www.springframework.org/schema/beans"
            xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
            xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">
         <bean id="uu" class="com.atguigu.boot.bean.User">
             <property name="name" value="zhangsan"></property>
             <property name="age" value="18"></property>
         </bean>
         <bean id="pp" class="com.atguigu.boot.bean.Pet">
             <property name="name" value="cat"></property>
         </bean>
     </beans>
    ```
    ```Java
     @ImportResource("classpath:beans.xml")
     @Configuration(proxyBeanMethods = true)
     public class MyConfig {
     }
    ```
    ```Java
     @SpringBootApplication()
     public class MainApplication {
         public static void main(String[] args) {
             ConfigurableApplicationContext run = SpringApplication.run(MainApplication.class, args);
             System.out.println(run.containsBean("uu"));  //true
             System.out.println(run.containsBean("pp")); //true
             //组件 uu 和 pp 导入成功
         }
     }
    ```
# 二、装配条件
## 1. 条件装配 @Conditional 🙋‍♂️
1. 它的作用是按照设定的条件进行判断，把满足判断条件的 bean 注册到Spring容器
2. **@Conditional 的使用**
    1. @Conditional 可以作用在方法上，也可以作用在类上；如果是类和方法都加了@Conditional 注解，最终在方法上的注解为最终的条件
    2. 使用的时候需要传入实现 **`Condition`** 接口类数组
    3. 如果绑定的 `Condition` 条件返回 `true` 则将标注的 Bean 加入容器，反之不会加入容器
3. **使用场景：**
    1. 在一些需要条件满足才是实例化的类中，使用此注解
    2. 比如项目中**需要根据不同的场景使用不同的消息中间件**，在消息队列的实例化 bean 上，使用 **`@Conditional`** 注解，**根据配置文件的不同，来判断哪个消息中间件的 bean 需要加载到Spring容器中**
4. 使用示例：
    1. 定义两个实现 `Condition` 接口的类：`CNCondition` 、`ZHCondition`

    ```Java
    public class CNCondition implements Condition {
        public boolean matches(ConditionContext conditionContext, AnnotatedTypeMetadata annotatedTypeMetadata) {
            // 只有当配置文件中 user.language=CN ，该条件才会返回 true，进而被标注的 Bean 定义方法才会生效
            return conditionContext.getEnvironment().getProperty("user.language").contains("cn");
        }
    }
    ```
    ```Java
    public class ZHCondition implements Condition {
        public boolean matches(ConditionContext conditionContext, AnnotatedTypeMetadata annotatedTypeMetadata) {
            return conditionContext.getEnvironment().getProperty("user.language").contains("zh");
        }
    }
    ```
    2. 使用 `@Conditional` 加载类，传入实现 `Condition` 接口类数组

    ```Java
    @Configuration
    @Conditional(ZHCondition.class)
    public class TestConditionalConfig {
        @Bean
        @Conditional(ZHCondition.class)//中文
        public Book bookZH()    {
            return new Book("红楼梦","中国古典书籍",99.9);
        }
        @Bean
        @Conditional(CNCondition.class)//英文
        public Book bookCN()    {
            return new Book("哈姆雷特","外国经典名著",66.6);
        }
    }
    ```
## 2. @Conditional 的扩展注解
1. 汇总：

    ```Java
    @ConditionalOnBean：仅在当前上下文中存在某个对象时，才会实例化一个Bean。
    @ConditionalOnClass：某个class位于类路径上，才会实例化一个Bean。
    @ConditionalOnExpression：当表达式值为true的时候，才会实例化一个Bean。
    @ConditionalOnMissingBean：仅仅在当前上下文中不存在某个对象时，才会实例化一个Bean。
    @ConditionalOnMissingClass：某个class类路径上不存在的时候，才会实例化一个Bean。
    @ConditionalOnNotWebApplication：非web应用，才会实例化一个Bean。
    @ConditionalOnBean：当容器中有指定Bean的条件下进行实例化。
    @ConditionalOnMissingBean：当容器里没有指定Bean的条件下进行实例化。
    @ConditionalOnClass：当classpath类路径下有指定类的条件下进行实例化。
    @ConditionalOnMissingClass：当类路径下没有指定类的条件下进行实例化。
    @ConditionalOnWebApplication：当项目是一个Web项目时进行实例化。
    @ConditionalOnNotWebApplication：当项目不是一个Web项目时进行实例化。
    @ConditionalOnProperty：当指定的属性有指定的值时进行实例化。
    @ConditionalOnExpression：基于SpEL表达式的条件判断。
    @ConditionalOnJava：当JVM版本为指定的版本范围时触发实例化。
    @ConditionalOnResource：当类路径下有指定的资源时触发实例化。
    @ConditionalOnJndi：在JNDI存在的条件下触发实例化。
    @ConditionalOnSingleCandidate：当指定的Bean在容器中只有一个，或者有多个但是指定了首选的Bean时，才会触发实例化。
    ```
2. **`@ConditionalOnBean`** 使用示例：

    ```Java
     @Configuration(proxyBeanMethods = true)
     public class MyConfig {
         @Bean
         //⭐ConditionalOnBean 表示当有"pet01"组件时，才会创建 user01 的组件
         @ConditionalOnBean(name = "pet01")
         public User user01(){
             User zhangsan = new User("zhangsan", 18);
             zhangsan.setPet(pet01());
             return zhangsan;
         }
         //@Bean
         public Pet pet01(){
             return new Pet("cat");
         }
     }
    ```
    ```Java
     @SpringBootApplication()
     public class MainApplication {
         public static void main(String[] args) {
             ConfigurableApplicationContext run = SpringApplication.run(MainApplication.class, args);
             System.out.println(run.containsBean("pet01"));  //false
             System.out.println(run.containsBean("user01")); //false
             // 可以看到当前既没有pet01组件，也没有user01组件
         }
     }
    ```
# 三、给组件的属性绑定配置文件中的值
1. **可以使用** **`@ConfigurationProperties`** **注解，根据 properties文件来给组件的属性赋值**（有时一些 bean 组件的值，是在properties文件中写好的）
2. 使用示例：
    1. 方法一：==**@Component + @ConfigurationProperties**==
        1. 配置文件

    ```Java
     # application.properties 文件
     server.port=8888
     mycar.brand = "BYD"
     mycar.price = 10000
    ```
        2. 在entity上写@Component 和 @ConfigurationProperties

    ```Java
     //1️⃣只有在容器中的组件，才会拥有SpringBoot提供的强大功能
     @Component
     //2️⃣prefix 表示取application.properties 文件中 mycar前缀后的属性，与下面Car类中的属性相匹配
     @ConfigurationProperties(prefix = "mycar")
     public class Car {
         private String brand;
         private Integer price;
             ...
     }
    ```
    ```Java
     @RestController
     public class HelloController {
         @Autowired
         Car car;
         @RequestMapping("/car")
         public Car getCar(){
             return car;
         }
     }
    ```

    ![[IMG-20260405035413858.png|800]]

    2. 方法二：**@EnableConfigurationProperties + @ConfigurationProperties**
        1. 配置文件

    ```Java
     # application.properties 文件
     server.port=8888
     mycar.brand = "YD"
     mycar.price = 10000
    ```
        2. 在entity上写@ConfigurationProperties

    ```Java
     //2️⃣这里不需要 @Component 了
     @ConfigurationProperties(prefix = "mycar")
     //prefix 表示去
     public class Car {
         private String brand;
         private Integer price;
             ...
     }
    ```
        3. 在配置类上写@EnableConfigurationProperties

    ```Java
     //1️⃣@EnableConfigurationProperties有两个功能
     //1、开启Car配置绑定功能
     //2、把这个Car这个组件自动注册到容器中
     @EnableConfigurationProperties(Car.class)
     public class MyConfig {
         ...
     }
    ```
    ```Java
     @RestController
     public class HelloController {
         @Autowired
         Car car;
         @RequestMapping("/car")
         public Car getCar(){
             return car;
         }
     }
    ```
