# 一、依赖管理
## 1. 自动版本仲裁机制
1. 使用父项目做依赖管理

    ```XML
     <parent>
         <groupId>org.springframework.boot</groupId>
         <artifactId>spring-boot-starter-parent</artifactId>
         <version>2.3.4.RELEASE</version>
     </parent>
    ```
2. 按住 ctrl 点进 **`spring-boot-starter-parent`** 发现他还有一个父项目

    ```XML
     <parent>
         <groupId>org.springframework.boot</groupId>
         <artifactId>spring-boot-dependencies</artifactId>
         <version>2.3.4.RELEASE</version>
     </parent>
    ```
3. 再进入 **`spring-boot-dependencies`** ，里面几乎声明了所有开发中常用的依赖的版本号，称为**自动版本仲裁机制**

    ![[IMG-20260404031827543.png|800]]

4. **自动版本仲裁的目的**
    1. ==无需关注版本号，自动版本仲裁。引入依赖默认都可以不写版本==
    2. 但是，引入**非版本仲裁的 jar**，要写版本号、
        1. 如：原来的 mysql 版本

    ```XML
     <!-- spring-boot-dependencies 文件中-->
     <mysql.version>8.0.21</mysql.version>
    ```
        2. 在 pom 里配置mysql

    ```XML
     <dependency>
      <groupId>mysql</groupId>
      <artifactId>mysql-connector-java</artifactId>
     </dependency>
    ```
        3. 上面的配置没有说明版本号，故自动引入的也是**`spring-boot-dependencies`**里的版本

    ![[IMG-20260404031827627.png|383]]

        4. **那如何使用其他版本的依赖？**
            1. 访问 maven 仓库：
    > ℹ️ Maven Repository: Search/Browse/Explore
    > [https://mvnrepository.com/](https://mvnrepository.com/)
            2. 找到想要的依赖包
            3. 找到对应的版本号

    ![[IMG-20260405035420257.png|800]]

            4. 在当前项目里面重写配置

    ```XML
     <!--直接添加在pom文件里即可（mvn有 就近原则，优先引入自定义的5.1.43）-->
     <properties>
         <mysql.version>5.1.43</mysql.version>
     </properties>
    ```
            5. 刷新 mvn 后，引入的mysql已经是5.1.43

    ![[IMG-20260404031827750.png|394]]

## 2. starter 场景启动器
1. 见到很多 **`spring-boot-starter-*`** ：其中 **`*`** 就是某种场景
2. 如入门项目里的web场景下的 starter：

    ```XML
     <dependencies>
         <dependency>
             <groupId>org.springframework.boot</groupId>
             <artifactId>spring-boot-starter-web</artifactId>
         </dependency>
     </dependencies>
    ```
3. ==**只要引入 starter，这个场景的所有常规需要的依赖我们都自动引入**==
4. SpringBoot所有支持的场景
    > ℹ️ Developing with Spring Boot
    > [https://docs.spring.io/spring-boot/docs/current/reference/html/using-spring-boot.html#using-boot-starter](https://docs.spring.io/spring-boot/docs/current/reference/html/using-spring-boot.html#using-boot-starter)
5. 如果见到： **`-spring-boot-starter`**，则是第三方为我们提供的简化开发的场景启动器
6. **所有场景启动器最底层的依赖**

    ```XML
     <!--SpringBoot 自动配置的核心依赖-->
     <dependency>
       <groupId>org.springframework.boot</groupId>
       <artifactId>spring-boot-starter</artifactId>
       <version>2.3.4.RELEASE</version>
       <scope>compile</scope>
     </dependency>
    ```
    - **spring-boot-starter-web**
    - **spring-boot-starter-jdbc**
    - **spring-boot-starter-data-redis**
    - **spring-boot-starter-data-elasticsearch**
    - **spring-boot-starter-data-jdbc**
    - **spring-boot-starter-websocket**
