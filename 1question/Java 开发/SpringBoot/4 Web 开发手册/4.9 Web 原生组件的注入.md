# 1. **方式1：使用 Servlet API 注入（推荐）**
使用注解：`**@ServletComponentScan**` **+** `**@WebServlet**`**、**`**@WebFilter**`**、**`**@WebListener**`
## 1.1 **注入 Servlet**
1. servlet
    
    ```Java
     @WebServlet(urlPatterns = "/myServlet")
     public class MyServlet extends HttpServlet {
         @Override
         protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
             resp.getWriter().write("666");
         }
     }
    ```
    
1. 开启组件扫描
    
    ```Java
     @ServletComponentScan(basePackages = "com.atguigu.admin")
     @SpringBootApplication
     public class SpringBootAdminApplication {
     
         public static void main(String[] args) {
             SpringApplication.run(SpringBootAdminApplication.class, args);
         }
     
     }
    ```
    
1. 访问 请求，且不会经过之前添加的登录判断拦截器
    
    ![[Attachment/1question/大数据/Java 开发/SpringBoot/4 Web 开发手册/IMG-20260405035413873.png|Untitled 332.png]]
    
    如果多个 Servlet 都能处理到同一层路径，则使用**精确优选原则**
    
    1. 我们自己写的 MyServlet 处理 `/myServlet`，而Spring 自动配置的 DispatchServlet 处理 `/`
    
    1. 此时 请求为 `/myServlet` 时 根据优先原则，会交由 MyServlet 处理，而 DispatchServlet 不会处理，故自然**不会经过之前添加的登录判断拦截器**

## 1.2 **注入 Filter**
1. filter
    
    ```Java

     @WebFilter(urlPatterns = {"/css/*","/images/*"})

             log.info("myFilter初始化完成");
    ```

# 2. **方式2：使用RegistrationBean**
1. 将前写的MyServlet、 myFiltermyFilter、myServletContextListener 上的 servle 的注解 注释掉
1. 编写一个 配置类
    
    ```Java
     @Configuration(proxyBeanMethods = true) //保证依赖的组件都是单实例的，避免对象冗余
     public class MyRegistConfig {
     
         @Bean
         public ServletRegistrationBean myServlet(){
             //new一个自己写的 MyServlet 对象
             MyServlet myServlet = new MyServlet();
    
             //将 MyServlet 对象 传入 ServletRegistrationBean的构造器 并返回
             return new ServletRegistrationBean(myServlet,"/myServlet");
         }
     
         @Bean
         public FilterRegistrationBean myFilter(){
             myFilter myFilter = new myFilter();
             FilterRegistrationBean filterRegistrationBean = new FilterRegistrationBean(myFilter);
     
             filterRegistrationBean.setUrlPatterns(Arrays.asList("/css/*","/images/*"));
             return filterRegistrationBean;
         }
     
         @Bean
         public ServletListenerRegistrationBean myListener(){
             myServletContextListener myServletContextListener = new myServletContextListener();
             return new ServletListenerRegistrationBean(myServletContextListener);
         }
     }
    ```
    
1. 实现效果与 2.9.1 相同
# 3. **DispatchServlet 的注册原理**
1. DispatchServlet 底层也是通过 使用 **方式二：RegistrationBean** 来注入的
1. 容器中自动配置了 DispatcherServlet 属性绑定到 WebMvcProperties，对应的配置文件配置前缀是 **spring.mvc**
    
    ```Java
      WebMvcProperties webMvcProperties
    ```
    
1. **通过 ServletRegistrationBean**<DispatcherServlet> 把 DispatcherServlet 配置进来
    
    ```Java
     @Bean(
         name = {"dispatcherServletRegistration"}
     )
     @ConditionalOnBean(
         value = {DispatcherServlet.class},
         name = {"dispatcherServlet"}
     )
     public DispatcherServletRegistrationBean dispatcherServletRegistration(DispatcherServlet dispatcherServlet, WebMvcProperties webMvcProperties, ObjectProvider<MultipartConfigElement> multipartConfig) {
    
         //通过 ServletRegistrationBean把 DispatcherServlet 配置进来
         DispatcherServletRegistrationBean registration = new DispatcherServletRegistrationBean(dispatcherServlet, webMvcProperties.getServlet().getPath());
    
         ....
    
         return registration;
     }
    ```
    
1. 默认映射的是 `**/**` 路径
    
    ```Java
     webMvcProperties.getServlet().getPath()
    ```
