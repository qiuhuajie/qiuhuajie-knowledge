## **@PathVariable 路径变量**
1. SpringMVC路径中的占位符常用于 RESTful 风格中
    
    - **请求路径中会将某些数据通过路径的方式传输到服务器中**
        
        ```HTML
         <!--这里1就是id参数，admin 就是 username参数-->
         
         <a th:href="@{/car/1/owner/admin}">测试</a><br>
        ```
        
    
    - 在相应的 @RequestMapping 注解的value属性中通过占位符 `**{xxx}**` 表示传输的数据
    
    - 在通过 @PathVariable注解，将占位符所表示的数据（路径变量）赋值给控制器方法的形参
    
1. 代码示例：
    
    ```Java
     @RestController
     public class parameterTestController {
         @GetMapping("/car/{id}/owner/{username}")
         public Map<String,Object> getCar(@PathVariable("id") Integer id,
                                          @PathVariable("username") String name,
                                          @PathVariable Map<String,Object> pv){
             HashMap<String, Object> map = new HashMap<>();
             map.put("id",id);
             map.put("username",name);
     
             // @PathVariable Map<String,Object> 可以将路径中的变量自动放到 Map中
             map.put("pv",pv);
             return map;
         }
     }
    ```
    
    ```HTML
     <a href="/car/3/owner/zhangsan">car/{id}/owner/{username}</a>
    ```

## **@MatrixVariable**
1. 查询字符串的语法： 请求路径：`**/cars/{path}?属性1=值1&属性2=值2**`
1. 矩阵变量的写法：`**/cars/{path;属性1=值1;属性2=值2}**`
    
    > 面试题：cookie 被禁用了，该如何获得session中的属性值？
    > 
    > 1. session中的设置属性值，session.set(a,b)；每个session有一个 jsessionid ，这个 jsessionid 会放到 cookie 中，而每次发请求都会携带 cookie ，故可以从cookie 中读到 session 的属性值
    > 
    > 1. 如果 cookie 被禁用了，cookie 携带的jsessionid获取不到了，可以把 cookie 的值使用矩阵变量的方式传递，这样即使没有cookie能获取到jsessionid
    > 
    > 1. 解决方法：url重写：/abc;jsessionid=xxxxx，
    
1. SpringBoot默认是**禁用**了矩阵变量的功能
    
    1. 底层源码：对于路径的处理，使用 UrlPathHelper 进行解析，UrlPathHelper 中 removeSemicolonContent = true（移除分号内容），故不能识别矩阵变量
    
    1. **需要手动开启：在配置类中自定义组件 webMvcConfigurer**
        
        ```Java
         //方式二：配置类实现接口 WebMvcConfigurer，重写 configurePathMatch 方法
         @Configuration(proxyBeanMethods = false)
         public class WebConfig implements WebMvcConfigurer {
         
             //WebMvcConfigurer接口中的方法是默认实现的，故不需要全部重写
             @Override
             public void configurePathMatch(PathMatchConfigurer configurer) {
         
                 UrlPathHelper urlPathHelper = new UrlPathHelper();
         
                 //将urlPathHelper中的RemoveSemicolonContent设置为false
                 urlPathHelper.setRemoveSemicolonContent(false);
         
                 configurer.setUrlPathHelper(urlPathHelper);
             }
         }
        ```
        
        ```Java
         //方式二：不使用继承接口的方式，直接创一个自定义的 WebMvcConfigurer 组件
         @Configuration(proxyBeanMethods = false)
         public class WebConfig{
         
             @Bean
             public WebMvcConfigurer webMvcConfigurer(){
                 return new WebMvcConfigurer() {
                     @Override
                     public void configurePathMatch(PathMatchConfigurer configurer) {
                         UrlPathHelper urlPathHelper = new UrlPathHelper();
                         //将urlPathHelper中的RemoveSemicolonContent设置为false
                         urlPathHelper.setRemoveSemicolonContent(false);
        
                         configurer.setUrlPathHelper(urlPathHelper);
                     }
                 };
             }
         }
        ```
        
    
1. **矩阵变量必须有 url 路径变量才能被解析**
    
    > 请求路径是：/cars/sell;low=34;brand=byd,audi
    > 
    > 相当于从分号开始后边是矩阵变量，而且矩阵变量需要绑定在路径变量 sell 中
    
    ```Java
     @Controller
     public class RequestController {
     
         @ResponseBody
         @GetMapping("/cars/{path}")
         public Map carSell(@MatrixVariable Integer low,
                            @MatrixVariable List<String> brand,
                            @PathVariable("path") String path){
             HashMap<String, Object> map = new HashMap<>();
             map.put("low",low);
             map.put("brand",brand);
             map.put("path",path);
             return map;
         }
     }
    ```
    
    ```HTML
     <a href="/cars/sell;low=34;brand=byd,audi">carsSell_1</a>
     
     <a href="/cars/sell;low=34;brand=byd;brand=audi">carsSell_2</a>
    ```

# 2. **使用 Servlet API 传入参数**
1. WebRequest、ServletRequest、MultipartRequest、 HttpSession、javax.servlet.http.PushBuilder、Principal、InputStream、Reader、HttpMethod、Locale、TimeZone、ZoneId
1. 故：HttpServletRequest request, HttpServletResponse response 这两个参数，可以直接使用
1. 示例：
    
    ```Java
     @Controller
     public class RequestController {
     
         @GetMapping("/goto")
    
         //这里的 HttpServletRequest request 域就是 Servlet API 作为参数
         public String goToPage(HttpServletRequest request){
     
             request.setAttribute("msg","成功了");
             request.setAttribute("code",200);
     
             return  "forward:/success";
         }
     }
    ```

1. 自动封装自定义类参数的原理
    
    1. 使用的参数解析器：**ServletModelAttributeMethodProcessor**
        
        ![[IMG-20260405035532954.png|Untitled 10 54.png]]
        
    
    1. 数据绑定器：
        
        ```Java
         WebDataBinder binder = binderFactory.createBinder(webRequest, attribute, name);
        ```
        
        1. 将请求参数的值绑定到指定的JavaBean里面
        
        1. 在绑定的时候，由于传入的数据都是 String 类型的（request带来参数的字符串），WebDataBinder 利用它里面的 **Converters** 将请求数据转成指定的数据类型（如Integer）
        
        1. GenericConversionService：在设置每一个值的时候，找它里面的所有 **converter （124个）**那个可以将这个数据类型转换到指定的类型
        
    
1. 自定义 **converter**
    
    > [!important] 如果 request 参数传入时有规定：
    > 
    > ```HTML
    >  <form action="/saveuser" method="post">
    >       姓名： <input name="userName" value="zhangsan"/> <br/>
    >       年龄： <input name="age" value="18"/> <br/>
    >       生日： <input name="birth" value="2019/12/10"/> <br/>
    >  
    >       <!--宠物姓名：<input name="pet.name" value="阿猫"/><br/>-->
    >       <!--宠物年龄：<input name="pet.age" value="5"/>-->
    >  
    >       <!--不使用级联属性传入参数-->
    >       宠物姓名、年龄：<input name="pet" value="阿猫,3">
    >  
    >       <input type="submit" value="提交">
    >   </form>
    > ```
    > 
    > 则会在转换类型时报错
    > 
    > 就需要自己写转换规则
    
    1. WebDataBinder里面放自己的Converter
        
        1. 在数据绑定时，会进行数据转换，而 124个 converter 原来找不到合适的转换器
        
        1. 但现在成了 125个了，有了新自定义的converter ，自定义写的就是能处理 String ----> Pet 的类型转换
        
        1. 底层就会使用这个converter 来进行数据转换
        
    
    1. 在配置类的WebConfig.class 中的webMvcConfigurer组件里重写addFormatters方法
        
        ```Java
         @Configuration(proxyBeanMethods = false)
         public class WebConfig{
         
             @Bean
             public WebMvcConfigurer webMvcConfigurer(){
                 return new WebMvcConfigurer() {
                     @Override
                     public void addFormatters(FormatterRegistry registry) {
                         //添加一个新的自定义的Converter
                         registry.addConverter(new Converter<String, Pet>() {
         
                             @Override
                             //source是页面提交过来的值
                             public Pet convert(String source) {
                                 if (!StringUtils.isEmpty(source)){
                                     Pet pet = new Pet();
                                     String[] split = source.split(",");
                                     pet.setName(split[0]);
                                     pet.setAge(Integer.parseInt(split[1]));
                                     return pet;
                                 }
                                 return null;
                             }
                         });
                     }
                 };
             }
         }
        ```
