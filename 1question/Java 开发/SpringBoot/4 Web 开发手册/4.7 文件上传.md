# 1. 前端表单设置
1. 前端表单要求：为了能上传文件，必须将表单的 **method设置为 POST**，并将 **enctype设置为 multipart/form-data**
    
    > 只有在这样的情况下，浏览器才会把用户选择的文件以二进制数据发送给服务器
    
1. 对表单中的 enctype 属性做个详细的说明：
    
    1. **application/x-www=form-urlencoded：**默认方式，只处理表单域中的 value 属性值，采用这种编码方式的表单会将表单域中的值处理成 URL 编码方式。
    
    1. **multipart/form-data**：这种编码方式会以二进制流的方式来处理表单数据，这种编码方式会把文件域指定文件的内容也封装到请求参数中，不会对字符编码
    
    1. **text/plain：**除了把空格转换为 "+" 号外，其他字符都不做编码处理，这种方式适用直接通过表单发送邮件
    
1. 一旦设置了enctype为multipart/form-data，浏览器即会采用**二进制流的方式来处理表单数据**，而对于文件上传的处理则涉及在服务器端解析原始的HTTP响应
# 2. **文件上传使用**
1. 文件上传表单
    
    ```HTML
     <form role="form" th:action="@{/upload}" method="post" enctype="multipart/form-data"> ⭐
         <div class="form-group">
             <label for="exampleInputEmail1">邮箱</label>
             <input name="email" type="email" class="form-control" placeholder="Enter email">
         </div>
         <div class="form-group">
             <label for="exampleInputPassword1">名字</label>
             <input name="username" type="text" class="form-control" placeholder="Password">
         </div>
         <div class="form-group">
             <label for="exampleInputFile">头像</label>
             <input name="headImage" type="file" id="exampleInputFile">
         </div>
         <div class="form-group">
             <label for="exampleInputFile">图集</label>
             <input name="photos" type="file" multiple> ⭐
         </div>
         </div>
         <button type="submit" class="btn btn-primary">提交</button>
     </form>
    ```
    
1. handler：**@RequestPart**
    
    ```Java
     @Slf4j
     @Controller
     public class formTestController {
         @GetMapping("/form_layouts")
         public String FormTestController(){
             return "form/form_layouts";
         }
     
         @PostMapping("/upload")
         public String upload(@RequestParam("email") String email,
                              @RequestParam("username") String username,
                              @RequestPart("headImage")MultipartFile headImage, //headImage 头像，photos 生活照
                              @RequestPart("photos") MultipartFile[] photos) throws IOException {
             log.info("上传信息：email={},username={},headImage={},photos={}",
                     email,username,headImage.getSize(),photos.length);
     
             //🟡将上传的文件封装成一个 MultipartFile ，MultipartFile提供了很多API供操作
    
             if (!headImage.isEmpty()){
                 String originalFilename = headImage.getOriginalFilename();
    
                 //保存文件
                 //这里的地址是本地，但也可以换成服务器上的地址（模拟将文件上传在服务器中）
                 headImage.transferTo(new File("F:\\fileTest\\"+originalFilename));
             }
     
             if (photos.length > 0){
                 for (MultipartFile photo : photos) {
                     if (!photo.isEmpty()){
                         String originalFilename = photo.getOriginalFilename();
                         photo.transferTo(new File("F:\\fileTest\\photos\\"+originalFilename));
                     }
                 }
             }
             return "main";
         }
     }
    ```

# 3. **文件上传原理**
1. 文件上传自动配置类 **MultipartAutoConfiguration**
1. 使用**MultipartProperties.class** 绑定配置类 MultipartAutoConfiguration
1. 底层自动配置好了 **StandardServletMultipartResolver** （文件上传解析器）
    
    > springMVC则需要在上下文中配置 MultipartResolver，才能实现文件上传
    
1. 原理步骤：
    
    1. 请求进来使用 文件上传解析器 判断是否是文件上传请求（**isMultipart( )**）
        
        ```Java
         \#通过表单上的 enctype 识别
         enctype="multipart/form-data"
        ```
        
    
    1. 使用方法resolveMultipart() ，将文件上传请求封装，封装为 **MultipartHttpServletRequest**
    
    1. 参数解析器来 解析 请求中的文件内容封装成 **MultipartFile** （@RequestPart 注解）
    
    1. 将request中文件信息封装为一个Map（**MultiValueMap<String, MultipartFile>**）
        
        > 也可以使用 FileCopyUtils 来实现文件流的拷贝
