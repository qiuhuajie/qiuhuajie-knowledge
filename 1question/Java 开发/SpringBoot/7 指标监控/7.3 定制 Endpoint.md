# 1. **自定义 Health 的信息**
1. **Health 中的 status 的判断逻辑：**
    
    ![[IMG-20260405035413880.png|Untitled 340.png]]
    
    1. 所有 components 的状态判断都为健康，则返回 up
    
    1. components 中有一个的状态判断为不健康，则返回 down
    
1. **自定义需求：**
    
    1. 当前有一个新加的组件，他的健康状态我们很看重
    
    1. 就需要将他添加进 components，将他的健康状态作为判断整个服务的状态的依据
    
1. **实现方式：**
    
    1. 配置文件
        
        ```Plain
         management:
           endpoints:
             web:
               exposure:
                 include: '*'
           endpoint:
             health:
               show-details: always
        ```
        
    
    1. **编写一个类实现 AbstractHealthIndicator 类（类名截去HealthIndicator，剩下的就是监控中component 的名字）**
        
        ```Plain
         @Component //记得加载组件
         public class MyHealthIndicator extends AbstractHealthIndicator {
             @Override
             protected void doHealthCheck(Health.Builder builder) throws Exception {
                 Map<String,Object> map = new HashMap<>();
                 //我们看重的新添加的组件 状态判断完成后，返回一些值
                 //这些值作为状态判断的条件（当前模拟使用 1 == 2）
                 if(1 == 2){
                     builder.status(Status.UP);
                     map.put("count",1);
                     map.put("ms",100);
                 }else {
                     builder.status(Status.OUT_OF_SERVICE);
                     map.put("err","连接超时");
                     map.put("ms",3000);
                 }
                 builder.withDetail("code",100)  //withDetail()可以写入单个键值对
                         .withDetails(map);      //withDetails(map)可以写入多个键值对
             }
         }
        ```
        
    
1. **测试：**
    
    [http://localhost:3344/actuator/health](http://localhost:3344/actuator/health)
    
    ![[IMG-20260404031837419.png|Untitled 1 257.png]]
    
    现在，只有我们新加的组件判断也为健康，整个服务的 status 才为 UP

# 3. **自定义 Metrics 的信息**
1. SpringBoot支持自动适配的Metrics
    
    - JVM metrics, report utilization of:
    
    - Various memory and buffer poolsStatistics related to garbage collectionThreads utilizationNumber of classes loaded/unloaded
    
    - CPU metrics
    
    - File descriptor metrics
    
    - Kafka consumer and producer metrics
    
    - Log4j2 metrics: record the number of events logged to Log4j2 at each level
    
    - Logback metrics: record the number of events logged to Logback at each level
    
    - Uptime metrics: report a gauge for uptime and a fixed gauge representing the application’s absolute start time
    
    - Tomcat metrics (`**server.tomcat.mbeanregistry.enabled**` must be set to `**true**` for all Tomcat metrics to be registered)
    
    - [Spring Integration](https://docs.spring.io/spring-integration/docs/5.4.1/reference/html/system-management.html#micrometer-integration) metrics
    
1. **增加定制Metrics**
    
    1. 需求：统计一个请求被访问了多少次？
    
    1. 原先的业务逻辑：
        
        1. **service**
            
            ```Plain
             @Service
             public class myService {
             
                 public String getName() {
                     return "张三（业务逻辑）";
                 }
             }
            ```
            
        
        1. **controller**
            
            ```Plain
             @RestController
             public class myController {
             
                 @Autowired
                 myService myService;
             
                 @GetMapping("testMyMetrics")
                 public String testMyMetrics(){
                     return myService.getName();
                 }
             }
            ```
            
        
    
    1. **添加定制 Metrics 监控 统计代码**
        
        ```Plain
         @Service
         public class myService {
        
             //import io.micrometer.core.instrument.Counter;
             Counter counter;
        
             public myService(MeterRegistry meterRegistry){
        
                 //将计数器注册在 Metrics 监控项中，并命名
                 counter = meterRegistry.counter("com.springcloud.service.myService.getName");
             }
         
             public String getName() {
        
                 //每执行一次业务逻辑就将 计数器加一
                 counter.increment();
                 return "张三（业务逻辑）";
             }
         }
        ```
        
    
    1. **测试**
        
        1. [http://localhost:3344/testMyMetrics](http://localhost:3344/testMyMetrics)（刷新访问多次）
            
            ![[IMG-20260405035420283.png|Untitled 2 215.png]]
            
        
        1. [http://localhost:3344/actuator/metrics/com.springcloud.service.myService.getName](http://localhost:3344/actuator/metrics/com.springcloud.service.myService.getName)
            
            可以统计到 上面的业务逻辑被访问了 8 次

# 4. **自定义 Endpoint**
1. 上面的只是修改了 提供的 Endpoint 的信息
1. 能否直接自定义一个 Endpoint ？
    
    ```Plain
     @Component
     @Endpoint(id = "container") //添加为端点，并命名
     public class DockerEndpoint {
     
         @ReadOperation
         public Map getDockerInfo(){
             return Collections.singletonMap("info","docker started...");
         }
     
         @WriteOperation
         private void restartDocker(){
             System.out.println("docker restarted....");
         }
     
     }
    ```
