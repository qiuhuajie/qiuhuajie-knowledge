# 一、总结 🙋‍♂️
1. SpringBoot 的自动配置可以帮助开发人员快速创建出一个可用的应用程序
2. 主要依靠一个核心注解 **`@EnableAutoConfiguration`** 来实现，这个注解通过 **`@SpringBootApplication`** 被间接地标记在了 Spring Boot 的启动类上
3. `@EnableAutoConfiguration` 也是一个**组合注解**，它包含了 **`@Import(AutoConfigurationImportSelector.class)`** 注解
    1. **`AutoConfigurationImportSelector`** 类则是 Spring Boot 自动装配的核心类
    2. 这个类会**通过 ClassLoader 获取 classpath 中的配置文件** **`META-INF/spring.factories`** **（**==**SPI机制**==**）**
    3. 这个 `spring.factories` 文件中是以 `key = value` 形式组织的，**从中筛选出以 `EnableAutoConfiguration` 为key 的、符合条件的配置类**
    4. 接着再根据每个配置类上的 **`@Conditional` 注解所配置的条件**，按需**过滤**掉一些不必要的自动配置类

# 二、从 @SpringBootApplication() 开始分析

![[IMG-20260404031828519.png|800]]

1. 从SpringBoot 核心配置开始分析

    ```Java
     @SpringBootApplication()
     public class MainApplication {
         public static void main(String[] args) {
    ```
2. **`@SpringBootApplication()`** 是下面三个注解配置合起来的替换

    ```Java
     //查看源码
     @SpringBootConfiguration
     @EnableAutoConfiguration
     @ComponentScan()
    ```
## 1. @SpringBootConfiguration
1. 和 @Configuration 一样代表当前是一个配置类
2. 因此，SpringBoot 启动时，也会将 `@SpringBootApplication()` 注解的启动类也作为配置加载，称为 **源配置类**
## 2. @ComponentScan()
1. 指定去哪个包下扫描Spring注解
2. 无需以前的组件扫描配置

    ```XML
     <context:component-scan base-package="com.atguigu"></context:component-scan>
    ```
3. **主程序所在包 及其 下面的所有子包里面的组件**都会被默认扫描进来

    ![[IMG-20260405035420369.png|469]]

4. 如果想要改变扫描路径：设置 `scanBasePackages` 参数

    ```Java
     @SpringBootApplication(scanBasePackages = "com.atguigu.boot")
     public class MainApplication {
     }
    ```

    相当于

```Java
 //@SpringBootApplication 等同于下面三个注解
 @SpringBootConfiguration
 @EnableAutoConfiguration
 @ComponentScan("com.atguigu.boot")
 public class MainApplication {
 }
```
## 3. @EnableAutoConfiguration ⭐
1. **这个注解是最核心的，他会导入** **`AutoConfigurationImportSelector`****，这个类会将所有符合条件的 `@Configuration` 配置都进行加载**
2. 点击查看 @EnableAutoConfiguration 源码，`@EnableAutoConfiguration` 又是下面两个注解合起来的替换

    ```Java
     @AutoConfigurationPackage
     @Import({AutoConfigurationImportSelector.class})
    ```
### 3.1 注解 1：@AutoConfigurationPackage
1. 字面意思：自动配置包，指定了默认的包规则
2. 查看源码：

    ```Java
     @Import({Registrar.class})
     public @interface AutoConfigurationPackage {
         String[] basePackages() default {};
         Class<?>[] basePackageClasses() default {};
     }
    ```

    再进入Registrar.class的源码

```Java
 static class Registrar implements ImportBeanDefinitionRegistrar, DeterminableImports {
         Registrar() {
         }
         public void registerBeanDefinitions(AnnotationMetadata metadata, BeanDefinitionRegistry registry) {
             //⭐
             AutoConfigurationPackages.register(registry, (String[])(new AutoConfigurationPackages.PackageImports(metadata)).getPackageNames().toArray(new String[0]));
         }
         ...
     }
```
    1. `AnnotationMetadata metadata` 中包含注解的所有元信息

    ![[IMG-20260405035422095.png|800]]

    2. `new AutoConfigurationPackages.PackageImports(metadata)).getPackageNames()` 使用注解元信息 metadata，获取到当前类所在的包名（也就是main程序所在的包）

    ![[IMG-20260405035434340.png|596]]

    3. 这也解释了之前 1.5.2 中：主程序所在包 及其 下面的所有子包里面的组件都会被默认扫描进来（源码就是这样写的规则）
    4. 得到当前类所在的包名后，`AutoConfigurationPackages.register()` 会把包下的所有组件封装到数组中，再批量注册注册到容器当中

### 3.2 注解 2：@Import({AutoConfigurationImportSelector.class}) ⭐

![[IMG-20260405035413990.png|800]]

1. `@Import` 导入了一个 **==AutoConfigurationImportSelector==**
    1. 这个`@Import` 注解通过 `@EnableAutoConfiguration` 注解，再通过 `@SpringBootApplication` 被间接的标记在了 Spring Boot 的启动类上
    2. 因此在 `SpringApplication.run(...)` 的内部就会执行 AutoConfigurationImportSelector 的 `selectImports()` 方法，找到所有需要自动配置类的全限定名对应的 class
    3. 下面分析一下这个 `selectImports()` 方法
2. AutoConfigurationImportSelector 的 `selectImports()` 方法通过 ==SpringFacories 机制==的 `SpringFactoriesLoader.loadFactoryNames()` ==**扫描所有具有 META-INF/spring.factories 的 jar 包**==
3. `spring-boot-autoconfigure-x.x.x.x.jar`里就有一个这样的`spring.factories`文件

    ![[IMG-20260405035445633.png|800]]

    - 这个`spring.factories`文件也是一组一组的 key=value 的形式
    - **其中有一个 key 是 `EnableAutoConfiguration` 类的全类名。自动配置时，只会获取这个 key 对应的 value**
        - 而它的 value 是一个`xxxxAutoConfiguration`的类名的列表
        - 这些类名以逗号分隔
4. `selectImports()` 方法中的 `getAutoConfigurationEntry()` 就是 SpringBoot 的自动配置入口方法

    ![[IMG-20260405035505354.png|800]]

5. 查看 `getAutoConfigurationEntry()` 方法源码

    ![[IMG-20260405035512152.png|800]]

    - 分析第二步：查看 `getCandidateConfigurations()` 方法的源码，发现底层确实是使用了 SpringFacories 机制的 `SpringFactoriesLoader.loadFactoryNames()`

    ![[IMG-20260405035520658.png|800]]

    - 分析第四步：此时拿到了所有的可配置类名，即 127 个场景的所有自动配置启动的时候默认全部加载
        - 但是在项目中并不是全都用到，**需要的会在 `.properties` 配置文件中配置开启**
        - 故在自动加载时，会通过条件注解，**只配置需要开启的类**
        - 例如 AopAutoConfiguration 中写有条件，会按照按照条件装配规则（**@Conditional**）按需配置

    ![[IMG-20260405035520731.png|800]]

6. **至此 SpringBoot 获取到了所有需要自动配置的类名对应的 class**
7. 做到了⭐==**用户只需要引入 jar 包即可，至于 jar 包中有哪些自动配置类，类名是什么等等，这些信息用户都不需要关心**==
