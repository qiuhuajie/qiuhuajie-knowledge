# 一、介绍
1. 🙋‍♂️**`final`** **关键字的使用**
    1. **final 修饰一个类：**此类==不能被其他类所继承==，比如 String 类、System 类
    2. **final 修饰一个方法：**此方法==可以被重载==，但此方法==不能被子类重写==
        > private 方法隐式地被指定为 final，如果在子类中定义的方法和基类中的一个 private 方法签名相同，此时子类的方法不是重写基类方法，而是在子类中定义了一个新的方法。
    3. **final 声明一个数据为常量：**可以是编译时常量，也可以是在运行时被初始化后不能被改变的常量
        - 对于基本类型，final 使数值不变
        - 对于引用类型，final 使引用不变，也就不能引用其它对象，但是被引用的对象本身是可以修改的
2. 🙋‍♂️**`final`** **和** **`abstract`** **能一起使用吗**❓
    1. 不可以
    2. 在Java编程语言中，`final` 和 `abstract`这两个关键字是有**矛盾**的
        1. 因为`final` 表示最终的、不可改变的
        2. 而`abstract` 表示抽象的、未实现的
    3. 如果**一个方法被声明为** **`final`** **，它将不能被子类重写，而** **`abstract`** **则要求子类实现该方法**。因此，这两个关键字不能同时使用
    4. 类也是如此，**一个类被声明为** **`final`** **，意味着它不能被继承，而一个类被声明为** **`abstract`** **，则要求子类实现其抽象方法**
    5. 因此，如果你尝试在同一个类或方法上同时使用final和abstract关键字，编译器将会抛出错误

3. **对** `final` **变量读取**
    1. 当变量值较小时，会复制一份在**栈**内存中
    2. 当变量值超过了短整型的最大值，会复制一份在**常量池**中
    3. 如果不加 `final` 修饰变量，会存在**堆**中，读取时的效率要比前两种慢

# 二、原理
1. 理解了 `volatile` 原理，再对比 `final` 的实现就比较简单了
2. **对** `final` **变量赋值**
    1. 示例

    ```Java
    public class TestFinal {
    		final int a = 20;
    }
    ```

        字节码

```Java
0: aload_0
1: invokespecial #1 // Method java/lang/Object."<init>":()V
4: aload_0
5: bipush 20
7: putfield #2 // Field a:I
<-- 写屏障⭐
10: return
```
    2. 发现 final 变量的赋值也会通过 **`putfield`** 指令来完成，**同样在这条指令之后也会加入**==**写屏障**==，**保证在其它线程读到它的值时，会读到主存中的最新值**，不会出现为 0 的情况
