# 一、实参与形参
1. **形参：**
    1. 形式参数
    2. 定义方法的时候使用的参数，目的是用来接收调用者传递的参数的
2. **实参：**
    1. 实际参数
    2. 在调用有参方法时传入的参数，用于调用时传递给方法的参数
    3. 实参在传递给别的方法之前是要被预先赋值的
3. 示例：

    ```Java
    public class Demo {
        public static void main(String[] args) {
            String name = "Java";
    				// 这里的name就是实际参数
            update(name);
            System.out.println(name);
        }
        // 这里方法参数列表中name就是形式参数
        private static void update(String name) {
            // doSomething
        }
    ```
# 二、值传递与引用传递⭐
1. **值传递（pass by value）**
    1. 是指在调用方法时**将实际参数拷贝一份传递到方法中**
    2. 这样在方法中如果对参数进行修改，将不会影响到实际参数
2. **引用传递（pass by reference）**
    1. 是指在调用方法时**将实际参数的地址直接传递到方法中**
    2. 那么在方法中对参数所进行的修改，将影响到实际参数

# 三、Java 的传递方式
1. 在Java中到底是引用传递还是值传递呢？其实这个问题也一直是争论不断，而且官方也没给个确切答案
2. 结论：==Java是== **==值转递==**
## 1. 当传递的是基本类型
1. 代码示例：

    ```Java
    public class Demo {
        public static void main(String[] args) {
            int count = 0;
            update(count);
            System.out.println("main方法中count：" + count);
        }
        private static void update(int count) {
            count++;
            System.out.println("update方法中count：" + count);
        }
    }
    ```

    结果：

```Java
update方法中count：1
main方法中count：0
```
2. 可以看到虽然update方法修改了形参count的值，但是main方法中实参count的值并没有变
3. 原因解析：
    > 💡 **前提知识**
    > [[2.2 虚拟机栈]][[2.2 虚拟机栈]][[2.2 虚拟机栈]]
    > 1. 虚拟机栈中存放着栈帧，方法调用的过程，就是栈帧在栈中入栈、出栈的过程
    > 2. 一个方法中的局部变量是存在栈中的
    >     3. 如果是**基本类型**的变量则**直接存的是这个变量的值**
    >     4. 如果是**引用类型**的变量则存的是**值的地址**，指向堆中具体的值
    1. 当执行main方法的时候，就往虚拟机栈中压入一个栈帧，栈帧中存储的局部变量信息是`count=0`

    ![[IMG-20260620222652307.png|237]]

    2. 当执行update方法的时候，再往虚拟机栈中压入一个栈帧，栈帧中存储的局部变量信息是`count=0`

    ![[IMG-20260620222652384.png|231]]

    3. 本例中，**==调用update方法传递的count，其实是count变量的拷贝，不是真正的count，在update方法中改变的是拷贝，对真正的count是没有影响的==**
    4. 因此验证是**值传递**

## 2. 当传递的是引用类型
### 2.1 代码示例一
```Java
public class Demo {
    public static void main(String[] args) {
        User user = new User();
        user.setId(0);
        update(user);
        System.out.println("main方法中user：" + user);
    }
    private static void update(User user) {
        user.setId(1);
        System.out.println("update方法中user：" + user);
    }
}
```

结果：

```Java
update方法中user：User(id=1)
main方法中user：User(id=1)
```
1. 当执行main方法时，会在堆内存中开辟一块内存，在栈内存中压入一个栈帧，栈帧中存储一个引用，指向堆内存中的地址

    ![[IMG-20260620222652433.png|433]]

2. 当调用update方法时，会把main方法的栈帧**拷贝一份**，再压入栈内存中，指向同一个堆内存地址

    ![[IMG-20260620222652479.png|480]]

3. 可以看出update方法修改user对象的属性，main方法中user对象也跟着变了
4. ⭐这个看起来像是引用传递
    1. 因为update中的修改对main中的对象产生了影响
    2. 但是这并不是引用传递，**仍然是值传递**
        1. 因为引用传递说的是 “ 在调用方法时==将实际参数的地址==**==直接传递==**==到方法中==，那么在方法中对参数所进行的修改，将影响到实际参数”
        2. 而例子中这里在参数传递的过程中，只是把实参的地址**拷贝**了一份传递给形参（使用了值传递），==update方法中只修改了参数地址里面的内容，并没有对形参本身进行修改==，因此属于值传递
        3. 上面说到：**引用类型**的变量在栈中存的是**值的地址**，故这里拷贝的只是栈中的引用地址，并不会拷贝堆中的数据
        4. 因此，虽然变量是拷贝，但是指向的地址是同一个，因此对变量中的数据修改时，还是会影响到原来真实的变量
    3. 因此，并不能只根据实参对象是否受到了影响，来判断是值传递或引用传递

### 2.2 代码示例二
```Java
public class Demo {
    public static void main(String[] args) {
        User user = new User();
        user.setId(0);
        update(user);
        System.out.println("main方法中user：" + user);
    }
    private static void update(User user) {
				// 注意这里新创建了一个对象，并赋值给了形参，相当于修改了形参中的引用地址值
        user = new User();
        user.setId(1);
        System.out.println("update方法中user：" + user);
    }
}
```

结果：

```Java
update方法中user：User(id=1)
main方法中user：User(id=0)
```
1. 可以看到示例二中的update方法修改user对象的属性，main方法中user对象并没有改变
2. 原因分析
    1. 当执行main方法时，会在堆内存中开辟一块内存，在栈内存中压入一个栈帧，栈帧中存储一个引用，指向堆内存中的地址

    ![[IMG-20260620222652524.png|448]]

    2. 当调用update方法时，会把main方法的栈帧拷贝一份，再压入栈内存中，指向同一个堆内存地址

    ![[IMG-20260620222652555.png|466]]

    3. 当执行update方法，**重新初始化user对象，并重新赋值的时候。会在堆内存中再开辟一块内存，再把栈内存中update栈帧指向新的堆内存地址**，并修改新的堆内存中的数据

    ![[IMG-20260620222652616.png|475]]

    4. 示例二中的也符合**值传递**，由于update中的`user = new User();` 语句，使得update中修改的对象并不是实参本身（形参中存储的引用地址值被修改了），所以和示例一不同：实参对象没有被影响
