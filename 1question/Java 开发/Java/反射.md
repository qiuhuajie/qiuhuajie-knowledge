# 1. 介绍
1. **Java 语言的反射机制：**
    - 在运行时（动态编译）==获知任意一个对象所属的类==
    - 在运行时，==构造任意一个类的对象==
    - 在运行时，==获知任意一个类所具有的成员变量和方法==
    - 在运行时，==调用任意一个对象的方法和属性==
2. **反射的优缺点：**
    1. 优点：比较==灵活==，能够在运行时动态获取类的实例
    2. 缺点：
        1. ==性能瓶颈==：反射相当于一系列解释操作，通知 JVM 要做的事情，性能比直接的 Java 代码要慢很多
        2. ==安全问题==：反射机制破坏了封装性，因为通过反射可以获取并调用类的私有方法和字段
3. **反射的应用：**实际编程中其实并不会直接大量的使用
    - ==动态代理机制==（[[代理模式 ⭐]]）
    - 使用 ==JDBC 连接数据库==（[[MyBatis 面试题]]）
    - ==Spring 框架==
        - **Spring IoC**（[[2.6 获取 Bean 的方式]]）
        - **Spring AOP**（[[3.4 AOP 实现原理]]，实际上是因为使用了动态代理，所以才和反射机制有关）

    1. 这个信息跟踪着每个对象所属的类的完整结构信息（==`instanceKlass`==），包括包名、类名、实现的接口、拥有的方法和字段等
    2. 可以通过 `Class` 类对象（称为类的类型对象，`==_java_mirror==` ）访问这些信息，一个 `Class` 对象对应一个加载到 JVM 中的一个 `.class` 文件
3. **==反射：==**
    1. **那么在加载完一个类后，堆内存就产生了一个 `Class` 对象**，包含了完整的类的结构信息，我们可以通过这个 `Class` 对象看到类的结构
    2. 在通常情况下，一定是先有类再有对象，我们把这个通常情况称为 “正”。那么反射中的这个 “反” 我们就可以理解为根据对象找到对象所属的类（对象的出处）
    3. 通过反射，也就是调用了 **`getClass()`** 方法后，我们就获得了一个类对应的 `Class` 对象，看到了这个类的结构，即找到了对象的出处

# 4. 通过反射构造一个类的实例
1. 成功获取 `Class` 类对象后，我们就需要构造对应类的实例
2. 方式一：**使用 Class.newInstance**

    ```Java
    Date date1 = new Date();
    Class alunbarClass2 = date1.getClass();
    Date date2 = alunbarClass2.newInstance(); // 创建一个与 alunbarClass2 具有相同类类型的实例
    ```
3. 方式二：**通过反射先获取构造方法再调用**
    1. 方式一的局限性：由于不是所有的类都有无参构造函数又或者类构造器是 `private` 的，在这样的情况下，如果我们还想通过反射来实例化对象，**`Class.newInstance`** 是无法满足的
    2. 此时，我们可以使用 `Constructor` 的 **`newInstance`** 方法来实现，先获取构造函数，再执行构造函数

        ```Java
        Class clazz = Class.forName("Student");
        Constructor con = clazz.getConstructor(null);
        Object obj = con.newInstance(); 
        ```
    3. 获取构造函数的方法：

        ```Java
        // 获取所有公有构造方法
        Constructor[] conArray = clazz.getConstructors();
        // 获取所有的构造方法(包括：私有、受保护、默认、公有)
        conArray = clazz.getDeclaredConstructors();
        // 获取一个指定参数类型的公有构造方法
        Constructor con = clazz.getConstructor(Class... parameterTypes);
        // 获取公有、无参的构造方法
        Constructor con = clazz.getConstructor(null);
        // 获取一个指定参数类型的"构造方法"，可以是私有的，或受保护、默认、公有的构造方法
        Constructor con = clazz.getDeclaredConstructor(Class... parameterTypes);
        ```

3. 代码示例：

    ```Java
    // 获取 Class 对象
    Class stuClass = Class.forName("fanshe.field.Student");
    // 获取所有公有的字段
    Field[] fieldArray = stuClass.getFields();
    // 获取所有的字段 (包括私有、受保护、默认的)
    fieldArray = stuClass.getDeclaredFields();
    // 获取指定名称的公有字段
    Field f = stuClass.getField("name");
    Object obj = con.newInstance(); // 调用构造函数，创建该类的实例
    f.set(obj, "刘德华"); // 为 Student 对象中的 name 属性赋值
    // 获取私有字段
    f = stuClass.getDeclaredField("phoneNum");
    f.setAccessible(true); // 暴力反射，解除私有限定
    f.set(obj, "18888889999"); // 为 Student 对象中的 phoneNum (private)属性赋值
    ```

    2. 获取所有的成员方法，包括私有的（不包括继承的）

        ![[IMG-20260405035510621.png|Untitled 9 13.png]]

2. **单个获取**：
    1. 获取一个指定方法名和参数类型的成员方法：

        ![[IMG-20260405035517804.png|Untitled 10 13.png]]

3. **调用：**
    1. `invoke` 方法
    2. 包含两个参数：
        - obj：哪个对象要来调用这个方法
        - args：调用方法时所传递的实参
4. 代码示例：

    ```Java
    public class Student {
    	public void show1(String s){...}
    	private String show4(int age){... return "abcd";}
    }
    // 获取 Class对象
    Class stuClass = Class.forName("Student");
    // 获取公有的无参构造函数
    Constructor con = stuClass.getConstructor();
    // 获取所有公有方法
    Method[] methodArray = stuClass.getMethods();
    // 获取所有的方法，包括私有的
    methodArray = stuClass.getDeclaredMethods();
    // 获取公有的show1()方法
    Method m = stuClass.getMethod("show1", String.class);
    Object obj = con.newInstance(); // 调用构造函数，实例化一个 Student 对象
    m.invoke(obj, "小牛肉");
    // 获取私有的show4()方法
    m = stuClass.getDeclaredMethod("show4", int.class);
    m.setAccessible(true); // 解除私有限定
    Object result = m.invoke(obj, 20);
    System.out.println("返回值：" + result);
    ```
