# 一、异常概述
1. 开发的过程中，**很多问题不是靠代码能够避免的**，比如： 客户输入数据的格式， 读取文件是否存在， 网络是否始终保持通畅等等
2. 异常：在Java语言中， 将程序执行中发生的不正常情况称为“异常”
    > 💡 注意：开发过程中的语法错误和逻辑错误不是异常
# 二、异常分类

![[IMG-20260620222445662.png|765]]

1. 异常可以分为两类：
    1. **`Error`**
        1. Java虚拟机无法解决的严重问题
        2. 如： JVM系统内部错误、 资源耗尽等严重情况，栈溢出 StackOverflowError 和堆溢出 OutOfMemoryError
        3. 一般不编写针对性的代码进行处理
    2. **`Exception`**
        1. 其它因编程错误或偶然的外在因素导致的一般性问题， 可以使用针对性的代码进行处理
        2. 例如：空指针访问、试图读取不存在的文件、数组角标越界
2. 体系结构：
    - java.lang.`Throwable`
        - java.lang.`Error`：一般不编写针对性的代码进行处理
        - java.lang.`Exception`：可以进行异常的处理
            - **编译时异常**（checked）
                - IOException
                    - FileNotFoundException
                - ClassNotFoundException
            - **运行时异常**（unchecked）
                - NullPointerExceptionkon 空指针
                - ArrayIndexOutOfBoundsException 位越界
                - ClassCastException
                - NumberFormatException
                - InputMisamatchException 输入不匹配
                - ArithmaticException 算数异常

# 三、异常处理机制
1. 异常处理概述：抓抛模型
    1. 抛：程序在正常执行的过程中，一旦出现异常，就会在异常代码处生成一个对应异常类的对象，并将此对象抛出，一旦抛出异常后，其后的代码不再执行
    2. 抓：可以理解为异常的处理方式
        1. try-catch-finally
        2. throws + 异常类型

    ![[IMG-20260620222445726.png|490]]

## 1. try-catch-finally
1. 说明：
    1. 使用try将可能出现的异常包装起来，在执行的过程中，一旦出现异常，就会生成一个对应异常类的对象，根据此对象的类型，取catch中进行匹配
    2. 一旦 try 中的异常匹配到一个catch，就进入catch中进行异常的处理
    3. catch中的异常没有子父类关系，则谁声明的前后顺序无所谓；若catch中的异常有子父类关系，要求子类一定在父类的上面，否则报错
    4. finally的使用
        1. 即使 1️⃣ catch中又出现了异常，2️⃣ try中有return语句，3️⃣ catch中有return语句，finally中声明的语句一定会被执行
        2. 因此可以用于资源释放

## 2. throws + 异常类型
1. `throws + 异常类型` ==用于在方法声明中标识该方法可能会抛出的异常==
2. 一旦当方法体执行时，出现异常，仍会在异常代码处生成一个异常类的对象，此对象满足 throws 后异常类型时，就会被抛出

    ![[IMG-20260620222445812.png|587]]

3. ==throws 只是将异常抛给了调用者，并没有真正的处理异常==
    1. 异常代码后面的代码就不会被执行了，因为在本方法内并没有被处理，而是被抛给了上层方法
    2. 而 try-catch-finally 是真正的将异常给处理掉了

# 四、手动抛出异常：throw
1. Java 异常类对象除在程序执行过程中出现异常时由系统自动生成并抛出， 也可根据需要使用==人工创建并抛出==
2. 示例：

    ```Java
    public void regist(int id) throws Exception{
           if(id > 0){
               this.id = id;
           }else {
               //太low了
               //System.out.println("输入的id不合法");
               //如果抛一个RuntimeException，编译不会报错，但不会处理
               //throw new RuntimeException("输入的id不合法");
               //如果抛一个Exception，需要抛给main方法处理异常
               throw new Exception("输入的id不合法");
           }
       }
    ```
