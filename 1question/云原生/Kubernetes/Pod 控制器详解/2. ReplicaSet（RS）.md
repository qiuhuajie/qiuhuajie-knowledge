# 1. 介绍
1. ==**主要作用：**==
    
    1. **保证总有期望数量的 pod 在正常运行**
    
    1. 它会持续监听这些Pod的运行状态，一旦Pod发生故障，就会重启或重建
    
    1. 同时它还支持对pod数量的扩缩容和镜像版本的升降级
    
    ![[IMG-20260404031955165.png|Untitled 399.png]]
    
1. ReplicaSet的资源清单文件
    
    ```YAML
     apiVersion: apps/v1 # 版本号
     kind: ReplicaSet # 类型
     metadata: # 元数据
       name: # rs名称
       namespace: # 所属命名空间
       labels: \#标签
         controller: rs
     spec: # 详情描述
       replicas: 3 # 副本数量
       selector: # 选择器，通过它指定该控制器管理哪些pod
         matchLabels:      # Labels匹配规则
           app: nginx-pod
         matchExpressions: # Expressions匹配规则
           - {key: app, operator: In, values: [nginx-pod]}
       template: # 模板，当副本数量不足时，会根据下面的模板创建pod副本
         metadata:
           labels:
             app: nginx-pod
         spec:
           containers:
           - name: nginx
             image: nginx:1.17.1
             ports:
             - containerPort: 80
    ```
    
1. 在这里面，需要新了解的配置项就是`spec`下面几个选项：
    
    - **`replicas`**：指定副本数量，其实就是当前rs创建出来的pod的数量，默认为1
    
    - **`selector`**：选择器
        
        - 它的作用是建立pod控制器和pod之间的关联关系，采用的Label Selector机制
        
        - 在pod模板上定义label，在控制器上定义选择器，就可以表明当前控制器能管理哪些pod了
        
    
    - **`template`**：模板，就是当前控制器创建pod所使用的模板板，里面其实就是前一章学过的pod的定义
    
# 2. 操作
## 2.1 **创建 RS**
1. 创建pc-replicaset.yaml文件
    
    ```Plain
     apiVersion: apps/v1
     kind: ReplicaSet
     metadata:
       name: pc-replicaset
       namespace: dev
     spec:
       replicas: 3
       selector:
         matchLabels:
           app: nginx-pod
       template:
         metadata:
           labels:
             app: nginx-pod
         spec:
           containers:
           - name: nginx
             image: nginx:1.17.1
    ```
    
1. 创建 RS
    
    ```Plain
     [root@master ~]# kubectl create -f pc-replicaset.yaml -n dev
     replicaset.apps/pc-replicaset created
    ```
    
1. 查看 RS
    
    ```Plain
     [root@master ~]# kubectl get rs pc-replicaset -n dev -o wide
     NAME            DESIRED   CURRENT   READY   AGE   CONTAINERS   IMAGES         SELECTOR
     pc-replicaset   3         3         3       9s    nginx        nginx:1.17.1   app=nginx-pod
    ```
    
    - `DESIRED`:期望副本数量
    
    - `CURRENT`:当前副本数量
    
    - `READY`:已经准备好提供服务的副本数量
    
1. 查看当前控制器创建出来的pod
    
    ```Plain
     [root@master ~]# kubectl get pod -n dev
     NAME                        READY   STATUS    RESTARTS   AGE
     pc-replicaset-kpgnf         1/1     Running   0          20s
     pc-replicaset-th5fn         1/1     Running   0          20s
     pc-replicaset-xjr95         1/1     Running   0          20s
    ```
    
## 2.2 **扩缩容**
1. ==**动态的修改 pod 副本的数量**==
1. 编辑 rs 的副本数量，修改 `spec:replicas: 6`
    
    ```Plain
     [root@master ~]# kubectl edit rs pc-replicaset -n dev
     replicaset.apps/pc-replicaset edited
    ```
    
1. 查看pod
    
    ```Plain
     [root@master ~]# kubectl get pod -n dev
     NAME                        READY   STATUS    RESTARTS   AGE
     pc-replicaset-bc8hp         1/1     Running   0          6s
     pc-replicaset-k8kxw         1/1     Running   0          6s
     pc-replicaset-kpgnf         1/1     Running   0          35m
     pc-replicaset-th5fn         1/1     Running   0          35m
     pc-replicaset-xjr95         1/1     Running   0          35m
     pc-replicaset-xpdvh         1/1     Running   0          6s
    ```
    
1. 也可以直接使用命令实现：**使用`scale`命令实现扩缩容， 后面`--replicas=n`直接指定目标数量即可**
    
    ```Plain
     [root@master ~]# kubectl scale rs pc-replicaset --replicas=2 -n dev
     replicaset.apps/pc-replicaset scaled
    ```
    
1. 查看pod （已经有4个开始准备退出了）
    
    ```Plain
     [root@master ~]# kubectl get pod -n dev
     NAME                        READY   STATUS        RESTARTS   AGE
     pc-replicaset-bc8hp         0/1     Terminating   0          23s
     pc-replicaset-k8kxw         1/1     Terminating   0          23s
     pc-replicaset-kpgnf         1/1     Terminating   0          36m

     pc-replicaset-xpdvh         0/1     Terminating   0          23s
    ```

1. 稍等片刻，就只剩下2个了
    

## 2.3 **镜像升降级**
1. ==**动态修改容器镜像的版本**==
1. 编辑 rs 的容器镜像，修改`image: nginx:1.17.2`
    
    ```Plain
     [root@master ~]# kubectl edit rs pc-replicaset -n dev
     replicaset.apps/pc-replicaset edited
    ```
    
1. 再次查看，发现镜像版本已经变更了
    
    ```Plain
     [root@master ~]# kubectl get rs -n dev -o wide
     NAME                  DESIRED   CURRENT   READY   AGE   CONTAINERS   IMAGES         SELECTOR
     nginx-app-d6ff45774   3         3         3       47h   nginx        nginx          app=nginx-app,pod-template-hash=d6ff45774
     pc-replicaset         2         2         2       37m   nginx
     nginx:1.17.2   app=nginx-pod
    ```
    
1. 也可以使用命令完成：`kubectl set image rs rs名称 容器=镜像版本 -n namespace`
    
    ```Plain
     [root@master ~]# kubectl set image rs pc-replicaset nginx=nginx:1.17.1  -n dev
     replicaset.apps/pc-replicaset image updated
    ```
    
1. 再次查看，发现镜像版本已经变更了
    
    ```Plain
     [root@master ~]# kubectl get rs -n dev -o wide
     NAME                  DESIRED   CURRENT   READY   AGE   CONTAINERS   IMAGES         SELECTOR
     nginx-app-d6ff45774   3         3         3       47h   nginx        nginx          app=nginx-app,pod-template-hash=d6ff45774
     pc-replicaset         2         2         2       37m   nginx        nginx:1.17.1   app=nginx-pod
    ```
    
## 2.4 **删除 RS**
1. 使用`kubectl delete`命令==**会删除此 RS 以及它管理的 Pod**==
1. 在kubernetes删除RS前，会将RS的replicasclear调整为0，等待所有的Pod被删除后，在执行RS对象的删除
    
    ```Plain
     [root@master ~]# kubectl delete rs pc-replicaset -n dev
     replicaset.apps "pc-replicaset" deleted
     [root@master ~]# kubectl get pod -n dev -o wide
     No resources found in dev namespace.
    ```
    
1. 如果希望**仅仅删除 RS 对象**（保留Pod），可以使用kubectl delete命令时添加**`-cascade=false`**选项**（不推荐）**
    
    ```Plain
     [root@master ~]# kubectl create -f pc-replicaset.yaml -n dev
     replicaset.apps/pc-replicaset created
     [root@master ~]# kubectl get pod -n dev -o wide
     NAME                        READY   STATUS    RESTARTS   AGE     IP               NODE     NOMINATED NODE
     pc-replicaset-54kbk         1/1     Running   0          2s      10.244.140.110   node02   <none>
     pc-replicaset-fn4fn         1/1     Running   0          2s      10.244.140.111   node02   <none>
     pc-replicaset-hgk5l         1/1     Running   0          2s      10.244.140.109   node02   <none>
     
     [root@master ~]# kubectl delete rs pc-replicaset -n dev --cascade=false
     warning: --cascade=false is deprecated (boolean value) and can be replaced with --cascade=orphan.
     replicaset.apps "pc-replicaset" deleted
     
     # Pod 还都在
     [root@master ~]# kubectl get pod -n dev -o wide
     NAME                        READY   STATUS    RESTARTS   AGE     IP               NODE     NOMINATED NODE        pc-replicaset-54kbk         1/1     Running   0          15s     10.244.140.110   node02   <none>
     pc-replicaset-fn4fn         1/1     Running   0          15s     10.244.140.111   node02   <none>
     pc-replicaset-hgk5l         1/1     Running   0          15s     10.244.140.109   node02   <none>
     
     # RS 都b
     [root@master ~]# kubectl get rs -n dev -o wide
     NAME                  DESIRED   CURRENT   READY   AGE   CONTAINERS   IMAGES   SELECTOR
     nginx-app-d6ff45774   3         3         3       47h   nginx        nginx    app=nginx-app,pod-template-hash=d6ff45774
    ```
    
1. 也可以**使用 yaml 直接删除（推荐）**
    
    ```Plain
     [root@master ~]# kubectl delete -f pc-replicaset.yaml
    ```
