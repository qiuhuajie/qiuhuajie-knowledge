# 一、k8s 基本概念
1. 简称 K8s， 是用 8 代替 8 个字符“ubernete” 而成的缩写
2. kubernetes的本质是**一组服务器集群**
3. 目标：==**让部署容器化的应用简单并且高效，实现资源管理的自动化**==
4. 作用：
    1. **一个容器编排引擎**
    2. **在集群的每个节点上运行特定的程序，来对节点中的容器进行管理**
    3. **支持自动化部署、 大规模可伸缩、应用容器化管理**

# 二、k8s 功能
![[IMG-20260404031956697.png|800]]
1. **自动装箱**
    - 基于容器对应用运行环境的资源配置要求自动部署应用容器
2. **自我修复**
    - 当容器失败时，会对容器进行重启，或者 某一个容器崩溃，能够在 1 秒中左右迅速启动新的容器
    - 当所部署的 Node 节点有问题时， 会对容器进行重新部署和重新调度
    - 当容器未通过监控检查时， 会关闭此容器直到容器正常运行时，才会对外提供服务
3. **弹性伸缩**
    - 可以根据需要，自动对集群中正在运行的容器数量进行调整：并发访问量大时，水平扩展出更多的服务；并发访问量变小时，恢复到原先的少量服务，节省资源
    - 通过简单的命令、用户 UI 界面或基于 CPU 等资源使用情况， 对应用容器进行规模扩大或规模剪裁
4. **服务发现**
    - 对外提供统一的服务调度入口，且服务也可以通过自动发现的形式找到它所依赖的服务
    - 用户不需使用额外的服务发现机制，就能够基于 Kubernetes 自身能力实现服务发现和负载均衡
5. **负载均衡**
    - 如果一个服务起动了多个容器，能够自动实现请求的负载均衡
6. 滚动更新
    - 可以根据应用的变化，对应用容器运行的应用，进行一次性或批量式更新
7. **版本回退**
    - 如果发现新发布的程序版本有问题，可以立即回退到原来的版本
8. 密钥和配置管理
    - 在不需要重新构建镜像的情况下，可以部署和更新密钥和应用配置，类似热部署。
9. **存储编排**
    - 自动实现存储系统挂载及应用，特别对 有状态（Session、Cookie）应用实现数据持久化非常重要
    - 存储系统可以来自于本地目录、网络存储、公共云存储服务
10. 批处理
    - 提供一次性任务，定时任务； 满足批量数据处理和分析的场景
