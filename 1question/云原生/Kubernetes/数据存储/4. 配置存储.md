# 一、ConfigMap
1. ConfigMap是一种比较特殊的存储卷，它的主要作用是**用来**==**存储配置信息**==**的**
2. 创建configmap.yaml，内容如下：

    ```Plain
     apiVersion: v1
     kind: ConfigMap
     metadata:
       name: configmap
       namespace: dev
     data:                   # ⭐ 此ConfigMap存储的配置信息 data 是一个键值对形式：这里 info是键，两个信息键值对信息是值
       info: |               # 这里的键会成为配置文件的文件名
         username:admin
         password:123456
    ```
3. 接下来，使用此配置文件创建configmap

    ```Plain
     # 创建configmap
     [root@k8s-master01 ~]# kubectl create -f configmap.yaml
     configmap/configmap created
     # 查看configmap详情
     [root@k8s-master01 ~]# kubectl describe cm configmap -n dev
     Name:         configmap
     Namespace:    dev
     Labels:       <none>
     Annotations:  <none>
     Data
     ====
     info:
     ----
     username:admin
     password:123456
     Events:  <none>
    ```
4. 接下来**创建一个Pod**：pod-configmap.yaml，将**上面创建的configmap挂载进去**

    ```Plain
     apiVersion: v1
     kind: Pod
     metadata:
       name: pod-configmap
       namespace: dev
     spec:
       containers:
       - name: nginx
         image: nginx:1.17.1
         volumeMounts: # 将configmap挂载到目录
         - name: config
           mountPath: /configmap/config  # ⭐ ConfigMap 中的配置信息文件会存在pod容器的此目录中
       volumes: # 引用configmap
       - name: config
         configMap:
           name: configmap       # ⭐ 使用 ConfigMap 的名字将ConfigMap和Pod绑定
    ```
    ```Plain
     # 创建pod
     [root@k8s-master01 ~]# kubectl create -f pod-configmap.yaml
     pod/pod-configmap created
     # 查看pod
     [root@k8s-master01 ~]# kubectl get pod pod-configmap -n dev
     NAME            READY   STATUS    RESTARTS   AGE
     pod-configmap   1/1     Running   0          6s
     \#进入容器
     [root@k8s-master01 ~]# kubectl exec -it pod-configmap -n dev /bin/sh
     # cd /configmap/config/
     # ls
     info
     # more info
     username:admin
     password:123456
    ```
    - 可以看到映射已经成功，每个configmap都映射成了一个目录
        - key--->文件 value---->文件中的内容
        - 此时如果更新configmap的内容, 容器中的值也会**动态更新**

            ```Plain
             \#修改配置文件内容
             [root@k8s-master01 ~]# kubectl edit cm comfigmap -n dev
             comfigmap/pod-comfigmap edited
             \#进入容器
             [root@k8s-master01 ~]# kubectl exec -it pod-configmap -n dev /bin/sh
             # cd /configmap/config/
             # ls
             info
             # more info
             username:admin
             password:1111111   # 内容已经修改（这里需要等待一会才能看到更新）
            ```
# 二、Secret
1. 在kubernetes中，还存在一种和ConfigMap非常类似的对象，称为Secret对象
2. **它主要用于**==**存储敏感信息，例如密码、秘钥、证书**==**等等**
3. 首先使用base64对数据进行编码（保存的时候不再使用明文）

    ```Plain
     [root@k8s-master01 ~]# echo -n 'admin' | base64         \#将 username 编码
     YWRtaW4=
     [root@k8s-master01 ~]# echo -n '123456' | base64        \#将 password 编码
     MTIzNDU2
    ```
4. 接下来编写secret.yaml，并创建Secret

    ```Plain
     apiVersion: v1
     kind: Secret
     metadata:
       name: secret
       namespace: dev
     type: Opaque
     data:
       username: YWRtaW4=        # ⭐写入编码后的数据，Secret会自动解码
       password: MTIzNDU2
    ```
    ```Plain
     # 创建secret
     [root@k8s-master01 ~]# kubectl create -f secret.yaml
     secret/secret created
     # 查看secret详情
     [root@k8s-master01 ~]# kubectl describe secret secret -n dev
     Name:         secret
     Namespace:    dev
     Labels:       <none>
     Annotations:  <none>
     Type:  Opaque
     Data
     ====
     password:  6 bytes          # ⭐ 并不会像ConfigMap一样显示地展示配置数据
     username:  5 bytes
    ```
5. 创建pod-secret.yaml，将上面创建的secret挂载进去：

    ```Plain
     apiVersion: v1
     kind: Pod
     metadata:
       name: pod-secret
       namespace: dev
     spec:
       containers:
       - name: nginx
         image: nginx:1.17.1
         volumeMounts: # 将secret挂载到目录
         - name: config
           mountPath: /secret/config
       volumes:
       - name: config
         secret:
           secretName: secret
    ```
    ```Plain
     # 创建pod
     [root@k8s-master01 ~]# kubectl create -f pod-secret.yaml
     pod/pod-secret created
     # 查看pod
     [root@k8s-master01 ~]# kubectl get pod pod-secret -n dev
     NAME            READY   STATUS    RESTARTS   AGE
     pod-secret      1/1     Running   0          2m28s
     # ⭐ 进入容器，查看secret信息，发现已经自动解码了
     [root@k8s-master01 ~]# kubectl exec -it pod-secret /bin/sh -n dev
     / # ls /secret/config/
     password  username
     / # more /secret/config/username
     admin
     / # more /secret/config/password
     123456
    ```
