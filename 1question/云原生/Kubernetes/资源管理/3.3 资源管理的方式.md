# 一、命令式对象管理
1. 直接使用**命令**去操作kubernetes资源（==很少使用==）

    ```Plain
     kubectl run nginx-pod --image=nginx:1.17.1 --port=80
    ```
2. **案例：namespace / pod的创建和删除**
    1. 创建一个 namespace

        ```Plain
         [root@master ~]# kubectl create namespace dev
         namespace/dev created
        ```
    2. 查看 namespace

        ```Plain
         [root@master ~]# kubectl get ns
         NAME              STATUS   AGE
         default           Active   6d14h        # 不加名称空间的操作，默认是去 default 名称空间下找
         dev               Active   11s
         kube-node-lease   Active   6d14h
         kube-public       Active   6d14h
         kube-system       Active   6d14h
        ```
    3. 此namespace 下创建并运行一个 nginx 的 Pod

        ```Plain
         [root@master ~]# kubectl run pod --image=nginx:1.17.1 -n dev
         pod/pod created
        ```
    4. 查看新创建的 Pod

        ```Plain
         [root@master ~]# kubectl get pod
         NAME                     READY   STATUS    RESTARTS   AGE
         nginx-6799fc88d8-xpc28   1/1     Running   3          6d13h     # 这是之前测试集群时创建的 default 下的 nginx
         [root@master ~]# kubectl get pod -n dev
         NAME   READY   STATUS    RESTARTS   AGE
         pod    1/1     Running   0          21s
        ```
    5. 查看新建 Pod 的详细信息

        ```Plain
         [root@master ~]# kubectl describe pod -n dev
        ```

        ![[IMG-20260404031957975.png|800]]

    6. 删除指定的 Pod

        ```Plain
         [root@master ~]# kubectl delete pods pod
         Error from server (NotFound): pods "pod" not found          # 不加名称空间的操作，默认是去 default 名称空间下找
         [root@master ~]# kubectl delete pods pod -n dev
         pod "pod" deleted
        ```
    7. 删除指定的 namespace（删除名称空间时，其下面的所有资源都会被删除）

        ```Plain
         [root@master ~]# kubectl delete ns dev
         namespace "dev" deleted
         [root@master ~]# kubectl get ns
         NAME              STATUS   AGE
         default           Active   6d14h
         kube-node-lease   Active   6d14h
         kube-public       Active   6d14h
         kube-system       Active   6d14h
        ```
# 二、命令式对象配置 ⭐
1. 通过 **命令配置** 和 **配置文件** 去操作kubernetes资源（==参数不在命令行写了，而是写在了yaml配置文件中==）

    ```Plain
     kubectl create/patch -f nginx-pod.yaml
    ```
2. 使用命令配合配置文件一起来操作kubernetes资源
3. **案例：namespace / pod的创建和删除**
    1. 创建一个nginxpod.yaml

        ```Plain
         apiVersion: v1
         kind: Namespace
         metadata:
           name: dev

         ---
         apiVersion: v1
         kind: Pod
         metadata:
           name: nginxpod
           namespace: dev
         spec:
           containers:
           - name: nginx-containers
             image: nginx:latest
        ```
    2. 执行create命令，创建资源

        ```Plain
        [root@master ~]# kubectl create -f nginxpod.yaml
        namespace/dev created
        pod/nginxpod created
        ```
    3. 执行get命令，查看资源

        ```Plain
         [root@master ~]# kubectl get ns dev
         NAME   STATUS   AGE
         dev    Active   19s
         [root@master ~]# kubectl get pod -n dev
         NAME       READY   STATUS    RESTARTS   AGE
         nginxpod   1/1     Running   0          27s
        ```
    4. 执行delete命令，删除资源

        ```Plain
         [root@master ~]# kubectl delete -f nginxpod.yaml
         namespace "dev" deleted
         pod "nginxpod" deleted
        ```
# 三、声明式对象配置 ⭐
1. 通过 **`apply`命令** 和 **配置文件** 去操作 kubernetes 资源（==只用于创建和更新资源==）

    ```Plain
     kubectl apply -f nginx-pod.yaml
    ```
2. 声明式对象配置跟命令式对象配置很相似，但是它只有一个命令apply

    ```Plain
     # 首先执行一次kubectl apply -f yaml文件，发现创建了资源
     [root@master ~]#  kubectl apply -f nginxpod.yaml
     namespace/dev created
     pod/nginxpod created
     # 再次执行一次kubectl apply -f yaml文件，发现说资源没有变动
     [root@master ~]#  kubectl apply -f nginxpod.yaml
     namespace/dev unchanged
     pod/nginxpod unchanged
    ```
3. 与命令式对象配置的区别
    1. 使用命令式对象配置操作资源：
        1. 如果资源不存在，就创建，相当于 kubectl create
        2. 如果资源已存在，则操作失败
    2. 使用声明式对象配置操作资源：
        1. 如果资源不存在，就创建，相当于 kubectl create
        2. 如果资源已存在，就更新，相当于 kubectl patch

# 四、三种管理方式总结
1. **比较：**

| 类型 | 操作对象 | 使用情形 | 优点 | 缺点 |
| --- | --- | --- | --- | --- |
| 命令式对象管理 | 对象 | 测试中临时性的查看 | 简单 | 只能操作活动对象，无法审计、跟踪 |
| 命令式对象配置 | 文件 | 开发（对资源增删改查） | 可以审计、跟踪 | 项目大时，配置文件多，操作麻烦 |
| 声明式对象配置 | 目录 | 开发（仅创建和更新资源） | 支持目录操作 | 意外情况下难以调试 |

2. **使用情形总结：（只是推荐，具体使用时，只要完成需求即可）**
    1. 创建/更新资源时，使用声明式对象配置

        ```Plain
         kubectl apply -f XXX.yaml
        ```
    2. 删除资源时，使用命令式对象配置

        ```Plain
         kubectl delete -f XXX.yaml
        ```
    3. 查询资源时，使用命令式对象管理

        ```Plain
         kubectl get(describe) 资源名称
        ```
