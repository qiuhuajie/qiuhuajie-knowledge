# 1. 方式 1：命令行
1. **创建并运行**
    
    1. 命令格式： `kubectl run (pod控制器名称) [参数]`
    
    1. 参数：
        
        1. `-image` 指定Pod的镜像
        
        1. `-port` 指定端口
        
        1. `-namespace` 指定namespace
        
    
    ```Plain
     [root@master ~]# kubectl run nginx --image=nginx:latest --port=80 --namespace dev
     pod/nginx created
    ```
    
1. **查看**
    
    1. 查看指定名称空间下的所有 Pod （不指定时为 default ）
        
        ```Plain
         [root@master ~]# kubectl get pods -n dev -o wide
         NAME    READY   STATUS    RESTARTS   AGE    IP               NODE     NOMINATED NODE   READINESS GATES
         nginx   1/1     Running   0          4m8s   10.244.196.135   node01   <none>           <none>
        ```
        
        > [!important]
        > 
        > - `READY`：运行的容器数/Pod中总容器数（注意：pause 容器是默认有的，不会算在里面）
        > 
        > - 使用参数 `-o wide` 可以获得 Pod 的IP
        
    
    1. 查看指定 Pod 的详细信息
        
        ```Plain
         kubectl describe pod nginx -n dev
        ```
        
    
1. **访问**
    
    ```Plain
     [root@master ~]# curl 10.244.196.135:80
     <!DOCTYPE html>
     <html>
     <head>
     <title>Welcome to nginx!</title>
     <style>
     html { color-scheme: light dark; }
     ...
    ```
    
1. **删除**
    
    > 注意：使用 run 命令运行的 Pod 好像不会带 deployment ，应该是课程是2020年的，新版本有所变化
    > 
    > 由于当前创建的 Pod 没有带 deployment ，所以下面的删除后重新拉起也不会有了
    > 
    > 推荐创建 Pod 时，直接创建 deployment，由 deployment 来创建 Pod
    
    ```Plain
     # 删除指定Pod
     [root@master ~]# kubectl delete pod nginx -n dev
     pod "nginx" deleted
     
     # 此时，显示删除Pod成功，但是再查询，发现又新产生了一个
     [root@master ~]# kubectl get pods -n dev
     NAME    READY   STATUS    RESTARTS   AGE
     nginx   1/1     Running   0          21s
     
     # ⭐这是因为当前Pod是由Pod控制器创建的，控制器会监控Pod状况，一旦发现Pod死亡，会立即重建
     # 此时要想删除Pod，必须删除Pod控制器
     
     # 先来查询一下当前namespace下的Pod控制器
     [root@master ~]# kubectl get deploy -n  dev
     NAME    READY   UP-TO-DATE   AVAILABLE   AGE
     nginx   1/1     1            1           9m7s
     
     # 接下来，删除此PodPod控制器
     [root@master ~]# kubectl delete deploy nginx -n dev
     deployment.apps "nginx" deleted
     
     # 稍等片刻，再查询Pod，发现Pod被删除了
     [root@master ~]# kubectl get pods -n dev
     No resources found in dev namespace.
    ```
    
# 2. 方式 2：Yaml **配置文件**
1. 最常用，详情见（）
1. 创建一个 pod-nginx.yaml，内容如下：
    
    ```Plain
     apiVersion: v1
     kind: Pod
     metadata:
       name: nginx
       namespace: dev
     spec:
       containers:
       - image: nginx:latest
         name: pod
         ports:
         - name: nginx-port
           containerPort: 80
           protocol: TCP
    ```
    
1. 然后就可以执行对应的创建和删除命令了：
    
    1. 创建：`kubectl create -f pod-nginx.yaml`
    
    1. 删除：`kubectl delete -f pod-nginx.yaml`
