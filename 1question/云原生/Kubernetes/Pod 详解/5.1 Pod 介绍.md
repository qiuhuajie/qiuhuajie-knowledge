# 一、基本概念
1. Pod 是 k8s 系统中**可以创建和管理的最小单元，由一个或多个容器组成**，**同一个 Pod 中的容器共享 网络资源 和 存储资源**

    ![[IMG-20260404031955812.svg|668]]
2. 可以将 Pod 理解为 ==**容器运行的环境**====，一个Pod中可以存在一个或者多个容器==，**k8s 不会直接处理容器， 而是处理 Pod**
3. kubernetes在集群启动之后，集群中的各个组件也都是以Pod方式运行的

    ![[IMG-20260404031955872.png|800]]
4. ==其他的资源对象都是用来支撑或者扩展 Pod 对象功能的==
    1. 控制器对象是用来管控 Pod 对象的
    2. Service 或者 Ingress 资源对象是用来暴露 Pod 的
    3. PersistentVolume 资源对象是用来为 Pod 提供存储的

# 二、Pod 特性
1. 资源共享

    **一个 Pod 里的多个容器可以 共享存储 和 网络**， 可以看作一个逻辑的主机
    - 多个容器共享同一 network namespace， 由此在一个 Pod 里的多个容器共享 Pod 的 IP 和 端口
    - 一个 Pod 里的多个容器可以共享存储卷， 这个存储卷会被定义为 Pod 的一部分， 并且可以挂载到该 Pod 里的所有容器的文件系统上
2. 生命周期短暂
    1. Pod 属于生命周期比较短暂的组件
    2. 比如，当 Pod 所在 node 节点**发生故障**，那么该节点上的 **Pod 会被调度到其他节点**， 但需要注意的是：被重新调度拉起的 Pod 是一个**全新的 Pod**，跟之前的 Pod 没有关系
3. 平坦的网络
    1. K8s 集群中的所有 Pod 都在同一个共享网络地址空间中
    2. 也就是说**每个 Pod 都可以通过其他 Pod 的 IP 地址来实现访问**

# 三、Pod 作用
1. 创建容器使用 Docker ，Docker 是单进程设计：一个docker 对应一个容器，一个容器有进程，一个容器一个应用程序
2. 而 Pod 是多进程设计，**一个 Pod 可以运行多个应用程序**（一个 Pod 中有多个 Docker 容器，故可以运行多个应用程序）
3. 更重要的意义是：当业务中有**多个紧密耦合且需要共享资源的应用程序**时，就需要使用 Pod 了，同一个 Pod 中的容器 可以共享网络 和 共享存储
4. 需要 Pod 的情况：
    1. 两个应用之间发生文件交互
    2. 两个应用需要通过127.0.0.1或者socket通信
    3. 两个应用需要发生频繁的调用

# 四、Pod 中容器的分类
![[IMG-20260404031955948.png|800]]
1. **`Pause`** **容器：**每一个 Pod 都有一个特殊的被称为” 根容器 “的 Pause容器
    1. 可以以它为依据，评估整个Pod的健康状态
    2. 可以在根容器上设置Ip地址，其它容器都此 Ip（Pod IP），以实现 Pod 内部的网路通信

        （这里是Pod内部的通讯，Pod的之间的通讯采用虚拟二层网络技术来实现，当前环境用的是Flannel）
2. **业务容器：**每个 Pod 还包含一个或多个紧密相关的用户业务容器
# 五、Pod 中容器的网络共享
1. k8s 使用 `Pause`容器（`infra-` `_infrastucture_` 容器）来实现 Pod 中容器的网络共享
2. ==实现原理：==
    1. 创建 Pod 的时候会**先创建 infra container 容器**，启动好之后然后再将实际创建的 Pod 加入到这个 infra container 容器当中
    2. infra container 容器实现的==主要目的是维护了 Pod 的网络，没有跑任何的业务逻辑==，只是启动了一个容器罢了
    3. **其他 业务容器** 的创建都让其**连接至该 infra container 容器的 network namespace**当中
    4. 这样一来同一个 Pod 当中所有的容器看到的网络设备，网卡，IP，Mac地址都是同一个了（因为**所有的容器都在同一个** **`network namespace`**）
    5. 这样就解决了网络共享的问题（==实际上 Pod 的 IP 就是== `infra container` ==的 IP==）

    ![[IMG-20260404031955984.png|570]]
3. 多个容器之间可以通过 localhost 来进行通信，注意：不同容器不要有端口冲突
4. 示例：创建一个名为 `test-netshare` 的 Pod，其中有两个容器：hello 和 web
