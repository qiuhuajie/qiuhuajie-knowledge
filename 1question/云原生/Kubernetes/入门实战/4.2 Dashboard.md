# 一、Dashboard 介绍
- 之前在kubernetes中完成的所有操作都是**通过命令行工具kubectl完成**的
- 其实，为了提供更丰富的用户体验，kubernetes还开发了一个==基于web的用户界面（Dashboard）==
- 用户可以使用Dashboard部署容器化的应用，还可以监控应用的状态，执行故障排查以及管理kubernetes中各种资源
# 二、部署 Dashboard
1. 下载yaml

    ```Plain
     [root@k8s-master01 ~]# wget  https://raw.githubusercontent.com/kubernetes/dashboard/v2.0.0/aio/deploy/recommended.yaml
    ```
2. 修改kubernetes-dashboard的Service类型

    ```YAML
     # 修改kubernetes-dashboard的Service类型
     kind: Service
     apiVersion: v1
     metadata:
       labels:
         k8s-app: kubernetes-dashboard
       name: kubernetes-dashboard
       namespace: kubernetes-dashboard
     spec:
       type: NodePort  # 新增 Service 对外暴露的方式
       ports:
         - port: 443
           targetPort: 8443  # DashboardService与Pod服务绑定的端口
           nodePort: 30009  # 新增 :DashboardService与node节点绑定的端口
       selector:
         k8s-app: kubernetes-dashboard
    ```
3. 部署 Dashboard

    ```Plain
     [root@master ~]# kubectl create -f recommended.yaml
    ```
4. 查看kubernetes-dashboard下的资源

6. 创建访问账户

    ```Plain

     serviceaccount/dashboard-admin created
    ```
7. 给账户授权

    ```Plain
     [root@master ~]# kubectl create clusterrolebinding dashboard-admin-rb --clusterrole=cluster-admin --serviceaccount=kubernetes-dashboard:dashboard-admin
     clusterrolebinding.rbac.authorization.k8s.io/dashboard-admin-rb created
    ```

# 三、使用 Dashboard
## 1. 查看

选择指定的命名空间`dev`，然后点击`Deployments`，查看dev空间下的所有deployment

![[IMG-20260404031957102.png|800]]

## 2. 扩缩容

在`Deployment`上点击`规模`，然后指定`目标副本数量`，点击确定

![[IMG-20260404031957180.png|800]]

## 3. 编辑

在`Deployment`上点击`编辑`，然后修改`yaml文件`，点击确定

![[IMG-20260404031957214.png|800]]

## 4. 查看 Pod

点击`Pods`, 查看pods列表

![[IMG-20260404031957262.png|800]]

## 5. 操作 Pod

选中某个Pod，可以对其执行日志（logs）、进入执行（exec）、编辑、删除操作

![[IMG-20260404031957311.png|800]]
