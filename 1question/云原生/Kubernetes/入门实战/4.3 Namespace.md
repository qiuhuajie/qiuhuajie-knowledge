# 1. **概述**
1. 将集群内部的资源分配到不同的 Namespace 中，可以形成逻辑上的 "组"，以方便不同的组的资源进行隔离使用和管理
1. 用来实现 **多套环境的资源隔离** 或者 **多租户的资源隔离**
    
    1. **多套环境的资源隔离**
        
        1. 默认情况下，kubernetes集群中的所有的 Pod 都是可以相互访问的
        
        1. 但是在实际中，可能==不想让两个Pod之间进行互相的访问==，那此时就可以==将两个 Pod 划分到不同的 namespace 下==
            

            ![[IMG-20260404031957415.png|Untitled 397.png]]

            
        
    
    1. **多租户的资源隔离**
        
        1. 可以通过 kubernetes 的授权机制，==将不同的 namespace 交给不同租户进行管理==，这样就实现了多租户的资源隔离
        
        1. 还能结合 kubernetes 的资源配额机制，限定不同租户能占用的资源，例如CPU使用量、内存使用量等等，来实现租户可用资源的管理
        
    
# 2. **操作**
1. **创建**
    
    ```Plain
     [root@master ~]# kubectl create namespace dev
     namespace/dev created
    ```
    
1. **查看**
    
    1. 查看所有名称空间
        
        ```Plain
         [root@master ~]# kubectl get ns
         NAME              STATUS   AGE
         default           Active   6d14h
         dev               Active   11s
         kube-node-lease   Active   6d14h
         kube-public       Active   6d14h
         kube-system       Active   6d14h
        ```
        
        > [!important] kubernetes在集群启动之后，会默认创建几个namespace
        > 
        > 1. `default`：所有未指定Namespace的对象都会被分配在default命名空间
        > 
        > 1. `kube-node-lease`：集群节点之间的心跳维护，v1.13开始引入
        > 
        > 1. `kube-public`：此命名空间下的资源可以被所有人访问（包括未认证用户）
        > 
        > 1. `kube-system`：所有由Kubernetes系统创建的资源都处于这个命名空间
        
    
    1. 查看指定名称空间的详细信息
        
        ```Plain
         [root@master ~]# kubectl describe ns dev
         Name:         dev
         Labels:       <none>
         Annotations:  <none>
         Status:       Active
         
         No resource quota.              # ResourceQuota 针对namespace做的资源限制
         
         No LimitRange resource.      # LimitRange针对namespace中的每个组件做的资源限制
        ```
        
    
1. **删除**
    
    ```Plain
     [root@master ~]# kubectl delete ns dev
     namespace "dev" deleted
    ```
    
1. **以配置文件的方式操作**
    
    1. 首先准备一个yaml文件：ns-dev.yaml
        
        ```Plain
         apiVersion: v1
         kind: Namespace
         metadata:
           name: dev
        ```
        
    
    1. 然后就可以执行对应的创建和删除命令了
        
        1. 创建：`kubectl create -f ns-dev.yaml`
        
        1. 删除：`kubectl delete -f ns-dev.yaml`
