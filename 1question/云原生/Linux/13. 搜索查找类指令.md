# 一、搜索查找类指令
> 💡 快速区分
> - `find`：遍历目录，按条件查找文件或目录，结果最新，但速度相对较慢。
> - `locate`：查询文件名数据库，速度快，但结果可能不是最新。
> - `which`：查看命令的可执行文件路径。
> - `grep`：查找文本内容；配合管道符 `|` 可以过滤命令输出。
## 1. 指令对比

| 指令 | 作用 | 特点 | 常见场景 |
| --- | --- | --- | --- |
| `find` | 按条件查找文件或目录 | 直接遍历文件系统，结果实时 | 按名称、大小、拥有者查找文件 |
| `locate` | 快速定位文件路径 | 查询数据库，速度快 | 只想快速知道文件在哪里 |
| `which` | 查看命令所在路径 | 常用于确认命令是否存在 | 查 `ls`、`java`、`python` 等命令的位置 |
| `grep` | 查找匹配的文本行 | 支持过滤文本内容 | 在文件内容或命令输出中搜索关键字 |
## 2. Find
- 基本语法：`find [搜索路径] [查找条件]`
- 功能描述：从指定目录开始递归向下查找，输出满足条件的文件或目录。

### 2.1 常用条件

| 条件 | 说明 |
| --- | --- |
| `-name 文件名` | 按文件名查找 |
| `-user 用户名` | 查找属于指定用户的文件 |
| `-size 大小` | 按文件大小查找，例如 `+200M`、`-10k` |
### 2.2 常用示例
1. 按文件名查找 `/home` 目录下的 `aaa.txt`

```bash
find /home -name "aaa.txt"
```
输出示例：
```text
/home/aaa.txt
/home/bbb/aaa.txt
```
2. 查找 `/opt` 目录下属于 `root` 用户的文件

```bash
find /opt -user root
```
3. 查找整个 Linux 系统中大于 `200M` 的文件

```bash
find / -size +200M
```
可能出现类似下面的报错：
```text
find: '/proc/xxxx/fd/5': No such file or directory
```
这是因为 `/proc` 目录中的内容是动态变化的，属于正常现象。只关心结果时，可以这样写：
```bash
find / -size +200M 2>/dev/null
```
## 3. Locate
- 基本语法：`locate 文件名`
- 功能描述：通过系统预先建立的文件名数据库快速定位文件路径。
- 特点：无需遍历整个文件系统，所以查询速度很快。

### 3.1 特别说明
- `locate` 依赖数据库，不是实时扫描。
- 第一次使用前，通常需要执行 `updatedb` 创建或更新数据库。
- 如果数据库没有及时更新，查询结果可能不准确。

### 3.2 常用示例
```bash
updatedb
locate aaa.txt
```
输出示例：
```text
/home/aaa.txt
/home/bbb/aaa.txt
```
## 4. Which
- 基本语法：`which 命令名`
- 功能描述：查看某个命令对应的可执行文件路径。

### 4.1 常用示例
```bash
which ls
```
输出示例：
```text
/usr/bin/ls
```
有些系统或 shell 环境中，也可能先显示别名信息，例如：
```text
alias ls='ls --color=auto'
/usr/bin/ls
```
## 5. Grep 与管道符
- `grep` 基本语法：`grep [选项] "查找内容" 文件`
- `grep` 功能描述：在文件中查找匹配的内容，并输出匹配行。
- 管道符 `|`：把前一个命令的输出结果，交给后一个命令继续处理。

### 5.1 常用选项

| 选项 | 说明 |
| --- | --- |
| `-n` | 显示匹配行及行号 |
| `-i` | 忽略大小写 |
| `-A 行数` | 显示匹配行及后面的若干行 |
| `-B 行数` | 显示匹配行及前面的若干行 |
| `-C 行数` | 显示匹配行前后若干行 |
### 5.2 常用示例
1. 在 `/home/a.txt` 中查找 `yes`，并显示行号

推荐写法：
```bash
grep -n "yes" /home/a.txt
```
输出示例：
```text
2:yes
4:yes
5:yes
```
2. 忽略大小写搜索日志中的 `error`

```bash
grep -in "error" app.log
```
3. 配合管道符过滤命令输出

```bash
ps -ef | grep nginx
```
说明：
- 如果只是搜索文件内容，直接使用 `grep` 更简洁。
- 如果要过滤前一个命令的输出，再配合管道符 `|` 使用。

### 5.3 日志排查：显示匹配行及上下文
异常栈、报错日志通常是多行连续输出的，只搜关键词时，往往只能看到第一行。这时可以配合上下文参数把前后关联日志一起带出来。
```bash
# 显示匹配行 + 后面 20 行（通常够看大部分堆栈了）
grep -A 20 "你的关键词" logfile.log
# 同时显示前面 5 行 + 后面 20 行
grep -B 5 -A 20 "你的关键词" logfile.log
# 如果日志很大，先看最近的部分（推荐）
tail -n 10000 logfile.log | grep -A 30 "NullPointerException"
```
- `-A` = After，显示匹配行后面的若干行
- `-B` = Before，显示匹配行前面的若干行
- `-C` = Context，显示匹配行前后若干行
