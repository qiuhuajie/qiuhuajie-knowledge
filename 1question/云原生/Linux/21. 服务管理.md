# 一、介绍
1. **服务(service) 本质就是进程**，但是是运行在后台的，通常都会**监听某个端口**，等待其它程序的请求
2. 比如 mysql , sshd 防火墙，因此我们又称为守护进程，是Linux中非常重要的知识点
3. **原理图：**==后台程序 = 守护进程 = 服务==

    ![[IMG-20260404031959881.png|557]]

# 二、service 管理指令
1. `service 服务名 [start | stop | restart | reload | status]`
2. ==在CentOS7.0后 不再使用 service== ，而是 systemctl
3. CentOS7.0后service管理服务指令有：
4. 应用案例：

    ```Plain
     [root@hadoop001 ~]# service network stop
     Stopping network (via systemctl):                          [  确定  ]
     [root@hadoop001 ~]# service network start
     Starting network (via systemctl):                          [  确定  ]
    ```

# 四、服务的运行级别（runlevel）
1. Linux系统有7种运行级别(runlevel)： 常用的是级别3和5
    - 运行级别0： 系统停机状态，系统默认运行级别不能设为0，否则不能正常启动
    - 运行级别1： 单用户工作状态， root权限，用于系统维护，禁止远程登陆
    - 运行级别2： 多用户状态(没有NFS)，不支持网络
    - **运行级别3： 完全的多用户状态(有NFS)，无界面，登陆后进入控制台命令行模式**
    - 运行级别4： 系统未使用，保留
    - **运行级别5： X11控制台，登陆后进入图形GUI模式**
    - 运行级别6： 系统正常关闭并重启，默认运行级别不能设为6，否则不能正常启动

# 五、chkconfig 指令
1. 介绍：
    1. 通过chkconfig 命令可以给每个服务的各个运行级别设置自启动/关闭
    2. chkconfig指令管理的服务在/etc/init.d查看
    3. 注意：Centos7.0后，很多服务使用systemctl管理
2. 基本语法：
    1. 查看服务：`chkconfig --list | grep xxx`
    2. 查看指定的服务：`chkconfig 服务名 --list`
    3. 将某个服务，在指定级别打开或关闭：`chkconfig --level 5 服务名 on/off`

# 七、systemctl 设置服务的自启动状态（永久效果）
1. 基本语法：
    1. `systemctl list-unit-files [| grep 服务名]`（查看服务开机启动状态）
    2. `systemctl enable 服务名`（设置服务开机自启）
    3. `systemctl disable 服务名`（关闭服务开机自启）
    4. `systemctl is-enabled 服务名`（查询某个服务是否是自启动的）
2. 应用案例：

    ```Plain
     [root@hadoop001 ~]# systemctl list-unit-files | grep firewalld
     firewalld.service
     enabled
     [root@hadoop001 ~]# systemctl is-enabled firewalld
     enabled
     [root@hadoop001 ~]# systemctl is-enabled sshd
     enabled
    ```
# 八、防火墙（fierwalld）
1. 关闭和启用防火墙，会立即生效，但这种方式只是临时生效，当重启系统后，还是会回归以前的服务设置
2. 若想让某个服务永久失效，或永久启用，则要设置开机自启systemctl enable|disable 服务名
3. 防火墙原理图：

    ![[IMG-20260404032000113.png|800]]

4. 测试防火墙的作用：
    1. 使用netstat指令，查看当前网络状态，看到systemd的端口号为111

        ![[IMG-20260404032000183.png|800]]

    2. 在cmd中`ping 192.168.200.130 111`，并不能ping通
    3. 此时在Linux中关闭防火墙，便可以在cmd中ping通了

# 九、防火墙打开或者关闭指定的端口
1. 真正的生产环境，往往需要将防火墙打开，但是外部请求数据包就不能跟服务器监听端口通讯，此时就需要打开指定的端口
2. firewall指令（**❗ firewall-cmd中间没空格啊，真的晕眩**）
    1. 打开端口：`firewall-cmd --permanent --add-port=端口号/协议号`
    2. 关闭端口：`firewall-cmd --permanent --remove-port=端口号/协议号`
    3. 重写载入才能生效：`firewall-cmd --reload`
    4. 查询端口是否开放：`firewall-cmd --query-port=端口号/协议号`
3. 应用案例：

    ```Plain
     [root@hadoop001 ~]# firewall-cmd --permanent --add-port=111/tcp
     success
     [root@hadoop001 ~]# firewall-cmd --reload
     success
     [root@hadoop001 ~]# firewall-cmd --query-port=111/tcp
     yes
         //关闭111端口
     [root@hadoop001 ~]# firewall-cmd --permanent --remove-port=111/tcp
     success
     [root@hadoop001 ~]# firewall-cmd --query-port=111/tcp
     yes         //不reload不会生效
     [root@hadoop001 ~]# firewall-cmd --reload
     success
     [root@hadoop001 ~]# firewall-cmd --query-port=111/tcp
     no          //已关闭
    ```
