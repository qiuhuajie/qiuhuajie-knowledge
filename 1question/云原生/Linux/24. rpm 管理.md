# 24. Rpm 管理
1. 介绍：
    1. 一种**用于互联网下载包的打包及安装工具**，它包含在某些Linux分发版中
    2. 它生成具有**.RPM**扩展名的文件
    3. RPM是RedHat Package Manager（RedHat软件包管理工具）的缩写，**类似windows setup.exe**，这一文件格式名称虽然打上了RedHat的标志，但理念是通用的
    4. Linux的分发版本都有采用（suse,redhat, centos 等等），可以算是公认的行业标准了
2. **rpm 包的**==**简单查询指令**==
    1. 基本语法：查询已安装的rpm列表： `rpm –qa | grep xx`
    2. 应用案例：

        ```Bash
         [root@hadoop001 ~]# rpm -qa | grep firefox
         firefox-60.2.2-1.el7.centos.x86_64
        ```
        - 名称：firefox
        - 版本号：60.2.2-1
        - 适用操作系统：l7.centos.x86_64
        - 表示`centos6.x`的64位系统
        - 如果是`i686`、 `i386`表示32位系统， `noarch`表示通用
3. **rpm 包的**==**其它查询指令**==
    1. `rpm -qa` ：查询所安装的所有rpm软件包
    2. `rpm -qa | more`：分页显示
    3. `rpm -qa | grep xxx` ：过滤查找
    4. `rpm -q 软件包名`：查询软件包是否安装
    5. `rpm -qi 软件包名` ：查询软件包信息
    6. `rpm -ql 软件包名` ：查询安装到哪里了，包含哪些文件
    7. `rpm -qf 文件全路径名` ：查询某个文件在之前安装时所属的软件
4. ==**卸载**==**rpm包**
    1. 基本语法：`rpm -e RPM包的名称`
    2. 细节讨论：
        1. 如果其它软件包依赖于您要卸载的软件包，卸载时则会产生错误信息
            如： rpm -e foo
            removing these packages would break dependencies:**foo is needed by bar-1.0-1**
        2. 如果我们就是要删除 foo这个rpm 包，可以增加参数 `-nodeps` ，就可以**强制删除**，但是一般不推荐这样做， 因为依赖于该软件包的程序可能无法运行
            如： rpm -e --nodeps foo
5. ==**安装**==**rpm包**
    1. 基本语法：`rpm -ivh RPM包全路径名称`
    2. 参数说明：
        1. `i`：install 安装
        2. `v`：verbose 提示
        3. `h`：hash 进度条
