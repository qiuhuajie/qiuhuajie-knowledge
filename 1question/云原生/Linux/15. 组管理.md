# 1. **Linux组基本介绍**
1. 在linux中**每个用户**必须属于一个组，不能独立于组外
1. 在linux中**每个文件**有所有者、所在组、其它组的概念
    
    1. 所有者：一般为文件的创建者,谁创建了该文件， 就自然的成为该文件的所有者
    
    1. 所在组：默认为所有者所在的组，对该文件有一定的权限
    
    1. 其他组：除了所在组的其他的组，对该文件有一定的权限

# 2. **文件/目录 所有者**
1. 查看文件的所有者：`ls -ahl`
    
    ```Plain
    
    1
    [root@hadoop1 home]# ls -ahl2
    总用量 900K3
    drwxr-xr-x.  6 root root    4.0K 6月  26 00:27 .4
    dr-xr-xr-x. 18 root root    4.0K 6月  25 20:02 ..5
    -rw-r--r--.  1 root root      12 6月  25 22:13 aaa.txt6
    -rw-r--r--.  1 root root      27 6月  26 00:09 a.txt7
    drwxr-xr-x.  2 root root    4.0K 6月  25 20:07 bbb8
    -rw-r--r--.  1 root root       0 6月  25 20:06 ccc.txt9
    drwx------.  5 jack wudang  4.0K 6月  22 17:17 cool10
    -rw-r--r--.  1 root root     792 6月  25 22:21 homeinfo.txt11
    drwx------.  3 jack shaolin 4.0K 6月  22 17:54 jack
    ```
    
1. 修改文件所有者：`chown 用户名 文件名`
    
    ```Plain
    
    1
    [root@hadoop1 home]# touch c.txt2
    [root@hadoop1 home]# ll3
    -rw-r--r--.  1 root root         0 6月  26 15:33 c.txt4
    5
    [root@hadoop1 home]# chown tom c.txt6
    [root@hadoop1 home]# ll7
    -rw-r--r--.  1 tom  root         0 6月  26 15:33 c.txt
    ```
    
# 3. **组的创建**
1. 基本语法：`groupadd 组名`
1. 应用实例：
    
    ```Plain
    
    1
    [root@hadoop1 home]# groupadd monster2
    3
    [root@hadoop1 home]# useradd -g monster fox4
    [root@hadoop1 home]# id fox5
    uid=1002(fox) gid=1003(monster) 组=1003(monster)
    ```
    
# 4. **文件/目录 所在组**
当某个用户创建了一个文件后，这个文件的所在组就是该用户所在的组
1. 查看文件/目录所在组`ls –ahl`
    
    ```Plain
    
    1
    [fox@hadoop1 ~]$ pwd2
    /home/fox3
    [fox@hadoop1 ~]$ touch ok.txt4
    [fox@hadoop1 ~]$ ll5
    总用量 06
    -rw-r--r--. 1 fox monster 0 6月  28 23:22 ok.txt7
        //ok.txt文件所有者是fox，所在的组就是fox所在的monster8
    9
    //或使用ls –ahl查看10
    [fox@hadoop1 ~]$ ls -ahl11
    总用量 36K12
    drwx------. 5 fox  monster 4.0K 6月  28 23:22 .13
    drwxr-xr-x. 7 root root    4.0K 6月  28 23:18 ..14
    -rw-r--r--. 1 fox  monster    0 6月  28 23:22 ok.txt
    ```
    
1. 修改文件所在的组`chgrp 组名 文件名`
    
    ```Plain
    
    1
    [root@hadoop1 ~]# cd /home2
    [root@hadoop1 home]# ll3
    总用量 8964
    -rw-r--r--.  1 root root         0 6月  25 20:06 ccc.txt5
    6
    [root@hadoop1 home]# chgrp monster ccc.txt7
    8
    [root@hadoop1 home]# ll9
    总用量 89610
    -rw-r--r--.  1 root monster      0 6月  25 20:06 ccc.txt
    ```
    
# 5. **其它组**
除文件的所有者和所在组的用户外，系统的其它用户都是文件的其它组
# 6. **改变用户所在组**
在添加用户时，可以指定将该用户添加到哪个组中，同样的用root的管理权限可以改变某个用户所在的组
1. 基本语法：
    
    1. `usermod –g` 组名 用户名
    
    1. `usermod –d` 目录名 用户名 改变该用户登陆的初始目录
        
        （特别说明：用户需要有进入到新目录的权限）
        
    
1. 应用实例：
    
    ```Plain
    
    1
    [root@hadoop1 home]# id jack2
    uid=1001(jack) gid=1002(shaolin) 组=1002(shaolin)3
        //此时jack在shaolin组，现在要改到tom组4
    5
        //先查查有没有tom组6
    [root@hadoop1 home]# cat /etc/group | grep tom7
    tom:x:1000:tom8
    9
    [root@hadoop1 home]# usermod -g tom jack10
    [root@hadoop1 home]# ll11
    总用量 89612
    drwx------.  3 jack tom       4096 6月  22 17:54 jackshell
    ```
