# 一、基本介绍
1. 在LINUX中，每个执行的程序（代码）都称为一个进程（动态概念）
2. 每一个进程都分配一个ID号
3. 每一个进程，都会对应一个父进程，而这个父进程可以复制多个子进程。例如www服务器
4. 每个进程都可能以两种方式存在的：前台与后台
    1. 前台进程：就是用户目前的屏幕上可以进行操作的
    2. 后台进程：则是实际在操作，但由于屏幕上无法看到的进程，通常使用后台方式执行
5. 一般系统的服务都是以后台进程的方式存在，而且都会常驻在系统中。直到关机才才结束
# 二、显示系统执行的进程
1. 基本介绍：

    `ps`命令是用来查看目前系统中，有哪些正在执行，以及它们执行的状况。可以不加任何参数
2. 常用选项：
    - `-a`：显示当前终端的所有进程
    - `-u`：以用户的格式显示进程信息
    - `-x`：显示后台进程运行的参数
    - `-e` 显示所有进程
    - `-f` 全格式
3. 显示的信息选项：

    ![[IMG-20260404031959726.png|800]]
    - `System` V展示风格
    - `USER`：用户名称
    - `PID`：进程号
    - `%CPU`：进程占用CPU的百分比
    - `%MEM`：进程占用物理内存的百分比
    - `VSZ`：进程占用的虚拟内存大小（单位： KB）
    - `RSS`：进程占用的物理内存大小（单位： KB）
    - `TT`：终端名称，缩写
    - `STAT`：进程状态，其中S-睡眠， s-表示该进程是会话的先导进程， N-表示进程拥有比普通优先级更低的优先级， R-正在运行， D-短期等待， Z-僵死进程， T-被跟踪或者被停止等等
    - `STARTED`：进程的启动时间
    - `TIME`： CPU时间，即进程使用CPU的总时间
    - `COMMAND`：启动进程所用的命令和参数，如果过长会被截断显示
4. **实例一：查看当前系统有没有目标进程**
    1. 语法：`ps –aux | grep xxx`
    2. 应用实例：

        ```Bash
         [root@hadoop001 ~]# ps -aux | grep "sshd"
         root       7753  0.0  0.2 112756  4320 ?        Ss   16:31   0:00 /usr/sbin/sshd -D
         root       8622  0.0  0.2 160848  5608 ?        Ss   16:35   0:00 sshd: root@pts/0
         root      12067  0.0  0.2 156636  5484 ?        Ss   18:04   0:00 sshd: root@pts/1
         root      12796  0.0  0.0 112728   988 pts/1    S+   18:24   0:00 grep --color=auto sshd
        ```
5. **实例二：以全格式显示当前所有的进程，查看进程的父进程**
    1. 语法：`ps –ef | grep xxx`

# 三、终止进程 kill 和 killall
1. 介绍：

    若是某个进程执行一半需要停止时，或是已消了很大的系统资源时，此时可以考虑停止该进程。使用kill命令来完成此项任务
2. 基本语法：
    1. `kill [选项] 进程号`（功能描述：通过进程号杀死进程）
    2. `killall 进程名称` （功能描述：通过进程名称杀死进程，也支持通配符，这在系统因负载过大而变得很慢时很有用）
3. 常用选项：
    `9`：表示强迫进程立即停止
4. **最佳实践：**
    1. 案例1：踢掉某个非法登录用户

        ```Bash
         kill 16704
        ```
    2. 案例2: 终止远程登录服务sshd, 在适当时候再次重启sshd服务

# 四、查看进程树 pstree
1. 基本语法：

    `pstree [选项]`，可以更加直观的来看进程信息
    ```Java
     [root@hadoop001 ~]# pstree
     systemd─┬─ModemManager───2*[{ModemManager}]
             ├─NetworkManager───2*[{NetworkManager}]
             ├─VGAuthService
             ├─2*[abrt-watch-log]
             ├─abrtd
             ├─accounts-daemon───2*[{accounts-daemon}]
             ├─alsactl
             ├─at-spi-bus-laun─┬─dbus-daemon───{dbus-daemon}
             │                 └─3*[{at-spi-bus-laun}]
             ├─at-spi2-registr───2*[{at-spi2-registr}]
             ├─atd
             ├─auditd─┬─audispd─┬─sedispatch
     ...
    ```
2. 常用选项：
    1. `p` ：显示进程的PID
    2. `u` ：显示进程的所属用户
3. 应用实例：
    1. 案例1：请你树状的形式显示进程的pid

        ```Java
         [root@hadoop001 ~]# pstree -p
         systemd(1)─┬─ModemManager(6931)─┬─{ModemManager}(6978)
                    │                    └─{ModemManager}(7001)
                    ├─NetworkManager(7186)─┬─{NetworkManager}(7285)
                    │                      └─{NetworkManager}(7298)
                    ├─VGAuthService(7006)
                    ├─abrt-watch-log(7009)
                    ├─abrt-watch-log(7013)
                    ├─abrtd(7008)
                    ├─accounts-daemon(6927)─┬─{accounts-daemon}(6973)
        ```
    2. 案例2： 请你树状的形式进程的用户id

        ```Java
         [root@hadoop001 ~]# pstree -u
         systemd─┬─ModemManager───2*[{ModemManager}]
                 ├─NetworkManager───2*[{NetworkManager}]
                 ├─VGAuthService
                 ├─2*[abrt-watch-log]
                 ├─abrtd
                 ├─accounts-daemon───2*[{accounts-daemon}]
                 ├─alsactl
                 ├─at-spi-bus-laun─┬─dbus-daemon───{dbus-daemon}
        ```
