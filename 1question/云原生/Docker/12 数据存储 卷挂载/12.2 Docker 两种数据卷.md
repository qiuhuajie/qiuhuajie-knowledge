# 一、自管理卷

**自管理数据卷的指定 需要在 DockerFile 中声明**

1. **示例验证：**查看 wordpress 的自管理卷
    1. 查看本地目录：

        ```Plain
         [root@docker001 ~]# cd /var/lib/docker
         [root@docker001 docker]# ls
         containers  image  network  overlay  plugins  swarm  tmp  trust  volumes
         [root@docker001 docker]# cd volumes
         [root@docker001 volumes]# ls
         # 当前 volumes 为空
        ```
    1. 启动一个wordpress 容器：

        ```Plain
         [root@docker001 volumes]# docker run --name worpress -d wordpress:latest
         8564db523649ee3da98e66474769b2339c2a6bbc8c46905635b85783bb0c6664
         [root@docker001 volumes]# docker inspect wordpress
         ...
         "Volumes": {
                         "/var/www/html": {}
                     },
         ...
         # wordpress 的描述信息中显示，容器在启动时会启动一个自管理卷
         # 这个镜像时官网下载的，在制作镜像的时候就将这些写好了
        ```
    1. 在去查看 /var/lib/docker 的内容：

        ```Plain
         [root@docker001 volumes]# ls
         a116055ae2e99062bf28af364af9733be134af83124bae1e23b39e0ef15df007
         # 这个就是wordpress 容器的自管理卷
        ```
    1. 进去看看：

        ```Plain
         [root@docker001 volumes]# cd a116055ae2e99062bf28af364af9733be134af83124bae1e23b39e0ef15df007/
         [root@docker001 a116055ae2e99062bf28af364af9733be134af83124bae1e23b39e0ef15df007]# ls
         _data
         [root@docker001 a116055ae2e99062bf28af364af9733be134af83124bae1e23b39e0ef15df007]# cd _data/
         [root@docker001 _data]# ls
         index.php    readme.html      wp-admin            wp-comments-post.php  wp-config-sample.php  wp-cron.php  wp-links-opml.php  wp-login.php  wp-settings.php  wp-trackback.php
         license.txt  wp-activate.php  wp-blog-header.php  wp-config-docker.php  wp-content            wp-includes  wp-load.php        wp-mail.php   wp-signup.php    xmlrpc.php
        ```
    1. 做一个标记：

        ```Plain
         [root@docker001 _data]# touch coolcoolcool
         [root@docker001 _data]# ls
         coolcoolcool  readme.html      wp-blog-header.php    wp-config-sample.php  wp-includes        wp-login.php     wp-signup.php
         index.php     wp-activate.php  wp-comments-post.php  wp-content            wp-links-opml.php  wp-mail.php      wp-trackback.php
         license.txt   wp-admin         wp-config-docker.php  wp-cron.php           wp-load.php        wp-settings.php  xmlrpc.php
        ```
    1. 再去wordpress 容器中，查看 对应的文件系统：

        ```Plain
         [root@docker001 _data]# docker exec -it worpress /bin/bash
         root@8564db523649:/var/www/html# ls
         coolcoolcool  readme.html      wp-blog-header.php    wp-config-sample.php  wp-includes        wp-login.php     wp-signup.php
         index.php     wp-activate.php  wp-comments-post.php  wp-content        wp-links-opml.php  wp-mail.php      wp-trackback.php
         license.txt   wp-admin         wp-config-docker.php  wp-cron.php       wp-load.php        wp-settings.php  xmlrpc.php
        ```
    1. **验证结果：**
        1. **容器中的** **`/var/www/html`** **确实与** **`/var/lib/docker/volumes/xxxx`** **是挂载在一起的**
        1. **删除容器后，宿主主机上的持久化数据也不会被删除**，如果想要删除数据卷：

            ```Plain
             docker rm -f -v 容器名
            ```
1. 制作镜像时，使用 Docker File 中的 VOLUME 指定自管理卷
    1. 写一个DockerFile

        ```Plain
         FROM hub.c.163.com/public/centos:6.7
         RUN touch /tmp/1.txt
         RUN mkdir /data
         VOLUME /data   # ⭐将容器内的目录 /data 与宿主主机的目录 /var/lib/docker/volumes/ 挂载起来
                         # 即标明 容器中的 /data 需要持久化
         CMD tail -f /tmp/1.txt
        ```
    1. build 镜像

        ```Plain
         [root@docker001 tomcat]# docker build -t test:v0.1 .
        ```
    1. 启动镜像为容器 test

        ```Plain
         root@docker001 tomcat]# docker run --name test -d test:v0.1
        ```
    1. 查看当前 宿主主机的挂载点：自管理卷固定为`/var/lib/docker/volumes/`

        ```Plain
         [root@docker001 tomcat]# cd /var/lib/docker/volumes/
         [root@docker001 volumes]# ls
         34b9727514f23868d74a55201d316b343b0bda2e62dce11da86d710a3de61544  a116055ae2e99062bf28af364af9733be134af83124bae1e23b39e0ef15df007  coolcoolcool  metadata.db
         [root@docker001 volumes]# cd 34b9727514f23868d74a55201d316b343b0bda2e62dce11da86d710a3de61544/
         [root@docker001 34b9727514f23868d74a55201d316b343b0bda2e62dce11da86d710a3de61544]# cd _data/
         [root@docker001 volumes]# ls
         # 当前是空的
        ```
    1. 进入容器，在 容器的挂载点 添加数据

        ```Plain
         [root@docker001 volumes]# docker exec -it test /bin/bash
         [root@97363e3f2f7c /]# cd  /data/
         [root@97363e3f2f7c data]# ls
         [root@97363e3f2f7c data]# touch coolcool
         [root@97363e3f2f7c data]# ls
         coolcool
         [root@97363e3f2f7c data]# exit
        ```
    1. 再查看 宿主主机的挂载点

        ```Plain
         [root@docker001 volumes]# cd 34b9727514f23868d74a55201d316b343b0bda2e62dce11da86d710a3de61544/
         [root@docker001 34b9727514f23868d74a55201d316b343b0bda2e62dce11da86d710a3de61544]# cd _data/
         [root@docker001 _data]# ls
         coolcool
        ```
# 二、自绑定卷

**自绑定数据卷的指定 需要在执行 run 时，使用参数 -v 进行声明**

1. 如果一个镜像的 DockerFile 并没有使用 VOLUME 来定义一个自管理卷
1. 在启动镜像时，也可以指明挂载

    ```Plain
     docker run --name test -v /data:/data -d test:v0.1
     # 宿主主机
    ```
1. 🔴**如果及在DockerFile 中声明了自管理卷，又在run 声明了绑定卷。则自绑定卷会优先生效**
1. 🔴**如果两个容器绑定到宿主主句的同一个目录，则可以实现两个容器共享同一个数据目录**
1. 🔴**如果不希望通过宿主主机来实现共享目录，则可以让容器2 共享 容器1 的自管理卷，来实现数据共享**
    1. 前提：两个容器的镜像需要都先使用VOLUME来实现自管理卷
    1. 创建容器1：

        ```Plain
         [root@docker001 tomcat]# docker run --name test_1 -d test:v0.1
         fc2ef539affdaa3c8fb7003972751f52203d44562a6f1b10ee22392177486b3d
        ```
    1. 创建容器2：

        ```Plain
         # --volumes-from test_1 指明容器2中/data 要共享容器1中/data的 自管理卷
         [root@docker001 tomcat]# docker run --name test_2 --volumes-from test_1 -d test:v0.1
         7914d6b55dbaca4b9cd573bff9177a107f7e64f6803d192b2e1248cea99ec41b
        ```
    1. 取默认的自管理卷目录查看：

        ```Plain
         [root@docker001 tomcat]# cd /var/lib/docker/volumes/
         [root@docker001 volumes]# ls
         34b9727514f23868d74a55201d316b343b0bda2e62dce11da86d710a3de61544
         # 可以看到只有一个
        ```
    1. 去容器1 的/data修改数据，再去容器2 的/data 查看

        ```Plain
         [root@docker001 volumes]# docker exec -it test_1 /bin/bash
         [root@fc2ef539affd /]# cd /data/
         [root@fc2ef539affd data]# ls
         [root@fc2ef539affd data]# touch coolcoolcool
         [root@fc2ef539affd data]# exit
         exit
         [root@docker001 volumes]# docker exec -it test_2 /bin/bash
         [root@7914d6b55dba /]# cd /data/
         [root@7914d6b55dba data]# ls
         coolcoolcool
        ```
