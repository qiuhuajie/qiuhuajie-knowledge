1. 原来的部署方式：

    将java应用打成 jar 包，再使用 java 命令启动应用
    ```Plain
     java -jar springBootAdmin-0.0.1-SNAPSHOT.jar
    ```
2. **存在的问题：需要服务器已经安装了相应版本的java环境、mysql环境等等**
3. 解决方法：
    1. 容器化部署
    2. 所有机器都安装Docker，任何应用都是镜像，所有机器都可以运行
4. 创建一个SpringBoot程序

    ```Java
     @RestController
     public class testController {
         private int count = 0;
         @RequestMapping("/test")
         public String test(){
             return "有【"+ count++ +"】 人访问了这个页面";
         }
     }
    ```
5. ⭐**==编写 Dockerfile==**

    ```Bash
     FROM openjdk:8-jdk-slim
     COPY target/*.jar /app.jar  # 类似于ADD，将我们文件拷贝到镜像中
     ENTRYPOINT ["java","-jar","app.jar"]
    ```
    > 💡 去 DockerHub 上找想要的镜像
    >
    > ![[IMG-20260404031953155.png|800]]
    >
    > 之前有了小疑惑：为啥这里只有 JDK ，而不需要 maven？
    >
    > - maven 并不需要写在Dockerfile中，应为maven这里作用就是一个打包工具
    >
    > - 在windows打jar包的过程中已经将需要的以来下载下来，并打进jar包里了，所以项目部署环境里并不需要maven
    >
    > - 仔细一想确实不需要
    >
    > - 但如果复杂项目中，需要调用的运行着mysql、redis的容器，这些怎么配还不太清楚...
6. 使用maven 打包，只保留 Dockerfile 与 jar包 dockerSpringBootDemo-0.0.1-SNAPSHOT.jar
7. 传到服务器上：

    ```Plain
     [root@docker002 ~]# ls
      dockerDemo  ...
     [root@docker002 ~]# cd dockerDemo/
     [root@docker002 dockerDemo]# ls
     Dockerfile  target
     [root@docker002 dockerDemo]# cd target/
     [root@docker002 target]# ls
     dockerSpringBootDemo-0.0.1-SNAPSHOT.jar
    ```
