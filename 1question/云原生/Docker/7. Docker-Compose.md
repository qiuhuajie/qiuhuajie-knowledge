# 1. **存在的问题**
1. Docker 提倡的理念时“一个容器一个进程”，假设一个服务需要多个进程组成，就需要多个容器组成一个系统，相互分工和配合对外提供完整服务
2. 如博客系统：
    - 组件1 - `mariadb`
    - 组件2 - `wordpress`的`appache web`
3. 在启动容器时，同一台主机下如果两个容器之间需要数据交流，使用 --link 选项建立两个容器之间的互联，前提是建立 `mariadb` 已经开启
    1. `docker start db` （先开启`mariadb`）
    2. `dickaer start MywordPress`
    3. 停止时，需要先关闭`MywordPress`
4. 但 **如果有上百个容器，在开启或关闭时需要按一定规则取操作，人工实现会很困难**
# 2. **解决方法**
> [!important] **Docker-Compose**
> 
> （了解），==**使用 K8S，功能更强大、方便**==
1. **容器编排工具**
2. 允许用户**在一个模板（YARL格式）中定义一组相关联的容器**，会根据 --link 等参数，对启动的优先级进行排序
3. **Compose安装：**

# 3. **Docker-Compose 的使用**
1. yaml文件：

    ```YAML
     version: '2'
     services:
         db:
             image: mysql:5.7
             restart: always
             environment:
                 MYSQL_ROOT_PASSWORD: somewordpress
                 MYSQL_DATABASE: wordpress
                 MYSQL_USER: wordpress
                 MYSQL_PASSWORD: wordpress
         wordpress:
             depends_on:
                 - db
             image: wordpress:latest
             restart: always
             ports:
                 - "8000:80"
             environment:
                 WORDPRESS_DB_HOST: db:3306
                 WORDPRESS_DB_USER: wordpress
                 WORDPRESS_DB_PASSWORD: wordpress
    ```
2. 使用compose启动容器

    ```Bash
     [root@docker001 ~]# ls
     anaconda-ks.cfg  docker  initial-setup-ks.cfg  公共  模板  视频  图片  文档  下载  音乐  桌面
     [root@docker001 ~]# vim wordpress.yaml
     [root@docker001 ~]# ls
     anaconda-ks.cfg  docker  initial-setup-ks.cfg  wordpress.yaml  公共  模板  视频  图片  文档  下载  音乐  桌面
     [root@docker001 ~]# mv wordpress.yaml docker-compose.yaml
     [root@docker001 ~]# docker-compose up -d
     Creating root_db_1 ... done
     Creating root_wordpress_1 ... done
    ```
3. compose 参数：
    1. `f` ：指定使用的 yaml 文件位置
    2. `ps`：显示所有的容器

        ```Bash
         [root@docker001 ~]# docker-compose ps
               Name                    Command               State          Ports
         --------------------------------------------------------------------------------
         root_db_1          docker-entrypoint.sh mysqld      Up      3306/tcp, 33060/tcp
         root_wordpress_1   docker-entrypoint.sh apach ...   Up      0.0.0.0:8000->80/tcp
        ```
    3. `logs`：查看日志信息
    4. `up -d`：启动容器项目
    5. `config -q`：验证yaml 文件是否正确
