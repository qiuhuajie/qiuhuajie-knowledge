# 一、语法
1. `FROM`：基础镜像，一切从这里开始构建
2. `MAINTAINER`：镜像是谁写的，姓名+邮箱
3. `RUN`：可以运行任何被基础 image 所支持的命令，通常用来运行镜像构建的时候需要运行的命令

    ```Bash
     RUN apt-get install -y openssh-server apt-utils tomcat7
    ```
4. `ADD`：**从 src 复制文件（源码包）到 container 的 dest 路径**

    ```Bash
     ADD docker-tomcat7 /etc/init.d/tomcat7
    ```
5. `USER`：设置container容器的用户，默认时root用户

    ```Bash
     USER daemom = ENTRYPOINT /etc/init.d/tomcat7 start && /usr/sbin/sshd -D
    ```
6. `WORKDIR`：镜像的工作目录

    ```Bash
     WORKDIR /var/www
    ```
7. `VOLUME`：挂载的目录
8. `EXPOSE`：暴露端口配置

    ```Bash
     EXPOSE 22
     EXPOSE 8080
    ```
9. `CMD`：指定这个容器启动的时候要运行的命令，只有最后一个会生效，可被替代

    ```Bash
     CMD ["/usr/bin/supervisord"]
    ```
10. `ENTRYPOINT`：**指定这个容器启动的时候要运行的命令，可以追加命令**

    ```Bash
     ENTRYPOINT /etc/init.d/tomcat7 start && /usr/sbin/sshd -D
    ```
11. `ONBUILD`：当构建一个被继承 Dockerfile 这个时候就会运行ONBUILD 的指令
12. `COPY`：**类似ADD，将我们文件拷贝到镜像中**
13. `ENV`：构建的时候设置环境遍量

    ```Bash
     ENV JAVA_HOME /root/jdk1.7.0_25
    ```
