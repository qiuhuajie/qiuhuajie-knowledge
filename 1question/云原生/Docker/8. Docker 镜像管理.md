# 一、镜像介绍
1. 容器创建时需要指定镜像，每个镜像都由唯一标示 ImageID，和容器的 Container ID 一样，默认 128 位，可以使用前 16 位缩略形式
2. 也可以使用镜像名和版本号组合成唯一标识符，如果省略版本号，默认使用最新版本标签（latest）
# 二、镜像的分层

Docker 的镜像通过联合文件系统（union filesystem）将各层文件系统叠加在一起

1. **bootsfs**：用于系统引导的文件系统，包括bootloader 和 kernel （linux内核），容器启动完成后会被卸载以节省内存资源
2. **rootfs**：位于 bootfs 之上，表现为 Docker 容器根文件系统；包含了一般系统上的常见目录结构，类似于/dev, /proc, /bin等等以及一些基本的文件和命令

    ![[IMG-20260414230546233.png|800]]

    1. 传统模式中，系统启动时内核挂载 rootfs 时会首先将其挂载为”只读”模式，完整性自检完成后，将其挂载为读写模式
    2. **Docker 中 rootfs 由内核始终挂载为只读模式，而后通过联合文件系统技术挂载一个“可写”层（真正的写入层）**
    3. **image是只读的，container部分则是可写的**。如果用户想要修改底层只读层上的文件，这个文件就会被先拷贝到上层，修改后驻留在上层，并屏蔽原有的下层文件
    4. 最后一部分是容器(container)， 容器是可读写的；既然是可读写的，就会有状态，容器共有两种状态：running 和 exited
    5. 在系统启动时，Docker会首先一层一层的加载image，直到最先面的base image，而这些image都是只读的
    6. 最后，在最上层添加可读写的一个容器， 这里存放着诸如unique id，网络配置之类的信息

        ![[IMG-20260414230546298.png|800]]

    7. Base Image：
        1. 一个镜像中需要安装一些环境，环境之上部署应用程序，再打包；而基础操作系统内核 kernel 是共享的
        2. 但内核之上还有一些不同的 lib 库及周边工具，这些就是 Base Image 基础镜像

            ![[IMG-20260414230546318.png|800]]

# 三、镜像特性
1. **已有的分层只能读不能修改**：只要读入容器，就不是最上层，容器中最上层永远是读写层
2. **上层镜像优先级大于底层镜像**：例如--修改的数据，以上层的修改为准
