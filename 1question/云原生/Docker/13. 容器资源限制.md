1. CGroup （ control groups ）是Linux内核提供的一种可以限制、记录、隔离进程组所使用的物理资源的机制，如 cpu、内存和 io 资源
1. docker 容器对于宿主操作系统来说，就是一个进程，故可以使用进程控制来对容器的资源进行限制
1. 默认情况下，如果不对容器做任何限制，容器能够占用当前系统能给容器提供的所有资源
1. Docker 限制可以从内存，CPU和IO三方面
1. OOME机制：out of memory exception
    
    1. 一旦发生 OOME，任何进行都有可能被杀死，包括 docker daemon 在内
    
    1. 为此Docker 调整了docker daemon 的 OOM 优先级，以免被内核杀死
    
# 1. **对内存资源限制**
1. 在docker 启动参数时和内存相关的一些参数
    
    1. m --memory：对容器使用内存大小的硬限制
    
    1. -memory-swap：对容器使用的内存大小和交换分区大小的总和的上限
    
    1. -memory-swappiness：主机可以把容器使用的匿名页swap出来的比例
    
    1. -memory-reservation：对容器使用内存大小的软限制
    
    1. -kernel-memory：容器能够使用内核的内存大小上限
    
    1. -oom-kil-disable：是否运行 OOM 的时候杀死容器
    
# 2. **对CPU资源限制**
1. Docker 提供的CPU资源限制选项可以在多核系统上限制容器能利用哪些 CPU
1. 而对容器最多能使用的CPU时间有两种限制方式:
    
    1. 一是有多个CPU密集型的容器竞争CPU时，设置各个容器能使用的CPU时间相对比例
    
    1. 二是以绝对的方式设置容器在每个调度周期内最多能使用的CPU时间
    
1. 选项参数：
    
    1. -cpuset-cpus=""：允许使用的CPU集，值可以为 "0-3" 或 "0, 1"
    
    1. c,--cpu-shares=0：CPU共享权值（相对权重)，默认值1024
        
        ```Plain
         container1 :--cpu-shares=2048
         container2 :不设置--cpu-shares （默认1024）
         container3 :--cpu-shares=2048
         # 则三个容器会 分别使用 CPU资源的 2/5, 1/5, 2/5
        ```
        
    
    1. -cpuset-mems=""：允许在上执行的内存节点(MEMs)
    
    1. 🔴--cpu-period=0：即可设置调度周期，CFS周期的有效范围是1ms~1s，对应的-cpu-period的数值范围是1000~1000000
    
    1. 🔴--cpu-quota=0：设置在每个周期内容器能使用的CPU时间，容器的CPU配额必须不小于1ms，即--cpu-quota的值必须>=1000，单位微秒
    
    1. -cups：能够限制容器可以使用的主机的CPU个数，并且值可以使用1.5等小数
    
1. 示例：
    
    ```Plain
     docker run -it --cpu period=50000 --cpu-quota=25000 ubuntu:16.04 /bin/bash
     
     docker run -it --cpu-period=10000 --cpu-quota=20000 ubuntu:16.04 /bin/bash
    ```
    
# 3. **资源限制实验**
1. 容器可使用的总内存资源：
    
    ```Plain
     [root@docker001 ~]# free -m
                   total        used        free      shared  buff/cache   available
     Mem:           3931         686        2437          23         807        2939
     Swap:          4095           0        4095
    ```
    
1. 内存限制：
    
    ```Plain
     [root@docker001 ~]# docker run --name stressTest -it --rm -m 156m lorel/docker-stress-ng:latest stress -vm 2
     # 容器使用内存上限为 156m
    ```
    
    ```Plain
     # 新会话
     [root@docker001 ~]# docker stats stressTest
    ```
