# 一、示例

![[IMG-20260414230509961.png|800]]

1. 下载`wordpress`和`mariadb`镜像

    ```Bash
    docker pull wordpress
    docker pull mariadb
    ```

    ![[IMG-20260404031954428.png|800]]

2. 查看`pull`好的镜像

    ```Bash
     [root@docker001 docker]# docker images
     REPOSITORY          TAG                 IMAGE ID            CREATED             SIZE
     wordpress           latest              ccd598bc2c91        13 days ago         616 MB
     mariadb             latest              12e05d5da3c5        2 weeks ago         409 MB
     hello-world         latest              feb5d9fea6a5        6 weeks ago         13.3 kB
    ```
3. 分别将两个镜像运行起来

    ```Bash
     [root@docker001 docker]# docker run --name db --env MYSQL_ROOT_PASSWORD=example -d mariadb
     40dfa03f445c3f568d7d833899782b0bd762a3f134a6dc623778a1c635452d3f
     [root@docker001 docker]# docker run --name MyWordPress --link db:mysql -p 8080:80 -d wordpress
     c6610f7ed6d87e03ab8ba9fb2c5fb7274aea73f035bda075c21128d3d8340975
     # --name 为容器起别名
     # --link 设置链接关系
     # -p 进行端口映射
     # -d 放在后台运行
     # wordpress 镜像名
    ```
4. 查看此时的进程运行情况

    ```Bash
     [root@docker001 docker]# docker ps
     CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS              PORTS                  NAMES
     c6610f7ed6d8        wordpress           "docker-entrypoint..."   14 seconds ago      Up 13 seconds       0.0.0.0:8080->80/tcp   MyWordPress
     40dfa03f445c        mariadb             "docker-entrypoint..."   56 seconds ago      Up 55 seconds       3306/tcp               db
    ```
5. 通过 192.168.10.110:8080 即可访问`wordpress`服务
