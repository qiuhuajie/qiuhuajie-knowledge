# **docker run**
1. 起应用：将镜像展开为容器
1. `docker run [OPTIONS] ImageName [COMMAND] [ARG...]`
    
    **可选参数[OPTIONS]：**
    
    1. `p`：端口映射
        
        ![[IMG-20260404031954352.png|Untitled 387.png]]
        
    
    1. `d`：设置容器为后台运行
    
    1. `v`：挂载数据卷[[12. 数据存储 卷挂载]]
        
        将容器内的文件与Host主机的文件系统进行挂载绑定
        
        ![[IMG-20260404031954402.png|Untitled 1 292.png]]
        
    
    1. `-name`：给容器命名
    
    1. `-restart=always`：让容器随docker的重启而重启
        
        ```Bash
         [root@docker001 ~]# systemctl restart docker
         # 看到之前的容器全关闭了：Exited (0)
         [root@docker001 ~]# docker ps -a
         
         # 使用--restart=always让容器随docker的重启而重启
         [root@docker001 ~]# docker run --restart=always --name db1 --env MYSQL_ROOT_PASSWORD=example -d mariadb
         [root@docker001 ~]# docker ps -a
         [root@docker001 ~]# systemctl restart docker
         [root@docker001 ~]# docker ps -a
        ```
        
    
    1. `-cpu`、`--m` 对容器的CPU和内存进行限制（详见 3.15）
    
    1. `h x.xx.xx` ：设置主机名
    
    1. `-dns x.xx.xx` ：设置容器的dns服务器
    
    1. `-dns-search` ：DNS搜索设置
    
    1. `-add-host hostname:IP` ：注入hostname和IP的解析映射
        
        ```Bash
         [root@docker001 ~]# docker run --name db2 --add-host www.qhj.com:192.168.13.11 --env MYSQL_ROOT_PASSWORD=example -d mariadb
         26dba896ae83e222da37775ed7c6365bfdc38f1705e64c58293299aa5487d078
         
         # 进入 db2 容器的 bash 内
         [root@docker001 ~]# docker exec -it db2 /bin/bash
         
         root@26dba896ae83:/# cat /etc/hosts
         127.0.0.1   localhost
         ::1 localhost ip6-localhost ip6-loopback
         fe00::0 ip6-localnet
         ff00::0 ip6-mcastprefix
         ff02::1 ip6-allnodes
         ff02::2 ip6-allrouters
         192.168.13.11   www.qhj.com         \#此时db2容器中的hosts文件已添加相应的映射
         172.17.0.3  26dba896ae83
        ```
        
    
    1. `-rm` ：服务停止时自动删除
        
        ```Bash
         [root@docker001 ~]# docker run --name db3 --rm --env MYSQL_ROOT_PASSWORD=example -d mariadb
         6223136ae5cc8c37a83a5f955a0d2f4d132d85f938c2df6914f9b931b2157221
         [root@docker001 ~]# docker stop db3
         db3
         \#db3一结束，就被删除
         [root@docker001 ~]# docker ps -a
         CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS                      PORTS               NAMES
         26dba896ae83        mariadb             "docker-entrypoint..."   2 minutes ago       Up 2 minutes                3306/tcp            db2
         6f898c6093c9        mariadb             "docker-entrypoint..."   10 minutes ago      Up 10 minutes               3306/tcp            db1
        ```

```Bash
 richarvey/nginx-php-fpm           Container running Nginx + PHP-FPM capable ...   818                  [OK]
```
# **docker pull**
从镜像仓库下载镜像到本地
```Bash
 docker pull nginx
```
# **docker push**
推送本地镜像到 docker 远程仓库
```Bash
 docker login
 
 docker push new-repo:tagname
```
# **docker save**
将镜像保存成压缩包文件
```Bash
 docker save -o abc.tar imageName:tag
```
# **docker load**
在其他机器上加载这个镜像
```Bash
 docker load -i abc.tar
```
# **docker images**
查看本地镜像
```Bash
 \#镜像size不一定代表这个镜像使用了多少资源
 \#因为镜像分层且能够共用
 [root@docker001 ~]# docker images
 REPOSITORY          TAG                 IMAGE ID            CREATED             SIZE
 wordpress           latest              ccd598bc2c91        2 weeks ago         616 MB
 mariadb             latest              12e05d5da3c5        3 weeks ago         409 MB
 hello-world         latest              feb5d9fea6a5        6 weeks ago         13.3 kB
```
# **docker rmi**
镜像删除
```Bash
 \#需要加版本号
 docker rmi hello-world:latest
 
 \#不写版本的话默认是最新版本
 docker rmi hello-world
 
 \#或使用镜像ID的前几位
 docker rmi fce2
```
