# 一、Docker 介绍
1. Linux Container 是 Docker 的底层实现技术，是一种内核虚拟化技术，可以提供轻量级的虚拟化，以便隔离进程和资源
2. Docker 是 PaaS 提供商 dotCloud 开源的一个基于 LXC（Linux Container） 的高级容器引擎（管理工具），源代码托管在 GitHub 上，基于 go 语言并遵从 apache 2.0 开源协议
    > go 可以看成现代版的 c，天生支持高并发多线程
3. Docker 设想是交付运行环境如海运，OS 如同一个货轮，每一个在 OS 基础上的软件都如同一个集装箱，用户可以通过标准化手段自由组装运行环境，同时集装箱的内容可以由用户自定义，也可以由专业人员制造
# 二、解决的问题
1. 由Java、JavaScript、C++ 等不同语言开发出来的应用程序，最终打成的运行文件包各不相同，且安装启动的操作也不同
2. 如何制定一个统一的标准，不需要关注各个应用程序的差异性，使用一套操作就可以完成不同应用的安装与运行（如同 windows 中的 .exe文件）
# 三、应用部署方式的演变
## 1. 传统的部署方式

![[IMG-20260404031953462.png|588]]

1. 互联网早期，会直接将应用程序部署在物理机上
2. **优点：**简单，不需要其它技术的参与
3. **缺点：**不能为应用程序定义资源使用边界，很难合理地分配计算资源，而且程序之间容易产生影响
## 2. 虚拟化部署方式

![[IMG-20260404031953562.png|603]]

1. 通过使用 **中间层（Hypervisor）** 将一台或多台独立的虚拟机器运行于物理硬件之上，也被称为**操作系统级虚拟化**
2. 每台虚拟机相互独立，且都有自己的宿主OS
3. **优点：**
    - 隔离性强：应用程序运行在各自的宿主OS 上，程序环境不会产生互相影响，实现了**基于Linux内核的隔离**，提供了一定程度的安全性
4. **缺点：**
    - 每台虚拟机都有自己的宿主 OS，每个宿主 OS 都有自己的类库（Libs/Bins），因此，宿主 OS 本身也会消耗很多资源（甚至宿主 OS 的镜像可以达到 GB 级别）
    - 且创建使用稍微复杂、启动速度慢（分钟级）、移植与分享不方便

## 3. 容器化技术

![[IMG-20260404031953629.png|600]]

1. 没有虚拟化中间层了，而是提供了一个 容器的运行时环境，可以让多个独立的容器运行在同一个宿主机上
2. **优点：**
    - 应用程序运行在容器中，而容器则不需要宿主OS ，故节省的资源用于支持应用程序使用（即相同的硬件情况下，容器技术可以支持更多的访问）
    - 基础镜像 MB 级别、创建简单、启动速度秒级、移植和分析方便
    - 隔离性强：可以保证每个容器拥有自己的文件系统、CPU、内存、进程空间等
    - 容器化的应用程序可以跨云服务商、跨Linux操作系统发行版进行部署
3. **But 存在**==**容器编排**==**问题：**
    1. **一个容器故障停机了，怎么样让另外一个容器立刻启动去替补停机的容器**
        > 举例：一个容器宕机了，里面的应用程序也就无法对外提供服务了，如何重新启用一个容器，让其顶替当掉的应用对外提供服务
    2. **当并发访问量变大的时候，怎么样做到横向扩展容器数量**
        > 举例：一个容器中运行着 nginx，本来有 1000 的访问量，nginx 是可以顶住的，突然并发访问量激增为 5000，怎么立刻做出反应增加 4 个 nginx 容器，来应对；或者
4. ==产生了一些====**容器编排的软件：**==
    - **Swarm**：Docker自己的容器编排工具
    - **Mesos**：Apache的一个资源统一管控的工具，需要和Marathon结合使用
    - ⭐**Kubernetes**：Google开源的的容器编排工具
