# 1. **Docker 客户端 和 服务器**
![[IMG-20260404031953953.png|Untitled 385.png]]
1. Docker 是一个客户端/服务器（C/S）架构的程序
1. Docker 客户端向Docker 服务器或守护进程发出请求，服务器或守护进程将完成的所有工作返回结果
1. Docker 守护进程有时也称为Docker 引擎
1. Docker 提供了一个命令行工具 docker 以及一整套API来与守护进程交互
1. 用户可以在同一台宿主主机上运行Docker 守护进程和客户端，也可以从本地的Docker 客户端连接到运行在另一台宿主主机上的远程 Docker 守护进程
# 2. **Docker 镜像**
1. Docker 镜像就是一个封装好的运行环境
1. Docker 镜像运行后变成容器，镜像是**静态的**，可以把镜像看作**容器的“源代码”**，体积很小，便于分享储存和更新
1. 镜像是基于联合文件系统的一种曾是结构，由一系列指令一步一步构建出来
# 3. **Docker 仓库（Registry）**
1. Docker Hub：
    
    > [!info] Docker Hub Container Image Library | App Containerization  
    >  
    > [https://hub.docker.com/](https://hub.docker.com/)  
    
1. 仓库分为公有和私有两种：
    
    1. Docker 公司运营的公有仓库叫做Docker Hub，用户可以分享和保存自己的镜像
    
    1. 用户也可以在Docker Hub上保存自己的私有镜像
    
    1. 用户甚至可以夹着自己的私有仓库，私有的仓库可以受到防火墙的保护，满足安全的特殊需求
    
# 4. **Docker 容器**
1. Docker 容器是**镜像运行完成后得到的运行着的**==**运行环境**==，用户只需要把自己的应用程序或服务打包放进容器即可
1. 容器是基于镜像启动起来的，容器中可以运行一个或多个进程
