# 2. Docker **网络模式修改**

> 即在docker run 时，可以添加一些参数对 docker 网络进行修改
1. `-net`
    
    1. bridge：Docker 默认方式，即Docker0网桥
    
    1. none：容器没有网络栈，比如有的容器只有计算功能，结果只需要保存在本主机即可，如果有网络暴露反而不安全
    
    1. container：直接使用其他容器的网络，即加入其他容器的 network namespace
    
    1. Host：表示容器使用 Host 主机的网路，（如tomcat启动时，设置host，直接访问本机ip:8080即可访问tomcat主页，不再需要对口进行映射）
        
        ```Plain
         docker run --name tomcat --net=Host -d tomcat:v1.0
         # 192.168.10.110:8080 即可访问
         
         # docker run --name tomcat -p 80:8080 -d tomcat:v1.0
        ```
        
    
1. `p`
    
    1. p :<containerPort>：将指定的容器端口映射至不确定IP地址的主机的一个动态端口
    
    1. p <hostPort>:<containerPort>：映射至指定的主机端口（80:8080）**（常用）**
    
    1. p <IP>:<containerPort>：将指定的容器端口映射至指定IP地址的主机的一个动态端口
    
    1. p <IP>:<hostPort>:<containerPort>映射至指定IP地址主机的主机端口**（常用）**
    
    1. P （大写）：暴露需要的所有端口
    
    > 命令：
    > 
    > ```Plain
    >  # docker port 容器名 ：查看容器的端口映射关系
    >  [root@docker002 ~]# docker run --name tomcat -p 80:8080 -d 3406d0282edb
    >  681d2f8888d20d3d3c56682ada69bc6099bd4e4280707d73757ffca0e388c167
    >  [root@docker002 ~]# docker port tomcat
    >  8080/tcp -> 0.0.0.0:80
    > ```
    
# 3. **网络隔离**
1. 在默认情况下两个容器使用的时 网桥 模式，可以互相通信
    
    ```Plain
     # 启动两个容器
     [root@docker002 ~]# docker run --name tomcat_1 -p 80:8080 -d 3406d0282edb
     [root@docker002 ~]# docker run --name tomcat_2 -p 81:8080 -d 3406d0282edb
     
     # 分别进入两个容器的 /bin/bash 查看IP
     ifconfig
     tomcat_1: 172.17.0.2
     tomcat_2: 172.17.0.3
     
     # 在tomcat_2 ping tomcat_1
     [root@ccc5608d22eb /]# ping -c 4 172.17.0.2
     PING 172.17.0.2 (172.17.0.2) 56(84) bytes of data.
     64 bytes from 172.17.0.2: icmp_seq=1 ttl=64 time=0.048 ms
     64 bytes from 172.17.0.2: icmp_seq=2 ttl=64 time=0.120 ms
     64 bytes from 172.17.0.2: icmp_seq=3 ttl=64 time=0.124 ms
     64 bytes from 172.17.0.2: icmp_seq=4 ttl=64 time=0.129 ms
     
     # 可以ping通
    ```
    
1. 基础命令说明：
    
    1. 查看当前可用的网络类型：
        
        ```Plain
         [root@docker002 ~]# docker network ls
         NETWORK ID          NAME                DRIVER              SCOPE
         3783951dcea9        bridge              bridge              local
         e18e704839d9        host                host                local
         05d2e7e8c18b        none                null                local
         a37ed99266a7        root_default        bridge              local
        ```
        
    
    1. 开创新的网络命名空间：
        
        ```Plain
         [root@docker002 ~]# docker network create -d bridge lamp
         f5ecc2411cae89de1f8bc0e8bddb2ca00d34b188ffde68983fdbdd158d9c9064
        ```
        
    
1. 如何实现两个容器之间网络的隔离？
    
    1. 创建两个网络命名空间
        
        ```Plain
         [root@docker002 ~]# docker network create -d bridge lamp
         f5ecc2411cae89de1f8bc0e8bddb2ca00d34b188ffde68983fdbdd158d9c9064
         [root@docker002 ~]# docker network create -d bridge lnmp
         f3609752199afc1bb683063074ccae9c642c4366c3d4d7332d95d600ce26f953
         [root@docker002 ~]# docker network ls
         NETWORK ID          NAME                DRIVER              SCOPE
         3783951dcea9        bridge              bridge              local
         e18e704839d9        host                host                local
         f5ecc2411cae        lamp                bridge              local
         f3609752199a        lnmp                bridge              local
         05d2e7e8c18b        none                null                local
         a37ed99266a7        root_default        bridge              local
        ```
        
    
    1. 启动两个容器，分别指定两个不同的网络命名空间
        
        ```Plain
         [root@docker002 ~]# docker run --name tomcat_3 --network=lamp -p 82:8080 -d 3406d0282edb
         af1e693e98e5ac42a6e64292536b045c63a4e6f595acbc7ecf6171eda0c39e2b
         [root@docker002 ~]# docker run --name tomcat_4 --network=lnmp -p 83:8080 -d 3406d0282edb
         3cffaa4cd8b9ce22f2fbe834c3e43b52e35912cf150be867eb860b32d5555841
        ```
        
    
    1. 分别查看两个容器的IP地址：
        
        ```Plain
         tomcat_3 172.19.0.2
         tomcat_4 172.20.0.2
        ```
        
    
    1. 可以看到两个容器都不在同一个网段了，必然ping不通无法通信
    
1. 如何实现不同主机上的容器的通信？
    
    1. 让容器不使用docker0
    
    1. 为他创建一个新的网桥，赋予一个独立的IP地址
