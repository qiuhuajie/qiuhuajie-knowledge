1. 集群规划
    
    1. 在 hadoop102、 hadoop103 和 hadoop104 三个节点上都部署 Zookeeper
    
    1. 思考：如果是 10 台服务器，需要部署多少台 Zookeeper？**生产经验：安装奇数台**
        
        - 10 台服务器： 3 台 zk
        
        - 20 台服务器： 5 台 zk
        
        - 100 台服务器： 11 台 zk
        
        - 200 台服务器： 11 台 zk
        
    
    1. 服务器台数多：
        
        1. 好处，提高可靠性
        
        1. 坏处：提高通信延时
        
    
1. 拷贝 apache-zookeeper-3.5.7-bin.tar.gz 安装包到 Linux 系统下
1. 解压到指定目录
    
    ```Bash
     tar -zxvf apache-zookeeper-3.5.7-bin.tar.gz -C /opt/module/
    ```
    
1. 修改名称
    
    ```Bash
     mv apache-zookeeper-3.5.7-bin/ zookeeper-3.5.7/
    ```
    
1. **配置服务器编号**
    
    1. 在/opt/module/zookeeper-3.5.7/这个目录上创建zkData文件夹
        
        ```Bash
         [cool@hadoop102 zookeeper-3.5.7]$ mkdir zkData
         [cool@hadoop102 zookeeper-3.5.7]$ cd zkData/
         [cool@hadoop102 zkData]$ pwd
         /opt/module/zookeeper-3.5.7/zkData
        ```
        
    
    1. 在/opt/module/zookeeper-3.5.7/zkData 目录下创建一个myid文件，并在文件中添加与 server 对应的编号
        
        ```Bash
         vim myid[cool@hadoop102 zkData]$ vim myid
        ```
        
        ```Plain
         2
        ```
        
    
1. **修改配置文件**
    
    1. 将/opt/module/zookeeper-3.5.7/conf 这个路径下的 zoo_sample.cfg 修改为 zoo.cfg
        
        ```Bash
         mv zoo_sample.cfg zoo.cfg
        ```
        
    
    1. 打开 zoo.cfg 文件，修改 dataDir 路径
        
        ```Bash
         # The number of milliseconds of each tick
         tickTime=2000
         # The number of ticks that the initial
         # synchronization phase can take
         initLimit=10
         # The number of ticks that can pass between
         # sending a request and getting an acknowledgement
         syncLimit=5
         # the directory where the snapshot is stored.
         # do not use /tmp for storage, /tmp here is just
         # example sakes.
         
         #======修改 dataDir 路径===========
         \#tmp/ 下的文件会定时清理
         \#dataDir=/tmp/zookeeper
         dataDir=/opt/module/zookeeper-3.5.7/zkData
         
         # the port at which the clients will connect
         clientPort=2181
        ```
        
    
    1. 打开 zoo.cfg 文件，增加如下配置
        
        ```Bash
         ######################\#cluster##########################
         server.2=hadoop102:2888:3888
         server.3=hadoop103:2888:3888
         server.4=hadoop104:2888:3888
        ```
        
    
    1. **配置参数解读:**
        
        ```Bash
         server.A=B:C:D
        ```
        
        1. A 是一个数字，**表示这个是第几号服务器**；集群模式下配置一个文件myid，这个文件在dataDir目录下，这个文件里面有一个数据就是 A 的值，Zookeeper 启动时读取此文件，拿到里面的数据与zoo.cfg 里面的配置信息比较从而判断到底是哪个server
        
        1. B 是这个**服务器的地址**
        
        1. C 是这个服务器 Follower 与集群中的 Leader 服务器**交换信息的端口**
        
        1. D 是万一集群中的 Leader 服务器挂了，需要一个端口来**重新进行选举**，选出一个新的Leader，而这个端口就是用来执行选举时服务器相互通信的端口
        
    
1. 拷贝配置好的 zookeeper 到其他机器上
    
    ```Bash
     [cool@hadoop102 module]$ xsync zookeeper-3.5.7/
    ```
    
1. 分别在 hadoop103、 hadoop104 上修改 myid 文件中内容为 3、 4
    
    ```Bash
     [cool@hadoop103 zkData]$ vim myid
     [cool@hadoop103 zkData]$ cat myid
     3
    ```
    
    ```Bash
     [cool@hadoop104 zkData]$ vim myid
     [cool@hadoop104 zkData]$ cat myid
     4
    ```
    
1. ==**操作ZooKeeper**==
    
    1. **当只启动一个Hadoop104节点后**，查看Zookeeper状态
        
        ```Bash
         [cool@hadoop104 zookeeper-3.5.7]$ bin/zkServer.sh start
         ZooKeeper JMX enabled by default
         Using config: /opt/module/zookeeper-3.5.7/bin/../conf/zoo.cfg
         Starting zookeeper ... STARTED
         
         [cool@hadoop104 zookeeper-3.5.7]$ bin/zkServer.sh status
         ZooKeeper JMX enabled by default
         Using config: /opt/module/zookeeper-3.5.7/bin/../conf/zoo.cfg
         Client port found: 2181. Client address: localhost.
             //发生错误：集群中只有半数以上节点存活，Zookeeper集群才能正常服务
         Error contacting service. It is probably not running.
        ```
        
    
    1. 分别Hadoop102和Hadoop103启动 Zookeeper
        
        ```Bash
         [cool@hadoop102 zookeeper-3.5.7]$ bin/zkServer.sh start
         [cool@hadoop103 zookeeper-3.5.7]$ bin/zkServer.sh start
        ```
        
    
    1. 查看状态
        
        ```Bash
         [cool@hadoop102 zookeeper-3.5.7]$ bin/zkServer.sh status

         Using config: /opt/module/zookeeper-3.5.7/bin/../conf/zoo.cfg
         Client port found: 2181. Client address: localhost.
         Mode: follower
             //hadoop102为follower
        ```
        
        ```Bash
         [cool@hadoop103 zookeeper-3.5.7]$ bin/zkServer.sh status
         ZooKeeper JMX enabled by default
         Using config: /opt/module/zookeeper-3.5.7/bin/../conf/zoo.cfg
         Client port found: 2181. Client address: localhost.
         Mode: follower
             //hadoop103为follower
        ```
        
        ```Bash
         [cool@hadoop104 zookeeper-3.5.7]$ bin/zkServer.sh status
         ZooKeeper JMX enabled by default
         Using config: /opt/module/zookeeper-3.5.7/bin/../conf/zoo.cfg
         Client port found: 2181. Client address: localhost.
         Mode: leader
             //hadoop104为 leader
        ```
