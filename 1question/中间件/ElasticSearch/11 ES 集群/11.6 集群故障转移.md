# 一、集群故障转移
1. 集群的master节点会监控集群中的节点状态

    ![[IMG-20260620224143038.png|800]]

2. **如果发现有节点宕机：**master节点会立即将宕机节点的分片数据迁移到其它节点，确保数据安全，这个叫做故障转移

    ![[IMG-20260620224153926.png|800]]

    > 在迁移中，会保证分片和自己的副本不会分配到同一个node上
3. **如果是master节点自己发生故障：**Node2 被选举成master节点，Node2 会查看集群状态：发现数据R-1和P-0 是不安全的，则会迁移数据

    ![[IMG-20260620224201939.png|800]]

4. 实操：
    1. 最初 node 3 是主节点

    ![[IMG-20260620224208199.png|643]]

    2. 停掉 node3

    ![[IMG-20260620224213371.png|800]]

    3. 查看此时集群情况
        1. 集群状态变为黄色

    ![[IMG-20260620224216474.png|656]]

        2. 一段时间后，数据恢复，node1 被选为新的主节点

    ![[IMG-20260620224218688.png|662]]

    4. 再次启动 node3 ，数据恢复了，但node3 不再是主节点了

    ![[IMG-20260620224220058.png|654]]

5. **保证出现故障时数据的动态迁移，同时也可以实现新增节点时，原有数据的集群动态扩容**
