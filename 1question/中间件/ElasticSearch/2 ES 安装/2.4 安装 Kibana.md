> kibana可以给我们提供一个elasticsearch的可视化界面
# 1. **运行容器**
1. 运行docker命令，部署kibana
    
    ```Bash
     docker run -d --name kibana -e ELASTICSEARCH_HOSTS=http://es:9200 --network=es-net -p 5601:5601  kibana:7.12.1
    ```
    
1. 命令解释：
    
    - `-network es-net` ：加入一个名为es-net的网络中，与elasticsearch在同一个网络中
    
    - `e ELASTICSEARCH_HOSTS=http://es:9200"`：设置环境变量，设置elasticsearch的地址，因为kibana已经与elasticsearch在一个网络，因此可以用容器名直接访问elasticsearch
    
    - `p 5601:5601`：端口映射配置

# 3. **DevTools**
1. kibana中提供了一个DevTools界面
    

    ![[IMG-20260404031902935.png|Untitled 2 101.png]]

    
1. 这个界面中可以编写DSL来操作elasticsearch。并且对DSL语句**==有自动补全功能==**
    

    ![[IMG-20260404031902984.png|Untitled 3 76.png]]

    

    语法说明：

    
    - POST：**请求方式**
    
    - /_analyze：**请求路径**，这里省略了[http://192.168.150.101:9200](http://192.168.150.101:9200/)，有kibana帮我们补充
    
    - **请求参数**，json风格：
        
        - analyzer：分词器类型，这里是默认的standard分词器
        
        - text：要分词的内容
