> es在创建倒排索引时需要对文档分词；在搜索时，需要对用户输入内容分词。但默认的分词规则对中文处理并不友好
# 1. **查看 ES 容器运行挂载的数据卷**
1. 安装插件需要知道elasticsearch的plugins目录位置，而我们用了数据卷挂载，因此需要查看elasticsearch的数据卷目录，通过下面命令查看:
    
    ```Bash
     docker volume inspect es-plugins
    ```
    
    显示结果：
    
    ```JSON
     [
         {
             "CreatedAt": "2022-05-06T10:06:34+08:00",
             "Driver": "local",
             "Labels": null,
             "Mountpoint": "/var/lib/docker/volumes/es-plugins/_data",
             "Name": "es-plugins",
             "Options": null,
             "Scope": "local"
         }
     ]
    ```
    
1. 说明plugins目录被挂载到了：`/var/lib/docker/volumes/es-plugins/_data`这个目录中
1. 进入到这个目录下
# 2. **解压上传缩分词器安装包**
1. 下载地址：[https://github.com/medcl/elasticsearch-analysis-ik](https://github.com/medcl/elasticsearch-analysis-ik)
1. 把 IK 分词器解压缩，重命名为ik，放在`/var/lib/docker/volumes/es-plugins/_data`这个目录中
    
    ![[IMG-20260405035438292.png|Untitled 180.png]]
    
# 3. **重启容器**
```Bash
 docker restart es
```
# 4. **测试验证分词器**
1. **ik 分词器包含两种模式：**
    
    1. `ik_smart`：最少切分，粗粒度，如果是一个词，就不会再细分了**（程序员三个字会被分为：程序员）**，搜索程序，程序员并不会匹配显示，缺点：搜索概率会降低；优点是占用的内存少了
    
    1. `ik_max_word`：最细切分，细粒度**（程序员三个字会被分为：程序员、程序、员）**
    
1. 测试：
