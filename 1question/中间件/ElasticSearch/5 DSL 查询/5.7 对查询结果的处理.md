# 三、分页
## 1. 分页说明
1. elasticsearch **默认情况下只返回top10的数据**
2. 而如果要查询更多数据就**需要修改分页参数**了
3. elasticsearch中通过修改from、size参数来控制要返回的分页结果：

    ![[IMG-20260620224219731.png|453]]

4. **ES 分分页和 Spring 的分页不同，需要每次取出`from + size`之前的所有数据，再根据`size` 截取数据**

    ![[IMG-20260620224220797.png|497]]

## 2. 深度分页问题
1. ES是分布式的，所以会面临深度分页问题
2. 例如：按price排序后，获取`from = 990`，`size =10`的数据：
    1. 首先在每个数据分片上都排序并查询前1000条文档
    2. ==**将再所有节点的结果聚合，在内存中重新排序选出前1000条文档**==
    3. 最后从这1000条中，选取从990开始的10条文档

    ![[IMG-20260620224221490.png|716]]

3. 因此会产生深度分页问题：⭐**如果搜索页数过深，或者结果集（** **`from + size`** **）越大，对内存和 CPU 的消耗也越高**
    > 💡 **因此ES设定结果集查询的上限是10000**
    > ![[IMG-20260620224221549.png|282]]
    > ![[IMG-20260620224222180.png|710]]
4. **深度分页解决方案**
    1. 针对深度分页，ES提供了两种解决方案
        1. `search after`：分页时需要排序，原理是从上一次的排序值开始，查询下一页数据。==官方推荐使用的方式==
        2. `scroll`：原理将排序数据形成快照，保存在内存。官方已经不推荐使用
    2. [官方文档](https://www.elastic.co/guide/en/elasticsearch/reference/current/paginate-search-results.html)：
    > ℹ️ Paginate search results | Elasticsearch Guide [8.5] | Elastic
    > By default, searches return the top 10 matching hits.
    > [https://www.elastic.co/guide/en/elasticsearch/reference/current/paginate-search-results.html](https://www.elastic.co/guide/en/elasticsearch/reference/current/paginate-search-results.html)
