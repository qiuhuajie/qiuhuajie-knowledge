# 一、数据库操作
1. **查询所有数据库**
    ```SQL
     show databases  ;
    ```
2. **查询当前数据库**
    ```SQL
     select database() ;
    ```
3. **创建数据库：**
    1. 语法：
    ```SQL
     create database [ if not exists ] 数据库名 [ default charset 字符集 ] [ collate 排序规则 ] ;
    ```
    2. 通过if not exists 参数来解决这个问题，数据库不存在, 则创建该数据库，如果存在，则不创建
    ```SQL
     create database if not extists itcast;
    ```
    3. 创建一个itheima数据库，并且指定字符集
    ```SQL
     create database itheima default charset utf8mb4;
    ```
4. **删除数据库**
    ```SQL
     drop database [ if exists ] 数据库名 ;
    ```
5. **切换数据库**
    ```SQL
     use 数据库名 ;
    ```
6. Mysql数据库安装完成后，自带了以下四个数据库，具体作用如下

|---|---|

# 二、表操作
## 1. 查询创建
1. 查询当前数据库所有表
    ```SQL
     show tables;
    ```
2. 查看指定表结构
    ```SQL
     desc 表名 ;
    ```
3. 查询指定表的建表语句
    ```SQL
     show create table 表名 ;
    ```

    通过这条指令，主要是用来查看建表语句的，而有部分参数我们在创建表的时候，并未指定也会查询到，因为这部分是数据库的默认值，如：存储引擎、字符集等

4. 创建表结构
    ```SQL
     CREATE TABLE 表名(
     字段1 字段1类型 [ COMMENT 字段1注释 ],
     字段2 字段2类型 [COMMENT 字段2注释 ],
     字段3 字段3类型 [COMMENT 字段3注释 ],
     ......
     字段n 字段n类型 [COMMENT 字段n注释 ]
     ) [ COMMENT 表注释 ] ;
    ```

> 💡 **char 与 varchar**
> 1. 都可以描述字符串
> 2. char是定长字符串，指定长度多长，就占用多少个字符，和字段值的长度无关
> 3. 而varchar是变长字符串，指定的长度为最大占用长度
> 4. 相对来说，char的性能会更高些
> ```SQL
>  # 用户名 username ------> 长度不定, 最长不会超过50
>  username varchar(50)
>  # 性别 gender ---------> 存储值, 不是男,就是女
>  gender char(1)
>  # 手机号 phone --------> 固定长度为11
>  phone char(11)
> ```

## 3. 修改 ALTER
1. 添加字段
    1. 基本语法
    ```SQL
     ALTER TABLE 表名 ADD 字段名 类型 (长度) [ COMMENT 注释 ] [ 约束 ];
    ```
    2. 示例：
    ```SQL
     ALTER TABLE emp ADD nickname varchar(20) COMMENT '昵称';
    ```
2. 修改数据类型
    基本语法
```SQL
 ALTER TABLE 表名 MODIFY 字段名 新数据类型 (长度);
```
3. 修改字段名和字段类型
    1. 基本语法
    ```SQL
     ALTER TABLE 表名 CHANGE 旧字段名 新字段名 类型 (长度) [ COMMENT 注释 ] [ 约束 ];
    ```
    2. 示例：
    ```SQL
     ALTER TABLE emp CHANGE nickname username varchar(30) COMMENT '昵称';
    ```
4. 删除字段
    1. 基本语法
    ```SQL
     ALTER TABLE 表名 DROP 字段名;
    ```
    2. 示例：
    ```SQL
     ALTER TABLE emp DROP username;
    ```
5. 修改表名
    1. 基本语法
    ```SQL
     ALTER TABLE 表名 RENAME TO 新表名;
    ```
    2. 示例：
    ```SQL
     ALTER TABLE emp RENAME TO employee;
    ```
## 4. 删除 DROP
1. 删除表
    1. 基本语法
    ```SQL
     DROP TABLE [ IF EXISTS ] 表名;
    ```
    2. 示例
    ```SQL
     DROP TABLE IF EXISTS tb_user;
    ```
2. 删除指定表, 并重新创建表
    基本语法
```SQL
 TRUNCATE TABLE 表名;
```
## 5. 清空表数据 TRUNCATE

快速删除表中的数据【truncate比较重要，必须掌握】

1. delete语句删除数据
    ```SQL
     \#删除dept_bak表中的数据
     delete from dept_bak;   \#这种删除数据的方式比较慢
    ```

    delete语句删除数据的原理：delete属于DML语句

    1. 表中的数据被删除了，但是这个数据在硬盘上的真实存储空间不会被释放
    2. 这种删除缺点是：删除效率比较低
    3. 这种删除优点是：支持回滚，后悔了可以再恢复数据
2. 如何快速删除表中的数据
    ```SQL
     truncate table dept_bak; \#这种操作属于DDL操作
    ```
    1. 大表非常大，上亿条记录删除的时候，使用delete，也许需要执行1个小时才能删除完！效率较低
    2. **可以选择使用truncate删除表中的数据。只需要不到1秒钟的时间就删除结束。效率较高**
    3. 但是使用truncate之前，必须仔细询问客户是否真的要删除，并警告删除之后不可恢复
    4. truncate是删除表中的数据，表还在
    5. drop是把表删除
