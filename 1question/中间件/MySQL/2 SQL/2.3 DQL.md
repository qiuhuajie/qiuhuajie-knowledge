# **⭐执行顺序**
**将本节的关键字全部组合在一起，来看一下他们的执行顺序**
1. 语法顺序
    
    ```SQL
     select
         ...
     from
         ...
     where
         ...
     group by
         ...
     having
         ...
     order by
         ...
     limit
         ...
    ```
    
1. 执行顺序：
    
    - SELECT 由第一位降到了第五位
    
    - 前四个都是属于条件，后两个属于对结果集的筛选
    
    ```SQL
     from                 \#从某张表中查询数据
         ...
     where               \#先经过where条件筛选出有价值的数据
         ...
     group by         \#对这些有价值的数据进行分组
         ...
     having              \#分组之后可以使用having继续筛选
         ...
     select      ⭐   \#select查询出来
         ...
     order by         \#最后排序输出
         ...
     limit                \#分页显示
         ...
    ```
    
1. 综合案例：
    
    > 找出每个岗位的平均薪资，要求显示平均薪资大于1500的，除MANAGER岗位之外，要求按照平均薪资降序排
    
# 1. **基本语法**
1. DQL 查询语句，语法结构如下：
    
    ```SQL
     SELECT
         字段列表
     FROM
         表名列表
     WHERE
         条件列表
     GROUP BY
         分组字段列表
     HAVING
         分组后条件列表
     ORDER BY
         排序字段列表
     LIMIT
         分页参数
    ```
    
1. 将上面的完整语法进行拆分，分为以下几个部分：
    
    - 基本查询（不带任何条件）
    
    - 条件查询（WHERE）
    
    - 聚合函数（count、max、min、avg、sum）
    
    - 分组查询（group by）
    
    - 排序查询（order by）
    
    - 分页查询（limit）
    
# 2. **基础查询 SELECT**

> 在基本查询的DQL语句中，不带任何的查询条件，查询的语法如下：
1. 查询多个字段
    
    ```SQL
     SELECT 字段1, 字段2, 字段3 ... FROM 表名 ;
     
     SELECT * FROM 表名 ;
    ```
    
1. **==字段设置别名==**
    
    ```SQL
     SELECT 字段1 [ AS 别名1 ] , 字段2 [ AS 别名2 ] ... FROM 表名;
     
     SELECT 字段1 [ 别名1 ] , 字段2 [ 别名2 ] ... FROM 表名;
    ```
    
    1. 使用`as`关键字起别名，只是将显示的查询结果列名显示为deptname，原表列名还是叫：dname
    
    1. select语句是永远都不会进行修改操作的（只负责查询）
    
    1. `as`关键字可以省略
        
        ```SQL
         mysql> select deptno,dname deptname from dept;
        ```
        
    
    1. 如果别名里面有空格
        
        ```SQL
         select deptno,dname 'dept name' from dept; //加单引号
         
         select deptno,dname "dept name" from dept; //加双引号
        ```
        
        1. 注意：在所有的数据库当中，字符串统一使用单引号括起来，单引号是标准
        
        1. 双引号在oracle数据库中用不了，但是在mysql中可以使用
        
    
1. 字段可以使用数学表达式
    
    ```SQL
     mysql> select ename,sal*12 from emp; // 结论：字段可以使用数学表达式！
     
     mysql> select ename,sal*12 as yearsal from emp; //起别名
     
     mysql> select ename,sal*12 as '年薪' from emp; //别名是中文，用单引号括起来
    ```
    
1. ==**去除重复记录**==
    
    ```SQL
     SELECT DISTINCT 字段列表 FROM 表名;
    ```
    
    1. 注意：原表数据不会被修改，只是查询结果去重
    
    1. distinct 出现在 job,deptno 两个字段之前，表示两个字段联合起来去重
        
        ```SQL
         select distinct job,deptno from emp;
        ```
        
    
    1. 例如：统计一下工作岗位的数量
        
        ```SQL
         select count(distinct job) from emp;
        ```
        
    
# 3. **条件查询**
条件查询：不是将表中所有数据都查出来，是查询出来符合条件的
```SQL
 \#语法格式：
     select
         字段1,字段2,字段3....
```

### `**=**` **等于**
1. 查询薪资等于800的员工姓名和编号
    
    ```SQL
     select empno,ename from emp where sal = 800;
    ```
    
1. 查询SMITH的编号和薪资？
    
    ```SQL
     select empno,sal from emp where ename = 'SMITH'; \#字符串使用单引号
    ```
    
### `**<**` `**>**`**或**`**!=**` **不等于**
查询薪资不等于800的员工姓名和编号
```SQL
 select empno,ename from emp where sal != 800;select empno,ename from emp where sal <> 800; \#小于号和大于号组成的不等号
```
### `**<**` **小于**
查询薪资小于2000的员工姓名和编号
```SQL
 mysql> select empno,ename,sal from emp where sal < 2000;
```
### `**<=**` **小于等于**
查询薪资小于等于3000的员工姓名和编号
```SQL
 select empno,ename,sal from emp where sal <= 3000;
```
### `**>**` **大于**
查询薪资大于3000的员工姓名和编号
```SQL
 select empno,ename,sal from emp where sal > 3000;
```
### `**> =**` **大于等于**
查询薪资大于等于3000的员工姓名和编号
```SQL
 select empno,ename,sal from emp where sal >= 3000;
```
### `**between … and ….**` **两个值之间**
等同于 >= and <=  
查询薪资在2450和3000之间的员工信息？包括2450和3000
1. 第一种方式：>= and <= （and是并且的意思。）
    
    ```SQL
     select empno,ename,sal from emp where sal >= 2450 and sal <= 3000;
    ```
    
1. 第二种方式：between … and …
    
    ```SQL
     select  empno,ename,sal from    emp where   sal between 2450 and 3000;
    ```
    
    注意：
    
    1. 使用between and的时候，必须遵循左小右大
    
    1. between and是闭区间，包括两端的值
    
### `**is null**` **为空**
（is not null 不为空）
查询哪些员工的津贴/补助为null
```SQL
 mysql> select empno,ename,sal,comm from emp where comm = null;Empty set (0.00 sec)  \#查不到数据
```
```SQL
 mysql> select empno,ename,sal,comm from emp where comm is null;
```
注意：
1. 在数据库当中null不能使用等号进行衡量
1. 需要使用is null，因为数据库中的null代表什么也没有，它不是一个值，所以不能使用等号衡量
### `**and**`**并且**
查询工作岗位是MANAGER并且工资大于2500的员工信息
```SQL
 select  empno,ename,job,sal from    emp where   job = 'MANAGER' and sal > 2500;
```
### `**or**` **或者**
查询工作岗位是MANAGER和SALESMAN的员工
```SQL
 select  empno,ename,jobfrom empwhere    job = 'MANAGER' or job = 'SALESMAN';
```
and和or的优先级  
查询工资大于2500，并且部门编号为10或20部门的员工
```SQL
 select  *from   empwhere    sal > 2500 and deptno = 10 or deptno = 20;
```
1. and优先级比or高，以上语句会先执行and，然后执行or
1. 以上语句表示的含义：找出工资大于2500并且部门编号为10的员工，或者20部门所有员工找出来
1. 结论：and和or同时出现，and优先级较高。如果想让or先执行，需要加“小括号”
    
    ```SQL
     select
         *
     from
         emp
     where
         sal > 2500 and (deptno = 10 or deptno = 20);
    ```
    
### `**in**` **包含**
相当于多个 or （not in 不在这个范围中）
**==查询工作岗位是 MANAGER 和 SALESMAN 的员工==**
```SQL
 select empno,ename,job from emp where job = 'MANAGER' or job = 'SALESMAN';
 select empno,ename,job from emp where job in('MANAGER', 'SALESMAN');
 \#in不是一个区间，是一个集合，in后面跟的是具体的值
 
 \#not in 表示不在这几个值当中的数据
 select ename,sal from emp where sal not in(800, 5000, 3000);
```
### `**not**` **取非**
主要用在 is 或 in 中
1. is null
1. is not null
1. in
1. not in
### `**like**` **模糊查询**
支持%或下划线匹配
1. `%`：**==匹配任意多个字符==**
1. `_`：**==匹配任意一个字符==**
```SQL
 \#找出名字中含有O的
 select ename from emp where ename like '%O%';
 
 \#找出名字以T结尾的
 select ename from emp where ename like '%T';
 \#找出名字以K开始的
 select ename from emp where ename like 'K%';
 
 \#找出第二个字每是A的
 select ename from emp where ename like '_A%';
 
 \#找出第三个字母是R的
 select ename from emp where ename like '__R%';
 \#如果表中有一个名字：jack_son，找出名字中有“_”的
 select name from t_student where name like '%_%'; \#这样不行
 
 select name from t_student where name like '%\_%'; # 使用 \ 转义字符
 \#找出名字中不含有O的
 select ename from emp where ename not like '%O%';
 # 以先后顺序同时出现 toy 和 carrots 的产品
select prod_name, prod_desc
from Products
where prod_desc like '%toy%carrots%';
```
# 4. **聚合函数**
### `**count**` **计数**
```Plain
 \#计算员工数量
 select count(ename) from emp;
```

> [!important] ⭐
> 
> ==**分组函数中**== `**count(*)**` ==**和**== `**count(具体字段)**` ==**有什么区别**==❓
> 
> 1. `**count(具体字段)**`：表示统计**该字段下所有不为NULL的元素的总数**
>     
>     ```SQL
>      mysql> select count(comm) from emp;
>              +-------------+
>              | count(comm) |
>              +-------------+
>              |           4 |
>              +-------------+
>     ```
>     
> 
> 1. `**count(*)**`：统计表当中的**总行数**，==每一行记录不可能所有字段都为NULL==，故只要有一行数据则 `**count++**`
>     
>     ```SQL
>      mysql> select count(*) from emp;
>              +----------+
>              | count(*) |
>              +----------+
>              |       14 |
>              +----------+
>     ```
>     
### `**sum**` **求和**
```SQL
 \#计算工资和
 select sum(sal) from emp;
```
### `**avg**` **平均值**
```SQL
 \#计算平均工资
 select avg(sal) from emp;
```
### `**max**` **最大值**
```SQL
 \#找出最高工资
 select max(sal) from emp;
```
### `**min**` **最小值**
```SQL
 \#找出最低工资
 select min(sal) from emp;
```
### **⭐ 聚合函数使用注意点**
1. 分组函数自动忽略NULL，你不需要提前对NULL进行处理
1. **⭐聚合函数在使用的时候必须先进行分组（group by）**
    
    1. 换句话说 聚合函数不能出现在 where 条件中，因为有以下执行顺序：from、where、group by
    
    1. 如果你没有对数据进行分组，整张表默认为一组
    
    1. 找出比最低工资高的员工信息
        
        ```SQL
         # 报错：
        select ename,sal from emp where sal > min(sal);
         ERROR 1111 (HY000): Invalid use of group function
         
         # 正确写法：
         select ename,sal from emp where sal > (select min(sal) from emp) # 可以看到 分组函数是可以出现在 子查询语句中的
        ```
        
    
    1. 下面语句能正常执行，是因为 select 的执行顺序是在 group by 之后的，而 where 的执行顺序是在 group by 之前的
        
        ```SQL
         select count(ename) from emp;
        ```
        
    
1. 所有的聚合函数可以组合起来一起用
    
    ```SQL
     select sum(sal),min(sal),max(sal),avg(sal),count(*) from emp;
             +----------+----------+----------+-------------+----------+
             | sum(sal) | min(sal) | max(sal) | avg(sal)    | count(*) |
             +----------+----------+----------+-------------+----------+
             | 29025.00 |   800.00 |  5000.00 | 2073.214286 |       14 |
             +----------+----------+----------+-------------+----------+
    ```
    
# 5. **分组查询**
### **group by**
1. 在实际的应用中，可能有这样的需求，需要**先进行分组，然后对每一组的数据进行操作**。这个时候我们需要使用分组查询
1. 由于 `order by` 最后执行，所以**==可以使用列别名==**
1. **分组查询语法**
    
    ```SQL
     select
         ...
     from
         ...
     group by
         ...
    ```
    
1. 使用情况举例：
    
    1. 计算每个部门的工资和
    
    1. 计算每个工作岗位的平均薪资
    
    1. 找出每个工作岗位的最高薪资
    
    1. ......
    
1. **分组查询案例：**
    
    1. **案例一：找出每个工作岗位的工资和**
        
        ```SQL
         \#实现思路：按照工作岗位分组，然后对工资求和。
         select
             job,sum(sal)
         from
             emp
         group by
             job;
         \#以上这个语句的执行顺序：
                     \#先从emp表中查询数据
                     \#根据job字段进行分组
                     \#⭐然后对每一组的数据进行sum(sal) ----> 分组函数在group by之后执行
        ```
        
        > PS：
        > 
        > ```SQL
        >  select ename,job,sum(sal) from emp group by job;
        > ```
        > 
        > 1. 以上语句在mysql中可以执行，但是毫无意义
        > 
        > 1. 以上语句在oracle中执行报错，oracle的语法比mysql的语法严格
        > 
        > 1. 结论：**在一条select语句当中，如果有group by语句的话，select后面只能跟：参加分组的字段 或者 分组函数，其它的一律不能跟**
        
    
    1. **案例二：找出每个部门的最高薪资**
        
        ```SQL
         \#实现思路：按照部门编号分组，求每一组的最大值
         select deptno,max(sal) from emp group by deptno;
        ```
        
    
    1. **案例三：找出“每个部门，不同工作岗位”的最高薪资**
        
        ```SQL
         \#实现思路：两个字段联合成1个字段看（两个字段联合分组）
         select
             deptno, job, max(sal)
         from
             emp
         group by
             deptno, job;
        ```
        
    
    1. **案例四：统计每个部门的人员数**
        
        ```SQL
         select deptno,count(name) from emp group by deptno;
        ```
        
    
### **having**
1. 使用`**having**`可以对**分完组之后的数据进一步过滤，因为聚合函数必须在 group by 后，因此不能写在 where 条件中，所以需要一个可以在group by之后添加过滤条件的关键字**
1. having不能单独使用，**having 不能代替 where**，==**having 必须和 group by 联合使用**==**，**==**用于对聚合函数处理后的数据，进一步过滤**==
1. where和having，优先选择where，where实在完成不了了，再选择having
1. **having 使用案例（**==**看到有对统计结果再做条件的，要想到用 having**==**）**
    
    1. **案例一：找出每个部门平均薪资，要求显示**==**平均薪资高于**==**2500的**
        
        1. 尝试使用where
            
            ```SQL
             select
                 deptno,avg(sal)
             from
                 emp
             where
                 avg(sal) > 2500;    \#⭐分组函数不能再where后使用！
             group by
                 deptno
            ```
            
        
        1. 使用 having 解决
            
            ```SQL
             \#第一步：找出每个部门平均薪资
             select deptno,avg(sal) from emp group by deptno;
             
             \#第二步：要求显示平均薪资高于2500的
             select
                 deptno,avg(sal)
             from
                 emp
             group by
                 deptno
             having
                 avg(sal) > 2500; ⭐
            ```
            
        
    
    1. **案例二：找出每个部门最高薪资，要求显示**==**最高薪资大于**==**3000的**
        
        ```SQL
         # 第一步：找出每个部门最高薪资，按照部门编号分组，求每一组最大值
         select deptno,max(sal) from emp group by deptno;
             +--------+----------+
             | deptno | max(sal) |
             +--------+----------+
             |     10 |  5000.00 |
             |     20 |  3000.00 |
             |     30 |  2850.00 |
             +--------+----------+
        
         # 第二步：要求显示最高薪资大于3000
         select
             deptno,max(sal)
         from
             emp
         group by
             deptno
         having
             max(sal) > 3000;
         +--------+----------+
         | deptno | max(sal) |
         +--------+----------+
         |     10 |  5000.00 |
         +--------+----------+
         
         # 以上的sql语句执行效率是不是低？答：比较低，实际上可以这样考虑：先将大于3000的都找出来，然后再分组
         select
             deptno,max(sal)
         from
             emp
         where
             sal > 3000
         group by
             deptno;
        ```
        
    
# 6. **排序查询**
1. 查询所有员工薪资且排序
    
    ```SQL
     select
         ename,sal
     from
         emp
     order by
         sal; \#默认是升序
    ```
    
1. **指定 升序 和 降序**
    
    ```SQL
     select
         ename,sal
     from
         emp
     order by
         sal desc;   \#降序
    ```
    
    ```SQL
     select
         ename,sal
     from
         emp
     order by
         sal asc;    \#升序
    ```
    
1. ==**使用两个或多个字段排序**==
    
    - 查询员工名字和薪资，要求按照薪资升序
    
    - 如果薪资一样的话，再按照名字升序排列
    
    ```SQL
     select
         ename,sal
     from
         emp
     order by
         sal asc, ename asc;
     # sal在前，起主导，只有sal相等的时候，才会考虑启用ename排序
    ```
    
1. 综合案例：  
    找出工资在1250到3000之间的员工信息，要求按照薪资降序排列
    
    ```SQL
     select
         ename,sal
     from
         emp
     where
         sal between 1250 and 3000
     order by
         sal desc;
    ```
    
# 7. **分页查询**

> 分页操作在业务系统开发时，也是非常常见的一个功能，我们在网站中看到的各种各样的分页条，后台都需要借助于数据库的分页操作
1. 语法
    
    ```SQL
     SELECT 字段列表 FROM 表名 LIMIT 起始索引, 查询记录数 ;
    ```
    
    - 起始索引从0开始，起始索引 = （查询页码 - 1）* 每页显示记录数
    
    - 分页查询是数据库的方言，不同的数据库有不同的实现，MySQL中是LIMIT
    
    - 如果查询的是第一页数据，起始索引可以省略，直接简写为 limit 10
    
1. 示例：查询第2页员工数据, 每页展示10条记录 --------> (页码-1)*页展示记录数
    
    ```SQL
     select * from emp limit 10,10;
    ```
    
1. 在Java中写分页 sql 语句
    
    ```SQL
     /*
     每页显示3条记录
         第1页：limit 0,3       [0 1 2]
         第2页：limit 3,3       [3 4 5]
         第3页：limit 6,3       [6 7 8]
         第4页：limit 9,3       [9 10 11]
     
     每页显示pageSize条记录
         第pageNo页：limit (pageNo - 1) * pageSize  , pageSize
     
     记公式：
         limit (pageNo-1)*pageSize , pageSize
     */
     public static void main(String[] args){
         // 用户提交过来一个页码，以及每页显示的记录条数
         int pageNo = 5; //第5页
         int pageSize = 10; //每页显示10条
     
       //⭐
         int startIndex = (pageNo - 1) * pageSize;
         String sql = "select ...limit " + startIndex + ", " + pageSize;
     }
    ```
