# 一、MySQL 基本概念
## 1. 数据库
1. 英文单词DataBase，简称DB
2. 按照一定格式存储数据的一些文件的组合。顾名思义：存储数据的仓库，实际上就是一堆文件。这些文件中存储了具有特定格式的数据
## 2. 数据库管理系统
1. DataBaseManagement，简称DBMS
2. 数据库管理系统是专门用来管理数据库中数据的，数据库管理系统可以对数据库当中的数据进行增删改查
3. 常见的数据库管理系统：MySQL、Oracle、MS SqlServer、DB2、sybase等....
## 3. SQL 语句
1. Structured Query Language 结构化查询语言
2. 程序员需要学习SQL语句，程序员通过编写SQL语句，然后DBMS负责执行SQL语句，最终来完成数据库中数据的增删改查操作
3. SQL是一套标准，程序员主要学习的就是SQL语句，这个SQL在mysql中可以使用，同时在Oracle中也可以使用，在DB2中也可以使用
## 4. 三者之间的关系
1. DBMS--执行--> SQL --操作--> DB
    ![[IMG-20260620224137542.png|800]]
2. 先安装数据库管理系统MySQL，然后学习SQL语句怎么写，编写SQL语句之后，DBMS将SQL语句进行执行，最终来完成数据库的数据管理
# 二、安装 MySQL
## 1. 安装MySQL DBMS
1. 先安装，选择“经典版”
2. 需要进行MySQL数据库实例配置，一路下一步就行了
3. 需要注意的事项
    1. 端口号：
        1. 端口号port是任何一个软件/应用都会有的，端口号是应用的唯一代表
        2. 端口号通常和IP地址在一块，IP地址用来定位计算机的，端口号port是用来定位计算机上某个服务的/某个应用的
        3. 在同一台计算机上，端口号不能重复。具有唯一性
        4. mysql数据库启动的时候，这个服务占有的默认端口号是3306
    2. 字符编码方式：
        1. 设置mysql数据库的字符编码方式为 UTF8
        2. 一定要注意：先选中第3个单选按钮，然后再选择utf8字符集
    3. 服务名称默认是：MySQL
    4. 选择配置环境变量path：如果没有选择可以手动配置：path=其它环境变量;mysql中bin目录绝对路径
    5. mysql超级管理员用户名不能改，一定是：root，同时设置密码
    6. 设置密码的同时，可以激活root账户远程访问
        1. 激活：表示root账号可以在外地登录
        2. 不激活：表示root账号只能在本机上使用
## 2. 查看 MySQL 服务
1. 计算机-->右键-->管理-->服务和应用程序-->服务-->找mysql服务
2. MySQL的服务，默认是“启动”的状态，只有启动了mysql才能用
3. 默认情况下是“自动”启动，自动启动表示下一次重启操作系统的时候自动启动该服务
4. 可以在服务上点击右键：（启动、重启服务、停止服务...）
5. 可以改变服务的默认配置：服务上点击右键，属性，然后可以选择启动方式：（自动（延迟启动）、自动、手动、禁用）
6. 在windows操作系统当中使用命令来启动和关闭mysql服务，语法：
    1. net stop 服务名称;
    2. net start 服务名称;
    其它服务的启停都可以采用以上的命令
## 3. 连接数据库
1. 服务启动后，使用bin目录下的mysql.exe命令来连接mysql数据库服务器

# 三、数据模型
## 1. 关系型数据库（RDBMS）
1. 概念：建立在关系模型基础上，由多张相互连接的二维表组成的数据库
2. 而所谓二维表，指的是由行和列组成的表，如下图
    > 类似于Excel表格数据，有表头、有列、有行，还可以通过一列关联另外一个表格中的某一列数据
    ![[IMG-20260620224140267.png|800]]
3. MySQL、Oracle、DB2、SQLServer这些都是属于关系型数据库，里面都是基于二维表存储数据的
4. 简单说，**基于二维表存储数据的数据库就成为关系型数据库**，不是基于二维表存储数据的数据库，就是非关系型数据库
5. 特点：
    1. 使用表存储数据，格式统一，便于维护
    2. 使用SQL语言操作，标准统一，使用方便
