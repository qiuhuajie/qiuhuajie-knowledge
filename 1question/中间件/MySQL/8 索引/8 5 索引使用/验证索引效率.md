1. 先通过一个简单的案例，来验证一下索引，看看是否能够通过索引来提升数据查询性能
    
    1. 在演示的时候，表 tb_sku , 在这张表中准备了1000w 的记录
    
    1. 表中id为主键，有主键索引，而其他字段是没有建立索引
    
    ![[Attachment/1question/中间件/MySQL/8 索引/8 5 索引使用/IMG-20260405035438349.png|Untitled 444.png]]
    
1. 先根据 id 查询：
    
    ```SQL
     select * from tb_sku where id = 1\G;
    ```
    
    即使有1000w的数据，根据 id 进行数据查询，性能依然很快，因为主键id是有索引的
    
    ![[IMG-20260405035453513.png|Untitled 1 329.png]]
    
1. 再来根据 sn 字段进行查询，执行如下SQL
    
    ```SQL
     SELECT * FROM tb_sku WHERE sn = '100000003145001';
    ```
    
    可以看到根据sn字段进行查询，查询返回了一条数据，结果耗时 20.78sec，就是因为sn没有索引，而造成查询效率很低
    
    ![[IMG-20260405035453534.png|Untitled 2 268.png]]
    
1. 那么可以针对于 `sn` 字段，建立一个索引，建立了索引之后，再次根据 `sn` 进行查询，再来看一下查询耗时情况
    
    1. 创建索引
        
        ```SQL
         create index idx_sku_sn on tb_sku(sn) ;
        ```
        
    
    1. 再次查询
        
        ```SQL
         SELECT * FROM tb_sku WHERE sn = '100000003145001';
        ```
        
        `**sn**` 字段建立了索引之后，查询性能大大提升。建立索引前后，查询耗时都不是一个数量级的
        
        ![[IMG-20260405035508234.png|Untitled 3 200.png]]
