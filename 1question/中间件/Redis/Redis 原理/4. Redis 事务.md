# 1. Redis 事务介绍
1. Redis 事务
    
    1. **提供了一种**==**将多个命令请求打包**==**，然后**==**一次性、按顺序地执行多个命令的机制**==
    
    1. 并且在事务执行期间，服务器**不会中断事务**而改去执行其他客户端的命令请求，它==会将事务中的所有命令都执行完毕，然后才去处理其他客户端的命令请求==
    
1. Redis 支持事务机制
1. 在 Redis 中，事务是通过 `MULTI`、`EXEC`、`DISCARD` 和 `WATCH` 四个命令实现的
# 2. 执行流程
![[IMG-20260405035438334.png|Untitled 225.png]]
1. **==使用== `MULTI` ==命令开始一个事务，将客户端从非事务状态切换至事务状态==**
1. **==命令入队（FIFO 队列）==**
    
    1. 客户端处于**非事务状态**时，这个客户端发送的命令会立即被服务器执行
    
    1. 客户端处于**事务状态**时，普通命令会被放入事务队列，而事务相关的四条命令会被立即送去服务器执行
    
1. ==**当所有的命令都添加到队列后，使用**== **`EXEC`** ==**命令将队列中的所有命令打包发给 Redis 执行**==
    
    1. Redis 会按照命令的添加顺序遍历队列，依次执行每个命令，最后将执行命令所得的结果全部返回给客户端
        
        ```Java
        MULTI   A命令   B命令   C命令   EXEC
        ```
        
    
    1. 如果 `EXEC` 命令执行成功，那么整个事务被提交并且所有命令被执行
    
    1. 如果 `EXEC` 命令执行时，队列中的一个命令出现错误
        
        1. 命令入队时报错：例如输入了一个错误的命令，则服务器会放弃事务执行，保证原子性
        
        1. 命令入队时正常，但执行时出现某个中间命令错误，**仍会继续执行完后面的命令，且之前执行过的命令是不会回滚的**，所以这种情况下是不保证原子性的
        
    
1. **`DISCARD`** 命令可以在事务执行之前使用，用于取消当前事务
1. **`WATCH`** ==**命令是一个乐观锁（CAS），它可以在**== **`EXEC`** ==**命令执行之前，监视任意数量的数据库**== **`key`**
    
    1. 如果在 `EXEC` 命令执行时，如果发现被监视的 `key` 中只要有一个被修改过了，那么务器将拒绝执行事务，并向客户端返回代表事务执行失败的空回复
        
        ![[IMG-20260405035447370.png|Untitled 1 159.png]]
        
    
    1. `WATCH` 的实现原理
        
        1. redis 的 `watched keys` 字典：
            
            1. 字典的==键是某个被== ==`WATCH`== ==命令监视的== ==`key`==
            
            1. 而字典的==值则是一个链表，链表中记录了所有监视相应== ==`key`== ==的客户端（由== ==`WATCH`== ==添加）==
                
                ![[IMG-20260405035452414.png|Untitled 2 134.png]]
                
            
        
        1. ==当== `key` ==被修改时==，会将这个 `key` 后面的**所有**客户端的 **`REDIS_DIRTY_CAS`** 标识打开（通知）
        
        1. 当服务器接收到一个客户端发来的 `EXEC` 命令要操作某个 `key` 时，如果这个 `key` 的链中当前客户端的 **`REDIS_DIRTY_CAS`** 标识已经打开（当前事务不安全）
        
        1. 因此服务器不会执行该客户端提交的事务
        
    
# 2. Redis 事务的 ACID
- Redis事务并不能完全保证事务四大特性，使用的时候需要注意：
    
    - 原子性：（❌）Redis事务具有一定的原子性，但是==不支持回滚==（详见上面 `EXEC` 命令队列中间某个命令执行错误，会继续执行完所有命令）
    
    - 一致性：（✅）Redis事务保证一致性
    
    - 隔离性：（✅）Redis （==单线程==）的事务总是以串行的方式运行的，因此==天然具有隔离性==；同时使用 `WATCH` 保证隔离性
    
    - 持久性：（🚫）Redis事务==不保证持久性==。Redis持久化机制（RDB、AOF）都是==异步刷盘，存在数据丢失的情况==
    
# 3. Lua 脚本（另一种形式的 Redis 事务）
## 2.1 介绍
1. Lua是一种编程语言
    
    > [!info] Lua 教程  
    > Lua 是一种轻量小巧的脚本语言，用标准C语言编写并以源代码形式开放， 其设计目的是为了嵌入应用程序中，从而为应用程序提供灵活的扩展和定制功能。 Lua 是巴西里约热内卢天主教大学（Pontifical Catholic University of Rio de Janeiro）里的一个研究小组于 1993 年开发的，该小组成员有：Roberto Ierusalimschy、Waldemar Celes 和 Luiz Henrique de Figueiredo。 其设计目的是为了嵌入应用程序中，从而为应用程序提供灵活的扩展和定制功能。 轻量级: 它用标准C语言编写并以源代码形式开放，编译后仅仅一百余K，可以很方便的嵌入别的程序里。 可扩展: Lua提供了非常易于使用的扩展接口和机制：由宿主语言(通常是C或C++)提供这些功能，Lua可以使用它们，就像是本来就内置的功能一样。 其它特性: 支持面向过程(procedure-oriented)编程和函数式编程(functional programming)； 自动内存管理；只提供了一种通用类型的表（table），用它可以实现数组，哈希表，集合，对象； 语言内置模式匹配；闭包(closure)；函数也可以看做一个值；提供多线程（协同进程，并非操作系统所支持的线程）支持； 通过闭包和table可以很方便地支持面向对象编程所需要的一些关键机制，比如数据抽象，虚函数，继承和重载等。 游戏开发 独立应用脚本 Web 应用脚本 扩展和数据库插件如：MySQL Proxy 和 MySQL WorkBench 安全系统，如入侵检测系统 接下来我们使用 Lua 来输出"Hello World!  
    > [https://www.runoob.com/lua/lua-tutorial.html](https://www.runoob.com/lua/lua-tutorial.html)  
    
1. Redis 内置的 Lua 解释器，可以实现在 Redis 中运行 Lua 脚本
1. redis 使用 Lua 的优点
    
    - **减少网络开销**：将多个命令请求通过脚本的形式一次发送
    
    - **保证原子性**：Lua 脚本被当做一命令集合一起被执行
    
    - **可重复使用**：客户端发送的脚本会永久存在 redis 中，这样其他客户端可以复用这一脚本
    
    - **保证隔离性**：redis 是单线处理机制，因此不需要 `WATCH` 保证隔离性，天然具备隔离性
    
1. 缺点
    
    - **不支持回滚**：与 `MULTI`，`EXEC` 过程相似，如果在执行 redis 命令过程出现异常，之前正常执行的命令也不支持回滚
    
## 2.2 应用
[[原子性优化]]
- **Lua 脚本模式的身影几乎无处不在，比如分布式锁、延迟队列、抢红包等场景**
- 不过在编写 Lua 脚本时，要注意如下两点：
    
    1. 为了避免 Redis 阻塞，**Lua 脚本业务逻辑不能过于复杂和耗时**
    
    1. **仔细检查和测试 Lua 脚本** ，因为执行 Lua 脚本具备一定的原子性，不支持回滚
    
## 2.3 Lua 脚本的编写
- 这里重点介绍 **Redis 提供的调用函数 `redis.call()`**，语法如下
    
    - 执行redis命令
        
        ```Bash
         redis.call('命令名称','key','其他参数',...)
        ```
        
    
    - 执行set name jack 命令
        
        ```Bash
         redis.call('set','name','jack')
        ```
        
    
    - 执行set name Rose，再执行get name 命令
        
        ```Bash
         redis.call('set','name','jack')
         local name = redis.call('get','name')
         return name
        ```
