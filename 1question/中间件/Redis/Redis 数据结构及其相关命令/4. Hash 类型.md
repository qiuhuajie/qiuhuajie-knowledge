# 一、介绍
1. Hash 类型，也叫散列，其 `value` 是一个**无序字典（键值对形式）**，==类似于 Java 中的== ==**HashMap**== ==结构==[[Map 接口]]
2. **String 结构存在的问题：**
    1. String 结构是将对象序列化为 JSON 字符串后存储
    2. 当需要**修改对象某个字段**时很不方便

    ![[%E5%9B%BE%E5%83%8F_13 2.png|800]]
3. Hash 结构==可以将对象中的====**每个字段独立存储**====，可以====**针对单个字段做CRUD**====：==

    ![[IMG-20260620224127703.png|800]]
4. ⭐==**String 还是 Hash 存储对象数据更好呢**==❓
    - String
        - String 存储的是**==序列化后==**==的对象数据==，存放的是整个对象
        - String 存储相对来说更加**节省内存**，缓存相同数量的对象数据，String 消耗的内存约是 Hash 的一半。并且，存储具有**多层嵌套的对象**时也方便很多
        - **如果系统对性能和资源消耗非常敏感的话，String 就非常适合**
    - Hash
        - Hash 是对对象的每个字段单独存储，可以获取部分字段的信息，也可以修改或者添加部分字段，节省网络流量
        - 如果对象中某些字段需要经常变动或者经常需要单独查询对象中的个别字段信息，Hash 就非常适合
    - **在绝大部分情况，建议使用 String 来存储对象数据即可！**

# 二、Hash的常见命令
- `HSET` key field value：添加或者修改hash类型key的field的值

    ```Shell
     127.0.0.1:6379> HSET coolcool:user:3 name qqq
     (integer) 1
     127.0.0.1:6379> HSET coolcool:user:3 age 20
     (integer) 1
     127.0.0.1:6379> HSET coolcool:user:3 age 17
     (integer) 0
     127.0.0.1:6379> HGET coolcool:user:3 age
     "17"
    ```
- `HGET` key field：获取一个hash类型key的field的值

    ```Shell
     127.0.0.1:6379> HGET coolcool:user:3 age
     "17"
    ```
- `HMSET`：批量添加多个hash类型key的field的值

    ```Shell
     127.0.0.1:6379> HMSET coolcool:user:4 name lilei age 20 sex man
     OK
    ```
    - 在图形化客户端查看

    ![[%E5%9B%BE%E5%83%8F_15 2.png|800]]
- `HMGET`：批量获取多个hash类型key的field的值
- `HGETALL`：获取一个hash类型的key中的所有的field和value

    ```Shell
     127.0.0.1:6379> HGETALL coolcool:user:4
     1) "name"
     2) "lilei"
     3) "age"
     4) "20"
     5) "sex"
     6) "man"
    ```
- `HKEYS`：获取一个hash类型的key中的所有的field

    ```Shell
     127.0.0.1:6379> HKEYS coolcool:user:4
     1) "name"
     2) "age"
     3) "sex"
    ```
- `HVALS`：获取一个hash类型的key中的所有的value

    ```Shell
     127.0.0.1:6379> HVALS coolcool:user:4
     1) "lilei"
     2) "20"
     3) "man"
    ```
- `HINCRBY`：让一个hash类型key的字段值自增并指定步长

    ```Shell
     127.0.0.1:6379> HINCRBY coolcool:user:4 age 6
     (integer) 26
    ```
- `HSETNX`：添加一个hash类型的key的field值，前提是这个field不存在，否则不执行

    ```Shell
     127.0.0.1:6379> HSETNX coolcool:user:4 sex woman
     (integer) 0
     127.0.0.1:6379> HSETNX coolcool:user:3 sex woman
     (integer) 1
    ```
