# 一、介绍
1. Redis 中的 List 类型**与 Java 中的** **`LinkedList`** **类似，可以看做是一个==双向链表结构==**。既可以支持正向检索和也可以支持反向检索
2. 特征也与 LinkedList 类似：
    - **==有序==**
    - **==元素可以重复==**
    - 插入和删除快
    - 查询速度一般
3. 应用场景
    1. **用来存储一个有序数据**
    2. 也可以**用来做消息队列**[[Redis 消息队列]][[Redis 消息队列]][[Redis 消息队列]]

# 二、List的常见命令
- `LPUSH` key element ... ：向列表左侧插入一个或多个元素（L队首，R队尾）

    ```Shell
     127.0.0.1:6379> LPUSH users 1 2 3
     (integer) 3
    ```
- `LPOP` key：移除并返回列表左侧的第一个元素，没有则返回nil

    ```Shell
     127.0.0.1:6379> LPOP users
     "3"
    ```
- `RPUSH` key element ... ：向列表右侧插入一个或多个元素

    ```Shell
     127.0.0.1:6379> RPUSH users 4 5 6
     (integer) 6
    ```
- `RPOP` key：移除并返回列表右侧的第一个元素

    ```Shell
     127.0.0.1:6379> RPOP users
     "6"
    ```

    ![[IMG-20260620224127702.png|800]]

- `LRANGE` key star end：返回一段角标范围内的所有元素

    ```Shell
     127.0.0.1:6379> LRANGE users 2 4
     1) "4"
     2) "5"
    ```

    ![[%E5%9B%BE%E5%83%8F_16 2.png|800]]

- `BLPOP` 和 `BRPOP`：
    - **带有阻塞效果的弹出方式**
    - 与LPOP和RPOP类似，只不过在没有元素时等待指定时间，而不是直接返回nil

    ```Shell
     127.0.0.1:6379> help BLPOP
       BLPOP key [key ...] timeout
       summary: Remove and get the first element in a list, or block until one is available
       since: 2.0.0
       group: list
     127.0.0.1:6379> BLPOP user2 5
     (nil)
     (5.04s)
    ```
