# 一、介绍
1. String类型，也就是字符串类型，是 Redis 中最简单的存储类型
2. 其 value 是字符串，不过**根据字符串的格式不同，又可以分为 3 类：**
    1. **`string`**：普通字符串
    2. **`int`**：整数类型，可以做自增、自减操作
    3. **`float`**：浮点类型，可以做自增、自减操作

    ![[%E5%9B%BE%E5%83%8F_10 2.png|800]]

3. 不管是哪种格式，==底层都是====**字节数组**====形式存储==，只不过是**编码方式不同**
4. 字符串类型的最大空间不能超过512m
5. 如果 Value 是一个 **Java 对象**，则可以**将对象**==**序列化为 JSON 字符串**==后存储：

    ![[%E5%9B%BE%E5%83%8F_12 2.png|800]]

    - 示例：

    ```Shell
     127.0.0.1:6379> SET coolcool:user:1 '{"id":1, "name":"Jack", "age": 21}'
     OK
     127.0.0.1:6379> SET coolcool:product:1 '{"id":1, "name":"小米11", "price": 4999}'
     OK
    ```
    - 但是，当需要**修改对象某个字段**时很不方便（引出 Hash 结构）

# 二、String 的常见命令
- `SET`：添加或者修改已经存在的一个String类型的键值对

    ```Shell
     127.0.0.1:6379> SET age 24
     OK
    ```
- `GET`：根据key获取String类型的value

    ```Shell
     127.0.0.1:6379> GET name
     "qhjqhj"
    ```
- `MSET`：批量添加多个String类型的键值对
- `MGET`：根据多个key获取多个String类型的value
- `INCR`：
    - 让一个整型的key自增1
    - **==可用来实现 全局唯一ID==**[[4.1 全局唯一ID]]

    ```Shell
     127.0.0.1:6379> SET age 24
     OK
     127.0.0.1:6379> INCR age
     (integer) 25
    ```
- `INCRBY`：让一个整型的key自增并指定步长

    ```Shell
     127.0.0.1:6379> INCRBY age 3
     (integer) 28
     127.0.0.1:6379> INCRBY age -5
     (integer) 23
    ```
- `INCRBYFLOAT`：让一个浮点类型的数字自增并指定步长

    ```Shell
     127.0.0.1:6379> SET length 20.5
     OK
     127.0.0.1:6379> GET length
     "20.5"
     127.0.0.1:6379> INCRBYFLOAT length 9.9
     "30.4"
    ```
- `SETNX`：
    - 添加一个 String 类型的键值对，前提是这个key不存在，否则不执行
    - **==可用来实现 互斥锁==**[[Redis 实现分布式锁]]

    ```Shell
     # SET 重名key会覆盖value
     127.0.0.1:6379> GET k4
     "v44"
     127.0.0.1:6379> SET k4 v444
     OK
     127.0.0.1:6379> GET k4
     "v444"
     # SETNX 重名key执行失败
     127.0.0.1:6379> SETNX k4 v4444
     (integer) 0
     127.0.0.1:6379> GET k4
     "v444"
    ```
- `SETEX`：添加一个String类型的键值对，并且指定有效期

    ```Shell
     127.0.0.1:6379> SETEX k5 5 v5
     OK
     127.0.0.1:6379> KEYS *
     1) "age"
     2) "k5"
     3) "name"
     4) "k4"
     5) "length"
     127.0.0.1:6379> KEYS *
     6) "age"
     7) "name"
     8) "k4"
     9) "length"
    ```
