# 一、当前存在的问题：（考虑极端情况）
- 由于释放锁的操作是分两步进行的
    1. 查出锁中的标识
    2. 判断标识与自己的线程标识是否一致
- **==释放锁的两步操作并不具有原子性==**
- 如果再查出锁中标识后，且再比较之前，线程被阻塞
- ==而在此期间，锁又刚好被超时释放了==

    ![[IMG-20260620224127755.png|800]]

# 二、解决方案： 使用 Lua 脚本确保 Redis 命令执行具有原子性，实现基于Lua 脚本实现分布式锁的释放锁逻辑
1. unLock.lua

    ```Lua
     --锁的key
     key = KEY[1]
     --当前线程的标识
     threadId = ARGV[1]
     --获取锁中的线程标识
     id = redis.call('get', key)
     --比较线程标识与锁中的标识是否一致
     if(id == threadId) then
         --释放锁 del key
         return redis.call('del', key)
     end
     return 0
    ```
2. 修改后的锁（只修改了释放锁处的逻辑）

    ```Java
     public class SimpleRedisLock implements ILock{
         private static final String KEY_PREFIX =  "lock:";
         private static final String ID_PREFIX = UUID.randomUUID().toString(true) + "-";
         //⭐第一个参数需要传入一个Script对象
         private static final DefaultRedisScript<Long> UNLOCK_SCRIPT;
         //使用静态代码块对Script对象进行初始化
         static {
             UNLOCK_SCRIPT = new DefaultRedisScript<>();
             UNLOCK_SCRIPT.setLocation(new ClassPathResource("unLock.lua"));
             UNLOCK_SCRIPT.setResultType(Long.class);
         }
         private String name;
         private StringRedisTemplate stringRedisTemplate;
         public SimpleRedisLock(String name, StringRedisTemplate stringRedisTemplate) {
             this.name = name;
             this.stringRedisTemplate = stringRedisTemplate;
         }
         @Override
         public boolean tryLock(long timeoutSec) {
             // 获取线程标识
             String threadId = ID_PREFIX + Thread.currentThread().getId();
             // 获取锁
             Boolean success = stringRedisTemplate.opsForValue().setIfAbsent(KEY_PREFIX + name, threadId, timeoutSec, TimeUnit.SECONDS);
             return BooleanUtil.isTrue(success);
         }
         @Override
         public void unLock() {
             //⭐第二个参数需要一个存KEY的List
             ArrayList<String> list = new ArrayList<>();
             list.add(KEY_PREFIX + name);
             //⭐第三个参数为Value
             String threadId = ID_PREFIX + Thread.currentThread().getId();
             //⭐调用 Lua 脚本
             stringRedisTemplate.execute(UNLOCK_SCRIPT, list,threadId);
         }
     }
    ```
