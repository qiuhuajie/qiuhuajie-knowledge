# 1. 示例：实现查询点赞排行榜的接口
![[IMG-20260405035438365.png|Untitled 429.png]]
1. ==需求：按照点赞时间先后排序，返回Top5的用户==
    
    - 要求redis存储已点赞用户的数据结构具有的特性：
        
        - ==元素唯一性==
        
        - ==元素有序==
        
        - 在判断已登录用户是否点赞时，要查询该数据结构，因此==查询性能也要好==
        
    
    - 因此**==选择== `SortedSet` 存储哪些用户点赞了当前 blog （**[[7. SortedSet 类型]]**）**
        
        ![[IMG-20260405035454516.png|Untitled 1 316.png]]
        
        示例：
        
        ```Bash
         127.0.0.1:6379> ZADD sorted.set.test 3 node01
         (integer) 1
         127.0.0.1:6379> ZADD sorted.set.test 4 node02
         (integer) 1
         127.0.0.1:6379> ZADD sorted.set.test 1 node03
         (integer) 1
        ```

1. BlogController
    
    ```Java
     @GetMapping("/likes/{id}")
     public Result queryBlogLikes(@PathVariable("id") Long id) {
         return blogService.queryBlogLikes(id);
     }
    ```
    
1. BlogServiceImpl
    
    ```Java
     @Override
     public Result queryBlogLikes(Long id) {
         //1. 查询Top5的点赞用户 zrange key 0 4
         String key = "blog:liked:" + id;
         Set<String> top5 = stringRedisTemplate.opsForZSet().range(key, 0, 4);
         if (top5 == null || top5.isEmpty()) {
             return Result.ok(Collections.emptyList());
         }
     
         //2. zrange查出来的是score，因此还需要解析出用户Id
         List<Long> ids = top5.stream().map(Long::valueOf).collect(Collectors.toList());
     
         //3. 根据解析出的用户Id，查出用户信息
         //        List<User> users = userService.listByIds(ids);
         String idStr = StrUtil.join(",", ids);
    
         //⭐⭐自定义拼接一个sql语句，last()表示直接在sql语句后拼接语句
         List<User> users = userService.query().in("id", ids).last("ORDER BY FIELD(id,"+ idStr +")").list();
         List<UserDTO> userDTOS = users.stream().map(user -> BeanUtil.copyProperties(user, UserDTO.class)).collect(Collectors.toList());
     
         //4. 返回
         return Result.ok(userDTOS);
     }
    ```
    
    > [!important] 因为需要根据 sortedset 中的score字段对已点赞用户排序
    > 
    > - 但是如果使用`mybatisPlus` 提供的 `listByIds();` 来查询用户的话
    >     
    >     ==返回的== ==`user`== ==对象====`List`== ==中 对象的顺序不一定是按照传入的== ==`id`== ==的顺序排列的==
    >     
    > 
    > - why? `listByIds();` 底层执行的sql语句是 `in`，不能保证查出的对象list顺序与传入的id顺序一致
    >

    >     
    > 
    > - 解决方法：需要使用 `ORDER BY FIELD`
