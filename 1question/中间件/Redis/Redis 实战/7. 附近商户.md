# 一、GEO 介绍
1. `GEO` 就是Geolocation的简写形式，代表地理坐标。Redis在3.2版本中加入了对GEO的支持，允许存储地理坐标信息，帮助我们根据经纬度来检索数据。常见的命令有：
    - [GEOADD](https://redis.io/commands/geoadd)：添加一个地理空间信息，包含：经度（longitude）、纬度（latitude）、值（member）
    - [GEODIST](https://redis.io/commands/geodist)：计算指定的两个点之间的距离并返回
    - [GEOHASH](https://redis.io/commands/geohash)：将指定member的坐标转为hash字符串形式并返回
    - [GEOPOS](https://redis.io/commands/geopos)：返回指定member的坐标
    - [GEORADIUS](https://redis.io/commands/georadius)：指定圆心、半径，找到该圆内包含的所有member，并按照与圆心之间的距离排序后返回。6.2以后已废弃
    - [GEOSEARCH](https://redis.io/commands/geosearch)：在指定范围内搜索member，并按照与指定点之间的距离排序后返回。范围可以是圆形或矩形。6.2.新功能
    - [GEOSEARCHSTORE](https://redis.io/commands/geosearchstore)：与GEOSEARCH功能一致，不过可以把结果存储到一个指定的key。 6.2.新功能

## 1. 需求一：先将所有商户的地理信息存入Redis
1. 测试类中

    ```Java
     @Test
     public void loadShopLocationData(){
         //1. 查询店铺信息
         List<Shop> shopList = shopService.list();
         //2. ⭐把店铺分组，按照typeId分组，typeId一致的放到一个List集合
         //所有类型的shopList存在一个Map集合中
         Map<Long, List<Shop>> map = shopList.stream().collect(Collectors.groupingBy(Shop::getTypeId));
         //3. 分批完成写入Redis
         for (Map.Entry<Long, List<Shop>> entry : map.entrySet()) {
             //3.1 获取类型id
             Long shopTypeId = entry.getKey();
             String key = "shop:geo:" + shopTypeId;
             //3.2 获取同类型的店铺的集合
             List<Shop> shops = entry.getValue();
             List<RedisGeoCommands.GeoLocation<String>> locations = new ArrayList<>(shops.size());
             //3.3 写入redis GEOADD key 经度 纬度 member
             for (Shop shop : shops) {
                 locations.add(new RedisGeoCommands.GeoLocation<>(shop.getId().toString(),new Point(shop.getX(),shop.getY())));
             }
             // ⭐这里存入redis时使用到的是直接存入一个集合的API，如果在shops的遍历中，一个一个存入shop的信息，则每遍历一次就要与redis建立一次链接，效率不高
             // 因此先将所有shops中的信息封装成一个集合，只需建立一次redis连接，即可存入所有的数据
             stringRedisTemplate.opsForGeo().add(key, locations);
         }
     }
    ```
    - **技巧一：**==如果集合中的所有元素要遍历存入redis==，如果有直接存入一个集合的API，建议使用；避免建立太多redis连接，从而降低效率
    - **技巧二：**如果一个`list`集合中的元素，如果需要按照某个字段，将其分类为多个`list`时，可以使用强大的`StreamAPI`，最后得到一个`Map`

2. ShopServiceImpl

    ```Java
     @Override
     public Result queryShopByType(Integer typeId, Integer current, Double x, Double y) {
         //1. 判断是否需要根据坐标查询
         if (x == null || y == null) {
             // 不需要，根据类型分页查询
             Page<Shop> page = query()
                     .eq("type_id", typeId)
                     .page(new Page<>(current, SystemConstants.DEFAULT_PAGE_SIZE));
             // 返回数据
             return Result.ok(page.getRecords());
         }
         //2. 计算分页参数
         int from = (current - 1) * SystemConstants.DEFAULT_PAGE_SIZE;
         int end = current + SystemConstants.DEFAULT_PAGE_SIZE;
         //3. 查询redis，按照距离排序分页。结果：shopId、distance
         String key = "shop:geo:" + typeId;
         //3.1 先从redis中查出所有结果
         GeoResults<RedisGeoCommands.GeoLocation<String>> results = stringRedisTemplate.opsForGeo().search(
                 key,
                 GeoReference.fromCoordinate(x, y),
                 new Distance(5000),
                 RedisGeoCommands.GeoSearchCommandArgs.newGeoSearchArgs().includeDistance().limit(end)
         );
         //结果判空
         if (results == null){
             return Result.ok(Collections.emptyList());
         }
         //4. 解析出shopId
         List<GeoResult<RedisGeoCommands.GeoLocation<String>>> contentList = results.getContent();
         //判断此次请求的起始位置，是否超过了结果集合的长度，若超过则返回空集合
         if (from >= contentList.size()) {
             return Result.ok(Collections.emptyList());
         }
         //保存shopId的list，后面遍历得到shop对象
         ArrayList<String> shopIdList = new ArrayList<>();
         //保存店铺距离的map，后面遍历shop结果集，给shop的距离字段赋值
         HashMap<String, Distance> map = new HashMap<>();
         //从from开始截取结果集，这里使用stream的skip，可以直接跳过，避免拷贝集合，降低效率
         contentList.stream().skip(from).forEach(result ->{
             String shopId = result.getContent().getName();
             shopIdList.add(shopId);
             Distance distance = result.getDistance();
             map.put(shopId,distance);
         });
         //5. 根据shopId查询出shop
         String shopIdStr = StrUtil.join(",", shopIdList);
         List<Shop> shops = query().in("id", shopIdList).last("ORDER BY FIELD(id," + shopIdStr + ")").list();
         //6. 给每个shop的距离字段赋值
         for (Shop shop : shops) {
             shop.setDistance(map.get(shop.getId().toString()).getValue());
         }
         //7. 返回
         return Result.ok(shops);
     }
    ```
    - 当需要从集合的某一个位置读取元素时，**可以先将 List 转为 Map 结构**

    ```Java
     for (int i = from; i < contentList.size(); i++) {
         String shopId = contentList.get(i).getContent().getName();
         shopIdList.add(shopId);
         Distance distance = contentList.get(i).getDistance();
         map.put(shopId,distance);
     }
    ```
        可以使用 `stream().skip()` 来代替
```Java

     String shopId = result.getContent().getName();
     shopIdList.add(shopId);
     Distance distance = result.getDistance();
     map.put(shopId,distance);
 });
```
