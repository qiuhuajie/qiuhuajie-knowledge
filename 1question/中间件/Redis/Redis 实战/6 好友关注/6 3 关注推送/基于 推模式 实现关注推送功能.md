# 1. 业务需求
- 修改新增探店笔记的业务，**在保存 blog 到数据库的同时，推送到粉丝的**==**收件箱**==
![[IMG-20260405035438400.png|Untitled 551.png]]
# 2. 实现方案
## 需求一：排序
1. 可以根据时间戳排序，可以用 Redis 的 **`SortedSet`** 类型数据结构实现
  
## 需求二：分页
1. 传统的分页查询：Feed流中的数据会不断更新，所以数据的角标也在变化，传统的**分页**模式会有问题
    
    ![[IMG-20260405035459748.png|Untitled 1 409.png]]
    
1. ==Feed流的分页查询需要使用== **==根据分数倒序查询== `ZREVRANGEBYSCORE`**
    
    ![[IMG-20260405035509459.png|Untitled 2 331.png]]
    
    - ⭐`**ZREVRANGEBYSCORE key max min [WITHSCORES] [LIMIT offset count]**`
    
    - 可以根据上一次分页查询的结果，决定本次分页查询的起始时间戳以及页内偏移量
        
        ![[IMG-20260405035517306.png|Untitled 3 250.png]]
        
    
    - 命令详解
        
        ```Java
         127.0.0.1:6379> help ZREVRANGEBYSCORE
         
           ZREVRANGEBYSCORE key max min [WITHSCORES] [LIMIT offset count]
           summary: Return a range of members in a sorted set, by score, with scores ordered from high to low
           since: 2.2.0
        ```
        
        - 由于 `**SCORE**` **是时间戳**
            
            - `**max**` **：最大时间戳（**==**第一次查询为当前时间戳，后续查询为上次查询结果的最小时间戳**==**）**
            
            - `**min**` **：最小时间戳（**==**固定为 0**==**）**
            
        
        - `**offset**` **：为在** `**max，min**` **范围在查询的起始偏移量（**==**第一次查询为 0，后续查询为上一次的结果集中，与最小时间戳一样分数的元素的个数**==**，即把已经查到的跳过）**
        
        - `**count**` **：查询的数据量（**==**固定的分页大小**==**）**
        
        - `WITHSCORES` ：结果中带上分数
        
    
    - 分页示例：
        
        ```Bash
        # ⭐第一次查询，查出 0-1000 内的，第二个0代表<=1000 ，3代表查询三个
         127.0.0.1:6379> ZREVRANGEBYSCORE sortedSet.test 1000 0 WITHSCORES LIMIT 0 3
         1) "node07"
         2) "7"
         3) "node06"
         4) "6"
         5) "node05"
         6) "5"
         
         # 此时插入一个新数据，角标发生变化
         127.0.0.1:6379> ZADD sortedSet.test 8 node08
         (integer) 1
         
         # ⭐第二次查询，查出 0-5 内的，5即上次查询的最小值，1代表<5（1由set中分数为5的value个数决定） ，3代表查询三个
         127.0.0.1:6379> ZREVRANGEBYSCORE sortedSet.test 5 0 WITHSCORES LIMIT 1 3
         1) "node04"
         2) "4"
         3) "node03"
         4) "3"
         5) "node02"
         6) "2"
        ```
        
    
# 3. 代码实现
## 3.1 **UP主推送**
1. BlogController
    
    ```Java
     @PostMapping
     public Result saveBlog(@RequestBody Blog blog) {
         return blogService.saveBlog(blog);
     }
    ```
    
1. BlogServiceImpl
    
    ```Java
     @Override
     public Result saveBlog(Blog blog) {
         // 1. 获取登录用户
         UserDTO user = UserHolder.getUser();
         blog.setUserId(user.getId());
         // 2. 保存探店博文
         boolean isSuccess = save(blog);
         if (!isSuccess) {
             return Result.fail("新增笔记失败！");
         }
         // 3. 查询笔记作者的所有粉丝 select * from tb_follow where follow_user_id = ?
         List<Follow> follows = followService.query().eq("follow_user_id", user.getId()).list();
         // 4. 推送笔记 id 到所有粉丝的收件箱
         for (Follow follow : follows) {
             // 4.1 获取粉丝id
             Long userId = follow.getUserId();
             // 4.2 推送
             String key = "feed:" + userId;
             stringRedisTemplate.opsForZSet().add(key, blog.getId().toString(), System.currentTimeMillis());
         }
         // 5. 返回id
         return Result.ok(blog.getId());
     }
    ```

    - **`value`**：用户 id
    
    - **`score`**：时间戳（可保证按时间排序）
    
## 3.2 **粉丝从收件箱中获取推送**
1. ScrollResult 结果封装类
    
    ```Java
     @Data
     public class ScrollResult {
         private List<?> list;
         private Long minTime;
         private Integer offset;
     }
    ```
    
1. BlogController
    
    ```Java
     @GetMapping("/of/follow")
     public Result queryBlogOfFollow(
             @RequestParam(value = "lastId") Long max,
             @RequestParam(value = "offset", defaultValue = "0") Integer offset) {
         return blogService.queryBlogOfFollow(max, offset);
     }
    ```
    
1. BlogServiceImpl
    
    ```Java
     @Override
     public Result queryBlogOfFollow(Long max, Integer offset) {
         //1. 获取当前用户
         Long userId = UserHolder.getUser().getId();
         //2. 查询收件箱
         String key = "feed:" + userId;
         Set<ZSetOperations.TypedTuple<String>> typedTuples = stringRedisTemplate.opsForZSet().reverseRangeByScoreWithScores(key, 0, max, offset, 2);
         // 非空判断
         if (typedTuples == null || typedTuples.isEmpty()) {
             return Result.ok();
         }
         
         ArrayList<Long> blogIdList = new ArrayList<>(typedTuples.size());
    
         //3. 求出 minTime（最新时间戳），offset，封装在分页对象中，返回给前端，以继续下一页的查询
         long minTime = 0;
         int offsetCount = 1;
         for (ZSetOperations.TypedTuple<String> typedTuple : typedTuples) {
             //获取blogId
             String idStr = typedTuple.getValue();
             blogIdList.add(Long.valueOf(idStr));
    				 
             //offset 的获取，即为统计本次查出的结果集typedTuples中最小时间戳的个数，（如果在得出minTime后，再遍历一次typedTuples，那样时间复杂度太高）
             //故在一次遍历中，就边遍历，边统计所有具有相同时间戳的元素个数，并当下一个不重复时，将计数器归1
             //这样遍历到最后，即可得到最小时间戳的出现次数
             long time = typedTuple.getScore().longValue();
             if (time == minTime){
                 offsetCount ++;
             }else {
                 // 获取minTime（时间戳），当for循环走完，minTime保存的一定是最后一个元素的时间戳，也即最小时间戳（因为是按时间戳倒序排序）
                 minTime = typedTuple.getScore().longValue();
                 offsetCount = 1;
             }
         }
    
         //4. 根据id查询blog
         String blogIdStr = StrUtil.join(",", blogIdList);
         List<Blog> blogList = query().in("id", blogIdList).last("ORDER BY FIELD(id," + blogIdStr + ")").list();
     
         for (Blog blog : blogList) {
             //给blog赋值 关联的用户
             Long id = blog.getId();
             User user = userService.getById(id);
             blog.setName(user.getNickName());
             blog.setIcon(user.getIcon());
     
             // 给blog赋值 blog是否被当前用户点赞
             isBlogLiked(blog);
         }
    
         //5. 封装成 滚动分页对象ScrollResult，并返回
         ScrollResult scrollResult = new ScrollResult();
         scrollResult.setList(blogList);
         scrollResult.setMinTime(minTime);
         scrollResult.setOffset(offsetCount);
     
         return Result.ok(scrollResult);
     }
    ```
