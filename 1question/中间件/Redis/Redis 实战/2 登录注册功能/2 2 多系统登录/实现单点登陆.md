# 一、单点登陆介绍
1. 单点登录的英文名叫做：Single Sign On（简称**SSO**）
    1. 简单来说，单点登录就是==在多个系统中，用户只需一次登录，各个系统即可感知该用户已经登录==
    2. 比如阿里系的**淘宝和天猫**，很明显地可以知道这是两个系统，但是在使用的时候，登录了天猫，淘宝也会自动登录

    ![[Attachment/1question/中间件/Redis/Redis 实战/2 登录注册功能/2 2 多系统登录/IMG-20260620224127739.png|666]]

2. **单系统的登录流程：**
    1. 登录：将用户信息保存在 Session 对象中
        1. 如果在 Session 对象中能查到，说明已经登录
        2. 如果在 Session 对象中查不到，说明没登录（或者已经退出了登录）
    2. 注销（退出登录）：从Session中删除用户的信息
    3. 记住我 (关闭掉浏览器后，重新打开浏览器还能保持登录状态)：配合 Cookie 来用
3. **多系统登录流程：**
    1. 为了合理利用资源和降低耦合性，于是把单系统**拆分**成多个子系统
    2. ⭐因此，**如果要实现单点登录需要解决两个问题**
        1. ==**Cookie 跨域的问题**==
        2. ==**session 共享的问题**==

# 二、Cookie 跨域的问题
1. **我们使用的是同一个域名且使用的是 Redis 代替 Seesion，用不到 Cookie，所以不存在 Cookie 跨域的问题**
2. 问题描述：
    1. **Cookie是不能跨域的**
    2. 比如请求`https://www.google.com/`时，浏览器会自动把`google.com`的Cookie带过去给`google`的服务器
    3. 而不会把`https://www.baidu.com/`的Cookie带过去给`google`的服务器。
3. 解决方案，使用 SpringCloud Gateway[[Gateway 项目实战]]
# 三、Session 共享问题
1. 单系统登录功能主要是用 Session 保存用户信息来实现的
2. 但是，**多系统**可能有多个 Tomcat
    1. ⭐**在第一台tomcat上登录后，当再次发送请求时，**==**可能由于Nginx负载均衡算法，此次请求被打到了第二台tomcat上了**==
    2. 而 Session 是依赖当前系统的 Tomcat，多台 Tomcat 并不共享 session 存储空间，所以==**之前登录时在session中存的登录信息也就没法查看了！**==

    ![[IMG-20260620224131838.png|800]]

3. **session 共享问题的多种解决方案**
    - tomcat 针对这个session共享问题也提供了**session拷贝功能**，但是存在一些问题：🚫
        - 多台tomcat互相拷贝相同的数据，造成资源浪费
        - 数据的拷贝有延迟，不能满足需求
    - **源 IP Hash 算法**：指定的客户机只能访问指定的后端服务，这样容易造成负载不均匀 🚫
    - 可以专门创建一个 **session 数据库**，但这样对 mysql 压力太大不合适 🚫
    - 使用 redis 解决 ✅
4. 问题的解决思路：
    1. **需要找出一个替代 session 的东西**
    2. 应该满足：
        - 数据共享
        - 内存存储
        - key、value结构
    3. ==**Redis ：没错！就是我了**==
