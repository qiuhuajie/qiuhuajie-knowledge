# 一、为什么要校验
1. 如果不实现校验登录状态：
    1. 在未登录状态下，直接在地址栏写请求，就可以访问到，这显然是不允许的
    - 故需要实现登录校验
2. 可以利用**拦截器**实现此功能，在 springboo t学习时，实现的登录校验也是一样的实现方法，**拦截请求，判断 session 中用户是否为空**[[4.6 拦截器]][[4.6 拦截器]][[4.6 拦截器]]

2. 代码编写：
    1. 自定义一个登录拦截器： `LoginInterceptor`

    ```Java
     public class LoginInterceptor implements HandlerInterceptor {
         @Override
         public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
             //1. 获取session
             HttpSession session = request.getSession();
             //2. 获取session中的用户
             Object user = session.getAttribute("user");
             //3. 判断用户是否存在
             if (user == null){
                 //4. 不存在，拦截
                 response.setStatus(401);
                 return false;
             }
             //5. ⭐存在，保存用户信息到 ThreadLocal
             UserHolder.saveUser((UserDTO) user);
             //6. 放行
             return true;
         }
         @Override
         public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception {
         }
         @Override
         public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) throws Exception {
             //移除用户
             UserHolder.removeUser();
         }
     }
    ```
    > 💡 **ThreadLocal**
    > [[Threadlocal]]
    > 1. **请求 Tomcat 的每个请求都是一个线程，这些线程需要携带参数，为了各个线程之间不会造成干扰，就要**==**将参数存在本线程的内存空间内**==
    >     - 这里登录拦截器拦截到需要校验登陆状态请求后，会将查出的用户信息存入本次请求线程的内存空间，这样多个请求之间不会干扰
    >     - 同时在执行完请求后，会执行拦截器的 afterCompletion 方法，在改方法里需要将ThreadLocal存着的user信息删除掉！
    > 1. 这里有几点需要注意：
    >     - 因为每个 Thread 内有自己的实例副本，且该副本只能由当前 Thread 使用。这是也是 ThreadLocal 命名的由来
    >     - 既然每个 Thread 有自己的实例副本，且其它 Thread 不可访问，因此也**不存在多线程间共享的问题**
    > ![[IMG-20260620224130861.png|800]]
    > 1. 项目代码里已经提供了ThreadLocal的保存、获取、删除的方法：
    >     ```Java
    >      public class UserHolder {
    >          private static final ThreadLocal<UserDTO> tl = new ThreadLocal<>();
    >          public static void saveUser(UserDTO user){
    >              tl.set(user);
    >          }
    >          public static UserDTO getUser(){
    >              return tl.get();
    >          }
    >          public static void removeUser(){
    >              tl.remove();
    >          }
    >      }
    >     ```
    2. 写一个配置类，将自定义的拦截器注册到 WebMvcConfigurer 中

    ```Java
     @Configuration
     public class MvcConfig implements WebMvcConfigurer {
         @Override
         public void addInterceptors(InterceptorRegistry registry) {
             registry.addInterceptor(new LoginInterceptor()).excludePathPatterns(
                     "/shop/**",
                     "/voucher/**",
                     "/shop-type/**",
                     "/upload/**",
                     "/blog/hot",
                     "/user/code",
                     "/user/login"
             );
         }
     }
    ```
    3. UserController，`/me` 请求负责返回当前登录的用户信息，即从UserHolder中取出当前用户

    ```Java
     @GetMapping("/me")
     public Result me(){
         // 获取当前登录的用户并返回
         UserDTO user = UserHolder.getUser();
         return Result.ok(user);
     }
    ```
    > 💡 **这里的 UserDTO 是个啥？**
    > - 如果我们将查出来的所有用户信息都返回给前端，这是很危险的，可能包含密码信息、年龄等敏感信息
    > - 就需要在向session中存入用户之前，将其变为不含隐私信息的 UserDTO对象（使用user的一个简化实体来返回用户的信息）
    >     ```Java
    >      @Data
    >      public class UserDTO {
    >          private Long id;
    >          private String nickName;
    >          private String icon;
    >      }
    >     ```
    > - 可以将 查出来的 User 对应字段再赋值给 UserDTO 对象，可以使用 **hutool 工具类**提供的 **cn.hutool.core.bean.BeanUtil**
    >     ```Java
    >      UserDTO userDTO = BeanUtil.copyProperties(user, UserDTO.class);
    >     ```
