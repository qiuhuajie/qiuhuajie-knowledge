# 一、介绍
1. 就是数据交换的缓冲区（称作Cache [ kæʃ ] ），是存贮数据的**临时地方**，一般**读写性能较高**

    ![[IMG-20260620224127713.png|800]]

2. **==缓存的本质：一种使用就近存储来提升性能的一种工程方法==**
3. **缓存的作用**
    - 降低后端负载
    - 提高读写效率，降低响应时间
4. ==**共享缓存与私有缓存**==
    1. 私有缓存
        1. 临近用户，性能更好
        2. 存在缓存同步问题：在用户侧私有的缓存，如果其他应用引发了数据变化，无法感知到变化，也就无法做出相应的更改
    2. 共享缓存
        1. 数据一致性好，成本相应的也高
        2. 比私有缓存性能差一些，毕竟私有缓存离自己业务特别近
        3. 对共享缓存要求也很高，比如性能，比如高可用。因为一旦共享缓存很慢，那么所有使用这个缓存的业务都受影响
5. ==缓存的例子：==
    1. 动态规划、递归中使用的临时存储结构用来加速计算或者防止爆栈
    2. CPU 缓存 L1、L2、L3
    3. `volatile` 关键字，经常性的在做性能调优和加速，通过寄存器来缓存部分内容或者加速计算等
    4. 写文章 `command + s` 时，写盘的内容也是落到了磁盘的缓存中

# 二、缓存的命中率
1. 缓存的命中率是核心指标之一，但缓存的命中率和出问题的概率并没有直接关系，它只是代表了业务运行的效率
2. 一定要知道数据源在没有缓存的情况下抗压能力是多少，这个数据能够很好的帮助业务风险评估和决策，尤其是在出问题后，业务能很清楚自己的业务底线在哪里，可以评估动作的有效性
3. 需要能够通过业务的摸高来测试上线的业务，缓存的抗压程度和数据源仍需要摸到。这个对于热点、倾斜等早发现早治疗有奇效
