# 一、最终一致性策略
- 为了达到最终一致性，针对不同的场景，业界逐步形成了下面这几种应用缓存的策略
1. `Cache Through` 模式

    ![[IMG-20260620224127744.png|584]]

    1. 用户访问到缓存即返回，它不能够直接访问数据源
    2. 用户读写后，由缓存来保障数据在内存和数据源之间的一致性
    3. 可以把这种模式看成一个**强一致性的主备架构**，只不过**主和备是异构的**
    4. 但这种方式在互联网中使用并不广泛，**维护成本比较高**
2. `Cache Aside` 模式 ✅

    ![[IMG-20260620224132339.png|763]]

    1. 意为==旁路缓存模式==，是应用最为广泛的一种缓存策略，==**由缓存的调用者，在更新数据库的同时更新缓存**==
    2. 读写流程：
        1. **读操作：缓存命中则直接返回，缓存未命中则查询数据库，并写入缓存，设定超时时间（实现超时剔除）**
        2. **写操作：先写数据库，然后再删除缓存（实现主动更新）要确保数据库与缓存操作的原子性**
