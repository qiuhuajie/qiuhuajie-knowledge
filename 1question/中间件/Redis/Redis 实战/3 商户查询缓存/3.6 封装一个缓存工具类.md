# 一、需求
基于 `StringRedisTemplate` 封装一个缓存工具类，满足下列需求：
- 处理缓存空值穿透
    - 方法1：存----将任意Java对象序列化为json并存储在string类型的key中，并且可以设置TTL过期时间
    - 方法2：取----根据指定的key查询缓存，并反序列化为指定类型，利用缓存空值的方式解决缓存穿透问题
- 处理缓存单点击穿
    - 方法3：存----将任意Java对象序列化为json并存储在string类型的key中，并且可以设置逻辑过期时间，用于处理缓存击穿问题
    - 方法4：取----根据指定的key查询缓存，并反序列化为指定类型，需要利用逻辑过期解决缓存击穿问题

# 二、代码实现
1. 封装的工具类

    ```Java
     @Component  //⭐注入组件，将工具类交给Spring管理
     public class CacheClient {
         private final StringRedisTemplate stringRedisTemplate;
         public CacheClient (StringRedisTemplate stringRedisTemplate){
             this.stringRedisTemplate = stringRedisTemplate;
         }
         /**
          * @Description: 1️⃣方法1：将任意Java对象序列化为json并存储在string类型的key中，并且可以设置TTL过期时间
          * @param: key
          * @param: value    任意类型的对象
          * @param: time     TTL
          * @param: unit     TTL的单位
          */
         public void set(String key, Object value, Long time, TimeUnit unit){
             stringRedisTemplate.opsForValue().set(key, JSONUtil.toJsonStr(value),time,unit);
         }
         /**
          * @Description: 3️⃣方法3：将任意Java对象序列化为json并存储在string类型的key中，并且可以设置逻辑过期时间，用于处理缓存击穿问题
          * @param: key
          * @param: value    任意类型的对象
          * @param: time     期望的逻辑TTL
          * @param: unit     期望的逻辑TTL的单位
          */
         public void setWithLogicalExpire(String key, Object value, Long time, TimeUnit unit){
             RedisData redisData = new RedisData();
             redisData.setData(value);
             redisData.setExpireTime(LocalDateTime.now().plusSeconds(unit.toSeconds(time)));
             stringRedisTemplate.opsForValue().set(key,JSONUtil.toJsonStr(redisData));
         }
         /**
          * @Description: 2️⃣方法2：根据指定的key查询缓存，并反序列化为指定类型，利用缓存空值的方式解决缓存穿透问题
          * @param: id 要查询数据的id
          */
         //⭐ 泛型方法的使用
         public <R,ID> R queryWithPassThrough(String keyPrefix, ID id, Class<R> type, Function<ID,R> dbFallback, Long time, TimeUnit unit){
             //1. 从Redis中查询缓存
             String json = stringRedisTemplate.opsForValue().get(keyPrefix + id);
             //2. 判断是否存在
             if (StrUtil.isNotBlank(json)){
                 //3. 存在，直接返回
                 return JSONUtil.toBean(json, type);
             }
             //对空值穿透的处理
             if (json != null){
                 return null;
             }
             //4. 不存在，根据id查询数据库
             R r = dbFallback.apply(id);     //⭐ 函数式编程的使用
             if (r == null) {
                 //将控制写入redis,TTL设置的短点：2分钟
                 stringRedisTemplate.opsForValue().set(keyPrefix + id,"",2L, TimeUnit.MINUTES);
                 //5. 不存在，返回错误
                 return null;
             }
             //6. 存在，写入redis缓存
             stringRedisTemplate.opsForValue().set(keyPrefix + id,JSONUtil.toJsonStr(r),time, unit);
             //7. 返回数据
             return r;
         }
         /**
          * 创建线程池
          */
         private static final ExecutorService CACHE_REBUILD_EXECUTOR = Executors.newFixedThreadPool(10);
         /**
          * @Description: 4️⃣方法4：根据指定的key查询缓存，并反序列化为指定类型，需要利用逻辑过期解决缓存击穿问题
          * @param: id 要查询数据的id
          */
         public <R,ID> R queryWithLogicalExpire(String keyPrefix, String lockPrefix, ID id, Class<R> type, Function<ID,R> dbFallback, Long time, TimeUnit unit){
             //1. 从Redis中查询缓存
             String json = stringRedisTemplate.opsForValue().get(keyPrefix + id);
             //2. 判断是否存在
             if (StrUtil.isBlank(json)){
                 //2.1. 未命中，直接返回null
                 return null;
             }
             //2.2. 命中，需要先把json反序列化为对象
             RedisData redisData = JSONUtil.toBean(json, RedisData.class);
             R r = JSONUtil.toBean((JSONObject) redisData.getData(), type);
             LocalDateTime expireTime = redisData.getExpireTime();
             //3. 判断逻辑过期时间
             if (expireTime.isAfter(LocalDateTime.now())){
                 //3.1 未过期，直接返回
                 return r;
             }
             //3.2 过期，直接当前已经过期的信息，并且需要缓存重建
             //4 缓存重建
             //4.1 获取互斥锁
             boolean isLock = tryLock(lockPrefix + id);
             //4.2 判断锁是否获取成功
             if (isLock){
                 //4.3 获取成功，开启独立线程，实现缓存重建
                 CACHE_REBUILD_EXECUTOR.submit(() -> {
                     try {
                         //重建缓存
                         //a. 查数据库
                         R r1 = dbFallback.apply(id);
                         //b. 存入redis缓存
                         this.setWithLogicalExpire(keyPrefix + id, r1, time, unit);
                     } catch (Exception e) {
                         throw new RuntimeException(e);
                     }finally {
                         //finally 中完成锁的释放
                         unLock(lockPrefix + id);
                     }
                 });
             }
             //5. 执行到这里的都是过期，直接返回已经过期数据
             return r;
         }
         /**
          *
          * @param: key redis中锁的key
          * @Description: 获取锁
          */
         private boolean tryLock(String key){
             Boolean flag = stringRedisTemplate.opsForValue().setIfAbsent(key, "myLock", 10, TimeUnit.MINUTES);
             return BooleanUtil.isTrue(flag);
         }
         /**
          *
          * @param: key redis中锁的key
          * @Description: 释放锁
          */
         private void unLock(String key){
             stringRedisTemplate.delete(key);
         }
     }
    ```

# 三、工具类实现小技巧
1. 泛型的应用
    1. 由于是工具类，所以==不知道此方法操作的，和返回的是什么类型的对象==
    2. 比如上面：不一定是返回shop对象，可能是userService调用工具类方法，返回user对象
    3. 这里就需要使用泛型方法 （详见 Java 11. 泛型）
2. 函数式编程
    1. 由于工具类中需要使用 id 获取数据，但是==不同数据类型所需要查询不同的的表，在工具类方法中无法写死==
    2. 所以需要不同的数据类型在调用工具类方法时，**将各自查询数据库表的方法，以** **`Function`** **参数的方式传入**
3. 工具类要使用 `@Component` 标注
    1. ==将工具类的实例交给 Spring 管理==
    2. 在使用工具类时，注入工具类实例即可
