# 一、缓存空值穿透问题
1. **指**==**客户端请求的数据在缓存中和数据库中都不存在（非法数据）**==
2. 这样缓存永远不会生效，这些请求都会打到数据库
3. 如 [http://localhost:8080/shop-detail.html?id=0](http://localhost:8080/shop-detail.html?id=0)
    1. 查找 **`id = 0`** 的商铺信息**（违法id）**，缓存肯定不会命中
    2. 故==每次都会去数据库找，如果恶意发送大量此类请求....====**给数据库带来巨大压力**==

    ![[IMG-20260404031942610.png|547]]
# 二、解决方案
- 常见的解决方案有以下几种：
1. ==当第一次发生缓存空值穿透后，将一个空对象放在缓存中，然后给这个==**==空对象==**==设置一个过期时间== ✅

    ![[IMG-20260404031942686.png|548]]
    - 优点：实现简单，维护方便
    - **缺点：**
        - 缓存层需要提供更多的内存空间来存储空对象，额外的内存消耗（被动防御，如果有大量空值攻击，会造成内存不足）
        - 虽然空对象的过期时间设置得很短，但还是可能会造成缓存与数据库数据的短期不一致
2. ==布隆过滤==[[位图和布隆过滤器]]
    1. 布隆过滤器
        1. 实际上是**一个很长的二进制位图** 和 **一系列随机哈希函数**
        2. 可以**用于快速检测一个元素是否在一个集合中**，它的优点是空间效率和查询的世界都远远超过一 般的算法，缺点是**有一定的误识别率**
    2. **解决空值击穿：**
        1. **将所有可能存在的参数哈希到一个足够大的 bitmaps 中（需要参数可枚举**❗**）**
        2. 如果是一个非法的参数，它一定不存在 bitmaps 中，会被拦截掉，从而避免了对底层存储系统的查询压力
        3. 有一定误判率，所以需要配合一些接口流量的限制（限制同一用户一段时间内访问的频率）、权限校验、黑名单等来解决空值击穿问题
3. ==做好热点参数的限流== 详见：SpringCloud Alibaba Sentinel：16.3.3 热点规则 ✅
4. 增强 id 的复杂度，避免被猜测 id 规律
5. 做好数据的基础格式校验
6. 加强用户权限校验
# 三、示例：在店铺查询功能中做加强，避免缓存穿透
![[IMG-20260404031942780.png|800]]
1. ShopServiceImpl

    ```Java
     @Service
     public class ShopServiceImpl extends ServiceImpl<ShopMapper, Shop> implements IShopService {
         @Resource
         StringRedisTemplate stringRedisTemplate;
         @Override
         public Result queryById(Long id) {
             //1. 从Redis中查询商铺缓存
             String shopJson = stringRedisTemplate.opsForValue().get(RedisConstants.CACHE_SHOP_KEY + id);
             //2. 判断是否存在
             if (StrUtil.isNotBlank(shopJson)){
                 Shop shop = JSONUtil.toBean(shopJson, Shop.class);
                 //3. 存在，直接返回
                 return Result.ok(shop);
             }
             //StrUtil.isNotBlank 对 "" 也会返回false，所以上面不会命中 ""
             //⭐新增：判断是否是空字符串 ""，如果是则表示缓存中的空值命中了，当前是一个违法请求，返回错误信息
             if (shopJson != null){
                 return Result.fail("店铺不存在！");
             }
             //4. 不存在，根据id查询数据库
             Shop shop = getById(id);
             if (shop == null) {
                 //⭐新增：将空值写入redis,TTL设置的短点：2分钟
                 stringRedisTemplate.opsForValue().set(RedisConstants.CACHE_SHOP_KEY + id,"",2L, TimeUnit.MINUTES);
                 //5. 不存在，返回错误
                 return Result.fail("店铺不存在！");
             }
             //6. 存在，写入redis缓存
             ...
         }
         @Override
         @Transactional
         public Result update(Shop shop) {
             ...
         }
     }
    ```
2. 测试：访问 [http://localhost:8080/shop-detail.html?id=0](http://localhost:8080/shop-detail.html?id=0)
    1. `id=0` 的商品不存在，缓存和数据库都不会命中
    2. 则会将 `""` 存入redis缓存

    ![[IMG-20260404031942835.png|800]]
    3. 再次查 `id=0` ，则会直接在缓存中命中空对象，不会对数据库造成压力
