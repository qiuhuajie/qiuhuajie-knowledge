# 1. 存在的问题
- 缓存中的批量 Key 失效
    
    1. 是指**在**==**同一时段大量的缓存 key 由于 TTL 接近，而造成同时失效**==**或者** ==**Redis 服务宕机**==
    
    1. **导致大量请求到达数据库，带来巨大压力**
        
        ![[Attachment/1question/中间件/Redis/Redis 实战/3 商户查询缓存/IMG-20260405035438355.png|Untitled 424.png]]
        
    
# 2. **解决方案**
- **给不同的 Key 的**==**添加随机 TTL 值**== **（针对大量的缓存 key 同时失效）**
- 利用 ==Redis 集群==提高服务的可用性（针对 Redis 服务宕机）
- 给业务添加==多级缓存==：使用 Nginx 缓存 + Redis 缓存 + 其他缓存，不同层使用不同的缓存，可靠性更高
- 给缓存业务添加**降级限流策略** （不会打到数据库，而是有兜底的快速失败） 详见：SpringCloud Alibaba Sentinel：16.3.2 服务降级规则
