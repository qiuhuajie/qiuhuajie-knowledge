# 一、代码示例
- 修改根据 id 查询商铺的业务，基于**互斥锁**方式来解决**缓存击穿**问题
    ![[IMG-20260404031818551.png|800]]
1. ShopServiceImpl

    ```Java
     @Service
     public class ShopServiceImpl extends ServiceImpl<ShopMapper, Shop> implements IShopService {
         @Resource
         StringRedisTemplate stringRedisTemplate;
         @Override
         public Result queryById(Long id) {
             // 互斥锁解决缓存击穿
             Shop shop = queryWithMutex(id);
             if (shop == null){
                 return Result.fail("店铺不存在！");
             }
             //7. 返回数据
             return Result.ok(shop);
         }
         private Shop queryWithMutex(long id){
             //1. 从Redis中查询商铺缓存
             String shopJson = stringRedisTemplate.opsForValue().get(RedisConstants.CACHE_SHOP_KEY + id);
             //2. 判断是否存在
             if (StrUtil.isNotBlank(shopJson)){
                 //3. 存在，直接返回
                 return JSONUtil.toBean(shopJson, Shop.class);
             }
             //对空值穿透的处理
             if (shopJson != null){
                 return null;
             }
             //4. ⭐实现缓存重建
             //4.1 尝试获取互斥锁
             Shop shop = null;
             try {
                 boolean isLock = tryLock(RedisConstants.LOCK_SHOP_KEY + id);
                 //4.2 判断是否获取成功
                 if (!isLock){
                     //4.3 失败，则休眠并重试
                     Thread.sleep(50);
                     //⭐递归重试
                     queryWithMutex(id);
                 }
                 //4.4 成功，根据id查询数据库
                 shop = getById(id);
                 //⭐模拟重建缓存的延迟
                 Thread.sleep(200);
                 if (shop == null) {
                     //将空值写入redis,TTL设置的短点：2分钟
                     stringRedisTemplate.opsForValue().set(RedisConstants.CACHE_SHOP_KEY + id,"",2L, TimeUnit.MINUTES);
                     //5. 不存在，返回错误
                     return null;
                 }
                 //6. 存在，写入redis缓存
                 stringRedisTemplate.opsForValue().set(RedisConstants.CACHE_SHOP_KEY + id,JSONUtil.toJsonStr(shop),30L, TimeUnit.MINUTES);
             } catch (InterruptedException e) {
                 e.printStackTrace();
             }finally {
                 //7. ⭐释放互斥锁，释放锁写在 finally 中
                 unLock(RedisConstants.LOCK_SHOP_KEY + id);
             }
             //8. 返回数据
             return shop;
         }
         //⭐获取锁
         private boolean tryLock(String key){
             Boolean flag = stringRedisTemplate.opsForValue().setIfAbsent(key, "myLock", 10, TimeUnit.MINUTES);
             return BooleanUtil.isTrue(flag);
         }
         //⭐释放锁
         private void unLock(String key){
             Boolean flag = stringRedisTemplate.delete(key);
         }
     }
    ```
    > 💡 ⭐这里实现锁的方式并不是 synchronized 和 lock ，而是利用了 Redis 中的
    > **`SETNX`** 命令：**添加一个String类型的键值对，前提是这个key不存在，否则不执行**
    > ```Bash
    >  # SETNX 重名key执行失败
    >  127.0.0.1:6379> SETNX k4 v4444
    >  (integer) 0
    >  127.0.0.1:6379> GET k4
    >  "v444"
    > ```
    > 而对应到 stringRedisTemplate ，即为 `stringRedisTemplate.opsForValue().setIfAbsent()` 方法
    > - 获取锁：将 `key=mylock` 设置 `value`，如果已存在这个`key` 则其他请求设置失败，即获取不到锁
    > - 释放锁：删除 `key=mylock` 的键值对
