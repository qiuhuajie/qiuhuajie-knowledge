# 二、DelayExchange 插件原理
1. DelayExchange需要将一个交换机声明为delayed类型
2. 当我们发送消息到delayExchange时，流程如下：
    1. 接收消息
    2. 判断消息是否具备`x-delay`属性
    3. 如果有`x-delay`属性，说明是延迟消息，持久化到硬盘，读取`x-delay`值，作为延迟时间
    4. 返回`routing not found`结果给消息发送者
    5. `x-delay`时间到期后，重新投递消息到指定队列

# 三、代码实现
> 示例：使用DelayExchange 插件实现一个延迟交换队列
## 1. 声明 DelayExchange 交换机
```Java
 @Slf4j
 @Component
 public class SpringRabbitListener {
     @RabbitListener(bindings = @QueueBinding(
             value = @Queue(name = "delay.queue", durable = "true"),
             exchange = @Exchange(name = "delay.direct",delayed = "true"),
             key = "delay"   //⭐
     ))
     public void listenDelayQueue(String msg){
         log.info("消费者收到了 delay.queue 中的延迟消息！");
     }
 }
```
## 2. 发送消息
```Java
 @Slf4j
 @RunWith(SpringRunner.class)
 @SpringBootTest
 public class SpringAmqpTest {
     @Test
     public void testDelayMessage(){
         //定义一个消息对象
         Message message = MessageBuilder
                 .withBody("test ttl Message".getBytes(StandardCharsets.UTF_8))
                 .setDeliveryMode(MessageDeliveryMode.PERSISTENT)
                 .setHeader("x-delay",10000)             //⭐设置信息延时时间
                 .build();
         //发送消息
         rabbitTemplate.convertAndSend("delay.direct","delay",message);
         log.info("delay 消息发送成功！");
     }
 }
```
