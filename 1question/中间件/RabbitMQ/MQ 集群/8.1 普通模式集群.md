# 1. 简介🙋‍♂️
1. 普通集群，或者叫标准集群（classic cluster），具备下列特征：
    
    1. 会在集群的==各个节点间共享部分数据==，包括：
        
        1. ==交换机==
        
        1. ==队列元信息，但==**==不包含队列中的消息==**
        
    
    1. 当访问集群某节点时
        
        1. **==如果队列不在该节点，会根据队列的元数据，去找数据实际所在节点==**
        
        1. 再从数据所在节点传递到当前节点并返回
        
        ![[Attachment/1question/中间件/RabbitMQ/MQ 集群/IMG-20260405035438324.png]]
        
    
    1. 队列所在节点宕机，队列中的消息就会丢失
        
        ![[IMG-20260405035451756.png]]

>

## 2.2 **获取 Cookie**
1. RabbitMQ底层依赖于Erlang，而Erlang虚拟机就是一个面向分布式的语言，默认就支持集群模式。集群模式中的每个RabbitMQ 节点使用 cookie 来确定它们是否被允许相互通信
1. 要使两个节点能够通信，它们必须具有相同的共享秘密，称为**Erlang cookie**
1. cookie 只是一串最多 255 个字符的字母数字字符
1. 每个集群节点必须具有**相同的 cookie**。实例之间也需要它来相互通信
1. 先在之前启动的mq容器中获取一个cookie值，作为集群的cookie。执行下面的命令：
    
    ```Shell
     docker exec -it mq cat /var/lib/rabbitmq/.erlang.cookie
    ```
    
1. 可以看到cookie值如下：
    
    ```Shell
     NMQOJCGYYMPFCWRYGNIS
    ```
    
1. 接下来，停止并删除当前的mq容器，重新搭建集群
    
    ```Shell
     docker rm -f mq
    ```
    
## 2.3 **准备集群配置**
1. 在/tmp目录新建一个配置文件 rabbitmq.conf：
    
    ```Bash
     cd /tmp
     # 创建文件
     touch rabbitmq.conf
    ```
    
    文件内容如下：
    
    ```Bash
     loopback_users.guest = false
     listeners.tcp.default = 5672
     cluster_formation.peer_discovery_backend = rabbit_peer_discovery_classic_config
     cluster_formation.classic_config.nodes.1 = rabbit@mq1
     cluster_formation.classic_config.nodes.2 = rabbit@mq2
     cluster_formation.classic_config.nodes.3 = rabbit@mq3
    ```
    
1. 再创建一个文件，记录cookie
    
    ```Bash
     cd /tmp
     # 创建cookie文件
     touch .erlang.cookie
     # 写入cookie
     echo "FXZMCVGLBIXZCDEMMVZQ" > .erlang.cookie
     # 修改cookie文件的权限
     chmod 600 .erlang.cookie
    ```
    
1. 准备三个目录,mq1、mq2、mq3：
    
    ```Bash
     cd /tmp
     # 创建目录
     mkdir mq1 mq2 mq3
    ```
    
1. 然后拷贝rabbitmq.conf、cookie文件到mq1、mq2、mq3：
    
    ```Bash
     # 进入/tmp
     cd /tmp
     # 拷贝
     cp rabbitmq.conf mq1
     cp rabbitmq.conf mq2
     cp rabbitmq.conf mq3
     cp .erlang.cookie mq1
     cp .erlang.cookie mq2
     cp .erlang.cookie mq3
    ```
    
## 2.4 **启动集群**
1. 创建一个网络：
    
    ```Bash
     docker network create mq-net
    ```
    
1. 启动三个容器
    
    ```Bash
     docker run -d --net mq-net \
     -v ${PWD}/mq1/rabbitmq.conf:/etc/rabbitmq/rabbitmq.conf \
     -v ${PWD}/.erlang.cookie:/var/lib/rabbitmq/.erlang.cookie \
     -e RABBITMQ_DEFAULT_USER=itcast \
     -e RABBITMQ_DEFAULT_PASS=123321 \
     --name mq1 \
     --hostname mq1 \
     -p 8071:5672 \
     -p 8081:15672 \
     rabbitmq:3.8-management
    ```
    
    ```Bash
     docker run -d --net mq-net \
     -v ${PWD}/mq2/rabbitmq.conf:/etc/rabbitmq/rabbitmq.conf \
     -v ${PWD}/.erlang.cookie:/var/lib/rabbitmq/.erlang.cookie \
     -e RABBITMQ_DEFAULT_USER=itcast \
     -e RABBITMQ_DEFAULT_PASS=123321 \
     --name mq2 \
     --hostname mq2 \
     -p 8072:5672 \
     -p 8082:15672 \
     rabbitmq:3.8-management
    ```
    
    ```Bash
     docker run -d --net mq-net \
     -v ${PWD}/mq3/rabbitmq.conf:/etc/rabbitmq/rabbitmq.conf \
     -v ${PWD}/.erlang.cookie:/var/lib/rabbitmq/.erlang.cookie \
     -e RABBITMQ_DEFAULT_USER=itcast \
     -e RABBITMQ_DEFAULT_PASS=123321 \
     --name mq3 \
     --hostname mq3 \
     -p 8073:5672 \
     -p 8083:15672 \
     rabbitmq:3.8-management
    ```
