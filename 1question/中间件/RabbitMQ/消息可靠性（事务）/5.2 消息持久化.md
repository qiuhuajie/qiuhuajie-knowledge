> 💡 **防止消息列表丢失消息**
# 一、介绍
1. MQ **默认是**==**内存存储消息**==**，开启持久化功能可以确保缓存在MQ中的消息不丢失**
2. 持久化的交换机、队列会带有 `D`：

    ![[IMG-20260404031931058.png|800]]

# 二、实现消息持久化
1. **要想实现一条消息的持久化，就需要交换机、队列、消息**==**三者同时**==**做到持久化**
    1. 1️⃣声明一个==持久化的交换机==

    ```Java
    @Configuration
    public class CommonConfig {
        @Bean
        public DirectExchange simpleDirect(){
            return new DirectExchange("simple.direct",true,false);			//第二参数：是否持久化
        }
    }
    ```

    ![[IMG-20260404031931107.png|701]]

    2. 2️⃣声明一个==持久化的队列==

    ```Java
    @Configuration
    public class CommonConfig {
        @Bean
        public DirectExchange simpleDirect(){
            return new DirectExchange("simple.direct",true,false);
        }
        @Bean
        public Queue simpleQueue(){
            return QueueBuilder.durable("simple.queue").build();		//.durable()：声明持久化的队列
        }
    }
    ```

    ![[IMG-20260404031931227.png|437]]

    3. 3️⃣发送一条==持久化的消息==
        - 持久化的信息不再是一个简单的字符串了，而是一个 Message 实例

    ```Java
    @Test
    public void testDurableMessage(){
        //定义一个消息对象
        Message message = MessageBuilder
                .withBody("test DurableMessage".getBytes(StandardCharsets.UTF_8))   //设置消息体
                .setDeliveryMode(MessageDeliveryMode.PERSISTENT)                    //设置持久化
                .build();
        //发送消息
        rabbitTemplate.convertAndSend("simple.queue",message);
    }
    ```

        `2-xx`：表示持久化的消息

![[IMG-20260404031931258.png|347]]

2. **发送完消息后，将 mq 容器重启：（模拟 MQ 宕机）**

    ```Bash
    [root@docker002 ~]# docker restart 43d151cac90f
    43d151cac90f
    ```
3. 再次查看消息是否存在：持久化的消息依然存在
