# 一、消息堆积问题
1. **当**==**生产者发送消息的速度**====**超过了**====**消费者处理消息的速度**==
    1. **就会导致队列中的**==**消息堆积**==**，直到队列存储消息达到上限**
    2. 而那些**最早接收到的消息，可能就会成为死信，会被丢弃**，这就是消息堆积问题

    ![[Attachment/1question/中间件/RabbitMQ/IMG-20260405035438282.png|747]]

2. 解决消息堆积有三种种思路：
    1. **增加更多消费者，提高消费速度**，例如：
        - 消费者扩容，增加消费者的数量，来提高消费能力，尽快把积压的消息消费完
        - Queue 扩容：新建一个临时的 Topic（Exchange），设置一些新的 Queue，然后先用一些消费者把消费的数据转发（不用业务处理，很快）到临时的 Topic（Exchange），接下来用扩容的消费者去消费新的 Topic（Exchange） 里的数据，消费完了之后，恢复原状

    ![[IMG-20260405035448931.png|800]]

    2. **在消费者内开启线程池加快消息处理速度**（适合与大作业，否则线性上下文切换，导致资源浪费）
    3. **扩大队列容积，提高堆积上限**（要提升队列容积，把消息保存在内存中显然是不行的）
3. ==解决消息堆积问题的较好的解决方案：====**使用惰性队列（原理：提高队列的堆积上限）**==
    - 基于磁盘存储，**消息上限高**
    - 没有间歇性的 `page-out`，**性能比较稳定**

# 二、惰性队列
1. 从RabbitMQ的3.6.0版本开始，就增加了`Lazy Queues`的概念，也就是惰性队列
2. 惰性队列的特征如下：
    1. ==接收到消息后====**直接存入磁盘**====而非内存==
    2. 消费者要消费消息时才会从磁盘中读取并加载到内存
    3. 支持数百万条的消息存储
3. 对于一个已经存在的队列：可以通过命令行将一个运行中的队列修改为惰性队列：

    ```Java
     rabbitmqctl set_policy Lazy "^lazy-queue$" '{"queue-mode":"lazy"}' --apply-to queues
    ```

    ![[IMG-20260405035506677.png|800]]

4. 而对于要新声明队列，指定`x-queue-mode`属性为`lazy`即可，用SpringAMQP声明惰性队列分两种方式：
    1. @Bean的方式：

# 三、代码实现
## 1. 声明惰性队列
```Java
 @Configuration
 public class LazyQueue {
     @Bean
     public Queue myLazyQueue(){
         return QueueBuilder
                 .durable("lazy.queue")
                 .lazy()     //⭐
                 .build();
     }
     @Bean
     public Queue normalQueue(){
         return QueueBuilder
                 .durable("normal.queue")
                 .build();
     }
 }
```
## 2. 发送消息

写两个测试方法，分别向惰性队列和普通队列中发送10000条消息

```Java
 @Slf4j
 @RunWith(SpringRunner.class)
 @SpringBootTest
 public class SpringAmqpTest {
     @Test
     public void testLazyMessage1(){
         for (int i = 0; i < 1000000; i++) {
             //定义一个消息对象
             Message message = MessageBuilder
                     .withBody("test lazy Message".getBytes(StandardCharsets.UTF_8))   //设置消息体
                     .setDeliveryMode(MessageDeliveryMode.NON_PERSISTENT)
                     .build();
             //发送消息
             rabbitTemplate.convertAndSend("lazy.queue",message);
         }
     }
     @Test
     public void testLazyMessage2(){
         for (int i = 0; i < 1000000; i++) {
             //定义一个消息对象
             Message message = MessageBuilder
                     .withBody("test normal Message".getBytes(StandardCharsets.UTF_8))   //设置消息体
                     .setDeliveryMode(MessageDeliveryMode.NON_PERSISTENT)
                     .build();
             //发送消息
             rabbitTemplate.convertAndSend("normal.queue",message);
         }
     }
 }
```

# 四、惰性队列的优缺点
1. 惰性队列的==优点：==
    1. 基于磁盘存储，**消息上限高**
    2. 没有间歇性的`page-out`，**性能比较稳定**
2. 惰性队列的==缺点：==
    1. 基于磁盘存储，消息时效性会降低
    2. 性能受限于磁盘的`IO`
