# 一、正向代理
- 类似于 VPN
- 客户端将**请求转发给代理服务器**，代理服务器再负责转发给目标网站
- 响应时**服务端先响应给代理服务器**，代理服务器再转发给对应的客户端
- 正向代理可以但不限于为局域网内客户端做代理，同样可以选择公网服务器作为代理，它扮演的角色类似于 NAT

    ![[IMG-20260620224140032.png|750]]

# 二、反向代理
## 1. 概念
1. ==与正向代理其实只是看代理的角度不同，==**==反向代理就是反方向（在服务器端）提供代理服务器==**

    ![[IMG-20260620224145876.png|800]]

2. 代理服务器==本质上就是一个====**网关**==，提供服务的服务器集群被分成不同的组，每个组使用各自的网关让外界进行访问
    - 请求：
        - 反向代理是为服务端转发请求
        - **客户端将请求发送至 Nginx 反向代理服务器**，反向代理服务器再将请求转发给真正的服务器以处理请求
    - 响应：
        - 响应时后端真正的服务器**将处理结果先发送给 Nginx 反向代理**
        - 再由反向代理构建响应并响应给客户端
3. **LVS（Linux Virtual Server）**
    1. 请求过代理服务器，但是**响应的数据不再经过代理服务器，直接由目标服务器发送给用户**
    2. LVS 就是这种模式的一个专业反向代理软件，但 LVS 功能较简单，直接被内置在CentOS系统中

## 2. 使用演示
1. 配置项：**`server.location.proxy_pass`**：配置代理的主机名或域名
2. 示例：使用 nginx 服务器，代理 百度官网

    ![[IMG-20260620224156123.png|800]]

    1. 修改配置文件

    ```Plain
     worker_processes  1;
     pid        logs/nginx.pid;
     events {
         worker_connections  1024;
     }
     http {
         include       mime.types;
         default_type  application/octet-stream;
         sendfile        on;
         keepalive_timeout  65;
         server {
             listen       80;
             server_name  localhost;
             location / {
                 # ⭐
                 proxy_pass http://www.baidu.com;
                 # root   /testNginx/www/qhjqhj;
                 # index  index.html index.htm;
             }
             error_page   500 502 503 504  /50x.html;
             location = /50x.html {
                 root   html;
             }
         }
     }
    ```
    2. 重新加载服务

    ```Plain
     [root@nginx conf]# systemctl reload nginx
    ```
    3. 测试访问：[http://192.168.10.191:80/](http://192.168.10.191/)
        - 可以看到地址栏仍然是 nginx 代理服务器的IP

    ![[IMG-20260620224203277.png|800]]

        - 查看此次请求状态

    ![[IMG-20260620224209072.png|677]]

        - 响应服务器也是 nginx 代理服务器，说明确实做到了代理

    ![[IMG-20260620224213798.png|346]]

    4. **注意**：
        1. 不可以写成：**`proxy_pass http://baidu.com;`**
            1. 这种情况下，代理服务器会将请求重定向：**重定向后 地址栏也不再是 nginx代理服务器的IP，也就意味着，接下来的页面操作根本不会再由 nginx 做代理了**，这个违背了使用 nginx 的目的，我们目的就是要让所有请求都被 nginx 服务器代理
            2. 查看请求：
                1. [http://192.168.10.191:80/](http://192.168.10.191/) 请求的响应服务器是nginx，但是请求返回的是 302 请求重定向

    ![[IMG-20260620224216959.png|644]]

                2. 重定向后的请求即 百度的域名

    ![[IMG-20260620224218882.png|641]]

        2. 且不支持 https ：**`proxy_pass https://www.baidu.com;`**（这种情况下，nginx 加载配置时会报错）

    ```Plain
     [root@nginx conf]# systemctl reload nginx
     Job for nginx.service failed because the control process exited with error code. See "systemctl status nginx.service" and "journalctl -xe" for details.
    ```
