# 1. **介绍**
1. **作用：**伪装请求 /itemServer?id=1001 ➡️ /item/1001.html
1. 语法格式：
    
    1. `**rewrite <regex> <replacement> [flag];**`
    
    1. `rewrite` 是实现URL重写的关键指令，根据 `regex` (正则表达式)部分内容，重定向到 `replacement`，结尾是 `flag` 标记
    
1. flag 标记说明：
    
    1. `last`：本条规则匹配完成后，继续向下匹配新的location URI规则
    
    1. `break` ：本条规则匹配完成即终止，不再匹配后面的任何规则
    
    1. `redirect`：返回302临时重定向，浏览器地址会显示跳转后的URL地址
    
    1. `permanent`：返回301永久重定向，浏览器地址栏会显示跳转后的URL地址
    
# 2. **示例**
1. 示例一：将 ^2.html 伪装成 /index.jsp?pageNum=2
    
    ```Plain
     server {
         listen       80;
         server_name  localhost;
         location / {
             # ⭐
             rewrite ^2.html     /index.jsp?pageNum=2    break;
             proxy_pass http://192.168.10.192:80;
         }
     }
    ```
    
1. 示例二：使用正则表达式
    
    ```Plain
     server {
         listen       80;
         server_name  localhost;
         location / {
             # ⭐
             rewrite ^/([0-9]+).html$        /index.jsp?pageNum=$1   break;
             # [0-9]+: 匹配多个数字
             # $: 正则结束符
    
             # replacement中的$1:匹配第一个正则规则
             proxy_pass http://192.168.10.192:80;
         }
     }
    ```
    
# 3. **URL Rewrite + 负载均衡**
1. 可以先将 nginx 代理服务器升级为一个网关服务器
    
    1. 此时 http://192.168.10.192:80/ 、 http://192.168.10.191:80/ 都能访问服务
    
    1. 想要的网关服务器效果：
        
        ![[IMG-20260404031924368.png|Untitled 248.png]]
        
        - 即直接访问服务 [http://192.168.10.192:80/](http://192.168.10.192/) 不能通
        
        - **只有通过 nginx 代理（网关）服务器这一条路 [http://192.168.10.191:80/](http://192.168.10.191/) ==（唯一大门）==**
        
        - 同时，==**在 URL Rewrite 上使用 负载均衡**==
        
    
1. 示例：
    
    1. 开启防火墙
        
        ```Plain

         [root@nginx02 html]# systemctl status firewalld

        ```
        
    
    1. 此时由于端口被关闭， [http://192.168.10.192:80/](http://192.168.10.192/) 、 [http://192.168.10.191:80/](http://192.168.10.191/) 都无法访问
    
    1. 将==**特定的端口和 IP 的访问开放**==
        
        ```Plain
         [root@nginx02 html]# firewall-cmd --permanent --add-rich-rule="rule family="ipv4" source address="192.168.10.191" port protocol="tcp" port="80" accept"
         success
        ```
        
    
    1. 重启防火墙
        
        ```Plain
         [root@nginx02 html]# systemctl restart firewalld
        ```
        
    
    1. 查看添加好的规则
        
        ```Plain
         [root@nginx02 html]# firewall-cmd --list-all
         public (active)
           target: default
           icmp-block-inversion: no

           protocols:

           icmp-blocks:
           rich rules:
                 rule family="ipv4" source address="192.168.10.191" port port="80" protocol="tcp" accept
        ```
        
    
    1. 此时直接访问 [http://192.168.10.192:80/](http://192.168.10.192/) 仍不能访问，但是 访问 [http://192.168.10.191:80/](http://192.168.10.191/) 可以（唯一网关入口）
    
    1. **在 rewrite 上使用 负载均衡**
        
        ```Plain
         upstream httpds {
             server 192.168.44.102 weight=8 down;
             server 192.168.44.103:8080 weight=2;
             server 192.168.44.104:8080 weight=1 backup;
         }
         location / {
             rewrite ^/([0-9]+).html$ /index.jsp?pageNum=$1 redirect;
             proxy_pass http://httpds ;
         }
        ```
