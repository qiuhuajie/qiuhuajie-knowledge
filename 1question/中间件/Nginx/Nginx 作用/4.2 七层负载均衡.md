# 二、Nginx 负载均衡算法
## 1. 轮询算法
1. 示例说明：
    1. 再开两台虚拟机192、193 作为模拟业务服务器，但为了简单也是跑的nginx服务
    2. 191 作为负载均衡器，将请求按一定算法（当前是轮询）打到两个服务器上

    ![[IMG-20260620224140034.png|800]]
2. 修改配置文件

    ```Plain
     worker_processes  1;
     pid        logs/nginx.pid;
     events {
         worker_connections  1024;
     }
    # ⭐ 7 层 http 负载
     http {
         include       mime.types;
         default_type  application/octet-stream;
         sendfile        on;
         keepalive_timeout  65;
         # ⭐ 当前是 RR
         upstream httpds{
             server 192.168.10.192:80;
             server 192.168.10.193:80;
         }
         server {
             listen       80;
             server_name  localhost;
             location / {
                 # ⭐ 会去找上面定义好的 httpds 组
                 proxy_pass http://httpds;
             }
             error_page   500 502 503 504  /50x.html;
             location = /50x.html {
                 root   html;
             }
         }
     }
    ```
3. 重新加载配置
4. 测试访问

    [http://192.168.10.191/](http://192.168.10.191/) 192、193 交替出现
![[IMG-20260404031923925.png|396]]
![[IMG-20260620224146037.png|394]]
## 2. 加权轮询
1. 示例说明：
    1. 给 192、193 不同的负载均衡权重 weight
    2. 即可以实现让硬件资源好的服务器承担更多的服务请求
2. 添加权重

    ```Plain
     worker_processes  1;
     pid        logs/nginx.pid;
     events {
         worker_connections  1024;
     }
     http {
         include       mime.types;
         default_type  application/octet-stream;
         sendfile        on;
         keepalive_timeout  65;
         upstream httpds{
             server 192.168.10.192:80 weight=8;     # ⭐ 当前是 加权RR
             server 192.168.10.193:80 weight=2;
         }
         server {
             listen       80;
             server_name  localhost;
             location / {
                 proxy_pass http://httpds;
             }
             error_page   500 502 503 504  /50x.html;
             location = /50x.html {
                 root   html;
             }
         }
     }
    ```
3. 重新加载配置
4. 测试访问
5. 可以设置一些注释：
    1. **`weight`**：负载均衡权重

    ```Plain
     upstream httpds{
         server 192.168.10.192:80 weight=8;
         server 192.168.10.193:80 weight=2;
     }
    ```
    2. **`down`**：让注释了的主机直接下线

    ```Plain
     upstream httpds{
         server 192.168.10.192:80 weight=8 down;
         server 192.168.10.193:80 weight=2;
     }
    ```
    3. **`backup`**：当192宕机时，backup193 就会顶上，但是192 正常时，backup193 并不会提供服务

    ```Plain
     upstream httpds{
         server 192.168.10.192:80 weight=8;
         server 192.168.10.193:80 weight=2 backup;
     }
    ```
## 3. 其他算法
- 除了上面的 轮询 和 加权轮询，还有其他的一些负载均衡策略（存在一些问题，故都不常用）
    - ip_hash：根据客户端的ip地址转发同一台服务器，可以保持会话（存在 IP 漂移，会话保持失败）
    - least_conn：最少连接访问
    - url_hash：根据用户访问的url定向转发请求，适用于访问固定的资源
    - fair：根据后端服务器响应时间转发请求
    - 另外可以使用
        - 哈希算法：解决集群模式下的负载均衡
        - 一致性哈希算法：解决分布式系统下的负载均衡 [[哈希表]]
- **实际生产中更多的是使用 LUA 脚本自定义负载均衡规则**
