# 签证 Agent 个人奖项提名

> [!note]
> 本提名基于 [[签证Agent]] 整理，默认按“项目核心负责人 / 核心建设者”口径撰写。以下数据均为脱敏后的阶段性结果，可直接用于内部提报。

## 建议提报方向

从现有材料看，==更推荐提报 `Top Agent Engineer`==。

原因有三点：

- 项目具备从 0 到 1 的完整建设过程，而不是局部优化。
- 有明确的 Before / After 对比，能够支撑“结构性提效”和“结构性降本”的表达。
- 现有文档已经包含产品效果图、系统架构图、评测闭环图和结果数据，适合直接配图提报。

若你所在场景更强调业务服务价值，也可以同步准备 `TOP Sales / TOP Service` 版本。

## 版本 A：Top Agent Engineer

### 提报标题

从 0 到 1 搭建签证 Agent，重构“咨询-准备-审核-结果校验”全链路，实现结构性提效与降本

### 一句话概述

我主导建设签证 Agent，从售前咨询到售后材料预审完成端到端落地，解决传统签证链路强依赖人工、响应慢、审核拥堵、规则迭代难的问题，让签证服务从“人海战术”升级为“AI 承接 + 人工兜底”的新模式。

### Before & After

| Before | 我的动作 | After |
|---|---|---|
| 传统签证流程强依赖人工，整体办理链路通常需要 3-5 天；简单咨询响应 10 分钟以上，材料准备平均耗时超 2 小时，审核阶段常见 T+2 天 | 从 0 到 1 设计并落地签证 Agent，覆盖售前咨询、政策问答、进度查询、材料智能上传、材料预审、出签结果校验 | 将关键环节前置自动化，把人工角色从“全量处理”转为“异常兜底”和关键决策，重塑全链路时效体验 |
| 早期 Agent 仅靠简单 Prompt 和无状态对话，复杂场景下容易答非所问、上下文漂移、召回不准 | 设计动态 Few-Shot、状态化会话上下文、Query 改写、Query 拆解 + 多路召回、Rerank + 后置过滤 | 复杂签证咨询场景的连贯性、准确性和泛化能力明显提升，支持真实多轮对话而非 demo 式问答 |
| 多审核点耦合在一次模型调用里，准确率不可控；Prompt 硬编码，更新慢、回滚难 | 设计四层审核架构，拆分子场景并行调度；将 Prompt 配置化、版本化、可灰度发布；补齐全链路追溯 | 审核系统从“不可控黑盒”升级为“可配置、可回放、可审计”的工程化系统 |
| 上线前后缺少稳定评测与数据闭环，线上 bad case 难发现、难复现 | 搭建“线上运行 -> 自动标注 -> 评测发现缺陷 -> 模型/规则更新 -> 回放验证”的评测飞轮 | 形成持续迭代机制，并通过多模型投票 + 置信度过滤 + 人工兜底，把人工标注依赖度目标降低 50%+ |
| 响应速度慢，澄清场景约 8 秒，政策咨询正文 15-17 秒，用户等待焦虑明显 | 推动流程并行、小模型分流、动态思维链反馈、流式输出截断等优化 | 澄清场景平均响应时间压缩到 4-5 秒，政策咨询正文压缩到 8-9 秒 |

### 我的关键贡献

- 负责签证 Agent 的整体方案设计，明确从“咨询 Agent”扩展到“材料预审 Agent”的双轮驱动方向。
- 将签证这种高复杂、低容错业务拆成可工程化治理的问题：数据治理、Prompt 具象化、泛化能力增强、评测闭环、系统可靠性。
- 主导搭建签证行业知识库，打通使领馆数据、客服聊天记录、第三方攻略等多源异构数据，支撑 RAG 能力落地。
- 设计并落地关键能力组件，包括动态 Few-Shot、状态化上下文、Query 重写、Query 拆解、多路召回、Rerank、Prompt 配置中心、任务状态机和全链路观测。
- 将材料审核从单次大模型“大而全判断”重构为“多子场景并行调度 + 规则融合”的架构，解决准确率、扩展性和性能三者无法兼得的问题。
- 推动评测体系前置建设，并把线上真实流量接入异步诊断和数据回流，建立 AI 应用可持续优化的基础设施。

### 结果数据（脱敏）

- 售前 AI 办签咨询全量后，用户 UV `1.9k+`，DAU `400+`，交互率 `98.56%`。
- AI 材料批量分类累计处理材料 `2.8w+`，分类准确率 `98.91%`。
- AI 单项材料审核已覆盖 `15+` 个校验点。
- 护照审核累计 `25w+`，预审拦截 `4.1w+`，拦截率 `17.13%`。
- 证件照审核累计 `14w+`，预审拦截率 `7.71%`。
- 出签结果 AI 审核累计 `1.5w+`，拦截率 `0.90%`。
- 澄清场景平均响应时间从 `8s` 压缩到 `4-5s`。
- 政策咨询正文链路平均响应时间从 `15-17s` 压缩到 `8-9s`。
- 多模型投票 + 自动化标注链路上线后，人工标注依赖度目标下降 `50%+`。

### 可直接用于提报表的精简版

我主导从 0 到 1 建设签证 Agent，围绕“咨询、材料准备、审核、结果校验”完成全链路 AI 化改造，解决了签证业务长期强依赖人工、规则复杂、时效慢、扩展难的问题。项目不是单点功能尝试，而是完整的 Agent 工程体系建设：前台落地售前咨询、政策问答、进度查询和材料预审，后台补齐知识库、状态化上下文、动态 Few-Shot、Query 重写与召回、Prompt 配置化、评测闭环和审核状态机，真正让业务从 demo 走向可规模化运行。

落地结果上，售前咨询 C 端全量后，脱敏口径下 UV 达 `1.9k+`、DAU `400+`、交互率 `98.56%`；材料分类累计 `2.8w+`、准确率 `98.91%`；护照审核累计 `25w+`、预审拦截 `4.1w+`；澄清场景响应时间从 `8s` 压缩到 `4-5s`，政策咨询从 `15-17s` 压缩到 `8-9s`。更重要的是，我把原本随人力线性扩张的签证服务链路，改造成“AI 承接高频标准问题 + 人工兜底异常问题”的新模式，形成了结构性提效与降本。

## 版本 B：TOP Sales / TOP Service

### 提报标题

用签证 Agent 重做用户服务体验，让签证咨询和材料预审从“排队等待”变成“实时承接”

### 高光故事

签证服务是一个典型的高复杂、强人工依赖场景。用户往往只想问一句“我能不能办”“需要准备什么”“我的签证到哪一步了”，但传统模式下却需要经历长时间排队、反复补材料和多轮人工沟通。面对这种“秒级需求、天级响应”的落差，我推动上线签证 Agent，把售前咨询、政策问答、进度查询和材料预审串成一条连续服务链路，让用户从第一次咨询开始就能得到实时反馈，也让商家从大量重复性事务里解放出来，聚焦高价值决策与兜底服务。

这个项目的高光不只在于“做了一个 AI 功能”，而在于真正把服务体验改写成了可以规模化承接的产品能力。用户在 CHAT 中可以直接表达办签需求，系统会自动识别国家、签证类型、出行时间和身份信息，给出商品推荐、材料说明和进度查询；在售后阶段，用户上传材料后，系统还能自动分类、自动预审并拦截明显问题，把原本要靠人工来回沟通的工作前置解决，显著减少等待、返工和盲等。

### 业绩结果数据（脱敏）

- 售前 AI 办签咨询 C 端全量后，用户 UV `1.9k+`，DAU `400+`，交互率 `98.56%`。
- 三类核心办签场景已形成稳定承接：指令办签、政策咨询、进度查询。
- AI 材料智能分类累计处理材料 `2.8w+`，分类准确率 `98.91%`。
- 护照审核累计 `25w+`，预审拦截 `4.1w+`，拦截率 `17.13%`。
- 证件照审核累计 `14w+`，预审拦截率 `7.71%`。
- 出签结果 AI 审核累计 `1.5w+`，拦截率 `0.90%`。
- 业务响应时效显著提升：澄清场景从 `8s` 优化到 `4-5s`，政策咨询从 `15-17s` 优化到 `8-9s`。
- 对商家侧而言，服务模式从“人海战术”升级为“AI 优先承接 + 人工兜底”，具备显著的结构性降本潜力。

### 可直接用于提报表的精简版

我推动签证 Agent 在售前咨询和售后预审场景落地，把原本高度依赖人工的签证服务改造成实时、连续、可规模化承接的智能服务链路。项目上线后，AI 不仅能完成国家政策咨询、材料说明和进度查询，还能在材料上传环节自动分类、自动预审，把大量重复问答和初筛动作前置自动化，显著改善用户等待和返工体验。
