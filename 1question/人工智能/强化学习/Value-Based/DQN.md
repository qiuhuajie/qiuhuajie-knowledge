# 1. Q 表的缺陷
1. 表格型方法存储的状态数量有限，当面对围棋或机器人控制这类有数不清的状态的环境时，表格型方法在存储和查找效率上都受局限
    

    ![[IMG-20260321173718239.png|800]]

2. `DQN` 使用**值函数近似**解决了这一局限
    1. 使用神经网络来近似替代 `Q` 表格
    2. Q 表格缺点：
        1. 表格可能占用极大内存
        2. 当表格极大时，查表效率低下
    3. 值函数近似：使用一个带参数的 Q 函数来近似这个 Q 表格
        

        ![[IMG-20260404032007607.png|800]]

    4. 优点：
        1. 仅需存储有限的参数
        2. 状态泛化，相似的状态可以输出一样

# 2. DQN
1. 本质上 `DQN` 还是一个 `Q-learning` 算法，更新方式一致
    

    ![[IMG-20260404032007908.png|800]]

    ![[IMG-20260404032007965.png|800]]

2. 为了更好的探索环境，同样的也采用 `ε-greedy` 方法训练
3. **DQN $\approx$ Q-learning + 神经网络**
    

    ![[IMG-20260405035358080.png|800]]

    1. 监督学习中
        1. 输入一批 X，输出 Y 的预测值 $\widehat{Y}$，将预测值和 Y 的真实值 $Y$ 做对比，希望预测值能尽可能得逼近真实值
        2. 通过求 $\widehat{Y}$ 与 $Y$ 的均方差，也就是 Loss，再将 Loss 送进优化函数中，就可以对这个网络自动地做更新和优化
    2. DQN 中
        1. 输入是一批状态，输出是对应的 Q 值，每个状态对应的输出 Q 值是一个向量，向量长度为动作维度，代表做什么动作
        2. 这里的真实值也就是目标值，计算方式就是 Q-learning 中的目标值的计算方法：$r+ γmax_{a^{'}}\widehat{q}(s^{'},a^{'},w)$
        3. 再计算输出 Q 值与目标 Q 值的均方差，作为 Loss，送入优化函数，进而自动更新网络
4. 在 Q-learning 的基础上，DQN 提出了两个技巧使得 Q 网络的更新迭代更稳定
    1. ==**经验回放 Experience Replay：**==
        1. Q-Learning 它是有一张Q表来保存所有的Q值的当前结果的，但是DQN是没有的，那么在做动作价值函数更新的时候，就需要其他的方法，这个方法就是经验回放
        2. 主要解决样本关联性和利用效率的问题
            1. 监督学习中输入的样本一般是独立的
            2. 而强化学习是序列决策问题，输入的样本状态前后可能是相互关联的，所以需要切断输入样本之间的联系
        3. 使用一个经验池存储多条经验 $s,a,r,s'$，再从中随机抽取一批数据送去训练，将与环境交互的经验存储起来重复利用
            

            ![[IMG-20260405035358576.png|800]]

        4. 代码
            

            ![[IMG-20260405035359347.png|800]]

    2. ==**固定Q目标 Fixed-Q-Target：**==
        1. 主要解决算法训练不稳定的问题
            1. 监督学习中输出预测值 $\widehat{Y}$ 逼近真实值 $Y$ ，真实值是稳定的
            2. 但在 DQN 中真实值 $Y$ （目标 Q 值）是不稳定的
            3. 因此，DQN 复制一个和原来Q网络结构一样的 Target Q 网络，在一段时间使用一个固定的 Q 目标值，同时，隔一段时间将网络参数同步到 Target Q 网络
            

    ![[IMG-20260405035400334.png|800]]

    4. 然后**将下一状态 $s_{t+1}$ 输入到 Target Q 网络中**，得到各种不同的动作的 Q 值

# 3. DDQN
> [!info] 强化学习（十）Double DQN (DDQN) - 刘建平Pinard - 博客园  
> 在强化学习（九）Deep Q-Learning进阶之Nature DQN中，我们讨论了Nature DQN的算法流程，它通过使用两个相同的神经网络，以解决数据样本和网络训练之前的相关性。但是还是有其他  
> [https://www.cnblogs.com/pinard/p/9778063.html](https://www.cnblogs.com/pinard/p/9778063.html)  
1. DQN 算法在深度强化学习领域取得了不俗的成绩，**不过其并不能保证一直收敛**，研究表明这种估计目标价值的算法过于乐观的高估了一些情况下的行为价值，导致算法会将次优行为价值一致认为最优行为价值，最终不能收敛至最佳价值函数
2. 一种使用双价值网络的 DDQN(double deep Q network) 被认为较好地解决了这个问题
3. DDQN 算法的网络结构与 DQN 相同，也有一样的两个 Q 网络结构
4. **DDQN 通过解耦目标 Q 值动作的选择和目标 Q 值的计算这两步，来消除过度估计的问题**
    1. 在DDQN这里，不再是直接在目标Q网络里面找各个动作中最大Q值，而是先在当前Q网络中先找出最大Q值对应的动作
    2. 然后利用这个选择出来的动作在目标网络里面去计算目标Q值
5. **Q 网络的更新流程**
    1. DDQN 与 DQN 大部分都相同，只有一步不同，那就是**⭐计算 Q 真实值（$Y_t$）时的方法不同**：见下面两个公式
    2. **==DQN 是将 S_t 输入 Q 网络直接选择 Q 网络的最大输出动作值==**
        

        ![[IMG-20260404032008554.png|347]]

    3. 而 DDQN不同
        

        ![[IMG-20260404032008649.png|486]]

        1. **==DDQN 首先将 S_t+1 输入 Q 网络，再从输出的结果中找到最大输出值的那个动作==**
        2. **==然后再找到这个动作对应的 Target Q 网络的输出值==**
    4. 用图像表示即为
        

        ![[IMG-20260404032008700.png|720]]

    5. 红色字母代表已知项
    6. 如上图所示，首先是根据 $s_t$ 和 $a_t$ 是可以确定当前的 $Q(s_t,a_t)$ 值的，即 $Q(s_t,a_1)$ 
    7. 然后**将下一状态 $s_{t+1}$ 输入到 Q 网络中**，得到各种不同的动作的 Q 值
    8. 接下来选择最大的 Q 值对应的动作，这里假设选择了动作 $a_1$
    9. 接着**将状态 $s_{t+1}$ 输入到 Target Q 网络中**，找到动作 $a_1$ 对应的 $Q$ 值 $Q(s_t,a_1)$ 
    10. 最后，以 $Q(s_t,a_1)$ 作为网络的预测值，而 $r_{t+1}+\gamma Q(s_{t+1},a_1)$ 作为网络的实际值，进行误差反向传播。Loss 函数可以选择方差等，把它当做一个有监督学习即可
    11. 过一段时间后，将Q网络中的参数**硬拷贝**到Target Q网络中
    
# 4. **Prioritized Replay DQN**
> [!info] 强化学习(十一) Prioritized Replay DQN - 刘建平Pinard - 博客园  
> 在强化学习（十）Double DQN (DDQN)中，我们讲到了DDQN使用两个Q网络，用当前Q网络计算最大Q值对应的动作，用目标Q网络计算这个最大动作对应的目标Q值，进而消除贪婪法带来的偏差。今天我  
> [https://www.cnblogs.com/pinard/p/9797695.html](https://www.cnblogs.com/pinard/p/9797695.html)  
1. Prioritized Replay DQN **根据每个样本的 TD 误差绝对值 $|δ(t)|$ ，给定该样本的优先级正比于 $|δ(t)|$ ，将这个**==**优先级**==**的值存入经验回放池**
2. 之前的DQN算法，我们仅仅只保存和环境交互得到的样本状态，动作，奖励等数据，没有优先级这个说法
3. 由于引入了经验回放的优先级，那么Prioritized Replay DQN 的经验回放池和之前的其他 DQN 算法的经验回放池就不一样了
    1. 因为这个**优先级大小会影响它被采样的概率**
    2. 在实际使用中，我们通常使用SumTree这样的二叉树结构来做我们的带优先级的经验回放池样本的存储
    
# 5. Dueling DQN
1. 在 Dueling DQN 中，尝试通过优化神经网络的结构来优化算法
    1. Dueling DQN考虑**将 Q 网络分成两部分**
        1. 第一部分是仅仅与状态 $S$ 有关，与具体要采用的动作 $A$ 无关，这部分我们叫做价值函数部分，记做 $V(S,w,\alpha)$
        2. 第二部分同时与状态状态 $S$ 和动作 $A$ 有关，这部分叫做优势函数（Advantage Function）部分,记为 $A(S,A,w,\beta)$
    2. 最终 Q 网络的输出由价格函数网络的输出和优势函数网络的输出**线性组合**得到
        

        $$Q(S,A,w,\alpha,\beta)=V(S,w,\alpha)+A(S,A,w,\beta)$$

    3. 其中，$w$ 是公共部分的网络参数，而 $\alpha$ 是价值函数独有部分的网络参数，而 $\beta$ 是优势函数独有部分的网络参数
2. **Dueling DQN 网络结构**
    1. 由于Q网络的价值函数被分为两部分，因此Dueling DQN的网络结构也和之前的DQN不同
    2. ==Dueling DQN在结构上与前半部分的原始DQN一致。在输出端，Dueling DQN 并不直接在卷积层之后连接一个全连通层，而是将输出映射到两个全连通层==
    3. 这两个完全连接的层将分别评估动作和状态的价值和优势
        

        ![[IMG-20260404032008753.png|800]]

3. 可以直接使用上面的的价值函数的组合公式得到我们的动作价值
    1. 但是这个式子无法辨识最终输出里面 $V(S,w,\alpha)$ 和 $A(S,A,w,\beta)$ 各自的作用
    2. 为了可以体现这种可辨识性，实际使用的组合公式如下：
        

        $$Q(S,A,w,\alpha,\beta)=V(S,w,\alpha)+(A(S,A,w,\beta)-\frac{1}{|A|}\sum_{a^{'}\in |A|} A(S,a^{'},w,\beta))$$

    3. 其实就是对优势函数部分做了中心化的处理
