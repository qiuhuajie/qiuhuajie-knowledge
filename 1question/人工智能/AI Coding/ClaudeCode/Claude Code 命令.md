# Summary

> [!note] Claude Code 隐藏命令极大提升使用体验，尤其是长任务和实验场景
> - /btw 允许并行提问不污染上下文，任务继续运行，回答后可直接清除。
> - /rewind 支持分别回退代码或对话，适合试错后保留有用历史。
> - /insights 每月生成使用习惯分析报告并推荐优化。
> - /model opusplan 让 Pro 用户在规划用 Opus、执行用 Sonnet，兼顾质量与额度。
> - /simplify 同时启动三个 Agent 审查代码复用、质量、效率，优于旧 /review。
> - /branch 分叉当前会话尝试不同方向，相当于平行宇宙。
> - /remote-control 生成 URL 让手机实时遥控终端会话，安全同步。
> - /export 快速把整段对话存为 Markdown，便于后续参考或跨工具协作。
> - 常用快捷键：Ctrl+V 直接粘贴截图、Ctrl+J 换行、Ctrl+R 搜索历史 Prompt。

# /btw

## 背景

1. /btw 这个命令是今年 3 月 11 号才出的。
2. Claude Code 负责人 Thariq 在 Twitter 上发了个帖子介绍，直接两百多万的阅读量，你就知道大家有多需要这个功能了。
	![[IMG-20260322195849663.jpg|396]]

## 怎么使用

1. /btw 是干嘛的呢，它让你在 Claude 正在干活的时候插一个问题进去，但这个问题不会被加入对话历史。
2. 以前你让 Claude Code 重构一个大模块，干着干着你突然想到一个问题，比如"诶那个测试文件在哪个目录来着"，你一问，Claude 停下来回答你，然后上下文窗口里就多了一段完全不相关的对话，它重新开始干活的时候可能就跑偏了。
3. 这就是所谓的上下文污染，用 Claude Code 时间长的人应该都被坑过。
4. 要不然就是呆呆的，等到整个任务跑完，再去问，但是其实也是污染。
5. 现在打 /btw，然后说一个问题，比如我在执行过程中，突然想知道现在我这个项目的抓取流程是啥，我就可以直接敲 /btw。
6. 你在敲一下空格，就可以直接在后面写问题了，然后发送。
	![[IMG-20260404032002917.jpg|608]]
7. 回答完以后，哦我知道了，那这段就没用了，你就可以按空格或者回车，直接把这一段消除掉。
	![[IMG-20260404032002979.jpg|642]]

## 使用效果

1. 这个回答，完全不会中断你之前发送的任务，上下两个进程，是纯粹的并行状态。
2. 之前的 Claude 的任务还是该干嘛继续干嘛，对话历史里干干净净，跟什么都没发生一样。
3. 而且几乎不费 token，因为复用了当前的提示缓存。
4. 究极好用，真的，我现在每个长会话里都会用好几次，属于用了就回不去的那种。

# /rewind

## 背景

1. /rewind，也就是按两下 Esc，可以把它理解成撤销或者回退，也就是很多设计软件里面的 Ctrl+Z。
2. 这个命令之前就有，但 2 月的时候升级了一个关键能力，就是代码和对话可以分别回退。
	![[IMG-20260404032003054.jpg|459]]

## 怎么使用

1. 以前你可能跟 Claude 说"比如试试这种写法"，改完发现不行，只能整段对话一起回退，连你之前的讨论也没了。
2. 而现在，打 /rewind 会弹出菜单让你选，是只回退代码还是只回退对话。
3. 比如我现在开发了一个功能，觉得不好想撤销，就可以先打 /rewind，然后把会话里面的历史拽出来。
4. 点击之前的会话，就会弹出一个菜单。
	![[IMG-20260404032003120.jpg|577]]
5. 菜单分别是回退代码和对话、回退对话但保留代码、回退代码但保留对话、从该点开始的对话压缩释放上下文窗口空间。

## 使用效果

1. 这就特别适合做实验，让 Claude 试一种新方案，不行的话代码回退，对话留着，这样 Claude 还记得你刚才聊了什么，知道这条路不通，可以直接换方向，不用重新解释一遍需求。
2. 真的，以前经常含泪 git reset，我又没那么懂，搞得乱七八糟。
3. 现在 /rewind 就回来了，贼好用，推荐大家一定要用。

# /insights

## 背景

1. /insights 这个命令我觉得被严重低估了。
2. 它会生成一份 HTML 报告，分析你过去一个月使用 Claude Code 的习惯，包括你最常用哪些命令，你有哪些重复性的操作模式，然后给你推荐一些自定义命令和 Skills。
	![[IMG-20260404032003180.jpg|336]]

## 怎么使用

1. 其实就是 Claude Code 在反向观察你。
2. 你直接打这个命令以后，它就会给你一份做好的本地 HTML 网页。
	![[IMG-20260404032003235.jpg|613]]

## 使用效果

1. 做得非常详细，有翻车现场，还推荐我改一下记忆，建议我怎么用。
	![[IMG-20260405214031442.jpg|736]]
2. 当时还有个抽象的事，他给我改服务器的链接然后记下来了，这种小事都记着了，很有趣。
	![[IMG-20260405214031504.jpg|518]]
3. 真的建议大家每个月跑一次 /insights，它会让你重新认识自己的一些习惯，非常有意思。

# /model Opusplan

## 背景

1. 这个功能我觉得对 20 刀的 Pro 订阅用户来说特别重要，就是 /model opusplan。
2. 它会在需要复杂推理时自动以 plan 模式使用 Claude Opus 4.6，然后切换到 Claude Sonnet 4.6 进行执行。
	![[IMG-20260405214031643.jpg|572]]

## 怎么使用

1. 而且这是正儿八经的隐藏命令，你直接 /model 切换模型，可是没有这个模式的。
	![[IMG-20260405214107512.jpg|617]]
2. 道理很简单，Pro 用户的 Opus 额度是有限的，真的很少，现在我都是 Max 才够用。
3. 你 20 刀的 Pro 会员，全程用 Opus 写代码，可能干到一半就开始被限速了，但规划和写代码对模型能力的要求是完全不一样的。

## 使用效果

1. 规划需要深度思考，需要理解整个项目的架构和各种依赖关系，这种事情 Opus 确实比 Sonnet 强很多。
2. 但到了具体写代码的环节，其实一些小的项目，Sonnet 真的完全够用了，而且快得多。
3. 如果你是那种确实只开 20 刀 Pro 会员的轻度用户，或者确实要省一些钱，那我真的建议开 /model opusplan 这个，真的，两全其美。
4. 非常牛逼，这对每个月 20 美刀 Pro 订阅的用户来说简直是福音。

# /simplify

## 背景

1. 这是今年 2 月底才出的内置 Skill，Boris 之前发了个帖子介绍说他自己每天都在用。
	![[IMG-20260405214107574.jpg|389]]
2. 但是我记得好像之前 1 月就开源过了，只不过是 2 月底才集成到 Claude Code 里面去的。

## 怎么使用

1. /simplify 可以理解成一个三合一的代码审查工具，本质上其实是个 Skills。
2. 你输入 /simplify 之后，Claude Code 会同时启动三个平行的 Agent，分别从代码复用、代码质量、运行效率三个角度审查你的改动。
	![[IMG-20260405214107672.jpg|597]]

## 使用效果

1. 然后汇总结果告诉你哪些地方可以优化，我是因为之前跑过了，所以比较干净。
	![[IMG-20260405214107742.jpg|615]]
2. 以前的 /review 命令现在几乎感觉已经没用了，我感觉 /simplify 好用的多。
3. 我现在的习惯是，每次跟 Claude Code 对话了很多轮，写了几个大的功能更新之后，都顺手跑一遍 /simplify。
4. AI 写的代码经常会有一些微妙的冗余，多余的 import、重复的逻辑、可以用更简洁写法替代的地方，/simplify 基本上都能挑出来。
5. 相当于找了三个同事帮你同时 review，还是很香的。

# /branch

## 背景

1. 这个命令以前叫 /fork，现在改名成 /branch 了，打 /fork 还是能用，会自动跳到 /branch。
2. 其实就是可以把当前对话分叉出一个新会话，原来的会话不受影响。
	![[IMG-20260405214107782.jpg|593]]
3. 其实就有点像 ChatGPT 的新聊天中的分支功能。

## 怎么使用

1. 这个适合你在跟 Claude 聊到一半，突然想试另一个方向，但又不想丢掉当前对话进度的时候。
2. 比如 Claude 刚帮你梳理完一个方案的思路，你想沿着这个思路试两种不同的实现方式，/branch 一下，两个会话各走一边，最后挑效果好的那个。
	![[IMG-20260405214107816.jpg|658]]

## 使用效果

1. 跟 /rewind 的区别是，/rewind 是回退，/branch 是分叉，你可以理解成一个是后悔药，一个是平行宇宙。

# /loop

## 背景

1. /loop 也是最近才出的，可以让 Claude 定时重复执行某个任务。
	![[IMG-20260405214107851.jpg|438]]

## 怎么使用

1. 用法是 /loop 后面跟时间间隔和你要它做的事情。
2. 比如 /loop 5m 检查一下部署状态，它会每五分钟帮你跑一次，不用你自己盯着，默认间隔是 10 分钟。
	![[IMG-20260405214107884.jpg|600]]
3. 有点类似于 OpenClaw 的心跳机制，Claude Code 以前确实没有这个功能。

## 使用效果

1. /loop 的好处是结果直接在对话上下文里，Claude 可以基于这些结果做判断和后续操作。
2. 然后因为很多任务其实都是短期循环的，所以 Claude Code 定期任务在创建 3 天后自动过期。
3. 任务会最后触发一次，然后自我删除，这其实就限制了被遗忘的循环能运行的最长时间。
4. 如果你希望一直运行下去，Claude 给到的方式是使用桌面版。
	![[IMG-20260405214107918.jpg|516]]

# /remote-control

## 背景

1. 这个功能是今年 2 月底出的，我觉得是 Claude Code 比较实用的功能之一。
2. 你在终端里打 /rc，或者打完整的 /remote-control，它会生成一个 URL。
	![[IMG-20260405214107958.jpg|615]]

## 怎么使用

1. 你用手机打开这个链接，你的整个 Claude Code 会话就出现在手机上了。
2. 而且是完全同步的，你在手机上发一条指令，终端那边也能看到，你在终端上操作，手机上也会实时更新。
3. 两边可以交替使用，对话历史完全一致。

## 使用效果

1. 代码始终在你的电脑上跑，手机只是一个遥控器，你的文件系统、MCP 服务器、项目配置，全部还在本地，手机只是给你提供了一个远程操作的窗口。
2. 所以除了方便之外，也非常的安全，很好用。

# /export

## 背景

1. 这个命令很简单，打 /export，当前的整段对话就会被导出成一个 Markdown 文件。

## 怎么使用

1. 听起来没什么大不了的对吧，但有的时候你会发现这个功能还挺重要。
2. 比如说你跟 Claude 讨论了半小时某个架构方案，中间有很多来回的推敲和判断，最后达成了一致。
3. 这些讨论如果不保存下来，你后面找起来就非常麻烦，不如导出存下来，作为未来更详细的 context。
	![[IMG-20260405214107992.jpg|655]]

## 使用效果

1. 而且还可以拿去跟 Codex 协同，比如直接导出然后扔给 Codex，说你看看 Claude Code 那个呆逼改了半天没改好，帮我看看到底错在了哪。
2. 有的时候，这种互相导一下，还挺有用的。

# 快捷键

## 背景

1. 除了这些命令之外，其实还有很多好用的快捷键，大家不知道，所以我顺便推几个我用的习惯的非常好用的快捷键。

## 怎么使用

1. **Ctrl+V** 可以直接粘贴截图，不需要先保存成文件再拖进去，Debug 的时候遇到报错，直接截屏粘过去，Claude 看图说话。
2. **Mac 用户注意，这里是 Ctrl+V 不是 Cmd+V**，别再傻傻的每次把截图保存到本地再拖进去了。
3. **Ctrl+J，Mac 用户也可以 Option+回车**，都可以直接实现换行。
	![[IMG-20260405214108083.jpg|604]]
4. **Ctrl+R** 可以搜索你之前输入过的所有 prompt 历史。
	![[IMG-20260405214108107.jpg|657]]
5. **Ctrl+U** 可以删除整行输入。
