# 一、大模型训练过程

![[IMG-20260404032005359.png|800]]

1. ==**第一个阶段就是模型的预训练**==，通过海量的数据来去训练一个大模型，得到的叫做通用的大模型，也叫预训练模型（pretrained model）
2. ==**第二个阶段就是模型的微调**==
    1. 有些能力可能通用大模型是没有具备的，所以这个时候需要用到一些领域的数据或者我们自己自有的私有化的数据来进行对大模型的改良，那这个过程我们一般把它叫做模型的微调。
    2. ==**模型微调也分为两个比较重要的过程：**==
        1. ==**第一个过程就是 SFT（supervise fine tuning）**==
            1. SFT阶段呢我们需要一些指令微调数据，那这个数据已长成什么样子呢？
            2. 每个数据呢由两个字段来构成：input + output，input其实就是用户的问题，output 就是标准的答案，这个标准的答案我们从哪儿获取呢？一种方式也是最常用的方法就是用户来标注
            3. **本质：告诉模型对于一个问题，我们**==**想要**==**什么样的答案，然模型去**==**模仿**==
        2. ==**第二个过程叫做对齐阶段**==
            1. SFT 模型上线完之后，实际上我们可以拿到一些用户的真实的反馈数据，比如说用户的点赞或者点踩
            2. 我们可以通过对齐的方式，对模型做一些纠正，让不太好的答案呢变得更好一些
                > 有一些常见的对齐方法：比如说像基于强化学习 PPO，训练起来会比较的有挑战；DPO 是一个比较主流的算法，训练过程会更加的简单一些
            3. 对齐过程的训练数据与 SFT 不同：input 是问题，问题有两个不同的答案，accept 接受的，reject 不接受的
            4. **本质：告诉模型对于一个问题，我们**==**不想要**==**什么样的答案，然模型去**==**纠正**==
3. **一个完整流程**
    1. 大模型没有办法满足我们场景的需要
    2. 经过 SFT 的过程，对这个模型的某一些方面能力做加强，得到了SFT模型
    3. 但是 SFT 模型本身呢它可能也有些缺陷，比如说它没有严格的去执行我们想要的一些指令
    4. 这些现象当我们把这个模型上线之后，会观测的会越来越多，这时再收集一批对齐的数据，然后告诉模型不要做什么事情，然后呢进行一个相应的训练
    5. 那我们最后其实可以得到一个优化之后的模型
    
# 二、模型微调
## 1. 介绍
1. 模型微调：在使用过程中，发现模型在某个方面能力不够，通过一些训练的方法把这个模型做一个更新，那更新后的模型在某一个方面上希望他的能力会提升，本质上是他对模型的一个改动。
2. 模型的改动本质上是对这个模型参数的改动
    

    ![[IMG-20260404032005419.png|800]]

    1. 模型实际上是由一堆的参数来构成的，也就是从原有的参数变成新的参数
    2. 通常可以把参数组织成几行几列的形式，比如三行三列的参数，总共有九个数字。
        1. 0.1 被改造成 0.2，也可以从另外一个角度，看作是 0.1 + 1 个偏移量0.1，然后得到0.2
        2. 因此，这个改变可以写成另外一种形式，即**对于原有的参数，再加了一个改动的量，得到新的参数。**
        3. **换句话说，我们对模型微调，本质上就是学习得到这个参数改动量矩阵**
        
## 2. 微调的方式
### 2.1 全量微调 (Full Fine-tuning)
1. 全量微调是一种传统的模型微调方法，它会更新模型中所有参数的权重。
2. 即我们需要学习到和原参数等量的一个参数改动量矩阵。
3. 这种方法具有以下特点：
    - 需要较大的计算资源和存储空间
    - 训练时间较长
    - 可以深度适应目标任务
    - 容易出现灾难性遗忘问题
    
### 2.2 LoRA 微调 (Low-Rank Adaptation)
1. 微调的本质：
    1. 通过调整参数，将模型的目标能力方法，同时保留模型的其他能力。
    2. 这里改动的量，我们可以认为就是针对我们想要提升的能力，所以不希望模型的改动太多，导致失去原有的能力（模型的能力遗忘）。
    3. 举例：一篇文章2000字，其中他有价值的部分可能只有 200 字，类比到参数模型，我们认为十亿个参数中，可能对我们想要增强的目标能力来说，只有部分的参数是有价值的，那么我们只需要对这些参数进行调整，来高效的提升目标能力。
2. LoRA 的思想
    1. 在一个矩阵中，是存在一些冗余数据的，如下的矩阵A、B，通过矩阵运算，其真正有价值的数据分别只有一行和两行
        

        ![[IMG-20260404032005480.png|800]]

    2. 那么，在模型微调时，参数改动量的参数矩阵，是否也类似的存在这样的冗余信息，可以对这个大的参数矩阵，转换成两个低秩矩阵的乘积。
        

        ![[IMG-20260404032005538.png|800]]

    3. 因此我们微调的目标，从学习得到一个包含大量参数的矩阵，变成了要学习两个低秩矩阵，来近似这个大参数菊展，从而大大降低学习的成本
    4. 其中，K 就是低秩矩阵的秩，如果原矩阵对于提升的目标能力的价值越低，选择的 K 就越小，两个低秩矩阵所包含的参数量也就越小。
3. LoRA 的优点：
    - ==**显著减少了可训练参数的数量，参数更新的范围从所有参数，变为了低秩矩阵参数**==
    - 降低了内存占用和计算开销
    - 训练速度更快
    - 可以保持原始模型的性能

# 三、模型遗忘
## 1. 介绍
1. **灾难性遗忘（Catastrophic Forgetting）** 是机器学习领域，尤其是**神经网络**中的一种现象：
    1. 当模型在学习新任务或数据时，会显著遗忘之前学过的知识。
    2. 这种现象在持续学习（Continual Learning）或增量学习（Incremental Learning）中尤为突出
2. **核心问题**
    - 模型无法同时保留旧任务的知识和学习新任务的能力。
    - 典型场景：模型先学习任务 A，再学习任务 B，最终在任务 A 上的性能急剧下降，尽管任务 B 的学习是成功的。
        

        ![[IMG-20260404032005750.png|800]]

3. **为什么会出现灾难性遗忘**❓
    1. **参数更新冲突**
        - 神经网络的参数在学习新任务时，会被调整以适应新数据，这可能导致旧任务的权重被破坏。
        - 例如，新任务的梯度方向可能与旧任务的梯度方向相反，导致参数更新抵消旧任务的知识。
    2. **权重共享问题**
        - 神经网络的参数通常是共享的（如全连接层），学习新任务时，共享参数可能被覆盖，导致旧任务的信息丢失。
    3. **优化目标单一化**
        - 传统训练方法（如 SGD）仅关注当前任务的损失函数，忽略历史任务的约束，导致模型过度拟合新数据。
        
## **2. 解决方法**
1. **正则化方法**
    - 弹性权重巩固（Elastic Weight Consolidation, EWC）：通过正则化约束，限制重要参数的更新幅度，防止旧任务知识丢失。
    - 学习重要性（Learning Importance, LwF）：在新任务中加入旧任务的损失约束，保留旧知识。
2. **动态网络结构**
    - 动态扩展网络（Dynamic Networks）：为新任务分配独立的子网络。
    - 模块化设计：将不同任务的模块解耦，减少参数共享冲突。
3. **数据回放（Experience Replay）**
    - 经验回放（Experience Replay）：存储旧任务的数据样本，在训练新任务时随机采样混合训练。
