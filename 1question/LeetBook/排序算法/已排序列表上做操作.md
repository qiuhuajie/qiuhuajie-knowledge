# 一、介绍
1. ==**某些算法需要在排序算法的基础上使用（即在排序数组上运行）**==
2. 例如：
    1. 二分查找： 根据数组已排序的特性，才能每轮确定排除两部分中的哪一部分
    2. 双指针： 例如合并两个排序链表，根据已排序特性，才能通过双指针移动在线性时间内将其合并为一个排序链表
3. 直接使用排序 API 即可（底层快排）

    ```Java
    Arrays.sort(nums);
    ```
# 二、例题
## 1. 📑169. 多数元素🔥
> ℹ️ 力扣
> [https://leetcode.cn/problems/majority-element/description/?favorite=2cktkvj](https://leetcode.cn/problems/majority-element/description/?favorite=2cktkvj)
1. 问题描述：

    ```Plain
    给定一个大小为 n 的数组 nums ，返回其中的多数元素。多数元素是指在数组中出现次数 大于 ⌊ n/2 ⌋ 的元素。
    你可以假设数组是非空的，并且给定的数组总是存在多数元素。
    ```
2. 示例：

    ```Plain
    示例 1：
    输入：nums = [3,2,3]
    输出：3
    示例 2：
    输入：nums = [2,2,1,1,1,2,2]
    输出：2
    ```
3. 代码：
    - **如果将数组 nums 中的所有元素按照单调递增或单调递减的顺序排序，那么下标为 `n/2` 的元素（下标从 0 开始）一定是众数**

    ```Java
     public int majorityElement(int[] nums) {
        Arrays.sort(nums);
        return nums[nums.length / 2];
    }
    ```
## 2. 📑406. 根据身高重建队列🔥
> ℹ️ 力扣
> [https://leetcode.cn/problems/queue-reconstruction-by-height/description/?favorite=2cktkvj](https://leetcode.cn/problems/queue-reconstruction-by-height/description/?favorite=2cktkvj)
1. 问题描述：

    ```Plain
     假设有打乱顺序的一群人站成一个队列，数组 people 表示队列中一些人的属性（不一定按顺序）。每个 people[i] = [hi, ki] 表示第 i 个人的身高为 hi ，前面 正好 有 ki 个身高大于或等于 hi 的人。
    请你重新构造并返回输入数组 people 所表示的队列。返回的队列应该格式化为数组 queue ，其中 queue[j] = [hj, kj] 是队列中第 j 个人的属性（queue[0] 是排在队列前面的人）。
    ```
2. 示例：

    ```Plain
    示例 1：
    输入：people = [[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]]
    输出：[[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]]
    解释：
    编号为 0 的人身高为 5 ，没有身高更高或者相同的人排在他前面。
    编号为 1 的人身高为 7 ，没有身高更高或者相同的人排在他前面。
    编号为 2 的人身高为 5 ，有 2 个身高更高或者相同的人排在他前面，即编号为 0 和 1 的人。
    编号为 3 的人身高为 6 ，有 1 个身高更高或者相同的人排在他前面，即编号为 1 的人。
    编号为 4 的人身高为 4 ，有 4 个身高更高或者相同的人排在他前面，即编号为 0、1、2、3 的人。
    编号为 5 的人身高为 7 ，有 1 个身高更高或者相同的人排在他前面，即编号为 1 的人。
    因此 [[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]] 是重新构造后的队列。
    ```
3. 思路：
    1. ⭐**套路：**==**一般这种数对，还涉及排序的，根据第一个元素正向排序，根据第二个元素反向排序，或者根据第一个元素反向排序，根据第二个元素正向排序，往往能够简化解题过程**==
    2. 本题目中，首先对数对进行排序，按照数对的元素 1 降序排序，按照数对的元素 2 升序排序
    3. 原因是，按照元素 1 进行降序排序，对于每个元素，在其之前的元素的个数，就是大于等于他的元素的数量，而按照第二个元素正向排序，为了保证在身高一样时结果的正确性
4. 代码：

    ```Java
     public int[][] reconstructQueue(int[][] people) {
        // 按照第一个元素降序排序，按照第二个元素升序排序
        Arrays.sort(people, new Comparator<int[]>() {
            @Override
            public int compare(int[] o1, int[] o2) {
                if (o1[0] != o2[0]) {
                    return o2[0] - o1[0];
                } else {
                    return o1[1] - o2[1];
                }
            }
        });
        List<int[]> list = new ArrayList<>();
        // 由于比当前元素大的元素都已经插入到list中了，只有这些已插入的元素会对当前元素的最终位置产生影响
        // 所以直接按照 k_i 直接插入即可，后边的元素因为都比当前元素小，所以后边元素不管怎么插，都不会对当前的元素位置产生影响
        for (int[] person : people) {
            list.add(person[1], person);
        }
        return list.toArray(new int[list.size()][]);
    }
    ```
