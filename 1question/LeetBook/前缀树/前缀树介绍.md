# 1**. 什么是前缀树**
1. 前缀树 是 `N`叉树 的一种特殊形式。通常来说，一个前缀树是用来 **存储字符串** 的
    
    1. 前缀树的每一个节点代表一个 字符串（前缀）
    
    1. 每一个节点会有多个子节点，通往不同子节点的路径上有着不同的字符
    
    1. **子节点上的字符串**是==由节点本身的== ==**原始字符串**== ==，以及 通往该子节点====**路径上所有的字符**== ==组成的==
    
1. 下面是前缀树的一个例子：
    
    [![](https://aliyun-lc-upload.oss-cn-hangzhou.aliyuncs.com/aliyun-lc-upload/uploads/2018/02/07/screen-shot-2018-01-31-at-163403.png)](https://aliyun-lc-upload.oss-cn-hangzhou.aliyuncs.com/aliyun-lc-upload/uploads/2018/02/07/screen-shot-2018-01-31-at-163403.png)
    
    1. 在上图示例中，在节点中标记的值是该节点对应表示的字符串
    
    1. 例如，从根节点开始，选择第二条路径 `'b'`，然后选择它的第一个子节点 `'a'`，接下来继续选择子节点 `'d'`，最终会到达的叶节点代表的字符串为 `"bad"`
    
    1. 节点的值是由从根节点开始，与其经过的路径中的字符按顺序形成的
    
    1. 注意：根节点表示 空字符串
    
    1. 前缀树的一个重要的特性是：**==节点所有的后代都与该节点相关的字符串有着共同的前缀==**
        
        1. 这就是 前缀树 名称的由来
        
        1. 例如：以节点 `"b"` 为根的子树中的节点表示的字符串，都具有共同的前缀 `"b"`。反之亦然，具有公共前缀 `"b"` 的字符串，全部位于以 `"b"` 为根的子树中，并且具有不同前缀的字符串来自不同的分支
        
    
# 2**. 如何表示一个前缀树**
1. 前缀树的特别之处在于字符和子节点之间的对应关系
1. 有许多不同的表示前缀树节点的方法，这里只介绍其中的两种方法
## 2.1 **存储子节点的方式一：数组**
1. 第一种方法是用 数组 存储子节点
1. 如果只存储含有字母 `a` 到 `z` 的字符串，可以在每个节点中声明一个大小为 `26` 的数组来存储其子节点
1. 对于特定字符 `c`，我们可以使用 `c - 'a'` 作为索引来查找数组中相应的子节点
    
    ![[IMG-20260405035400209.png|Untitled 105.png]]
    
1. 实现代码：
    
    ```Java
     class TrieNode {
         // change this value to adapt to different cases
         public static final int N = 26;
         public TrieNode[] children = new TrieNode[N];
    
         // you might need some extra values according to different cases
     };
    ```
    
1. 访问子节点十分快捷
    
    1. 访问一个特定的子节点比较容易 ，因为在大多数情况下，**很容易将一个字符转换为索引**
    
    1. 但并非所有的子节点都需要这样的操作，所以这可能会导致 **空间的浪费**
    
## 2.2 **存储子节点的方式二：Map**
1. 第二种方法是使用 `Hashmap` 来存储子节点
1. 我们可以在每个节点中声明一个 `Hashmap` 。`Hashmap` 的键是字符，值是相对应的子节点
1. 实现代码：
    
    ```Java
     class TrieNode {
         public Map<Character, TrieNode> children = new HashMap<>();
    
         // you might need some extra values according to different cases
     };
    ```
    
1. 通过相应的字符来访问特定的子节点 更为容易
    
    1. 但它可能比使用数组 稍慢一些，但是，由于只存储我们需要的子节点，因此**节省了空间**
    
    1. 这个方法也更加灵活 ，因为**不受到固定长度和固定范围的限制**
    
# 3. **📑前缀树的数据结构**
1. 这里使用的存储子节点的方式是使用数组
1. 除了存储子节点，还需要一些额外的值
    
    - `path`：表示有多少字符串途径了本节点
    
    - `end`：表示有多找字符串是以本节点为结尾的
    
1. 代码：
    
    ```Java
     class TrieNode{
         int path = 0;
         int end = 0;
         TrieNode[] next = new TrieNode[26]; //如果字符种类很多，可以换成hashMap：HashMap<char, TrieNode> map
     }
    ```
