# **1. 📑拓扑排序**
1. 拓扑序列：一些活动，其中某些活动必须在另一些活动完成之后才能开始，一定是无环的有向图，称为AOV网
1. 拓扑排序：
    
    1. 其实就是对一个有向图构造拓扑序列的过程
    
    1. 构造时会有两个结果：
        
        - 如果此网的全部结点都被输出，则说明其为不存在环的AOV网
        
        - 如果没有输出全部顶点数，则说明这个网存在回路，不是AOV网
        
    
1. 拓扑排序基本思路：
    
    1. 从AOV网中选择一个入度为0的顶点输出，然后删去此结点，并删除以此结点为尾的弧
    
    1. 删除一个入度为0的顶点，一定还会至少再出现一个入度为0的顶点
    
    1. 继续重复此步骤，直到输出全部顶点或者AOV网中不存在入度为0的顶点为止
    
1. 代码：
    
    ```Java
     public class TopologySort {
         public static List<Node> sortedTopology(Graph graph) {
     
             //使用一个map记录每个节点的入度
             HashMap<Node, Integer> inMap = new HashMap<>();
     
             //使用一个queue存放入度为0的节点
             Queue<Node> zeroInQueue = new LinkedList<>();
     
             //遍历图的所有顶点，将每个节点的入度记录在map中
             for (Node node : graph.nodes.values()) {
                 inMap.put(node, node.in);
                 if (node.in == 0){
                     zeroInQueue.add(node);
                 }
             }
     
             //结果集
             ArrayList<Node> result = new ArrayList<>();
             //遍历所有0入度队列中的顶点
             while (!zeroInQueue.isEmpty()){
                 Node p = zeroInQueue.poll();
                 //加入结果集
                 result.add(p);
                 //将与当前弹出的0入度顶点相连的顶点更新入度-1，同时如果发现有新的节点入度变为0，就需要要将其加入0入度队列
                 for (Node next : p.nexts) {
                     Integer nextIn = inMap.get(next);
                     inMap.put(next, nextIn - 1);
                     if (nextIn - 1 == 0){
                         zeroInQueue.add(next);
                     }
                 }
             }
     
             return result;
         }
     }
    ```
    
# **2. 📑kruskal 算法**
1. 用来求加权连通图的最小生成树的算法
1. 算法思想 ：**加边法**
    
    ![[IMG-20260405035400232.png]]
    
    1. 按照权值从小到大的顺序选择 `n-1` 条边，并保证这 `n-1` 条边不构成回路
    
    1. 如何保证这 `n-1` 条不会构成回路？
    
    1. 由于可能出现两个联通的部分（堪称两个集合），通过新加的一个边，从而合成一个集合的情况，因此可以使用一个并查集（[[并查集]]）
    
    ```Plain
     > ==并查集（Union Find）==也叫「不相交集合（Disjoint Set）」
     >
     > 1. 专门用于 动态处理 不相交集合的「查询」与「合并」问题
     >
     > 2. 所谓「动态」的意思是：要处理的数据不是一开始就确定好的，理解「并查集」动态处理数据的最好的例子是「最小生成树」算法
     >
     > 3. 并查集动态处理的两个问题
     >
     >    1. **查询（Find）**：查询图中的两个顶点是不是在同一个集合中
     >
     >       > 注意：并查集只回答两个顶点之间是否有一条路径连接，而不回答怎么连接
     >
     >    2. **合并（Union）**：将两个不相交集合进行合并
    ```
    
1. 代码：
```Java
public class Kruskal {
    public static Set<Edge> kruskalMST(Graph graph) {
        //准备一个并查集，初始时并查集中有 n 个集合，每个集合中是单独一个顶点
        UnionFind unionFind = new UnionFind();
        unionFind.makeSets(graph.nodes.values());
        //准备一个堆，存放所有边
        PriorityQueue<Edge> priorityQueue = new PriorityQueue<>(new Comparator<Edge>() {
            @Override
            public int compare(Edge o1, Edge o2) {
                return o1.weight - o2.weight;
            }
        });
        //将所有边加入堆
        for (Edge edge : graph.edges) {
            priorityQueue.add(edge);
        }
        Set<Edge> result = new HashSet<>();
        //堆顶出堆，即找出最小边
        while (!priorityQueue.isEmpty()) {
            Edge edge = priorityQueue.poll();
            //查看当前边的from和to是不是在同一个集合中
            if (!unionFind.isSameSet(edge.from, edge.to)) {
                //如果不在，将当前表加入结果集
                result.add(edge);
                //并将当前边的from和to所在的集合合并
                unionFind.union(edge.from, edge.to);
            }
            //如果不在则不需要处理，否则会出现环
        }
        return result;
    }
    // Union-Find Set
    public static class UnionFind {
        private HashMap<Node, Node> fatherMap;
        private HashMap<Node, Integer> rankMap;
        public UnionFind() {
            fatherMap = new HashMap<Node, Node>();
            rankMap = new HashMap<Node, Integer>();
        }
        private Node findFather(Node n) {
            Node father = fatherMap.get(n);
            if (father != n) {
                father = findFather(father);
            }
            fatherMap.put(n, father);
            return father;
        }
        public void makeSets(Collection<Node> nodes) {
            fatherMap.clear();
            rankMap.clear();
            for (Node node : nodes) {
                fatherMap.put(node, node);
                rankMap.put(node, 1);
            }
        }
        public boolean isSameSet(Node a, Node b) {
            return findFather(a) == findFather(b);
        }
        public void union(Node a, Node b) {
            if (a == null || b == null) {
                return;
            }
            Node aFather = findFather(a);
            Node bFather = findFather(b);
            if (aFather != bFather) {
                int aFrank = rankMap.get(aFather);
                int bFrank = rankMap.get(bFather);
                if (aFrank <= bFrank) {
                    fatherMap.put(aFather, bFather);
                    rankMap.put(bFather, aFrank + bFrank);
                } else {
                    fatherMap.put(bFather, aFather);
                    rankMap.put(aFather, aFrank + bFrank);
                }
            }
        }
    }
}
```
# **3. 📑Prim 算法**
1. 算法思想：**加点法**
    
    ![[IMG-20260405035401343.png|Untitled 108.png]]
    
    1. 对图`G(V,E)`设置集合`S`（`set`），存放已访问的顶点：例如上图（c）中，`S` 就是 `1,3,4`，`V-S` 就是 `2,4,5,6`
    
    1. 然后每次从集合`V-S`中选择与集合`S`的最短距离最小的一个顶点（记为`u`），访问并加入集合`S`
        
        > 这里找u 可以使用一个堆来实现，堆中记录着集合V-S与集合S之间相连的所有边，通过堆顶弹出，可以找到这些边中权重最小的那个
        > 
        > 同时，没有一个`u` 加入集合`S`，就需要将与 `u` 相连的所有边加入堆
        
    
    1. 之后，从新的集合`S`出发，再从集合`V-S`中选择与集合`S`的最短距离最小的一个顶点，访问并加入集合`S`
    
    1. 执行`n`次（`n`为顶点个数），直到集合`S`已包含所有顶点
    
1. 代码：
    
    ```Java
    public class Prim {
        public static Set<Edge> pr1m(Graph graph) {
    
            //用一个堆记录从当前已经联通的部分发出的所有边，用堆的话方便取出最小的边
            PriorityQueue<Edge> priorityQueue = new PriorityQueue<>(new Comparator<Edge>() {
                @Override
                public int compare(Edge o1, Edge o2) {
                    return o1.weight - o2.weight;
                }
            });
    
            //保存已经加在最小生成树中的顶点
            HashSet<Node> set = new HashSet<>();
            //记录当前生成的最小生成树的所有边
            HashSet<Edge> result = new HashSet<>();
    
            for (Node node : graph.nodes.values()) {
                //初始的node可以是随便选的一个点，上面的循环是防止如果给的是几个不连通的图的情况，即graph是一个森林
                if (!set.contains(node)){
                    //加入当前节点
                    set.add(node);
                    //此时node与之前的最小树部分并成一个部分了，那么就需要将与node相连的所有边加入堆
                    priorityQueue.addAll(node.edges);
                    //将堆顶最小的边弹出，直至堆空
                    while (!priorityQueue.isEmpty()){
                        //此时取出的边是 当前生成的最小树与外界顶点，所连边中权重最小的边
                        Edge edge = priorityQueue.poll();
                        //查看当前遍历的边的to节点是否已经加在了最小树中
                        Node toNode = edge.to;
                        if (!set.contains(toNode)) {
                            //如果不在，需要将其加入set，同时
                            set.add(toNode);
                            result.add(edge);
                            //有新的顶点加入，就需要将与新顶点所连的所有点加入堆
                            priorityQueue.addAll(toNode.edges);
                        }
                    }
                }
            }
            return result;
        }
    }
    ```
    
# **4. 📑Dijkstra 算法**
1. Dijkstra算法是解决**单源最短路径**问题的**贪心算法**
1. 它先求出长度最短的一条路径，再参照该最短路径求出长度更短的一条路径，直到求出从源点到其他各个顶点的最短路径
    
    ![[IMG-20260405035403506.png|Untitled 1 70.png]]
    
    从源点出发依次将最短距离的点（红框）加入结果集合，再继续依次找剩下点中，与结果集之间最短距离的点……。最后红框连起来的就是最短路径
    
1. 基本思想
    
    1. 使用两个集合`S`和`U`
        
        - `S`的作用是记录那些 与其相邻的所有节点最短距离完成更新的顶点 `selectedNodes`
        
        - 而`U`则是记录 从源点出发，到全部顶点的**当前**最短路径 `distanceMap`
        
    
    1. 首先，从`distanceMap`选择一个，不在`selectedNodes`中，且记录的距离最小的节点作为下一个循环的`minNode`，由于初始时 `distanceMap` 只有源节点，那么此时`minNode`就是源节点，
    
    1. 将与源节点相连的所有边遍历一遍，此时所有边的 `toNode` 肯定都不在 `distanceMap`中，那么就将所有的节点都加入 `distanceMap`，加入的距离为 `0` + 当前边的 权重
    
    1. 接着将源节点加入 `selectedNodes` ，再从`distanceMap`选择一个，不在`selectedNodes`中，且记录的距离最小的节点作为下一个循环的`minNode`
    
    1. 将与`minNode`相连的所有边遍历一遍，此时有的边的 `toNode` 可能已经在 `distanceMap`中了，那么如果当前发现存在更小的源节点到`toNode` 的距离，那么就需要更新 `distanceMap`中原来记录的最小距离了
    
    1. 将与当前`minNode`相连的所有边遍历一遍完后，将当前`minNode`加入 `selectedNodes` ，再从`distanceMap`选择一个，不在`selectedNodes`中，且记录的距离最小的节点作为下一个循环的`minNode`
    
    1. 循环直到，再也选不出一个新的`minNode`，即所有节点都加在 `selectedNodes` 中了
    
1. 代码实现：
    
    ```Java
    public class Dijkstra {
        public static HashMap<Node, Integer> dijkstra1(Node head) {
            //存储源节点到各个节点之间的距离（当前最短距离，有可能在遍历过程中更新）
            HashMap<Node, Integer> distanceMap = new HashMap<>();
            //首先加入源节点
            distanceMap.put(head, 0);
    
            //记录那些 与其相邻的所有节点最短距离已经更新的节点
            HashSet<Node> selectedNodes = new HashSet<>();
    
            //得到遍历的开始节点
            Node minNode = getMinDistanceAndUnselectedNode(distanceMap, selectedNodes);
    
            while (minNode != null){
                //distance是源节点到当前节点的距离
                Integer distance = distanceMap.get(minNode);
                //遍历当前节点的所有边
                for (Edge edge : minNode.edges) {
                    Node toNode = edge.to;
                    //如果当前边的to节点不在distanceMap中，则直接加入
                    if (!distanceMap.containsKey(toNode)){
                        distanceMap.put(toNode, distance + edge.weight);
                    }else {
                        //如果当前边的to节点已经在distanceMap中，则将 min(源节点到当前节点的最短距离 + 当前边的权重， 之前记录的最短距离) 加入map
                        distanceMap.put(toNode, Math.min(distance + edge.weight, distanceMap.get(toNode)));
                    }
                }
                //当minNode的所有相邻节点的距离都更新完成后，将minNode加入selectedNodes
                selectedNodes.add(minNode);
                //再从distanceMap选择一个，不在selectedNodes中，且记录的距离最小的节点作为下一个循环的minNode
                minNode = getMinDistanceAndUnselectedNode(distanceMap, selectedNodes);
            }
            //当再也选不出 minNode 后，也即所有节点都在selectedNodes中了，那么distanceMap中就是最后的结果
            return distanceMap;
        }
    
        //获得distanceMap中，且不在selectedNodes中的，且与源节点距离最小的那个节点
        public static Node getMinDistanceAndUnselectedNode(HashMap<Node, Integer> distanceMap,
                                                           HashSet<Node> selectedNodes) {
            Node minNode = null;
            int minDistance = Integer.MAX_VALUE;
    
            for (Map.Entry<Node, Integer> curEntry : distanceMap.entrySet()) {
                Node curNode = curEntry.getKey();
                Integer curDistance = curEntry.getValue();
                //如果满足两个条件：1. 在 distanceMap - selectedNodes中，2. 距离比当前最小距离小
                if (!selectedNodes.contains(curNode) && curDistance < minDistance){
                    //更新记录最小节点和最小距离
                    minNode = curNode;
                    minDistance = curDistance;
                }
            }
            return minNode;
        }
    }
    ```
