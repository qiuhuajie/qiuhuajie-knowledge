# **1. 简介**
1. **并查集** Disjoint-set data structure

    > 直译为不交集数据结构）是一种数据结构，用于处理一些不交集（Disjoint sets，一系列没有重复元素的集合）的合并及查询问题

    > 并查集 (不相交集) 是一种描述不相交集合的数据结构，即若一个问题涉及多个元素，它们可划归到不同集合，同属一个集合内的元素等价（即可用任意一个元素作为代表，比如上述的互为亲戚即互相等价），不同集合内的元素不等价

    ![[IMG-20260405035400234.png|Untitled 104.png]]

2. 并查集支持如下操作：

    - **==查询==**：查询某个元素属于哪个集合，通常是返回集合内的一个“代表元素”。这个操作是为了判断两个元素是否在同一个集合之中

    - ==**合并**==：将两个集合合并为一个

    - 添加：添加一个新集合，其中有一个新元素。添加操作不如查询和合并操作重要，常常被忽略

    > 由于支持查询和合并这两种操作，并查集在英文中也被称为联合-查找数据结构（Union-find data structure）或者合并-查找集合（Merge-find set）

3. 并查集的应用：

    1. 用于计算最小生成树的 迪杰斯特拉算法 中的关键。由于最小生成树在网络路由等场景下十分重要，并查集也得到了广泛的引用

    2. 此外，并查集在符号计算，寄存器分配等方面也有应用

public static class Element<T>{

        this.value = value;
    }

## 2.1 **初始化并查集**
初始化后，每个元素自成一个集合
![[IMG-20260405035401400.png|Untitled 1 67.png]]
```Java
 //1️⃣ 存储所有元素的 map <值, 包装的Element>
 public HashMap<T, Element<T>> elementMap;
 
 //2️⃣ 存储所有元素与其父元素对应关系的 map <子Element, 父Element>
 public HashMap<Element<T>, Element<T>> parentMap;
 
 //3️⃣ 存储所有root元素（族长）的 map <rootElement, rootElement所在集合的元素个数>
 public HashMap<Element<T>, Integer> rootMap;
 
 //初始化
 public UnionFindSet(List<T> list) {
     elementMap = new HashMap<>();
     parentMap = new HashMap<>();
     rootMap = new HashMap<>();
     for (T value : list) {
         Element<T> element = new Element<T>(value);
         elementMap.put(value, element);
         parentMap.put(element, element);
         rootMap.put(element, 1);
     }
 }
```
## 2.2 查询：返回所在集合的root元素
```Java
 //查找某个节点所在集合的 root 元素
 private Element getRoot(Element<T> a) {
     Element root = a;
     while (parentMap.get(root) != root){
         root = parentMap.get(root);
     }
     return root;
 }
```
==寻根优化==
- 优化后，每次向上寻根只需要一步，就可以从 `**parentMap**` 中找出 root
- 即将 root 的子元素 **扁平化**

    ![[IMG-20260405035403641.png|Untitled 2 54.png]]

```Java
 //优化
 private Element getRoot(Element<T> a) {
     //将寻根路上遇到的 parent 都放在栈里
     Stack<Element<T>> stack = new Stack<>();
     Element root = a;
     while (parentMap.get(root) != root){
         Element<T> parent = parentMap.get(root);
         stack.push(parent);
         root = parent;
     }
     //将栈里存着的 parent，的父都指向 root
     while (!stack.isEmpty()){
         parentMap.put(stack.pop(), root);
     }
     return root;
 }
```
## 2.3 判断两个元素是否在同一个集合
![[IMG-20260405035407513.png|Untitled 3 43.png]]
- 如果上图 `**3，5**` 要查询，则各自找到 root，分别为 `**9，6**`，发现不是一个 root ，则`**3，5**` 不在同一个集合内
```Java
 //判断两个节点是否在同一个集合内
 public boolean find(T a, T b){
     if (elementMap.containsKey(a) && elementMap.containsKey(b)) {
         //判断条件：两个节点的 root 是否为同一个
         return getRoot(elementMap.get(a)) == getRoot(elementMap.get(a));
     }
     return false;
 }
```
## 2.4 **合并两个集合**
- 如果下图 `**3，5**` 要查询，则各自找到 root，分别为 `**9，6**`，发现不是一个 root ，则`**3，5**` 可以合并

    ![[Attachment/1question/大数据/LeetBook/并查集/IMG-20260405035407906.png|Untitled 4 35.png]]

- 将小集合并到大集合里

    ![[IMG-20260405035409773.png|Untitled 5 30.png]]

```Java
 //将两个节点所在的集合合并为同一个
 public void union(T a, T b){
     if (elementMap.containsKey(a) && elementMap.containsKey(b)) {
         //获得两个节点的 root
         Element rootA = getRoot(elementMap.get(a));
         Element rootB = getRoot(elementMap.get(b));
 
         //如果不是同一个 root 则可以合并
         if (rootA != rootB){
 
             //找出较大的那个集合big，较小的那个集合small
             Element big = rootMap.get(rootA) > rootMap.get(rootB) ? rootA : rootB;
             Element small = rootA == big ? rootB : rootA;
 
             //将小root的父指向大root
             parentMap.put(small, big);
 
             //更新大root的所在集合的个数
             rootMap.put(big, rootMap.get(big) + rootMap.get(small));
 
             //将小root从 rootMap 里删除
             rootMap.remove(small);
         }
     }
 }
```
