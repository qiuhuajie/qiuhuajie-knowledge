# 1. 介绍
1. 快慢指针主要解决链表中的问题，例如一个经典问题：给定一个链表，判断链表中是否有环？
1. 快慢指针一般会初始化指向链表的头结点 head，前进时快指针 fast 在前，慢指针 slow 在后
    
    1. 有的情况下是**相对速度的快慢**
    
    1. 有的情况下是快指针先走，造成的**相对位置的快慢**

### **📑141. 环形链表（判断）**🔥✔️

> [!info] 力扣  
> 
> [https://leetcode.cn/problems/linked-list-cycle/](https://leetcode.cn/problems/linked-list-cycle/)  
1. 问题描述：
    
    ```Plain
     给定一个链表，判断链表中是否有环。
     
     如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。 为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环
     注意：pos 不作为参数进行传递，仅仅是为了标识链表的实际情况。
     
     如果链表中存在环，则返回 true 。 否则，返回 false 。
     
     进阶：
     你能用 O(1)（即，常量）内存解决此问题吗？
    ```
    
1. 示例：
    

    ![[IMG-20260405035400212.png|Untitled 90.png]]

    

    注意：有环链表一定不会是下面这种中间有环的情况：否则 5 节点会有两个 next 指针

    

    ![[IMG-20260405035401002.png|Untitled 1 55.png]]

    
    ```Plain
     head = [3,2,0,-4], pos = 1
     输出：true
     解释：链表中有一个环，其尾部连接到第二个节点。
     
     输入：head = [1,2], pos = 0
     输出：true
     解释：链表中有一个环，其尾部连接到第一个节点。
    ```
    
1. 分析：
    
    1. 有两个速度不同的跑步者。如果他们在直路上行驶，快跑者将首先到达目的地。但是，如果它们在圆形跑道上跑步，那么快跑者如果继续跑步就会追上慢跑者
    
    1. 这正是在链表中使用两个速度不同的指针时会遇到的情况：
        
        1. 如果没有环，快指针将停在链表的末尾
        
        1. 如果有环，快指针最终将与慢指针相遇
        
    
    1. 所以剩下的问题是：这**两个指针的适当速度**应该是多少？
        
        1. 一个安全的选择是每次移动慢指针一步，而移动快指针两步。每一次迭代，快速指针将额外移动一步
        
        1. 如果环的长度为 M，经过 M 次迭代后，快指针肯定会多绕环一周，并赶上慢指针
        
    
    1. 复杂度分析
        
        1. **空间复杂度**：如果只使用指针，而不使用任何其他额外的空间，那么空间复杂度将是 `**O(1)**`
        
        1. **时间复杂度**：需要分析运行循环的次数。在前面的查找循环示例中，假设每次移动较快的指针 2 步，每次移动较慢的指针 1 步
            
            1. 如果**没有循环**，快指针需要 **N/2 次才能到达链表的末尾**，其中 N 是链表的长度
            
            1. 如果**存在循环**，则快指针需要 **M 次才能赶上慢指针**，其中 M 是列表中循环的长度
            
            1. 显然，M <= N 。所以将循环运行 N 次。对于每次循环，只需要常量级的时间。因此，该算法的时间复杂度总共为 `**O(N)**`
            
        
    
1. 代码：
    
    1. **快慢指针：**
        
        ```Java
        public boolean hasCycle(ListNode head) {
            if (head == null || head.next == null){
                return false;
            }
            ListNode slow = head;
            ListNode fast = head;
            while (fast != null && fast.next != null) {
                fast = fast.next.next;
                slow = slow.next;
                if (fast == slow) return true;
            }
            return false;
        }
        ```
        
    
    1. 哈希表：
        

### **📑142. 环形链表（返回入环点）**🔥✔️

> [!info] 力扣  
> 
> [https://leetcode.cn/problems/linked-list-cycle-ii/](https://leetcode.cn/problems/linked-list-cycle-ii/)  
1. 问题描述：
    
    ```Plain
     给定一个链表，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。
     
     为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环
     注意，pos 仅仅是用于标识环的情况，并不会作为参数传递到函数中。
     
     说明：不允许修改给定的链表。
     
     进阶：
     你是否可以使用 O(1) 空间解决此题？
    ```
    
1. 示例：
    
    ```Plain
     输入：head = [3,2,0,-4], pos = 1
     输出：返回索引为 1 的链表节点（即返回 pos）
     解释：链表中有一个环，其尾部连接到第二个节点
     
     输入：head = [1,2], pos = 0
     输出：返回索引为 0 的链表节点
     解释：链表中有一个环，其尾部连接到第一个节点
    ```
    
1. 代码：
    
    1. **使用快慢指针：空间O(1)**
        
        1. 使用两个指针，fast 与 slow。它们起始都位于链表的头部
        
        1. 随后，slow 指针每次向后移动一个位置，而 fast 指针向后移动两个位置。如果链表中存在环，则 fast 指针最终将再次与 slow 指针在环中相遇
        
        1. 如下图所示，设链表中环外部分的长度为 a
            
            1. slow 指针进入环后，又走了 b 的距离与 fast 相遇
            
            1. 此时，fast 指针已经走完了环的 n 圈，因此它走过的总距离为 $a+n(b+c)+b=a+(n+1)b+nc$
            

            ![[IMG-20260405035403093.png|Untitled 2 44.png]]

            
        
        1. 根据题意，任意时刻，fast 指针走过的距离都为 slow 指针的 2 倍。因此，我们有
            

            $$a+(n+1)b+nc=2(a+b)\\a=c+(n−1)(b+c)$$

            
        
        1. 有了 $a=c+(n-1)(b+c)$ 的等量关系，发现：从相遇点到入环点的距离 c ，再加上 n−1 圈的环长b + c，恰好等于从链表头部到入环点的距离
        
        1. 因此，**当发现 slow 与 fast 相遇时，再把快慢指针的任意一个重新指向 head；随后两个指针**==**同时前进**==**。最终，它们会在入环点相遇**
            

            ![[IMG-20260405035405364.png|Untitled 3 36.png]]

            
        
        ```Java
        public ListNode detectCycle(ListNode head) {
            if (head == null || head.next == null){
                return null;
            }
            ListNode slow = head;
            ListNode fast = head;
            while (fast != null && fast.next != null) {
                fast = fast.next.next;
                slow = slow.next;
                if (fast == slow) {
                    // 两指针相遇后，一定存在环！开始寻找入环点
                    fast = head;
                    while (true) {
        								// 再次相遇即入环点
                        if (fast == slow) {
                            return slow;
                        }
                        fast = fast.next;
                        slow = slow.next;
                    }
                }
            }
            return null;
        }
        ```
        
    
    1. 使用哈希表 HashSet：
        
        ```Java
         //将遍历过的节点存到hashset中，再次遍历到的就是入环点
         public ListNode detectCycle(ListNode head) {
             ListNode point = head;
             HashSet<ListNode> set = new HashSet<>();
        
             while (point != null){
                 if (set.contains(point)){
                     return point;   //碰到重复的返回该节点即可
                 }else {
                     set.add(point);
                     point = point.next;
                 }
             }
             return null;
         }
        ```
        
    
### **📑234. 回文链表（链的中点）**🔥✔️

> [!info] 力扣  
> 
> [https://leetcode.cn/problems/palindrome-linked-list/](https://leetcode.cn/problems/palindrome-linked-list/)  
1. 问题描述：
    
    ```Plain
     给你一个单链表的头节点 head ，请你判断该链表是否为回文链表。如果是，返回 true ；否则，返回 false 。
    ```
    
1. 示例：
    
    ```Plain
     输入：head = [1,2,2,1]
     输出：true
    ```
    
1. 代码：
    
    - ⭐ **找到链表中点：可以使用快慢指针在一次遍历中找到**
        
        - 慢指针一次走一步，快指针一次走两步，快慢指针同 时出发
        
        - 当快指针移动到链表的末尾时，慢指针恰好到链表的中间
        
        - 当链表长度是奇数时，slow 恰好停在中点位置；当链表的长度是偶数时，slow 最终的位置是中间偏左
        
    
    ```Java
    public boolean isPalindrome(ListNode head) {
        if (head.next == null){
            return true;
        }
    
        // 1. 快慢指针找中点
        ListNode slow = head;
        ListNode fast = head;
        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;
        }
        ListNode posHead = null;
        if (fast == null) {
            posHead = slow;
        } else {
            posHead = slow.next;
        }
    
        // 2. 将后半部分链翻转，将前后两个链，从头开始遍历比较
        posHead = reverseList(posHead);   // 链表翻转后，旧 posHead 已经不再指向翻转后的链头结点，需要重新赋值
        ListNode p1 = head;
        ListNode p2 = posHead;    
        while (p2 != null) {
            if (p1.val != p2.val) return false;
            p1 = p1.next;
            p2 = p2.next;
        }
        //判断完成后，将链表还原回去
        reverseList(posHead);
        return true;
    
    }
    
    // 头插法反转
    public ListNode reverseList(ListNode head){
        ListNode pre = head;
        ListNode point = head.next;
        while (point != null){
            pre.next = point.next;
            point.next = head;
            head = point;
            point = pre.next;
        }
        return head;
    }
    ```

### **📑160. 两链表第一个公共节点（扩展）**
1. 问题描述：
    
    ```Plain
     给你两个单链表的头节点 headA 和 headB ，请你找出并返回两个单链表相交的起始节点。如果两个链表没有交点，返回 null 。
     
     注意，函数返回结果后，链表必须 保持其原始结构
    ```

    ```Java
     public class GetIntersectionNodeHasCycle {
         //得到某个链表的入环点
         public static ListNode getLoopNode(ListNode head){
             if (head == null || head.next == null){
                 return null;
             }
             ListNode slow = head;
             ListNode fast = head;
     
             //验证有无环：有环相遇，无环不相遇
             do{
                 if (fast == null || fast.next == null){
                     return null;
                 }
                 slow = slow.next;
                 fast = fast.next.next;
             }while (slow != fast);
     
             //有环：找入环点
             ListNode p = head;
             while (p != slow){
                 p = p.next;
                 slow = slow.next;
             }
             return p;
         }
     
         //两个链表都无环的情况下，和上题一样
         public static ListNode noLoop(ListNode headA, ListNode headB){
             if (headA == null || headB == null){
                 return null;
             }
     
             ListNode pA = headA;
             ListNode pB = headB;
             while (pA != pB){
                 if (pA == null){
                     pA = headB;
                 }else {
                     pA = pA.next;
                 }
     
                 if (pB == null){
                     pB = headA;
                 }else {
                     pB = pB.next;
                 }
             }
             return pA;
         }
     
         //两个链表都有环的情况下
         public static ListNode bothHasLoop(ListNode headA, ListNode headB){
             if (headA == null || headB == null){
                 return null;
             }
             ListNode pA = headA;
             ListNode pB = headB;
     
             ListNode loopA = getLoopNode(headA);
             ListNode loopB = getLoopNode(headB);
     
             //判断入环点是否一样
             if (loopA == loopB){
                 //入环点一点：又和上题相同，但只需要遍历入环点之前的即可
                 while (pA != pB){
                     if (pA == loopA.next){
                         pA = headB;
                     }else {
                         pA = pA.next;
                     }
     
                     if (pB == loopB.next){
                         pB = headA;
                     }else {
                         pB = pB.next;
                     }
                 }
                 return pA;
             }else {
                 //入环点不同：
                 ListNode p = loopA;
                 //从其中一个入环点开始遍历，看是否会遍历到另一个入环点：即为情况3
                 do{
                     if (p == loopB){
                         return loopA;
                     }
                     p = p.next;
                 }while (p != loopA);
     
                 //如果没有遍历到另一个入环点：即为情况2，两个链表各自成环
                 return null;
             }
         }

             if (headA == null || headB == null){
                 return null;
             }
     
             ListNode loopNodeA = getLoopNode(headA);
             ListNode loopNodeB = getLoopNode(headB);
             if (loopNodeA == null && loopNodeB == null){
                 return noLoop(headA, headB);
             }else if (loopNodeA != null && loopNodeB != null){
                 return bothHasLoop(headA, headB);
             }else {
                 return null;
             }
         }
     }
    ```

    1. 并且让 **fast 比 slow 超前 n 个节点**。当 fast 遍历到链表的末尾时，slow 就恰好处于倒数第 n 个节点
    
    ```Java
     public ListNode removeNthFromEnd(ListNode head, int n) {
         ListNode slow = head;
         ListNode fast = head;
         //slow 起始位置是head，故fast需要先走n+1步
         for (int i = 0; i < n + 1; i++) {
             //因为没有用到哨兵，这里这里需要特殊处理，当fast已经为空了，但还要继续向后走一步，则说明要删除的是头节点

                 return head.next;
             }
             fast = fast.next;
         }
         while (fast != null){
             fast = fast.next;
             slow = slow.next;
         }
    		 // slow.next 就是要删除的节点
         slow.next = slow.next.next;
         return head;
     }
    ```
    

    使用==**哨兵 pre**==，它的 next 指针指向链表的头节点。这样就**不需要对头节点进行特殊的判断了**

    ```Java
     public ListNode removeNthFromEnd(ListNode head, int n) {

    		 // ⭐ slow 从 pre 开始
         ListNode slow = pre;
         ListNode fast = head;

             fast = fast.next;
         }
         while (fast != null){
             fast = fast.next;
             slow = slow.next;
         }
         slow.next = slow.next.next;
         return pre.next;
     }
    ```

    ```Java
        int slow = 0;
        int fast = 0;
        while (fast < nums.length) {
            if (nums[fast] != 0) {
                nums[slow] = nums[fast];
                slow++;
                fast++;
            } else {
                fast++;
            }
        }
        while (slow < nums.length) {
            nums[slow++] = 0;
        }
    ```
