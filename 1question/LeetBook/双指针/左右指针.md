# 1. 介绍
1. 在上一章中，我们通过迭代数组来解决一些问题。通常，我们只需要一个指针进行迭代，即从数组中的第一个元素开始，最后一个元素结束。然而，有时我们会使用两个指针进行迭代
1. 示例：
    
    从一个经典问题开始：反转数组中的元素。比如数组为 ['l', 'e', 'e', 't', 'c', 'o', 'd', 'e']，反转之后变为 ['e', 'd', 'o', 'c', 't', 'e', 'e', 'l']。
    
    使用双指针技巧，其思想是分别将两个指针分别指向数组的开头及末尾，然后将其指向的元素进行交换，再将指针向中间移动一步，继续交换，直到这两个指针相遇
    
1. 左右指针一般用在数组问题中，实际是指两个索引值，一般初始化为 `left =0`，`right = len(nums - 1)`
# 2. 例题
## 二分搜索
1. 二分搜索是一种高效的查找算法，可以在已排序的数据集中查找特定元素。使用二分搜索的条件是==**数据集必须是已排序的**==
1. 当数据量较大且需要频繁查找时，使用二分搜索可以大大提高查找效率，因为二分搜索的时间复杂度为**`O(log n)`**，比线性搜索的时间复杂度**`O(n)`**要低得多
1. 当数据集规模较小时，使用二分搜索可能不如线性搜索快速，因为**排序数据集需要额外的时间和空间成本**
```Java
public static Integer binarySearch(int[] nums, int target) {
    int left = 0;
    int right = nums.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) {
            return mid;
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid - 1;
        }
    }
    return -1;
}
```

1. 问题描述：
    
    ```Plain
    给你一个按照非递减顺序排列的整数数组 nums，和一个目标值 target。请你找出给定目标值在数组中的开始位置和结束位置。
    
    如果数组中不存在目标值 target，返回 [-1, -1]。
    
    你必须设计并实现时间复杂度为 O(log n) 的算法解决此问题。
    ```
    
1. 示例：
    
    ```Plain
    示例 1：
    输入：nums = [5,7,7,8,8,10], target = 8
    输出：[3,4]
    
    示例 2：
    输入：nums = [5,7,7,8,8,10], target = 6
    输出：[-1,-1]
    ```
    
1. 代码：
    
    - **当 `target == nums[mid]` 时，并不直接返回**
        
        - ==**寻找下边界时**==
            
            - 当 `target == nums[mid]` 时，把 `nums[mid]` 看作比 `target` 大的数处理
            
            - 即将等于和大于合并在一起处理，最后返回 left （手动模拟下就知道返回哪个了）
            
        
        - ==**寻找上边界时**==
            
            - 当 `nums[mid] == target` 时，把 `nums[mid]` 看作比 `target` 小的数处理
            
            - 即将等于和小于合并在一起处理，最后返回 right
            
        
    
    - 因为 while 循环条件是 `left <= right`
        
        - 如果 target 不存在，二分结束后，left 指在第一个小于 target 的元素，right 指在第一个大于 target 的元素
        
        - 所以即使只有一个 target，在做了合并处理后，最后 left 和 right 还是会指向 target
        
    
    ```Java
     public static int[] searchRange(int[] nums, int target) {
    
        int l = lowerBound(nums, target);
        int r = highBound(nums, target);
        
        // l > r 时 target 不存在
        if (l > r) {
            return new int[]{-1, -1};
        } else {
            return new int[]{l, r};
        }
    }
    
    static int lowerBound(int[] nums, int target) {
    
        int left = 0, right = nums.length - 1;
    
        while (left <= right) {
            int mid = (left + right) >>> 1;
            if (nums[mid] >= target) {
                // 寻找下边界时，将等于和大于合并在一起处理
                right = mid - 1;
            }else if (target > nums[mid]) {
                left = mid + 1;
            }
        }
        return left;
    }
    
    static int highBound(int[] nums, int target) {
    
        int left = 0, right = nums.length - 1;
    
        while (left <= right) {
            int mid = (left + right) >>> 1;
            if (nums[mid] > target) {
                right = mid - 1;
            }else if {
                // 寻找上边界时，将等于和小于合并在一起处理
                left = mid + 1;
            }
        }
        return right;
    }
    ```

1. 示例：
    

    ```Java
     public boolean searchMatrix(int[][] matrix, int target) {
        int m = matrix.length;
        int n = matrix[0].length;
    
        // 依次二分每行
        for (int i = 0; i < m; i++) {
            boolean bs = bs(matrix[i], 0, n - 1, target);
            if (bs) return true;
        }
        return false;
    }
    
    private boolean bs(int[] nums, int left, int right, int target) {
        while (left <= right) {
            int mid = (right + left) >>> 1;
            if (nums[mid] == target) {
              return true;
            } else if (nums[mid] > target) {
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }
        return false;
    }
    ```

        ```Java
         public int minArray(int[] numbers) {
            int left = 0, right = numbers.length - 1;
            while (left <= right) {
                int mid = (left + right) >>> 1;

                    left = mid + 1;
                } else if (numbers[mid] < numbers[right]) {
                    right = mid;
                } else {
                    right--;
                }
            }
            return numbers[left];
        }
        ```

## **📑33. 搜索旋转排序数组**🔥✔️

> [!info] 力扣  
>  
> [https://leetcode.cn/problems/search-in-rotated-sorted-array/description/?favorite=2cktkvj](https://leetcode.cn/problems/search-in-rotated-sorted-array/description/?favorite=2cktkvj)  
1. 问题描述：
    
    ```Plain
    整数数组 nums 按升序排列，数组中的值 互不相同 。
    
    在传递给函数之前，nums 在预先未知的某个下标 k（0 <= k < nums.length）上进行了 旋转，使数组变为 [nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]（下标 从 0 开始 计数）。例如， [0,1,2,4,5,6,7] 在下标 3 处经旋转后可能变为 [4,5,6,7,0,1,2] 。
    
    给你 旋转后 的数组 nums 和一个整数 target ，如果 nums 中存在这个目标值 target ，则返回它的下标，否则返回 -1 。
    
    你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。
    ```
    
1. 示例：
    
    ```Plain
    示例 1：
    输入：nums = [4,5,6,7,0,1,2], target = 0
    输出：4
    
    示例 2：
    输入：nums = [4,5,6,7,0,1,2], target = 3
    输出：-1
    
    示例 3：
    输入：nums = [1], target = 0
    输出：-1
    ```
    
1. 代码：
    
    - 题目要求 O(logN) 的时间复杂度，基本可以断定本题是需要使用二分查找，怎么分是关键
        
        ![[IMG-20260405035402983.png|Untitled 2 45.png]]
        
    
    - 由于题目说数字了无重复，举个例子： 1 2 3 4 5 6 7 可以大致分为两类
        
        - 第一类 2 3 4 5 6 7 1 这种，也就是 `nums[left] <= nums[mid]`。此例子中就是 2 <= 5。 这种情况下，**前半部分** `[left, mid]` **有序**
            
            - 因此如果 `nums[left] <=target<nums[mid]`，则在前半部分找（由于有序，可以快速判断 target 是否在有序部分中）
            
            - 否则去后半部分找
                
                > [!important] 如果
                > 
                > `nums[left] <= nums[mid]`处不写`<=`而是写`<`的话，当数组为`[3, 1]`，target = 1时，`left = 0, mid = 0, right = 1`，这时会把`{nums[0], nums[1]}`当成有序数组，显然不对
                
            
        
        - 第二类 6 7 1 2 3 4 5 这种，也就是 `nums[left] > nums[mid]`。此例子中就是 6 > 2。 这种情况下，**后半部分** `[mid, right]` **有序**
            
            - 因此如果 `nums[mid] <target<=nums[right]`，则在后半部分找（由于有序，可以快速判断 target 是否在有序部分中）
            
            - 否则去前半部分找
            
        
    
    ```Java
     public int search(int[] nums, int target) {
        int left = 0;
        int right = nums.length - 1;
    
        while (left <= right) {
            int mid = left + (right - left) / 2;
    
            if (nums[mid] == target) {
                return mid;
            }
            // 注意此处用小于等于
            if (nums[left] <= nums[mid]) {
                // [left, mid] 有序
                // 由于有序，可以判断 target 是否在有序部分中
                if (nums[left] <= target && target < nums[mid]) {
                    right = mid - 1;
                }else {
                    left = mid + 1;
                }
            } else { 
                // [mid, right] 有序
                // 由于有序，可以判断 target 是否在有序部分中
                if (nums[mid] < target && target <= nums[right]) {
                    left = mid + 1;
                }else {
                    right = mid - 1;
                }
            }
        }
        return -1;
    }
    ```


    ```Java
     class Solution {
        public int[] exchange(int[] nums) {
           int i = 0;
           int j = nums.length - 1;
    
           while(i < j) {
               while(i < j && nums[i] % 2 == 1) {
                   i++;
               }
               while(i < j && nums[j] % 2 == 0) {
                   j--;
               }
               swap(nums, i, j);
           }
           return nums;
        }
    
        public void swap(int[] nums, int a, int b) {
            int tmp = nums[a];
            nums[a] = nums[b];
            nums[b] = tmp;
        }
    }
    ```
    
## **📑15. 三数之和**🔥✔️

> [!info] 力扣  
>  
> [https://leetcode.cn/problems/3sum/?favorite=2cktkvj](https://leetcode.cn/problems/3sum/?favorite=2cktkvj)  
1. 问题描述：
    
    ```Plain
    给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0 。请
    
    你返回所有和为 0 且不重复的三元组。
    
    注意：答案中不可以包含重复的三元组。
    ```
    
1. 示例：
    
    ```Plain
    示例 1：
    输入：nums = [-1,0,1,2,-1,-4]
    输出：[[-1,-1,2],[-1,0,1]]
    解释：
    nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0 。
    nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0 。
    nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0 。
    不同的三元组是 [-1,0,1] 和 [-1,-1,2] 。
    注意，输出的顺序和三元组的顺序并不重要。
    
    示例 2：
    输入：nums = [0,1,1]
    输出：[]
    解释：唯一可能的三元组和不为 0 。
    ```
    
1. 代码：
    
    - 固定 3 个指针中最左（最小）数字的指针 `i`，双指针 `left`，`right` 分设在数组索引 `(i，nums.length)` 两端，通过双指针交替向中间移动
    
    - 记录对于每个固定指针 i 的所有满足 `nums[i] + nums[left] + nums[right] == 0` 的 `left`，`right` 组合
    
    ```Java
     public List<List<Integer>> threeSum(int[] nums) {
        // 先对数组排序
        Arrays.sort(nums);
        // 结果集
        List<List<Integer>> res = new ArrayList<>();
        // 遍历排序好的数组
        // 当前元素为第一个元素，固定不变，其余两个元素为当前元素之后所有元素的首末元素
        for (int i = 0; i < nums.length - 2; i++) {
            // 之后不可能再出现正确结果了
            if (nums[i] > 0) break;
    
            // i 在移动时，要忽略重复元素
            if (i > 0 && nums[i - 1] == nums[i]) {
                continue;
            }
    
            // 定义当前双指针，向中间移动找后两个元素
            int left = i + 1;
            int right = nums.length - 1;
    
            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];
                if (sum == 0) {
                    // 记录结果（要先记录结果，再移动指针）
                    res.add(new ArrayList<Integer>(Arrays.asList(nums[i], nums[left], nums[right])));
    
                    // 左右同时向中间移动一步（同时要注意去重）
                    while (left < right && nums[left] == nums[++left]);
                    while (left < right && nums[right] == nums[--right]);
    
                } else if (sum < 0) {
                    // sum < 0，说明数不够大，故左指针往右移动一步（同时要注意去重）
                    while (left < right && nums[left] == nums[++left]);
                } else {
                    // sum > 0，说明数不够小，故左指针往左移动一步（同时要注意去重）
                    while (left < right && nums[right] == nums[--right]);
                }
            }
        }
        return res;
    }
    ```
    
    - ==想让一个指针==**==跳过重复元素==**==进行==**==左移或右移==**==：==`while (left < right && nums[right] == nums[--right]);`
    
    - **这两行是不一样的：**第一行是 `r` 与 `r-1` 比较，第二行是 `r-1` 与 `r-1` 比较

## **📑16. 最接近的三数之和**
1. 问题描述：
    
    ```Plain
    给你一个长度为 n 的整数数组 nums 和 一个目标值 target。请你从 nums 中选出三个整数，使它们的和与 target 最接近。
    
    返回这三个数的和。
    
    假定每组输入只存在恰好一个解。
    ```
    
1. 示例：
    
    ```Plain
    示例 1：
    输入：nums = [-1,2,1,-4], target = 1
    输出：2
    解释：与 target 最接近的和是 2 (-1 + 2 + 1 = 2) 。
    
    示例 2：
    输入：nums = [0,0,0], target = 1
    输出：0
    ```
    
1. 代码：
    
    ```Java
     public int threeSumClosest(int[] nums, int target) {
        Arrays.sort(nums);
        int res = Integer.MAX_VALUE;
        for (int i = 0; i < nums.length - 2; i++) {
            int left = i + 1;
            int right = nums.length - 1;
    
            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];
                res = Math.abs(sum - target) < Math.abs(res - target) ? sum : res;
                if (sum < target) {
                    left++;
                } else if (sum > target) {
                    right--;
                } else {
                    return res;
                }
            } 
        }
        return res;
    }
    ```

    ```Java
                res = Math.max(res, (right - left) * height[left]);
                left++;
            } else {
    ```

## **📑581. 最短无序连续子数组**🔥

> [!info] 力扣  
>  
> [https://leetcode.cn/problems/shortest-unsorted-continuous-subarray/description/?favorite=2cktkvj](https://leetcode.cn/problems/shortest-unsorted-continuous-subarray/description/?favorite=2cktkvj)  
1. 问题描述：
    
    ```Plain
     给你一个整数数组 nums ，你需要找出一个 连续子数组 ，如果对这个子数组进行升序排序，那么整个数组都会变为升序排序。
    
    请你找出符合题意的 最短 子数组，并输出它的长度。
    
    进阶：你可以设计一个时间复杂度为 O(n) 的解决方案吗？
    ```
    
1. 示例：
    
    ```Plain
    示例 1：
    输入：nums = [2,6,4,8,10,9,15]
    输出：5
    解释：你只需要对 [6, 4, 8, 10, 9] 进行升序排序，那么整个表都会变为升序排序。
    
    示例 2：
    输入：nums = [1,2,3,4]
    输出：0
    ```
    
1. 代码：
    
    1. 先排序，再与未排序的列表作比较，找到左右界，计算长度，但由于要先排序，所以时间复杂度肯定大于 **O(n)**
        
        ```Java
         public int findUnsortedSubarray(int[] nums) {
            int[] tmp = new int[nums.length];
            for (int i = 0; i < nums.length; i++) {
                tmp[i] = nums[i];
            }
            Arrays.sort(tmp);
            int left = 0;
            int right = nums.length - 1;
        
            // 找左边第一个不相等的元素
            while (left < nums.length) {
                if (tmp[left] == nums[left]) {
                    left++;
                } else {
                    break;
                }
            }
            // 找右边第一个不相等的元素
            while (right > 0 ) {
                if (tmp[right] == nums[right]) {
                    right--;
                } else {
                    break;
                }
            }
            // 计算长度
            if (left <= right) {
                return right - left + 1;
            } else {
                return 0;
            }
        }
        ```
        
    
    1. 进阶：**==设计一个时间复杂度为 O(n) 的解决方案==**
        
        - 思路
            
            - 可以假设把这个数组分成三段，`左段`和`右段`是标准的升序数组，`中段`数组虽是无序的，**但满足最小值大于`左段`的最大值，最大值小于`右段`的最小值**
                
                ![[IMG-20260405035410746.png|Untitled 6 24.png]]
                
            
            - 那么我们目标就很明确了，找中段的左右边界，我们分别定义为 begin 和 end，分两头开始遍历：
                
                - 从左到右维护一个最大值max，在进入右段之前，那么遍历到的nums[i]都是小于max的（即 max 为右端之前的最大值，红色线条），我们要求的end就是遍历中最后一个小于max元素的位置
                
                - 同理，从右到左维护一个最小值min，在进入左段之前，那么遍历到的nums[i]也都是大于min的（即 min 为左端之后的最小值，黄色线条），要求的begin也就是最后一个大于min元素的位置
                
            
        
        ```Java
        public int findUnsortedSubarray(int[] nums) {
            //初始化
            int len = nums.length;
            int min = Integer.MAX_VALUE;
            int max = Integer.MIN_VALUE;
            int begin = 0, end = -1;
            
            //遍历
            for(int i = 0; i < len; i++){
        
                // 从左到右维护最大值
                max = Math.max(max, nums[i]);
                // end 保存遍历中最后一个小于 max 元素的位置
                if(nums[i] < max){
                    end = i;
                }
        
                // 从右到左维护最小值
                min = Math.min(min, nums[len - i - 1]);
                // begin 保存遍历中第一个大于 min 元素的位置
                if(nums[len - i - 1] > min){
                    begin = len - i - 1;
                }
            }
            return end - begin + 1;
        }
        ```
