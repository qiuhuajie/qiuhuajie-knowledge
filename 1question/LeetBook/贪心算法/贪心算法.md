# **1. 贪心算法**
## **1.1 简介**
1. 贪心算法是对完成一件事情的方法的描述，贪心算法⭐==**每一次都做出当前看起来最好的选择**====，而不用考虑其它可能的选择==
1. 贪心策略并**不是从整体上考虑最优**
    
    1. 所做的选择只是当前看似最佳选择，必须**进一步证明**该算法最终导致问题能到达**整体最优解**
    
    1. 贪心算法的每一步上用作决策依据的选择准则被称为**最优度量标准**（这里的最优是局部最优）
    
    1. 在根据最优度量标准选择分量的过程中，还需要使用一个**可行判定函数**（约束条件）
    
1. 贪心算法的应用场景：
    
    1. 解决一个问题需要多个步骤，每一个步骤有多种选择
    
    1. 可以使用贪心算法解决的问题，每一步只需要解决一个子问题，只做出一种选择，就可以完成任务
    
## **1.2 使用条件**
可以使用「贪心算法」的问题需要满足的条件：
- 1️⃣ ⭐**贪心选择性质：**
    
    1. 指所求问题的**整体最优解**可以通过一系列**局部最优的选择**（贪心选择）来达到
    
    1. ==这是贪心算法可行的第一个基本要素，也是贪心算法与动态规划法的主要区别==
    
- 2️⃣ **无后效性：**后面阶段的求解不会修改前面阶段已经计算好的结果
- 3️⃣ **最优子结构：**
    
    1. 最优子结构规定的是子问题与原问题的关系：**规模较大的问题的解由规模较小的子问题的解组成**
    
    1. 区别于「动态规划」，可以使用「贪心算法」的问题「规模较大的问题的解」只由**其中一个**「规模较小的子问题的解」决定
        
        - 贪心算法每次只需要考虑**一个子问题**，并且通常是自底向上求解
            
            ![[Attachment/1question/人工智能/LeetBook/贪心算法/IMG-20260405035400275.png|Untitled 28.png]]
            
        
        - 动态规划需要考虑原问题的**所有子问题**
            
            ![[IMG-20260405035402040.png|Untitled 1 9.png]]
            
        
    
## **1.3 贪心算法经典问题**
贪心法能够解决多种领域中的问题：背包问题、最小代价生成树（Kruskal算法、Dijkstra算法）、哈夫曼树、文件的最佳合并树等
1. **背包问题：**
    
    1. 两类背包问题：
        
        1. 0/1背包问题：如果每一件物品不能分割，只能作为整体或者装入背包，或者不装入
        
        1. 一般背包问题或简称背包问题：如果物品是可以分割的，也就是允许将其中的一部分装入背包
        
    
    1. 一般背包问题描述：
        
        1. 解可以表示成一个元组：每个 x_i 是第 i 件物品装入背包的部分
            
            $$X=(x_0,x_1,...,x_{n-1})\\  
            0\leq x_i\leq 1\\0\leq i <n$$
            
        
        1. 判定可行解的约束条件是：
            
            $$\sum_{i=0}^{n-1}w_i*x_i\leq M$$
            
        
        1. 背包问题的最优解 必须使下列目标函数取最大值：
            
            $$max (\sum_{i=0}^{n-1} p_i*x_i)$$
            
        
        1. 选择最优度量标准：
            
            1. 每次选取价值最大的物品装入或每次选取重量最小的物体装包 \to 只能得到近似解，不能得到最优解
            
            1. **每次选 p_i/w_i 最大的物品装包** \to 标准最合理，得到最优解
            
        
    
1. **Kruskal算法：**
    
    ![[IMG-20260405035404848.png|Untitled 2 8.png]]
    
    1. 每次找最小权值边（局部最优解）
    
    1. 满足边上的两个顶点不在同一颗树上（约束条件）
    
    1. 满足以上两个才加入到最优解集中
    
    1. 加入 n-1 条边结束
    
## **1.4 贪心算法其他算法的区别**
1. **「解决一个问题需要多个步骤，每一个步骤有多种选择」这样的描述我们在「回溯算法」「动态规划」算法中都会看到**
1. ==它们的区别如下：==
    
    - 「回溯算法」需要记录每一个步骤、每一个选择，用于回答所有具体解的问题
    
    - 「动态规划」需要记录的是每一个步骤、所有选择的汇总值（最大、最小或者计数）
    
    - 「贪心算法」由于适用的问题，每一个步骤只有一种选择，一般而言只需要记录与当前步骤相关的变量的值
    
    对于不同的求解目标和不同的问题场景，需要使用不同的算法

## **📑55. 跳跃游戏**🔥

> [!info] 力扣  
>  
> [https://leetcode.cn/problems/jump-game/?favorite=2cktkvj](https://leetcode.cn/problems/jump-game/?favorite=2cktkvj)  
1. 问题描述：
    
    ```Plain
     给定一个非负整数数组 nums ，你最初位于数组的 第一个下标 。
    
    数组中的每个元素代表你在该位置可以跳跃的最大长度。
    
    判断你是否能够到达最后一个下标。
    ```
    
1. 示例：
    
    ```Plain
    示例 1：
    输入：nums = [2,3,1,1,4]
    输出：true
    解释：可以先跳 1 步，从下标 0 到达下标 1, 然后再从下标 1 跳 3 步到达最后一个下标。
    
    示例 2：
    输入：nums = [3,2,1,0,4]
    输出：false
    解释：无论怎样，总会到达下标为 3 的位置。但该下标的最大跳跃长度是 0 ， 所以永远不可能到达最后一个下标。
    ```
    
1. 代码：
    
    - 其实跳几步无所谓，关键在于可跳的覆盖范围！
    
    - 不一定非要明确一次究竟跳几步，每次取最大的跳跃步数，这个就是可以跳跃的覆盖范围
    
    - 每次移动取最大跳跃步数（得到最大的覆盖范围），每移动一个单位，就更新最大覆盖范围
    
    - 贪心算法**局部最优解：**每次取最大跳跃步数（取最大覆盖范围），**整体最优解：**最后得到整体最大覆盖范围，看是否能到终点
    
    ```Java
    public boolean canJump(int[] nums) {
        if (nums.length == 1) return true;
        // 能到达的最远位置
        int maxCover = nums[0];
        // 在能到达的范围内移动
        for (int i = 0; i <= maxCover; i++) {
            // 每次贪心选择能走的最远步长，更新能到达的最远位置
            maxCover = Math.max(maxCover, i + nums[i]);
            // 如果能到达最后一个位置，则返回 true
            if (maxCover >= nums.length - 1) {
                return true;
            }
        }
        return false;
    }
    ```

1. 问题描述：
    
    ```Plain
     给定一个长度为 n 的 0 索引整数数组 nums。初始位置为 nums[0]。
    
    每个元素 nums[i] 表示从索引 i 向前跳转的最大长度。换句话说，如果你在 nums[i] 处，你可以跳转到任意 nums[i + j] 处:
    
    0 <= j <= nums[i] 
    i + j < n
    返回到达 nums[n - 1] 的最小跳跃次数。生成的测试用例可以到达 nums[n - 1]。
    ```
    
1. 示例：
    
    ```Plain
    输入: nums = [2,3,1,1,4]
    输出: 2
    解释: 跳到最后一个位置的最小跳跃数是 2。
         从下标为 0 跳到下标为 1 的位置，跳 1 步，然后跳 3 步到达数组的最后一个位置。
    ```
    
1. 代码：
    
    ```Java
     public class Jump {
        public static int jump(int[] nums) {
            int count = 0;
            int cur = 0;
            int maxCover = nums[0];
            // 从 0 起向后跳
            while (cur < nums.length - 1) {
                int next = 0;
                if (cur != nums.length - 1 && cur + nums[cur] >= nums.length - 1) {
                    return count + 1;
                }
                // 每次从当前位置，选择可达范围内的，能够到达最远位置的 next 下标
                for (int i = 0; i <= nums[cur]; i++) {
                    if (cur + i + nums[cur + i] > maxCover) {
                        maxCover = cur + i + nums[cur + i];
                        next = cur + i;
                    }
                }
                cur = next;
                count++;
            }
            return count;
        }
    }
    ```

1. 问题描述：
    
    ```Plain
     给你一个字符串 s 。我们要把这个字符串划分为尽可能多的片段，同一字母最多出现在一个片段中。
    
    注意，划分结果需要满足：将所有划分结果按顺序连接，得到的字符串仍然是 s 。
    
    返回一个表示每个字符串片段的长度的列表。
    ```
    
1. 示例：
    
    ```Plain
    输入：s = "ababcbacadefegdehijhklij"
    输出：[9,7,8]
    解释：
    划分结果为 "ababcbaca"、"defegde"、"hijhklij" 。
    每个字母最多出现在一个片段中。
    像 "ababcbacadefegde", "hijhklij" 这样的划分是错误的，因为划分的片段数较少。
    ```
    
1. 代码：
    
    ```Java
     public class PartitionLabels {
        public List<Integer> partitionLabels(String s) {
            ArrayList<Integer> res = new ArrayList<>();
            HashMap<Character, Integer> map = new HashMap<>();
            // 统计每个字符在 s 中的最后一个位置
            for (int i = 0; i < s.length(); i++) {
                map.put(s.charAt(i), i);
            }
    
            // start 和 end 保存当前分段中的起始和结束位置
            int start = 0;
            int end = 0;
            for (int i = 0; i < s.length(); i++) {
                // 当前分段的结束位置由遍历到的字符在 s 中的最后出现的位置决定
                end = Math.max(end, map.get(s.charAt(i)));
                // 如果当前分段遍历完，则记录结果
                if (i == end) {
                    res.add(i - start + 1);
                    start = i + 1;
                }
            }
            return res;
        }
    }
    ```
    
## **📑455. 分发饼干**
1. 问题描述：
    
    ```Plain
     假设你是一位很棒的家长，想要给你的孩子们一些小饼干。但是，每个孩子最多只能给一块饼干。
     
     对每个孩子 i，都有一个胃口值 g[i]，这是能让孩子们满足胃口的饼干的最小尺寸；并且每块饼干 j，都有一个尺寸 s[j] 。如果 s[j] >= g[i]，我们可以将这个饼干 j 分配给孩子 i ，这个孩子会得到满足。你的目标是尽可能满足越多数量的孩子，并输出这个最大数值
    ```
    
1. 示例：
    
    ```Plain
     输入: g = [1,2,3], s = [1,1]
     输出: 1
     解释:
     你有三个孩子和两块小饼干，3个孩子的胃口值分别是：1,2,3。
     虽然你有两块小饼干，由于他们的尺寸都是1，你只能让胃口值是1的孩子满足。
     所以你应该输出1
     
     输入: g = [1,2], s = [1,2,3]
     输出: 2
     解释:
     你有两个孩子和三块小饼干，2个孩子的胃口值分别是1,2。
     你拥有的饼干数量和尺寸都足以让所有孩子满足。
     所以你应该输出2.
    ```
    
1. 代码：
    
    > 贪心策略：将两个数组排序，保证贪心地每次取出最小的饼干来满足最小胃口的孩子
    
    ```Java
     public int findContentChildren(int[] g, int[] s) {
         //将两个数组排序
         Arrays.sort(g);
         Arrays.sort(s);
         int count = 0;
         //i在g[]上遍历，j在s[]上遍历
         for (int i = 0,j = 0; i <g.length && j < s.length ; i++,j++) {
             //如果当前饼干大小小于当前孩子的胃口
             while (g[i] > s[j]){
                 //在s[]上向后找更大的饼干
                 j++;
                 //如果s[]遍历完了也没找到合适的饼干，跳出while循环
                 if (j >= s.length){
                     break;
                 }
             }
             if (j < s.length){
                 count++;
             }
         }
         return count;
     }
    ```
    
## **📑1754. 构造字典序最大的合并字符串**
1. 问题描述：
    
    ```Plain
     给你两个字符串 word1 和 word2 。你需要按下述方式构造一个新字符串 merge ：如果 word1 或 word2 非空，选择 下面选项之一 继续操作：
     
     如果 word1 非空，将 word1 中的第一个字符附加到 merge 的末尾，并将其从 word1 中移除。
         例如，word1 = "abc" 且 merge = "dv" ，在执行此选项操作之后，word1 = "bc" ，同时 merge = "dva" 。
     如果 word2 非空，将 word2 中的第一个字符附加到 merge 的末尾，并将其从 word2 中移除。
         例如，word2 = "abc" 且 merge = "" ，在执行此选项操作之后，word2 = "bc" ，同时 merge = "a" 。
     返回你可以构造的字典序 最大 的合并字符串 merge 。
     
     长度相同的两个字符串 a 和 b 比较字典序大小，如果在 a 和 b 出现不同的第一个位置，a 中字符在字母表中的出现顺序位于 b 中相应字符之后，就认为字符串 a 按字典序比字符串 b 更大。例如，"abcd" 按字典序比 "abcc" 更大，因为两个字符串出现不同的第一个位置是第四个字符，而 d 在字母表中的出现顺序位于 c 之后。
    ```
    
1. 示例：
    
    ```Plain
     输入：word1 = "cabaa", word2 = "bcaaa"
     输出："cbcabaaaaa"
     解释：构造字典序最大的合并字符串，可行的一种方法如下所示：
     - 从 word1 中取第一个字符：merge = "c"，word1 = "abaa"，word2 = "bcaaa"
     - 从 word2 中取第一个字符：merge = "cb"，word1 = "abaa"，word2 = "caaa"
     - 从 word2 中取第一个字符：merge = "cbc"，word1 = "abaa"，word2 = "aaa"
     - 从 word1 中取第一个字符：merge = "cbca"，word1 = "baa"，word2 = "aaa"
     - 从 word1 中取第一个字符：merge = "cbcab"，word1 = "aa"，word2 = "aaa"
     - 将 word1 和 word2 中剩下的 5 个 a 附加到 merge 的末尾
    ```
    
1. 代码：
    
    > 贪心策略：如果字符串中，当前字符串及其之后的字串 字典序较大，则将其加入结果，同时对应的遍历指针+1
    > 
    > 1. 采用贪心的思想，word1和word2分别设置从0开始的i，j指针进行比较
    >     
    >     1. 如果`word1[i]`和`word1[j]`字母不同，则选大的字母加入结果并后移指针，指类似归并排序的merge()
    >     
    >     1. 如果`word1[i]`和`word1[j]`字母相同，例如字符串分别为aaa和aab，`word1[0] == word2[0]`，此时应当选择`word2[0]`加入结果，因为只有这样才能最快访问到 b，从而使合并的字符串的字典序最大
    >     
    > 
    > 1. 所以实现时将两种情况合并，**直接比较以**`**word1[i]**`**和**`**word1[j]**`**开头的字符串的字典序，来判断拼接**`**word1[i]**`**还是**`**word1[j]**`
    
    ```Java
     public static String largestMerge(String word1, String word2) {
         if (word1 == null || word2 == null || word1.length() == 0 || word2.length() == 0){
             return "";
         }
         //两个遍历指针
         int i = 0;
         int j = 0;
     
         StringBuffer result = new StringBuffer();
     
         //遍历两个字符串，如果一个为空另一个不为空的话，要把剩余的字符加入结果中
         while (i <  word1.length() || j < word2.length()){
             //如果字符串中，当前字符串及其之后的字串 字典序较大，则将其加入结果，同时对应的遍历指针+1
             if (word1.substring(i).compareTo(word2.substring(j)) > 0){
                 result.append(word1.charAt(i));
                 i++;
             }else {
                 result.append(word2.charAt(j));
                 j++;
             }
         }
         return result.toString();
     }
    ```
    
## **📑1547. 切棍子的最小成本**

> 这题是一道 （区间动态规划）

## **📑切金条**
1. 问题描述：
    
    ```Plain
     一块金条切成两半， 是需要花费和长度数值一样的铜板的。 比如长度为20的金条， 不管切成长度多大的两半， 都要花费20个铜板。
     一群人想整分整块金条， 怎么分最省铜板?
     输入一个数组， 返回分割的最小代价
    ```
    
1. 示例：
    
    ```Plain
     例如,给定数组{10,20,30}， 代表一共三个人， 整块金条长度为10+20+30=60。金条要分成10,20,30三个部分。 如果先把长度60的金条分成10和50， 花费60；再把长度50的金条分成20和30， 花费50； 一共花费110铜板。
     但是如果先把长度60的金条分成30和30， 花费60； 再把长度30金条分成10和20，花费30； 一共花费90铜板。
    ```
    
1. 代码：
    
    > 贪心策略：每次都取当前最小的两个元素
    
    以给定数组构造一棵哈夫曼树，每次将最小两个叶节点的相加值，累加到最后的成本中
    
    ```Java
    public int minCost(int n, int[] cuts) {
        PriorityQueue<Integer> heap = new PriorityQueue<>(new Comparator<Integer>() {
            @Override
            public int compare(Integer o1, Integer o2) {
                return o1 - o2;
            }
        });
    
        if (cuts == null || cuts.length == 0){
            return 0;
        }
    
        //数组元素入堆
        for (int cut : cuts) {
            heap.add(cut);
        }
    
        int sum = 0;
        //每次弹出最小两个相加，得到cur，将cur累加到结果，并再次入堆
        while (heap.size() > 1){
            int cur = heap.poll() + heap.poll();
            sum += cur;
            heap.add(cur);
        }
        return sum;
    }
    ```
    
## **📑最多宣讲次数**
1. 问题描述：
    
    ```Plain
    一些项目要占用一个会议室宣讲， 会议室不能同时容纳两个项目的宣讲。
    给你每一个项目开始的时间和结束的时间(给你一个数组， 里面是一个个具体的项目)， 你来安排宣讲的日程，要求会议室进行的宣讲的场次最多。
    返回这个最多的宣讲场次。
    ```
    
1. 示例：
    
1. 代码：
    
    > 贪心策略：每次安排结束时间最早的那个项目
    
    ```Java
    public class Code04_BestArrange {
    
    	public static class Program {
    		public int start;
    		public int end;
    
    		public Program(int start, int end) {
    			this.start = start;
    			this.end = end;
    		}
    	}
    
        //按结束时间递增排序
    	public static class ProgramComparator implements Comparator<Program> {
    		@Override
    		public int compare(Program o1, Program o2) {
    			return o1.end - o2.end;
    		}
    
    	}
    
    	public static int bestArrange(Program[] programs, int start) {
            //按结束时间排序
    		Arrays.sort(programs, new ProgramComparator());
            //记录安排宣讲的场次
    		int result = 0;
    		for (int i = 0; i < programs.length; i++) {
                //同时保证当前时间要早于要安排的项目宣讲的开始时间
    			if (start <= programs[i].start) {
    				result++;
    				start = programs[i].end;
    			}
    		}
    		return result;
    	}
    }
    ```
    
## **📑最大收益**
1. 问题描述：
    
    ```Plain
    输入：
        正数数组costs
        正数数组profits
        正数k
        正数m
    含义：
        costs[i]表示i号项目的花费
        profits[i]表示i号项目在扣除花费之后还能挣到的钱(利润)
        k表示你只能串行的最多做k个项目
        m表示你初始的资金
    说明：
        你每做完一个项目， 马上获得的收益， 可以支持你去做下一个项目。
    输出：
        你最后获得的最大钱数。
    ```
    
1. 示例：
    
1. 代码：
    
    > 贪心策略：每次根据当前本钱，选出可以做的项目，再从这些项目中选一个利润最大的做，这样重复选择 k 次
    > 
    > 由于两个堆需要分别对成本和收益设定比较器：
    > 
    > 1. 这里使用的方法是，将成本和收益封装成一个 Node 类的属性，往堆中添加的是 Node 对象，`Node.c` 为成本，`Node.p` 为利润
    > 
    > 1. 还可以往堆中添加 `int[]`，里面只存两个数， `int[0]` 为成本， `int[1]` 为利润
    
    ```Java
    class Node {
        public int p;
        public int c;
    
        public Node(int p, int c) {
            this.p = p;
            this.c = c;
        }
    }
    
    public class FindMaximizedCapital {
        public static int findMaximizedCapital(int k, int W, int[] profits, int[] costs) {
    
            //定义一个小根堆，将node按成本存放
            PriorityQueue<Node> costHeap = new PriorityQueue<>(new Comparator<Node>() {
                @Override
                public int compare(Node o1, Node o2) {
                    return o1.c - o2.c;
                }
            });
    
            //定义一个大根堆，将node按利润存放
            PriorityQueue<Node> profitHeap = new PriorityQueue<>(new Comparator<Node>() {
                @Override
                public int compare(Node o1, Node o2) {
                    return o2.p - o1.p;
                }
            });
    
            Node[] nodes = new Node[costs.length];
    
            for (int i = 0; i < costs.length; i++) {
                nodes[i] = new Node(costs[i], profits[i]);
            }
    
            //所有node进入小根堆
            costHeap.addAll(Arrays.asList(nodes));
    
            //选出k个项目
            for (int i = 0; i < k; i++) {
                //先取出当前本钱能满足的所有项目，加入大根堆
                while (!costHeap.isEmpty() && costHeap.peek().c <= W) {
                    profitHeap.add(costHeap.poll());
                }
                if (profitHeap.isEmpty()) {
                    return W;
                }
                //将大根堆堆顶弹出，即选出当前本钱能满足的，且利润最大的一个项目
                //同时跟新本钱
                W += profitHeap.poll().p;
            }
            return W;
        }
    }
    ```
