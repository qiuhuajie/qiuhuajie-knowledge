# 一、📑28. 找出字符串中第一个匹配项的下标
1. 问题描述：

    ```Plain
     给你两个字符串 haystack 和 needle ，请你在 haystack 字符串中找出 needle 字符串的第一个匹配项的下标（下标从 0 开始）。如果 needle 不是 haystack 的一部分，则返回  -1 。
    ```
2. 示例：

    ```Plain
    示例1：
    输入：haystack = "sadbutsad", needle = "sad"
    输出：0
    解释："sad" 在下标 0 和 6 处匹配。
    第一个匹配项的下标是 0 ，所以返回 0 。
    示例2：
    输入：haystack = "leetcode", needle = "leeto"
    输出：-1
    解释："leeto" 没有在 "leetcode" 中出现，所以返回 -1 。
    提示：
    1 <= haystack.length, needle.length <= 104
    haystack 和 needle 仅由小写英文字符组成
    ```
# 二、问题分析
1. 字符串搜索（匹配子串）是一个很经典也具有实际应用场景的问题
2. 针对不同难度定位（数据范围）有不同的解法：
    - 如果只是某道题的其中一个环节的话，我们可以直接调用语言自带的 `indexOf()` 方法
    - 如果是一道简单题（数据范围 $10^3$ ~ $10^4$）的话，我们可以使用「双指针」解法
    - 如果是一道中等题（数据范围 $10^5$ ~ $10^6$）的话，则是在考察我们 KMP 等字符串匹配算法

# 三、问题解法
## 1. 解法 1 ：朴素解法
1. 直观的解法的是：枚举原串中的每个字符作为「发起点」，每次从原串的「发起点」和匹配串的「首位」开始尝试匹配：
    - 匹配成功：返回本次匹配的原串「发起点」
    - 匹配失败：枚举原串的下一个「发起点」，重新尝试匹配
2. 代码：

    ```Java
    public class KMP {
        public static void main(String[] args) {
            String str = "abcabcababaccc";
            String match = "ababa";
            System.out.println(strStr(str, match));
        }
        public static int strStr(String haystack, String needle) {
            char[] str1 = haystack.toCharArray();
            char[] str2 = needle.toCharArray();
            //1. 枚举原串的发起点
            for (int i = 0; i <= str1.length - str2.length; i++) {
                //2. 每次都从原串的发起点和匹配传的0点开始尝试匹配
                int a = i;
                int b = 0;
                //3. 遇到相同的字符则同时向后移
                while (b < str2.length && str1[a] == str2[b]){
                    a++;
                    b++;
                }
                //4. 当遇到不同字符时，判断当前匹配串是否走到了最后，如果是，则找到的想要的结果
                if (b == str2.length){
                    return i;
                }
            }
            //5. 原串所有发起点都没有找到可以匹配的结果
            return -1;
        }
    }
    ```
3. 复杂度分析：
    - 时间复杂度：`n` 为原串的长度，`m` 为匹配串的长度。其中枚举的复杂度为 $O(n-m)$，构造和比较字符串的复杂度为$O(m)$ 。整体复杂度为 $O((n-m)*m)$
    - 空间复杂度：$O(1)$

## 2. 解法 2 ：KMP 解法
### 2.1 KMP 算法
1. KMP 算法是一个快速查找匹配串的算法，它的**作用**其实就是本题问题：==**如何快速在「原字符串」中找到「匹配字符串」的下标**==
2. 上述的朴素解法，复杂度近似是 $O((n-m)*m)$ 的，而 KMP 算法的复杂度为 $O(m+n)$
3. **KMP 之所以能够在 $O(m+n)$ 复杂度内完成查找，是因为其能在「非完全匹配」的过程中提取到有效信息进行复用，以减少「重复匹配」的消耗**
4. 在模拟 KMP 匹配过程之前，我们先建立两个概念：
    - **前缀**：对于字符串 `abcxxxxefg`，我们称 `abc` 属于 `abcxxxxefg` 的某个前缀
    - **后缀**：对于字符串 `abcxxxxefg`，我们称 `efg` 属于 `abcxxxxefg` 的某个后缀

### 2.2 举例说明
1. 设原串为 `abeababeabf`，匹配串为 `abeabf`：

    ![[IMG-20260620205920252.png|596]]

2. **匹配过程：**
    1. 首先在「原串」和「匹配串」分别各自有一个指针指向当前匹配的位置： `p1` `p2`
    2. 首次匹配的「发起点」是第一个字符 `a`。显然，后面的 `abeab` 都是匹配的，两个指针会同时往右移动
    3. 在都能匹配上 `abeab` 的部分，「朴素匹配」和「KMP」并无不同
    4. 直到出现第一个不同的位置，如下图位置：

        ![[IMG-20260620205920314.png|564]]

    5. **接下来，正是「朴素匹配」和「KMP」出现不同的地方：**
        - **先看下「朴素匹配」逻辑：**
            1. 将原串的指针 `p1` 移动至本次「发起点」的下一个位置（`b` 字符处）；匹配串的指针 `p2` 移动至起始位置（ `0` 位置处）

                ![[IMG-20260620205920373.png|587]]

            2. 尝试匹配，发现对不上，原串的指针 `p1` 会一直往后移动，直到能够与匹配串对上位置，会走到下一个 `a` 字符位置：

                ![[IMG-20260620205920429.png|586]]

            3. 两个指针通知往后走后，发现也匹配不上，原串的指针 `p1` 继续往后移动，走到下一个 `a` 字符位置，继续匹配，最后匹配成功

                ![[IMG-20260620205920489.png|588]]

            4. 也就是说，对于「朴素匹配」而言，一旦匹配失败，将会将原串指针调整至下一个「发起点」，匹配串的指针调整至起始位置，然后重新尝试匹配
            5. 这也就不难理解为什么「朴素匹配」的复杂度是  **$O(m*n)$**  了
        - ⭐**然后再看「KMP 匹配」过程：**
            1. 首先匹配串会检查之前已经匹配成功的部分中里是否存在相同的「前缀」和「后缀」，即找出了 `p2` 之前的**==最长公共前后缀==**： `ab`
            2. 即 `next[p2] = 2` ，故 `p2` 直接跳到下标为 `2` 的位置

                ![[IMG-20260620205920543.png|588]]

            3. 如果存在，则 `p1` 不动， `p2` 跳转到「前缀」的下一个位置，继续往下匹配：

                ![[IMG-20260620205920612.png|594]]

                > 💡 这里为什么
                > `p2` 可以直接跳到「前缀」的下一个位置，而无需回到 `0` 位置开始匹配
                > 两个原因：
                > 1. 第一：跳转之前 `p1` `p2` 之前的字符是一路匹配过来的，所以可以保证 绿色部分和橙色部分一定是相同的，又因为 橙色部分和黄色部分是 `p2` 之前的最长公共前后缀，所以这两个也一定是相同的，故可以保证 `p1` 不动 ， `p2` 直接跳到「前缀」的下一个位置后，它俩之前的绿色部分和黄色部分一定是相同的，可以省去重复的匹配过程
                > 2. 第二：可以保证 `i` 与 `p1` 之间一定不存在一个位置可以产生一个更长的绿色部分使得其等于下面的黄色部分。证明过程：如果存在一个更长的绿色部分使得其等于下面的黄色部分，由于 `p1` `p2` 之前的字符是一路匹配过来的，所以可以保证存在一个更长的橙色部分与上面存在的更长的绿色部分相同，但是橙色部分和黄色部分是 `p2` 之前的最长公共前后缀，这与前面存在一个更长的黄橙部分冲突，故一定不存在一个更长的绿色部分使得其等于下面的黄色部分
            4. 跳转到下一匹配位置后，尝试匹配，发现两个指针的字符对不上，并且此时匹配串指针 `p2` 前面不存在相同的「前缀」和「后缀」，即 `next[p2] = 0` ，故 `p2` 直接跳到下标为 `0` 的位置

                ![[IMG-20260620205920651.png|600]]

            5. 继续匹配后，发现匹配成功

                ![[IMG-20260620205920683.png|600]]

    6. 如果第 4 步中 `p2` 与 `p1` 指向的字符也不同，只能是将 `p1` 后移一位（从下一个原串的「发起点」开始，进行下一轮匹配）， `p2` 不动，继续正常的朴素匹配逻辑

### 2.3 KMP 为什么相比于朴素解法更快
1. **因为 KMP 利用已匹配部分中相同的「前缀」和「后缀」来加速下一次的匹配**
2. **因为 KMP 的原串指针不会进行回溯（没有朴素匹配中回到下一个「发起点」的过程）**
### 2.4 next 数组的构建
1. 接下来，我们看看 `next` 数组是如何在  的复杂度内被预处理出来的
2. 假设有匹配串 `aaabbab`，对应的 `next` 是如何被构建出来的
    1. 第一个默认是 - 1
    2. 第二个默认是 0
    3. 之后每一个 i 位置都要靠前面的 i-1 位置的元素得出
    4. 对于 i 位置来说，如果 next 数组的 i-1 位置是 x，那么需要在 str2 里面的 x 位置的元素跟 i-1 位置的元素比较
    5. 如果一样，那么 next 数组中，**i 位置的元素是 x+1**
3. 这就是整个 `next` 数组的构建过程，时空复杂度均为 **$O(m)$**   
4. 至此整个 KMP 匹配过程复杂度是  **$O(m+n)$**   的
# 四、KMP 代码实现
```Java
public class KMP {
    public static int strStr(String s, String m) {
        if (s == null || m == null || m.length() < 1 || s.length() < m.length()) {
            return -1;
        }
        int p1 = 0;
        int p2 = 0;
        char[] str1 = s.toCharArray();
        char[] str2 = m.toCharArray();
        // 构造next数组
        int[] next = getNextArray(str2);
				// 开始匹配
        while (p1 < str1.length && p2 < str2.length) {
            // 相同：一起向后匹配
            if (str1[p1] == str2[p2]) {
                p1++;
                p2++;
            } else if (next[p2] == -1) {
                // p2 此时在匹配串的 0 位置，且 str1[p1] != str2[p2]，那么需要将 p1 移至下一个发起点
                p1++;
            } else {
                // 不相同：则 p1 不动， p2 跳转到「前缀」的下一个位置
                p2 = next[p2];
            }
            // 继续往下匹配
        }
				// p2是否走到了匹配串的最后
        return p2 == str2.length ? p1 - p2 : -1;
    }
    public static int[] getNextArray(char[] ms) {
        if (ms.length == 1) {
            return new int[] { -1 };
        }
        int[] next = new int[ms.length];
        // 0、 -1 位置是固定设置的
        next[0] = -1;
        next[1] = 0;
        // 从下标 2 位置开始设置值
        int i = 2;
        int cn = 0;
        while (i < next.length) {
            if (ms[i - 1] == ms[cn]) {
                next[i++] = ++cn;
            } else if (cn > 0) {
                cn = next[cn];
            } else {
                next[i++] = 0;
            }
        }
        return next;
    }
    public static void main(String[] args) {
        String str = "abcabcababaccc";
        String match = "ababa";
        System.out.println(strStr(str, match));
    }
}
```
