## 4. 视图
1. 视图（View）并不在数据库中实际存在，而是一种虚拟表

    ![[IMG-20260404032026136.png|800]]

2. ==视图的优点：==
    1. 视图能简化用户的操作
    2. 视图机制可以使用户以不同的方式查询同一数据
    3. 视图对数据库重构提供了一定程度的逻辑独立性
    4. 视图可以对机密的数据提供安全保护
3. 视图相当于一种多表关联的 SQL 查询语句，所以在执行时的效率很低
4. 可以利用物化视图解决上述问题：将视图的内容物理存储起来，其数据随原始表变化，同步更新

## 5. 分区分表分库

![[IMG-20260404032026571.png|800]]

1. 分区分表的差异

    ![[IMG-20260404032026633.png|800]]

2. 分区的方式

    ![[IMG-20260404032026670.png|800]]

3. 分区的优点：
    1. 相对于单个文件系统或是硬盘，分区可以存储更多的数据
    2. 数据管理比较方便，比如要清理或废弃某年的数据，就可以直接删除该日期的分区数据即可
    3. 精准定位分区查询数据，不需要全表扫描查询，大大提高数据检索效率
    4. 可跨多个分区磁盘查询，来提高查询的吞吐量
    5. 在涉及聚合函数查询时，可以很容易进行数据的合并
