# 一、在浏览器中输入 URL 之后，它会执行什么流程❓
1. **解析 URL**
    1. 浏览器会解析输入的URL
    2. 将其分成协议（protocol），主机名（hostname）、端口（port）、路径（path）和查询字符串（query string）等组成部分
2. **DNS 域名解析**
    1. （Domain Name System，域名系统）域名解析，**找到与域名相对应的 IP 地址**
    2. 浏览器会依次检查：**浏览器的 DNS 缓存 ➡️ 操作系统的缓存 ➡️ 本地 `host` 文件**
    3. 如果都没有找到对应的 IP 地址，就会向 DNS服务器发送一个查询请求来获取目标服务器的IP地址
        1. 会先请求**本地 DNS 服务器**，本地 DNS 服务器一般是由本地网络服务商提供，目前用的比较多的是谷歌提供的公用 DNS 是 `8.8.8.8` 和国内的公用 DNS 是 `114.114.114.114`
        2. 如果本地 DNS 服务器没有相应的记录，就会去**根域名服务器**查询了
            1. 全球一共有 13 组根域名服务器的逻辑实例
            2. **各层域名服务器**都了解该层以下分层所有的域名服务器的IP地址，因此它们**从根服务器开始呈现树状结构相互连接**
            3. 查询时，根据域名服务器开始顺序进行

                ![[IMG-20260315141441606.png|800]]

        > ❗ **为什么不直接访问 IP 地址来请求服务器？**
        >
        > 1. 因为 IP 地址很长，不方便记忆，而 URL 地址好记很多，所以会使用 URL 来替代 IP 地址
        >
        > 2. 而 URL 就像 IP 地址的别名一样，用它可以定位到相应的 IP 地址
3. **建立 TCP 连接：**
    1. 浏览器通过TCP/IP协议建立与目标服务器的连接
    2. 三次握手过程
4. **发送请求：**
    1. 浏览器向目标服务器发送一个HTTP请求
    2. 包括请求的**方法**（GET、POST等）、**路径**、**HTTP 版本号**、**请求头**（Accept、Accept-Encoding、User-Agent等）和**请求体**（对于POST请求）等信息[[7.1 HTTP 介绍]][[7.1 HTTP 介绍]][[7.1 HTTP 介绍]]

        ![[IMG-20260405035535212.png|800]]

5. **服务器处理请求**：目标服务器接收到请求后，会根据请求的信息执行相应的操作，例如读取数据库、生成动态HTML页面等
6. **服务器发送响应**：
    1. 服务器根据请求处理结果，生成HTTP响应
    2. 包括**HTTP版本号**、**状态码**（200、404等）、**响应头**（Content-Type、Content-Encoding等）和**响应体**（HTML、CSS、JavaScript、图片、视频等）等信息[[7.1 HTTP 介绍]][[7.1 HTTP 介绍]][[7.1 HTTP 介绍]]

    ![[IMG-20260405035547248.png|800]]

7. **断开 TCP 连接：**
    1. 浏览器通过TCP/IP协议断开与目标服务器的连接
    2. 四次挥手过程
8. **浏览器接收响应：**
    1. 浏览器接收到HTTP响应后，会根据响应头信息解析响应体
    2. 然后根据情况执行缓存、解压缩、重定向等操作
9. **渲染页面：**浏览器根据响应体中的HTML、CSS和JavaScript等内容，解析渲染页面，并将其呈现给用户
