# 一、HTTP 的风险
1. **明文缺点：**
    1. HTTP 的所有信息都暴露在了光天化日下，相当于信息裸奔
    2. 带来的严重的缺点就是不安全：
        1. ==**数据泄露：**==通信使用明文（不加密），内容可能会被泄露。比如，账号信息容易泄漏
        2. ==**身份伪装：**==不验证通信方的身份，因此有可能遭遇伪装。比如，访问假的淘宝
        3. ==**内容篡改：**==无法证明报文的完整性，所以有可能已遭篡改。比如，网页上植入垃圾广告

# 二、HTTPS
1. ⭐TTPS 并不是一种新的协议，而是==在 HTTP 协议的基础上添加了==**==加密机制 SSL==**==（Secure Socket Layer）或== **==TLS==**==（Transport Layer Security）==

    ![[IMG-20260405035535184.png|745]]

2. ==**HTTPS = HTTP + 混合加密 + 数字证书认证 + 校验和完整性保护**==
3. 端口号：
    1. HTTP 的端口号是 **`80`**
    2. HTTPS 的端口号是 **`443`**
4. ==HTTPS 在 TCP三次握手之后，还需进行== ==**SSL/TLS 的握手过程**====（非对称加密），才可进入==**==通信传输==**==（对称加密）==，可以很好的解决了上述的风险：
    1. **信息加密：**交互信息无法被窃取
    2. **身份证书：**证明淘宝是真的淘宝网
    3. **校验机制：**无法篡改通信内容，篡改了就不能正常显示

# 三、SSL/TLS 协议
1. **SSL 和 TLS 没有太大的区别，TLS 是基于 SSL 之上的升级版本**
2. 很多相关的文章都把这两者并列称呼（SSL/TLS），因为这两者可以视作同一个东西的不同阶段（加多宝和王老吉）
