# 1. 介绍

![[IMG-20260315213656178.png|Untitled 256.png]]

1. **介绍：**
    
    1. 抽象工厂模式（Abstract Factory Pattern）是围绕一个超级工厂创建其他工厂
    
    1. 抽象工厂模式就是对**简单工厂模式的改进（或者称为进一步的抽象）**：将工厂抽象成两层， AbsFactory（抽象工厂） 和 具体实现的工厂子类
    
    1. 程序员可以根据创建对象类型使用对应的工厂子类。 这样**将单个的简单工厂类变成了工厂簇，更利于代码的维护和扩展**
    
1. ==**意图：**==**==提供一个创建一系列相关或相互依赖对象的接口，而无需指定它们具体的类==**
1. ==**主要解决：**====**主要解决接口选择的问题**==
1. **何时使用：**系统的产品有多于一个的产品族，而系统只消费其中某一族的产品
1. **如何解决：**==就是当产品类型多了的时候，根据某个维度给产品分类，针对每种不同类型的产品（产品族）单独提供一个工厂，而所有工厂又有一个抽象大工厂，这样便于维护==
1. **关键代码：**在一个工厂里聚合多个同类产品
1. **优点：**当一个产品族中的多个对象被设计成一起工作时，它能保证客户端始终只使用同一个**产品族**中的对象
1. **缺点：**产品族扩展非常困难，要增加一个系列的某一产品，既要在抽象的 Creator 里加代码，又要在具体的里面加代码
1. ==**使用场景：**== 
    
    1. QQ 换皮肤，一整套一起换
    
    1. 生成不同操作系统的程序
    
1. **注意事项：**产品族难扩展，产品等级易扩展
# 2. 登场角色
## 2.1 Abstract Factory 模式中有以下登场角色
1. **AbstractProduct(抽象产品)**
    

    AbstractProduct 角色负责定义AbstractFactory 角色所生成的抽象零件和产品的接口(API)

    
1. **AbstractFactory(抽象工厂)**
    

    AbstractFactory角色负责定义用于生成抽象产品的接口(API)

    
1. **Client(委托者 )**
    

    Client角色仅会调用AbstractFactory 角色和AbstractProduct 角色的接口(API)来进行工作，对于具体的零件、产品和工厂一无所知

    
1. **ConcreteProduct(具体产品)**
    

    ConcreteProduct 角色负责实现AbstractProduct 角色的接口(API)

    
1. **ConcreteFactory( 具体工厂)**
    

    ConcreteFactory 角色负责实现 AbstractFactory 角色的接口(API)

# 3. 代码示例
## 3.1 新需求
1. 客户在点披萨时，可以点不同产地的披萨，比如 北京的奶酪pizza、北京的香肠pizza；或者是纽约的奶酪pizza、 纽约的香肠pizza
1. 思路1：
    
    1. 使用简单工厂模式，创建不同的简单工厂类，比如`BJPizzaSimpleFactory`、`NYPizzaSimpleFactory` 等等
    
    1. 从当前这个案例来说，也是可以的，但是考虑到项目的规模，以及软件的可维护性、可扩展性并不是特别好
    
1. 思路2：
    

    使用抽象工厂模式

    
## 3.2 使用抽象工厂模式实现
1. Pizza 抽象类 ➡️ **AbstractProduct(抽象产品)**
    
    ```Java
    public abstract class Pizza {
        protected String name;
    
        public abstract void bake();
    
        public void setName(String name) {
            this.name = name;
        }
    }
    ```
    
1. SausagePizz、SausagePizz 继承 Pizza ➡️ **ConcreteProduct(具体的产品)**
    
    ```Java
    public class CheesePizza extends Pizza {
        @Override
        public void bake() {
            System.out.println(name + " is baked");
        }
    }
    ```
    
    ```Java
    public class SausagePizza extends Pizza{
        @Override
        public void bake() {
            System.out.println(name + " is baked");
        }
    }
    ```
    
1. ⭐创建一个抽象工厂接口：`AbsPizzaFactory` ➡️ **AbstractFactory(抽象工厂)**
    
    ```Java
    public interface AbsPizzaFactory {
        Pizza createPizza(String type);
    }
    ```
    
1. 创建 北京、纽约两个工厂 ➡️ **ConcreteFactory( 具体工厂)**
    
    ```Java
    class BJPizzaFactory implements AbsPizzaFactory {
        public Pizza createPizza(String type) {
            Pizza pizza = null;
            if (type.equals("sausage")) {
                pizza = new SausagePizza();
                pizza.setName("BJ-香肠披萨");
            } else if (type.equals("cheese")) {
                pizza = new CheesePizza();
                pizza.setName("BJ-奶酪披萨");
            }
            return pizza;
        }
    }
    ```
    
    ```Java
    class NYPizzaFactory implements AbsPizzaFactory {
        public Pizza createPizza(String type) {
            Pizza pizza = null;
            if (type.equals("sausage")) {
                pizza = new SausagePizza();
                pizza.setName("NY-香肠披萨");
            } else if (type.equals("cheese")) {
                pizza = new CheesePizza();
                pizza.setName("NY-奶酪披萨");
            }
            return pizza;
        }
    }
    ```
    
1. 订单服务 A，在订单服务中创建 Pizza 实例 ➡️ **Client(委托者 )**
    
    > [!important] Client角色仅会调用AbstractFactory 角色和AbstractProduct 角色的接口(API)来进行工作，对于具体的零件、产品和工厂一无所知
    
    ```Java
    public class PizzaOrderA {
    	AbsPizzaFactory pizzaFactory = null;
    	Pizza pizza = null;
    
    	public PizzaOrderA(AbsPizzaFactory pizzaFactory, String orderType) {
    		this.pizzaFactory = pizzaFactory;
    
    		pizza = pizzaFactory.createPizza(orderType);
    		// 制作
    		this.pizza.bake();
    	}
    }
    ```
    
1. 订单服务 B，代码与订单服务 A 完全一样，也在订单服务中创建 Pizza 实例 ➡️ **Client(委托者 )**
    
    ```Java
    public class PizzaOrderB {
    
    	AbsPizzaFactory pizzaFactory = null;
      Pizza pizza = null;
    
    	public PizzaOrderB(AbsPizzaFactory pizzaFactory, String orderType) {
    		this.pizzaFactory = pizzaFactory;
    
    		pizza = pizzaFactory.createPizza(orderType);
    		// 制作
    		this.pizza.bake();
    	}
    }
    ```
    
1. `SimpleFactoryDemo`
    
    ```Java
    public class SimpleFactoryDemo {
        public static void main(String[] args) {
            BJPizzaFactory bjPizzaFactory = new BJPizzaFactory();
            new PizzaOrderA(bjPizzaFactory, "sausage");
        }
    }
    ```

     * 获得策略的标识

```Java
package com.alibaba.example.factoryDemo;
import java.util.Collection;
import java.util.HashMap;
import java.util.Map;
import org.springframework.beans.BeansException;
import org.springframework.beans.factory.InitializingBean;
import org.springframework.context.ApplicationContext;
import org.springframework.context.ApplicationContextAware;
/**
 * 抽象工厂
 * @author qiutian
 * @date 2023/9/12
 */
public abstract class StrategyFactory<T, S extends Strategy<T>>
        implements InitializingBean, ApplicationContextAware {
    private Map<T, S> strategyMap;
    private ApplicationContext appContext;
    /**
     * 根据策略 id 获得对应的策略的 Bean
     *
     * @param id 策略 id
     * @return 策略的 Bean
     */
    public S getStrategy(T id) {
        return strategyMap.get(id);
    }
    /**
     * 获取策略的类型（交给子类去实现）
     *
     * @return 策略的类型
     */
    protected abstract Class<S> getStrategyType();
    @Override
    public void afterPropertiesSet() {
        // 获取 Spring 容器中，所有 S 类型的 Bean
        Collection<S> strategies = appContext.getBeansOfType(getStrategyType()).values();
        strategyMap = new HashMap<>(strategies.size());
        // 将所有 S 类型的 Bean 放入到 strategyMap 中
        for (final S strategy : strategies) {
            T id = strategy.getId();
            strategyMap.put(id, strategy);
        }
    }
    @Override
    public void setApplicationContext(ApplicationContext applicationContext) throws BeansException {
        appContext = applicationContext;
    }
}
```
