# 1. 介绍

![[IMG-20260315213656163.png|Untitled 254.png]]

1. **介绍：**单例模式（Singleton Pattern）是 Java 中最简单的设计模式之一
1. ==**意图：**====**保证一个类仅有一个实例，并提供一个访问它的全局访问点**==
1. ==**主要解决：**====**一个全局使用的类频繁地创建与销毁**==
1. **何时使用：**当您想控制实例数目，节省系统资源的时候
1. **如何解决：**判断系统是否已经有这个单例，如果有则返回，如果没有则创建
1. **关键代码：**构造函数是私有的
1. **应用实例：**
    
    - 一个班级只有一个班主任
    
    - Windows 是多进程多线程的，在操作一个文件的时候，就不可避免地出现多个进程或线程同时操作一个文件的现象，所以所有文件的处理必须通过唯一的实例来进行
    
    - 一些设备管理器常常设计为单例模式，比如一个电脑有两台打印机，在输出的时候就要处理不能两台打印机打印同一个文件
    
1. **优点：**
    
    - 在内存里只有一个实例，减少了内存的开销，尤其是频繁的创建和销毁实例（比如管理学院首页页面缓存）
    
    - 避免对资源的多重占用（比如写文件操作）
    
1. **缺点：**没有接口，不能继承，与单一职责原则冲突，一个类应该只关心内部逻辑，而不关心外面怎么样来实例化
1. ==**使用场景：**==🙋‍♂️
    
    - WEB 中的计数器，不用每次刷新都在数据库里加一次，用单例先缓存起来
    
    - 创建的一个对象需要消耗的资源过多，比如：线程池、数据库配置、数据库连接池
    
    - 应用程序的日志
    
1. **注意事项：**`getInstance()` 方法中需要使用同步锁 `synchronized` (Singleton.class) 防止多线程同时进入造成 `instance` 被多次实例化

# 3. 代码示例
- ==**编写时的注意点：**==
    
    1. 1️⃣ **保证本类只能有一个实例：**通过==构造器==**==私有==**==化==实现
    
    1. 2️⃣ **它必须自行创建这个实例：**类定义中含有一个该类的**==静态+私有==**==对象==
    
    1. 3️⃣ **对外提供获取该实例对象的方式：**该类提供了一个**==静态+公有==**==的函数==用于==创建或获取==它本身的静态私有对象
    
## 3.1 **饿汉式（线程安全 + 非懒加载）**
1. **描述：**
    
    1. ==在类加载时已经创建好该单例对象，等待被程序使用==
        
        1. 它**基于 `classloader` （类加载）机制来保证初始化实例时只有一个线程**，进而避免了多线程的同步问题
        
        1. **依赖于 `static` 静态关键字**，类在加载时会在堆内存中创建一个 Singleton 对象，当类被卸载时，Singleton 对象也随之消亡了
        
    
    1. 优点：没有加锁，执行效率会提高
    
    1. 缺点：类加载时就初始化，容易产生垃圾对象，浪费内存
    
1. **代码示例：**
    
    ```Java
    public class Singleton {  
    		//创建 SingleObject 的一个对象
        private static Singleton instance = new Singleton();
    
        //让构造函数为 private，这样该类就不会被实例化
        private Singleton (){}  
    
        //获取唯一可用的对象
        public static Singleton getInstance() {  
    		    return instance;  
        }  
    }
    ```
    
## 3.2 **懒汉式（线程安全 + 懒加载）**
1. **描述：**不推荐
    
    1. ==在不加== `instance == null` ==的话，会存在线程安全问题==：**如果两个线程同时判断** `instance` **为空，那么它们都会去实例化一个** `instance` **对象，这就变成双例了**
    
    1. 故下面的这种懒汉式==使用同步锁== `synchronized` ==解决线程安全问题==
        
        1. 解决方法就是在方法上加锁或者使用同步代码块加锁
        
        1. 规避了两个线程同时创建 `instance` 对象的风险
        
    
    1. 🚨**但引来一个新问题：**
        
        1. 这种方式是直接将 `getInstance()` 方法锁住了
        
        1. 任何线程去获取对象都需要先获取锁，并发性能非常地差，极端情况下，可能会出现卡顿现象
        
    
1. **代码示例：**
    
    ```Java
    public class Singleton {  
        private static Singleton instance;  
        private Singleton (){}  
    
        public static synchronized Singleton getInstance() {  
            if (instance == null) {  
                instance = new Singleton();  
            }
            return instance;  
        }  
    }
    ```
    
## 3.3 **双重校验锁 DCL（线程安全 + 懒加载）⭐**
1. **描述：**==推荐使用==
    
    1. Double Check（双重校验） + Lock（加锁）：因为需要 **两次判空** `**if (singleton == null)**` **+ 对类对象加锁**
    
    1. 如果不加第一个判断条件的话，所有的创建请求都会去获取当前 synchronized 这个类锁，而加了第一个判空条件的话，在单例对象创建后，之后的代码就不会进如获取锁的代码了，提高性能
        
        1. 如果没有实例化对象则加锁创建
        
        1. 如果已经实例化了，在第一个判空条件后，直接获取实例，不需要加锁
        
    
    1. 同时，==对懒汉式创建单例对象线程安全问题进一步做了优化，双重检查锁==**==避免整个方法被锁住，而是对类对象加锁==**
    
    1. 完美地解决了并发安全 + 性能低效问题
    
1. **代码示例：**
    
    ```Java
    public class Singleton {  
        private volatile static Singleton singleton;    //⭐ static volatile
        private Singleton (){}                          //⭐ private
        public static Singleton getSingleton() {        //⭐ static
    		    if (singleton == null) {                    // 1️⃣
    		        synchronized (Singleton.class) {        //⭐ 锁住类对象
    		            if (singleton == null) {  
    		                singleton = new Singleton();    // 2️⃣
    		            }  
    		        }  
    		    }  
    		    return singleton;  
        }  
    }
    ```
    
    > [!important] **为什么单例一定要加**
    > 
    > `**volatile**`
    > 
    > 1. `volatile` 的两个作用：保证可见性和防止指令重排序[（详见 volatile）](https://www.notion.so/volatile-f0195f74770741e39bd671a640625457?pvs=21)
    > 
    > 1. 给私有变量 `singleton` 加 `volatile` 主要是为了⭐==防止 2️⃣ 处（====`singleton = new Singleton();`====）执行时的指令重排序==。这行代码的实际执行分为以下 3 步：
    > 
    >     1. 创建内存空间
    > 
    >     1. 在内存空间中初始化对象 `Singleton`
    > 
    >     1. 将内存地址赋值给 `instance` 对象（执行了此步骤，`instance` 就不等于 `null` 了）
    > 
    > 
    > 1. ==如果不加== ==`volatile`====，那么线程 1 在执行到 2️⃣ 处时就可能会执行指令重排序，将原本是 1、2、3 的执行顺序，重排为 1、3、2==
    > 
    > 1. ==单线程下是没有问题的，====但是====**多线程**====情况下：==
    > 
    >     1. 如果，线程 1 在执行完第 3 步之后，线程 2 执行到 1️⃣ 处时，判断 `instance` 对象已经不为 `null`，但此时线程 1 还未将对象实例化完
    > 
    >     1. ==那么线程 2 将会得到一个被实例化“一半”的对象==，从而导致程序执行出错
    >

## 3.4 **静态内部类（线程安全 + 懒加载） ⭐**
1. 这种方式能达到双检锁方式一样的功效，但实现更简单
1. **描述：**==推荐使用==
    
    1. 线程安全：
        
        1. 这种方式**同饿汉式一样利用了 `classloader` （类加载）机制来保证初始化 instance 时只有一个线程**
        
    
    1. 懒加载：
        
        1. 但它跟饿汉式不同的是，这种方式**利用了**==**静态内部类在被调用时，才被加载的特性，实现了懒加载**==
        
        1. 饿汉式只要 `Singleton` 类被装载了，那么 `instance` 就会被实例化（没有达到 lazy loading 效果）
        
        1. 而这种方式是 `Singleton` 类被装载了，`instance` 不一定被初始化
        
        1. 因为**静态内部类 `SingletonHolder` 类**中 **`Singleton INSTANCE` 被声明为一个静态变量**
            
            1. 只有在这个静态变量首次被访问时，这个内部类才会被初始化[[类的生命周期]]
            
            1. 即**只有通过显式调用 `getInstance` 方法时，才会显式装载 `SingletonHolder` 类**，从而实例化 `instance`
            
        
        1. 这种方式相比饿汉式就显得很合理，达到了 lazy loading 效果
        
    
1. **代码示例：**
    
    ```Java
    public class Singleton {  
        private static class SingletonHolder {  
    		    private static final Singleton INSTANCE = new Singleton();  
        }  
        private Singleton (){}  
        public static final Singleton getInstance() {  
            return SingletonHolder.INSTANCE;  
        }  
    }
    ```
    
## 3.5 **枚举（线程安全 + 非懒加载）⭐**
1. **JDK 版本：**JDK1.5 起
1. **描述：**==推荐使用==
    
    1. ==这种方式不仅能避免多线程同步问题，而且还能防止反射或者反序列化操作重新创建新的对象==
    
    1. 优点：
        
        1. 这种方式是 Effective Java 作者 Josh Bloch 提倡的方式
        
        1. 它不仅能避免多线程同步问题，而且还**自动支持序列化机制**，**防止反序列化时重新创建新的对象，绝对防止多次实例化**
        
    
    1. 不过，由于 JDK1.5 之后才加入 `enum` 特性，这种实现方式还没有被广泛采用，但这是实现单例模式的最佳方法
    
    1. 不能通过 `reflection attack` 来调用私有构造方法
    
1. **代码示例：**
    
    ```Java
    public enum Singleton {  
        INSTANCE;  
        public void whateverMethod() {  
        }  
    }
    ```
