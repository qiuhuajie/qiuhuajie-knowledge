# 一、介绍

![[IMG-20260315213656270.png|Untitled 260.png]]

1. **介绍：**
    1. 桥接（Bridge）是用于把抽象化与实现化解耦，使得二者可以独立变化。这种类型的设计模式属于结构型模式，它通过提供抽象化和实现化之间的桥接结构，来实现二者的解耦
    1. 这种模式涉及到一个作为桥接的接口，使得实体类的功能独立于接口实现类。这两种类型的类可被结构化改变而互不影响
    1. 我们通过下面的实例来演示桥接模式（Bridge Pattern）的用法。其中，可以使用相同的抽象类方法但是不同的桥接实现类，来画出不同颜色的圆。
1. ==**意图：**==**==将抽象部分与实现部分分离，使它们都可以独立的变化==**
1. ==**主要解决：**==**==在有多种可能会变化的情况下，用继承会造成类爆炸问题，扩展起来不灵活==**
1. **何时使用：**实现系统可能有多个角度分类，每一种角度都可能变化
1. **如何解决：**把这种多角度分类分离出来，让它们独立变化，减少它们之间耦合
1. **关键代码：**抽象类依赖实现类
1. **应用实例：**
    1. 猪八戒从天蓬元帅转世投胎到猪，转世投胎的机制将尘世划分为两个等级，即：灵魂和肉体，前者相当于抽象化，后者相当于实现化。生灵通过功能的委派，调用肉体对象的功能，使得生灵可以动态地选择
    1. 墙上的开关，可以看到的开关是抽象的，不用管里面具体怎么实现的
1. **优点：**
    1. 抽象和实现的分离
    1. 优秀的扩展能力
    1. 实现细节对客户透明
1. **缺点：**桥接模式的引入会增加系统的理解与设计难度，由于聚合关联关系建立在抽象层，要求开发者针对抽象进行设计与编程
1. **使用场景：**
    1. 如果一个系统需要在构件的抽象化角色和具体化角色之间增加更多的灵活性，避免在两个层次之间建立静态的继承联系，通过桥接模式可以使它们在抽象层建立一个关联关系
    1. 对于那些不希望使用继承或因为多层次继承导致系统类的个数急剧增加的系统，桥接模式尤为适用
    1. 一个类存在两个独立变化的维度，且这两个维度都需要进行扩展
1. **注意事项：**对于两个独立变化的维度，使用桥接模式再适合不过了
# 二、登场角色
## 1. Bridge 模式中有以下登场角色
1. **Abstraction(抽象化)**
    1. 该角色位于“类的功能层次结构”的最上层 ➡️ **充当了桥的作用**
    1. 它使用Implementor 角色的方法定义了基本的功能
    1. 该角色中保存了Implementor 角色的实例
1. **RefinedAbstraction(改善后的抽象化)**

    在Abstraction 角色的基础上增加了新功能的角色

1. **Implementor(实现者)**
    1. 该角色位于“类的实现层次结构”的最上层
    1. 它定义了用于实现Abstraction 角色的接口(API)的方法
1. **Concretelmplementor( 具体实现者)**

    该角色负责实现在Implementor 角色中定义的接口(API)

# 三、代码示例
## 1. 需求
- 需求：现在对不同手机类型的不同品牌实现操作编程（比如:开机、关机、上网，打电话等）
## 2. 传统方案解决手机操作问题

![[IMG-20260315213657499.png|Untitled 2 159.png]]

- 存在的问题：
    1. 扩展性问题(类爆炸)， **如果我们再增加手机的样式(旋转式)，就需要增加各个品牌手机的类，同样如果我们增加一个手机品牌，也要在各个手机样式类下增加**
    1. 违反了单一职责原则，当我们增加手机样式时，要同时增加所有品牌的手机，这样增加了代码维护成本.
- 解决方案：使用桥接模式 ⬇️
## 3. 桥接模式解决手机操作问题

![[IMG-20260315213658196.png|Untitled 3 120.png]]

1. `Brand` ➡️ **Implementor(实现者)**

    ```Java
    public interface Brand {
        void call();
    }
    ```
1. `Brand` 的实现类：`Vivo` 、`XiaoMi` ➡️ **Concretelmplementor( 具体实现者)**

    ```Java
    public class Vivo implements Brand{
        @Override
        public void call() {
            System.out.println("Vivo 手机 call");
        }
    }
    ```
    ```Java
    public class XiaoMi implements Brand{
        @Override
        public void call() {
            System.out.println("XiaoMi 手机 call");
        }
    }
    ```
1. `Phone` ➡️ **Abstraction(抽象化)**

    ```Java
    public abstract class Phone {
        private Brand brand;
        public Phone(Brand brand) {
            this.brand = brand;
        }
        protected void call(){
            brand.call();
        }
    }
    ```
1. `Phone` 的子类：`FoldedPhone` 、`UpRightPhone` ➡️ **RefinedAbstraction(改善后的抽象化)**

    ```Java
    public class FoldedPhone extends Phone{
        public FoldedPhone(Brand brand) {
            super(brand);
        }
        public void call(){
            super.call();
            System.out.println("折叠样式手机");
        }
    }
    ```
    ```Java
    public class UpRightPhone extends Phone{
        public UpRightPhone(Brand brand) {
            super(brand);
        }
        public void call(){
            super.call();
            System.out.println("滑盖样式手机");
        }
    }
    ```
1. `BridgeDemo`

    ```Java
    public class BridgeDemo {
        public static void main(String[] args) {
            FoldedPhone foldedPhone = new FoldedPhone(new XiaoMi());
            foldedPhone.call();
            System.out.println("-----------------");
            UpRightPhone upRightPhone = new UpRightPhone(new Vivo());
            upRightPhone.call();
        }
    }
    ```
1. 运行结果

    ```Java
    XiaoMi 手机 call
    折叠样式手机
    -----------------
    Vivo 手机 call
    滑盖样式手机
    ```
