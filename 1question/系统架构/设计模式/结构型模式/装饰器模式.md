# 1. 介绍
![[IMG-20260315213656284.png|Untitled 261.png]]
1. **介绍：**
    
    1. 装饰器模式（Decorator Pattern）允许向一个现有的对象添加新的功能，同时又不改变其结构。这种类型的设计模式属于结构型模式，它是作为现有的类的一个包装
    
    1. 这种模式创建了一个装饰类，用来包装原有的类，并在保持类方法签名完整性的前提下，提供了额外的功能
    
    1. 我们通过下面的实例来演示装饰器模式的用法。其中，我们将把一个形状装饰上不同的颜色，同时又不改变形状类
    
1. ==**意图：**==**==动态地给一个对象添加一些额外的职责。就增加功能来说，装饰器模式相比生成子类更为灵活==**
1. ==**主要解决：**==**==一般的，我们为了扩展一个类经常使用继承方式实现，由于继承为类引入静态特征，并且随着扩展功能的增多，子类会很膨胀==**
1. **何时使用：**在不想增加很多子类的情况下扩展类
1. **如何解决：**将具体功能职责划分，同时继承装饰者模式
1. **关键代码：**
    
    1. Component 类充当抽象角色，不应该具体实现
    
    1. 修饰类引用和继承 Component 类，具体扩展类重写父类方法
    
1. **应用实例：**
    
    1. 孙悟空有 72 变，当他变成"庙宇"后，他的根本还是一只猴子，但是他又有了庙宇的功能
    
    1. 不论一幅画有没有画框都可以挂在墙上，但是通常都是有画框的，并且实际上是画框被挂在墙上。在挂在墙上之前，画可以被蒙上玻璃，装到框子里；这时画、玻璃和画框形成了一个物体
    
1. **优点：**装饰类和被装饰类可以独立发展，不会相互耦合，装饰模式是继承的一个替代模式，装饰模式可以动态扩展一个实现类的功能
1. **缺点：**多层装饰比较复杂
1. ==**使用场景：**==
    
    1. 扩展一个类的功能
    
    1. 动态增加功能，动态撤销
    
1. **注意事项：**可代替继承
# 2. 登场角色
## 2.1 Decorator 模式中有以下登场角色
1. **Component(被装饰物)**
    
    1. 增加功能时的核心角色。举例子来说，被装饰前的蛋糕就是 Component角色
    
    1. Component角色只是定义了蛋糕的接口(API)
    
1. **ConcreteComponent( 具体的被装饰物)**
    
    该角色是实现了Component角色所定义的接口(API)的具体蛋糕
    
1. **Decorator(装饰物)**
    
    1. 该角色具有与Component角色相同的接口(API)
    
    1. ==在它内部聚合了被装饰对象 Component 角色==，Decorator 角色知道自己要装饰的对象
    
1. **ConcreteDecorator( 具体的装饰物)**
    
    该角色是具体的Decorator 角色

## 3.3 装饰器模式解决咖啡订单
1. `Coffee` ➡️ **Component(被装饰物)**
    
    ```Java
    @Data
    public abstract class Coffee {
    
        private String description;
        private float price = 0.0f;
    
        abstract public float cost();
    }
    ```
    
1. `Coffee` 子类：`ShortBlack` 、`Espresso` ➡️ **ConcreteComponent( 具体的被装饰物)**
    
    ```Java
    public class ShortBlack extends Coffee{
    
        public ShortBlack() {
            setDescription("美式咖啡");
            setPrice(5.0f);
        }
    
        @Override
        public float cost() {
            return super.getPrice();
        }
    }
    ```
    
    ```Java
    public class Espresso extends Coffee{
    
        public Espresso() {
            setDescription("意大利浓咖啡");
            setPrice(6.0f);
        }
    
        @Override
        public float cost() {
            return super.getPrice();
        }
    }
    ```
    
1. `Decorator` ➡️ **Decorator(装饰物)**
    
    ```Java
    public class Decorator extends Coffee{
    
        private Coffee coffee;    //⭐装饰物内部聚合了被装饰对象 Component 角色
    
        public Decorator(Coffee coffee) {
            this.coffee = coffee;
        }
    
        @Override
        public float cost() {
            return super.getPrice() + coffee.cost();
        }
    
        @Override
        public String getDescription() {
            return super.getDescription() + super.getPrice() + "+" + coffee.getDescription();
        }
    }
    ```
    
1. `Decorator` 子类：`Milk` 、`Chocolate` ➡️ **ConcreteDecorator( 具体的装饰物)**
    
    ```Java
    public class Milk extends Decorator{
        public Milk(Coffee coffee) {
            super(coffee);
            setDescription("牛奶");
            setPrice(2.0f);
        }
    }
    ```
    
    ```Java
    public class Chocolate extends Decorator{
    
        public Chocolate(Coffee coffee) {
            super(coffee);
            setDescription("巧克力");
            setPrice(3.0f);
        }
    }
    ```
    
1. `DecoratorDemo` ➡️
    
    ```Java
    public class DecoratorDemo {
        public static void main(String[] args) {
            Coffee order = new ShortBlack();
            System.out.println("当前订单描述：" + order.getDescription());
            System.out.println("当前费用：" + order.cost());
    
            order= new Milk(order);
            System.out.println("当前订单描述：" + order.getDescription());
            System.out.println("当前费用：" + order.cost());
    				
            // ⭐要对订货添料时，只需要一层一层包装即可
            order= new Chocolate(order);
            System.out.println("当前订单描述：" + order.getDescription());
            System.out.println("当前费用：" + order.cost());
        }
    }
    ```
    
1. 输出结果：
    
    ```Java
    当前订单描述：美式咖啡
    当前费用：5.0
    当前订单描述：牛奶2.0+美式咖啡
    当前费用：7.0
    当前订单描述：巧克力3.0+牛奶2.0+美式咖啡
    当前费用：10.0
    ```
