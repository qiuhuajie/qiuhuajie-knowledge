# 一、介绍
![[IMG-20260315213656288.png|Untitled 259.png]]
1. 介绍：
    1. 适配器模式（Adapter Pattern）是作为两个不兼容的接口之间的桥梁。这种类型的设计模式属于结构型模式，它结合了两个独立接口的功能
    1. 这种模式涉及到一个单一的类，该类负责加入独立的或不兼容的接口功能
    1. 举例子，如果想让额定工作电压是直流12伏特的笔记本电脑在交流 100伏特的 AC 电源下工作，会使用AC适配器
        1. 将家庭用的交流 100 伏特电转换成我们所需要的直流12伏特电压
        1. 这就是适配器的工作，它位于实际情况与需求之间，填补两者之间的差异

            ![[IMG-20260315213656479.png|Untitled 1 189.png]]
    1. Adapter模式有以下两种：
        - 类适配器模式（使用继承的适配器）
        - 对象适配器模式（使用委托的适配器）
1. ==**意图：**====**将一个类的接口转换成客户希望的另外一个接口。适配器模式使得原本由于接口不兼容而不能一起工作的那些类可以一起工作**==
1. ==**主要解决：**====**主要解决在软件系统中，常常要将一些"现存的对象"放到新的环境中，而新环境要求的接口是现对象不能满足的**==
1. **何时使用：**
    1. 系统需要使用现有的类，而此类的接口不符合系统的需要
    1. 想要建立一个可以重复使用的类，用于与一些彼此之间没有太大关联的一些类，包括一些可能在将来引进的类一起工作，这些源类不一定有一致的接口
    1. 通过接口转换，将一个类插入另一个类系中
    1. 比如：**老虎和飞禽，现在多了一个飞虎，在不增加实体的需求下，增加一个适配器，在里面包容一个虎对象，实现飞的接口**
1. **如何解决：**继承或依赖（推荐）
1. **关键代码：**适配器继承或依赖已有的对象，实现想要的目标接口
1. **应用实例：**
    1. 美国电器 110V，中国 220V，就要有一个适配器将 110V 转化为 220V
    1. JAVA JDK 1.1 提供了 Enumeration 接口，而在 1.2 中提供了 Iterator 接口，想要使用 1.2 的 JDK，则要将以前系统的 Enumeration 接口转化为 Iterator 接口，这时就需要适配器模式
    1. 在 LINUX 上运行 WINDOWS 程序
    1. JAVA 中的 jdbc
1. **优点：**
    1. 可以让任何两个没有关联的类一起运行
    1. 提高了类的复用
    1. 增加了类的透明度
    1. 灵活性好
1. **缺点：**
    1. 过多地使用适配器，会让系统非常零乱，不易整体进行把握、
        1. 比如，明明看到调用的是 A 接口，其实内部被适配成了 B 接口的实现，一个系统如果太多出现这种情况，无异于一场灾难
        1. 因此如果不是很有必要，可以不使用适配器，而是直接对系统进行重构
    1. 由于 JAVA 至多继承一个类，所以至多只能适配一个适配者类，而且目标类必须是抽象类
1. ==**使用场景：**==
    1. **==有动机地修改一个正常运行的系统的接口，这时应该考虑使用适配器模式==**
    1. Mabatis 多种日志框架的整合
    1. SpringMVC 适配器 `handlerAdapter` **（详见 下面的 4. 适配器模式在 SpringMVC 框架应用的源码分析）**
    1. 新老版本的兼容问题
1. **注意事项：适配器不是在详细设计时添加的，而是解决正在服役的项目的问题**
# 二、登场角色
## 1. Adapter 模式中有以下登场角色
1. **Target(对象)**
    1. 该角色负责定义所需的方法
    1. 即让手机正常工作所需的直流 5伏特电源
1. **Client(请求者)**
    1. 该角色负责使用 Target 角色所定义的方法进行具体处理
    1. 即直流 5 伏特电源所驱动的手机
1. **Adaptee(被适配)**
    1. 注意不是Adapt-er (适配)角色，而是Adapt-ee (被适配)角色
    1. Adaptee 是一个持有既定方法的角色
    1. 即交流 220 伏特电源
1. **Adapter(适配器)**
    1. Adapter模式的主人公
    1. 使用Adaptee 角色的方法来满足Target 角色的需求，这是Adapter 模式的目的，也是 Adapter 角色的作用
    1. Adapter 角色就是将交流 220 伏特电压转换为直流12伏特电压的适配器
    1. 在类适配器模式中，Adapter角色通过继承来使用Adaptee 角色，而在对象适配器模式中Adapter角色通过委托来使用Adaptee角色

# 三、类适配器模式（使用继承的适配器）
## 1. 代码示例
1. `Voltage220V`**(被适配)**

    ```Java
    public class Voltage220V {
        // 输出 220V 电压
        public int output220V(){
            int src = 220;
            System.out.println("output voltage " + src);
            return src;
        }
    }
    ```
1. `Voltage5V`**Target(对象)**

    ```Java
    public interface Voltage5V {
        int output5V();
    }
    ```
1. `VoltageAdapter`**Adapter(适配器) ==在类适配器模式中，Adapter角色通过继承来使用Adaptee 角色==**

    ```Java
    public class VoltageAdapter extends Voltage220V implements Voltage5V {   // ⭐在类适配器模式中，Adapter角色通过继承来使用Adaptee 角色
        @Override
        public int output5V() {
            int src = output220V();
            int dst = src / 44;
            return dst;
        }
    }
    ```
1. `Phone`**Client(请求者)**

    ```Java
    public class Phone {
        public void charging(Voltage5V voltage5V){
            int output = voltage5V.output5V();
            System.out.println("正在充电，电压为 " + output + "V");
        }
    }
    ```
1. `AdapterDemo`

    ```Java
    public class AdapterDemo {
        public static void main(String[] args) {
            Phone phone = new Phone();
            phone.charging(new VoltageAdapter());
        }
    }
    ```

    ```Java

    正在充电，电压为 5V
    ```

# 四、对象适配器模式（使用委托的适配器）
> ❗ **思考：适配器模式与策略模式的区别**
>
> 1. 适配器模式：
>
>     1. 是在想使用一个已经存在的类，但是他的接口并不符合要求，因为在编码过程中要遵循对扩展开放，对修改关闭的原则，所以不能对原有的类进行修改
>
>     1. 这时便需要使用适配器模式，将原有的类适配成自己需要的形式。有类适配器和对象适配器两种适配器
>
>     1. 举个简单的例子，==A 要使用 B，发现 B 不能直接拿来使用，因此用一个 实现了 A 的适配器 C，然后在 C 中调用 B 的方法，实现 A 使用 B 的功能==
>
>
> 1. 策略模式：
>
>     1. 定义了一系列的算法，并将每一个算法封装起来，而且使它们还可以相互替换。策略模式让算法独立于使用它的客户而独立变化； 使用的关键点是面对对象、面向接口编程
>
>     1. 举个例子，==武士可以不断更换武器==
>
## 1. 代码示例
1. `Voltage220V`**(被适配)**

    ```Java
    public class Voltage220V {
        // 输出 220V 电压
        public int output220V(){
            int src = 220;
            System.out.println("output voltage " + src);
            return src;
        }
    }
    ```
1. `Voltage5V`**Target(对象)**

    ```Java
    public interface Voltage5V {
        int output5V();
    }
    ```
1. `VoltageAdapter`**Adapter(适配器) ==在对象适配器模式中Adapter角色通过委托来使用Adaptee角色==**

    ```Java
    public class VoltageAdapter implements Voltage5V {
        private Voltage220V voltage220V;   // ⭐在对象适配器模式中Adapter角色通过委托来使用Adaptee角色
        public VoltageAdapter(Voltage220V voltage220V) {
            this.voltage220V = voltage220V;
        }
        @Override
        public int output5V() {
            int src = voltage220V.output220V();
            int dst = src / 44;
            return dst;
        }
    }
    ```

    ```Java
    public class Phone {
        public void charging(Voltage5V voltage5V){
            int output = voltage5V.output5V();
            System.out.println("正在充电，电压为 " + output + "V");
        }
    }
    ```
1. `AdapterDemo`

    ```Java
    public class AdapterDemo {
        public static void main(String[] args) {
            Phone phone = new Phone();
            phone.charging(new VoltageAdapter(new Voltage220V()));
        }
    }
    ```

    ```Java
    正在充电，电压为 5V
    ```

## 1. DispatcherServlet 源码
1. `DispatcherServlet` 在处理请求调用的过程中，调用到类中的 `doDispatch()` 方法[[处理请求(调用)的过程]][[处理请求(调用)的过程]][[处理请求(调用)的过程]]
1. **DispatcherServlet.class** ：⭐**`doDispatch()`**

    ```Java

         HttpServletRequest processedRequest = request;
         HandlerExecutionChain mappedHandler = null;
         boolean multipartRequestParsed = false;
         WebAsyncManager asyncManager = WebAsyncUtils.getAsyncManager(request);
         try {
             ModelAndView mv = null;
             Exception dispatchException = null;

                 processedRequest = checkMultipart(request);
                 multipartRequestParsed = (processedRequest != request);
                 // ⭐根据 request 找到对应处理请求的 mappedHandler
                 mappedHandler = getHandler(processedRequest);

                 }

                 HandlerAdapter ha = getHandlerAdapter(mappedHandler.getHandler());
                 ...
                 // 调用拦截器的 preHandle()

             }
             catch (Throwable err) {

             }
             // 后续处理：处理模型数据和渲染视图
    ```

## 2. ⭐这里为什么要使用 Adapter 模式
> ❗ 思考：
>
> 1. 这里 spring 的应用看起来更像是策略模式的思想
>
> 1. 而真正的传统的适配器思想是，==A 要使用 B，发现 B 不能直接拿来使用，因此用一个 实现了 A 的适配器 C，然后在 C 中调用 B 的方法，实现 A 使用 B 的功能==
>
> 1. ==如果强行解释的话，那么就是== **==handler 不能直接拿来用，因为不知道是哪一种 handler，所以要用一个 adpter 适配一下，也就是先判断是哪种 handler，然后在适配器里调用对应的 handler 处理方法==**
1. 会发现 Spring MVC 并没有一个名为 Handler 的类，**在所有源码用到 handler 的地方，它都是一个 `Object` 对象**
1. 因此要执行它，==**首先得判断它是一个什么类型的处理器**==
1. 而如果正常编写调用，就需要使用多个 `if else` 判断 `instance of`

    ```Java
    if(handler instanceof HttpRequestHandler){
    	 handle1();//HttpRequestHandler的处理方式
    } else if (handler instanceof Controller){
    	 handle2();//Controller的处理方式
    } else if (handler instanceof Servlet){
    	 handle3();//Servlet的处理方式
    } ...
    ```
1. 这种方法再添加新的处理器实现方式时，需要修改源码：新增 else 添加处理逻辑，不符合对扩展开放，对修改关闭原则
1. 使用了 `HandlerAdapter` 后，**当需要新增一种处理器（Controller）时，只需要新增一个对应的 `HandlerAdapter` 即可**，⭐**==判断逻辑在他的==** ==**`supports**`== **==方法中，处理逻辑在==** ==`**handle`**== **==中，不需要修改任何其他代码==**
1. ==**所以这里的适配器也可以看作高级版的**== **`if-else`**
## 3. HandlerAdapter
1. `HandlerAdapter` 接口有如下具体的处理请求适配器
    > 在 DispatcherServlet 初始化的过程中会将几种适配器初始化到`handlerAdapters` 中[[初始化的过程]][[初始化的过程]]

    ![[IMG-20260315213659578.png|Untitled 6 70.png]]
    - `SimpleControllerHandlerAdapter` ：当 Contrller 类继承了 Contrller 类时，请求处理时使用的适配器
    - `HttpRequestHandlerAdapter` ：处理 Http 请求时使用的适配器
    - `RequestMappingHandlerAdapter` ：当 Contrller 使用了 `@Controller` 时，请求处理时使用的适配器
1. `HandlerAdapter` 接口中定义了三个方法

    ![[IMG-20260315224944295.png|Untitled 7 60.png]]
    ```Java
    public interface HandlerAdapter {
        // 判断是否支持传入的handler
        boolean supports(Object handler);
        // 使用handler处理请求
        ModelAndView handle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception;
        // 获取资源的LastModified值
        long getLastModified(HttpServletRequest request, Object handler);
    }
    ```
    1. ⭐`==**supports()**==` ==**方法的用途：**==在 `getHandlerAdapter()` 方法中

        ```Java
        protected HandlerAdapter getHandlerAdapter(Object handler) throws ServletException {
            if (this.handlerAdapters != null) {
                Iterator var2 = this.handlerAdapters.iterator();
                while(var2.hasNext()) {
                    HandlerAdapter adapter = (HandlerAdapter)var2.next();
        						// 如果此适配器支持当前传入的handler，则返回该适配器
                    if (adapter.supports(handler)) {
                        return adapter;
                    }
                }
            }
            throw new ServletException("No adapter for handler [" + handler + "]: The DispatcherServlet configuration needs to include a HandlerAdapter that supports this handler");
        }
        ```
        - 这里会遍历 `handlerAdapters` 中的所有 `handlerAdapter` ，并通过每个遍历到的 `handlerAdapter` 的 `supports()` 方法来判断需要使用哪个类型的 `handlerAdapter`
        - 例如下面的 `HttpRequestHandlerAdapter` 的 `supports()` 方法，如果判断传入的 handler 是 `HttpRequestHandler` 类的实例，则表示 `HttpRequestHandler` 支持当前请求

            ```Java
            public boolean supports(Object handler) {
                return handler instanceof HttpRequestHandler;
            }
            ```
        - 如果 `adapter.supports(handler)` 返回 `true`，则 `getHandlerAdapter` 就会将支持当前请求的 `handlerAdapter` 返回
    1. ⭐**`==handle()==`** **==方法的用途：==**在 DispatcherServlet.class 的 `doDispatch()` 方法中

        ```Java
         protected void doDispatch(HttpServletRequest request, HttpServletResponse response) throws Exception {
             HttpServletRequest processedRequest = request;
             HandlerExecutionChain mappedHandler = null;
             ...
                     // ⭐根据 request 找到对应处理请求的 mappedHandler
                     mappedHandler = getHandler(processedRequest);
                     if (mappedHandler == null) {
                         noHandlerFound(processedRequest, response);
                         return;
                     }
                     // ⭐通过控制器方法创建相应的处理器适配器，调用所对应的控制器方法
                     HandlerAdapter ha = getHandlerAdapter(mappedHandler.getHandler());
                     ...
                     // ⭐⭐由处理器适配器调用具体的handle()控制器方法，最终获得ModelAndView对象
                     mv = ha.handle(processedRequest, response, mappedHandler.getHandler());
        	           ...
             ...
         }
        ```
        - 这里 `getHandlerAdapter` 返回支持当前请求的 `handlerAdapter` 后
        - `ha.handle(processedRequest, response, mappedHandler.getHandler());` 中**实际调用**的便是上面选中的具体的某个 `handlerAdapter` 实现类的 `handle()` 方法
        - 例如下面的 `HttpRequestHandlerAdapter` 的 `handle()` 方法

            ```Java
            @Nullable
            public ModelAndView handle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
                ((HttpRequestHandler)handler).handleRequest(request, response);
                return null;
            }
            ```
            - 因为 `handlerAdapter` 是之前执行过 `supports()` 方法选择出来的，故只要执行到 `HttpRequestHandlerAdapter` 的 `handle()` 方法，那么可以确定传入的 `handler` 一定是 `HttpRequestHandler` 类型的
            - **因此这里直接将传入的** `handler` **强转成** `HttpRequestHandler` **类型**
            - 故再调用 `handleRequest()` 后，一定是 `HttpRequestHandler` 中的 `handleRequest()` 方法

## 4. 手写简单的 HandlerAdapter 适配器模式
1. 有三种 Controller ：HttpController 、SimpleController 、AnnotationController

    ```Java
    public interface Controller {
    }
    class HttpController implements Controller {
        public void doHttpHandler() {
            System.out.println("http...");
        }
    }
    class SimpleController implements Controller {
        public void doSimplerHandler() {
            System.out.println("simple...");
        }
    }
    class AnnotationController implements Controller {
        public void doAnnotationHandler() {
            System.out.println("annotation...");
        }
    }
    ```
1. 现在的需求是根据不同的 Controller 执行其对应的 doxxxHandler 方法
1. 首先针对每一种不同的 Controller 创建一个实现了 HandlerAdapter 接口的 HandlerAdapter

    ```Java
    //定义一个Adapter接口
    public interface HandlerAdapter {
        boolean supports(Object handler);
        void handle(Object handler);
    }
    // 多种适配器类
    class SimpleHandlerAdapter implements HandlerAdapter {
        public boolean supports(Object handler) {
            return (handler instanceof SimpleController);
        }
        public void handle(Object handler) {
            ((SimpleController) handler).doSimplerHandler();
        }
    }
    class HttpHandlerAdapter implements HandlerAdapter {
        public boolean supports(Object handler) {
            return (handler instanceof HttpController);
        }
        public void handle(Object handler) {
            ((HttpController) handler).doHttpHandler();
        }
    }
    class AnnotationHandlerAdapter implements HandlerAdapter {
        public boolean supports(Object handler) {
            return (handler instanceof AnnotationController);
        }
        public void handle(Object handler) {
            ((AnnotationController) handler).doAnnotationHandler();
        }
    }
    ```
1. 实现一个简易版的 DispatchServlet

    ```Java
    public class DispatchServlet {
        public static List<HandlerAdapter> handlerAdapters = new ArrayList<HandlerAdapter>();
        // 初始化 handlerAdapters
        public DispatchServlet() {
            handlerAdapters.add(new AnnotationHandlerAdapter());
            handlerAdapters.add(new HttpHandlerAdapter());
            handlerAdapters.add(new SimpleHandlerAdapter());
        }
        public void doDispatch() {
            HttpController controller = new HttpController();
            //AnnotationController controller = new AnnotationController();
            //SimpleController controller = new SimpleController();
            // 得到对应适配器
            HandlerAdapter adapter = getHandlerAdapter(controller);
            // 通过适配器执行对应的 controller 中的 doxxxHandler 方法
            adapter.handle(controller);
        }
        public HandlerAdapter getHandlerAdapter(Controller controller) {
            //遍历：根据得到的controller(handler), 返回对应适配器
            for (HandlerAdapter adapter : this.handlerAdapters) {
                if (adapter.supports(controller)) {
                    return adapter;
                }
            }
            return null;
        }
        public static void main(String[] args) {
            new DispatchServlet().doDispatch(); // http...
        }
    }
    ```
