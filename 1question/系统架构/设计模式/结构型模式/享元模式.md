# 一、介绍
![[IMG-20260315213656259.png|Untitled 264.png]]
**共享对象，避免浪费**
1. **介绍：**
    1. Flyweight 是“轻量级”的意思，指的是拳击比赛中选手体重最轻的等级。顾名思义，该设计模式的作用是为了让对象变“轻”
    1. 对象在计算机中是虚拟存在的东西，它的“重”和“轻”并非指实际重量，而是它们“所使用的内存大小”。使用内存多的对象就是“重”对象，使用内存少的对象就是“轻”对象
    1. 为了能够在计算机中保存该对象，需要分配给其足够的内存空间。当程序中需要大量对象时如果都使用new关键字来分配内存，将会消耗大量内存空间
    1. ⭐**当需要某个实例时，并不总是通过 new 关键字来生成实例，而是尽量共用已经存在的实例**这就是Flyweight模式的核心内容
1. ==**意图：**====**运用共享技术有效地支持大量细粒度的对象**==
1. ==**主要解决：**==**==在有大量对象时，有可能会造成内存溢出，我们把其中共同的部分抽象出来，如果有相同的业务请求，直接返回在内存中已有的对象，避免重新创建==**
1. **何时使用：**
    1. 系统中有大量对象
    1. 这些对象消耗大量内存
    1. 这些对象的状态大部分可以外部化
    1. 这些对象可以按照内蕴状态分为很多组，当把外蕴对象从对象中剔除出来时，每一组对象都可以用一个对象来代替
    1. 系统不依赖于这些对象身份，这些对象是不可分辨的
1. **如何解决：**用唯一标识码判断，如果在内存中有，则返回这个唯一标识码所标识的对象
1. **关键代码：**用 HashMap 存储这些对象
1. **应用实例：**
    1. JAVA 中的 String，如果有则返回，如果没有则创建一个字符串保存在字符串缓存池里面
    1. 数据库的数据池
1. **优点：**大大减少对象的创建，降低系统的内存，使效率提高
1. **缺点：**提高了系统的复杂度，需要分离出外部状态和内部状态，而且外部状态具有固有化的性质，不应该随着内部状态的变化而变化，否则会造成系统的混乱
1. **使用场景：**
    1. 系统有大量相似对象
    1. 享元模式能够解决重复对象的内存浪费的问题，当系统中有大量相似对象，需要==**缓冲池**==时。 不需总是创建新对象，可以从缓冲池里拿。这样可以降低系统内存，同时提高效率
    1. 享元模式经典的应用场景就是池技术了， ==**String常量池**==[[StringTable（实现在堆中）]]==**、数据库连接池**==[**（详见 SpringBoot 3. 数据访问）**](https://www.notion.so/3-d32c79cade054444a7d6e92ffa780c39?pvs=21)==**、缓冲池**==等等都是享元模式的应用，享元模式是池技术的重要实现方式
    1. 像数据库连接池，里面都是创建好的连接对象，在这些连接对象中有我们需要的则直接拿来用，避免重新创建；如果没有我们需要的，则创建一个
1. **注意事项：**
    1. 注意划分外部状态和内部状态，否则可能会引起线程安全问题
    1. 这些类必须有一个工厂对象加以控制

# 二、登场角色
## 1. Flyweight 模式中有以下登场角色
1. **Flyweight(享元角色)**
    1. 按照通常方式编写程序会导致程序变重，所以如果能够共享实例会比较好，而Flyweight角色表示的就是那些实例会被共享的类
    1. 享元角色中包含内部状态和外部状态（使用享元模式时，注意**划分**内部状态和外部状态）
        1. **内部状态：**指对象**共享出来的信息**，存储在享元对象内部且不会随环境的改变而改变（例如 棋子的颜色）
        1. **外部状态：**指对象得以依赖的一个标记，是随环境改变而改变的、**不可共享**的状态（例如 棋子的位置）
1. **FlyweightFactory(享元角色创建工厂)**
    1. FlyweightFactory角色是生成Flyweight角色的工厂
    1. 在工厂中生成Fyweight 角色可以实现共享实例
1. **Client(请求者)**

    Client角色使用FlyweightFactory 角色来生成Flyweight 角色

# 三、代码示例
## 1. 需求描述
小型的外包项目，给客户A做一个产品展示网站， 客户B、客户C 也感觉效果不错，也希望做这样的产品展示网站，但是要求都有些不同：
1. 有客户要求以新闻的形式发布
1. 有客户人要求以博客的形式发布
1. 有客户希望以微信公众号的形式发布
## 2. 传统方案解决网站展现项目
1. 方案：直接复制粘贴一份，然后根据客户不同要求，进行定制修改
1. 存在的问题：需要的网站结构相似度很高，而且都不是高访问量网站，如果分成多个虚拟空间来处理，相当于一个相同网站的实例对象很多，造成服务器的资源浪费
1. 解决思路：整合到一个网站中，共享其相关的代码和数据，对于硬盘、内存、 CPU、数据库空间等服务器资源都可以达成共享，减少服务器资源
1. 上面的解决思路就可以使用 享元模式 来解决 ⬇️
## 3. 享元模式解决网站展现项目
1. `WebSite` ➡️ **享元角色的抽象类**

    ```Java
    public abstract class WebSite {
        public abstract void use(User user);
    }
    ```
1. `ConcreteWebSite` ➡️ **Flyweight(享元角色)**

    ```Java
    public class ConcreteWebSite extends WebSite{
        private String type = "";
        public ConcreteWebSite(String type) {
            this.type = type;
        }
        @Override
        public void use(User user) {
            System.out.println("网站的发布形式为：" + type + " " + user.getName());
        }
    }
    ```
1. `User` ➡️ 这里 `User` 就是**外部状态**；而享元角色自己的 `type` 属性就是**内部状态**

    ```Java
    @Data
    @AllArgsConstructor
    @NoArgsConstructor
    public class User {
        private String name;
    }
    ```
1. `WebSiteFactory` ➡️ **FlyweightFactory(享元角色创建工厂)**

    ```Java
    public class WebSiteFactory {
        HashMap<String, ConcreteWebSite> pool = new HashMap<>();    // 用一个 HashMap 来保存要被共享的对象
        public WebSite getWebSite(String type){
            if (!pool.containsKey(type)){
                pool.put(type, new ConcreteWebSite(type));
            }
            return (WebSite) pool.get(type);
        }
        public Integer getWebSiteCount(){
            return pool.size();
        }
    }
    ```
1. `FlyweightDemo` ➡️ **Client(请求者)**

    ```Java
    public class FlyweightDemo {
        public static void main(String[] args) {
            WebSiteFactory webSiteFactory = new WebSiteFactory();
            WebSite webSite1 = webSiteFactory.getWebSite("新闻");
            webSite1.use(new User("张三"));
            WebSite webSite2 = webSiteFactory.getWebSite("新闻");
            webSite2.use(new User("李四"));
            WebSite webSite3 = webSiteFactory.getWebSite("微薄");
            webSite3.use(new User("王五"));
            Integer webSiteCount = webSiteFactory.getWebSiteCount();
            System.out.println("网站的类型数：" + webSiteCount);
        }
    }
    ```
1. 运行结果

    ```Java
    网站的发布形式为：新闻 张三
    网站的发布形式为：新闻 李四
    网站的发布形式为：微薄 王五
    网站的类型数：2
    ```

# 四、享元模式在 JDK-Interger 的应用源码分析
1. 演示代码：

    ```Java
    Integer x = Integer.valueOf(127);
    Integer y = new Integer(127);
    Integer z = Integer.valueOf(127);
    Integer w = new Integer(127);
    System.out.println(x.equals(y)); // true
    System.out.println(x == y ); // false
    System.out.println(x == z ); // true
    System.out.println(w == x ); // false
    System.out.println(w == y ); // false
    ```
1. **`Integer.valueOf()`** 源码：

    ![[IMG-20260315213658101.png|Untitled 3 124.png]]
    - 当传入的 `int` 数据的数值范围 在`IntegerCache.low` 与 `IntegerCache.high` 之间，则会从享元角色数组（`IntegerCache.cache[]` ）中返回一个 `Interger` 对象
    - 如果不在这个返回则会新 `new` 一个 `Interger` 对象

        ```Java
        Integer x1 = Integer.valueOf(127);
        Integer x2 = Integer.valueOf(127);
        System.out.println(x1 == x2 );      // true
        Integer x3 = Integer.valueOf(200);
        Integer x4 = Integer.valueOf(200);
        System.out.println(x3 == x4 );      // false
        ```
1. **`IntegerCache`** 是 `Interger` 类的一个内部类 ➡️ **FlyweightFactory(享元角色创建工厂)**
    1. 这里初始化了 `IntegerCache.low` 与 `IntegerCache.high` 两个范围静态遍历：`-128，127`
    1. 同时初始化了**`cache[]`** 这个 `Interger` 类数组，**用与保存要被共享的对象**
