# 一、介绍

![[IMG-20260315213656261.png|Untitled 265.png]]

1. **介绍：**
    1. 在代理模式（Proxy Pattern）就是 **使用**==**代理对象**==**来代替对**==**真实对象**==**的访问，这样就可以在不修改原目标对象的前提下，**==**扩展目标对象的功能**==
    1. 这种类型的设计模式属于结构型模式
1. **意图：**为其他对象提供一种代理以控制对这个对象的访问
1. **主要解决：**
    1. 在直接访问对象时带来的问题，比如说：要访问的对象在远程的机器上
    1. 在面向对象系统中，有些对象由于某些原因（比如对象创建开销很大，或者某些操作需要安全控制，或者需要进程外的访问），直接访问会给使用者或者系统结构带来很多麻烦
    1. 我们可以在访问此对象时加上一个对此对象的访问层
1. **何时使用：**想在访问一个类时做一些控制
1. **如何解决：**增加中间层
1. **关键代码：**实现与被代理类组合
1. **应用实例：**
    1. Windows 里面的快捷方式
    1. 猪八戒去找高翠兰结果是孙悟空变的，可以这样理解：把高翠兰的外貌抽象出来，高翠兰本人和孙悟空都实现了这个接口，猪八戒访问高翠兰的时候看不出来这个是孙悟空，所以说孙悟空是高翠兰代理类
    1. 买火车票不一定在火车站买，也可以去代售点
    1. 一张支票或银行存单是账户中资金的代理。支票在市场交易中用来代替现金，并提供对签发人账号上资金的控制
    1. spring aop
1. **优点：**
    1. 职责清晰
    1. 高扩展性
    1. 智能化
1. **缺点：**
    1. 由于在客户端和真实主题之间增加了代理对象，因此有些类型的代理模式可能会造成请求的处理速度变慢
    1. 实现代理模式需要额外的工作，有些代理模式的实现非常复杂
1. **使用场景：**按职责来划分，通常有以下使用场景： 1、远程代理。 2、虚拟代理。 3、Copy-on-Write 代理。 4、保护（Protect or Access）代理。 5、Cache代理。 6、防火墙（Firewall）代理。 7、同步化（Synchronization）代理。 8、智能引用（Smart Reference）代理
1. **注意事项：**
    1. 和适配器模式的区别：适配器模式主要改变所考虑对象的接口，而代理模式不能改变所代理类的接口
    1. 和装饰器模式的区别：装饰器模式为了增强功能，而代理模式是为了加以控制

# 二、登场角色
## 1. Proxy 模式中有以下登场角色
1. **Subject(主体)**
    1. Subject角色定义了使Proxy角色和 RealSubject角色之间具有一致性的接口（**这里的描述严格来说是静态代理的特性，动态代理的见下方描述**）
    1. 由于存在Subjiect角色，所以**Client角色不必在意它所使用的究竟是 Proxy角色还是 RealSubject 角色**
1. **Proxy(代理人)**
    1. Proxy角色会尽量处理来自 Client角色的请求
    1. 只有当自己不能处理时，它才会将工作交给RealSubject角色。因此，Proxy角色只有在必要时才会生成RealSubject 角色
    1. Proxy角色实现了在 Subject角色中定义的接口(API)
1. **RealSubject(实际的主体)**
    1. “本人”RealSubject角色会在“代理人”Proxy角色无法胜任工作时出场
    1. 它与Proxy角色一样也实现了在 Subject角色中定义的接口(API)

# 三、️⃣ 静态代理
## 1. 静态代理介绍
1. 静态代理就是**在代码运行之前，通过修改 `class` 文件来完成代理**
1. 三种实现方式：
    1. 方式一：在代码编译时，通过 AspectJ 编译器重新编译生成一个新的代理类，然后把新类也会作为一个正常的类加载到 JVM 中
    1. 方式二：在代码编译之后，通过字节码技术重新构建 class 和 jar 文件（通常在代理第三方 jar 时使用）
    1. 方式三：在 JVM 加载类的时候植入代理
1. 静态代理在使用时，⭐需要定义接口或者父类，**被代理对象**（即目标对象）与**代理对象（）**一起**实现相同的接口**或者是**继承相同父类**
1. 静态代理优缺点
    1. ==优点：==
        - 功能几乎可以无限扩大
        - 在运行时，无需做额外的任何事情，运行效率通常比动态代理快出数十倍
    1. ==缺点：==
        - 每一个代理类只能为一个接口服务，这样一来程序开发中必然产生过多的代理
        - 而且，因为代理对象需要与目标对象实现了一样的接口，因此，**一旦接口增加方法，目标对象与代理对象都要维护**

## 2. 代码示例
1. `Factory` ➡️ **Subject(主体)**

    ```Java
    interface Factory{
        void produce();
    }
    ```
1. `ConcreteFactory` ➡️ **RealSubject(实际的主体)**

    ```Java
    class ConcreteFactory implements Factory{
        @Override
        public void produce() {
            System.out.println("A工厂生产一批产品");
        }
    }
    ```
1. `ProxyFactory` ➡️ **Proxy(代理人)**

    ```Java
    class ProxyFactory implements Factory{
        private Factory factory;
        //构造代理对象时，需要将被代理对象传入
        public ProxyFactory(Factory factory) {
            this.factory = factory;
        }
        @Override
        public void produce() {
            System.out.println("代理工厂做一些准备工作或增强功能");
            factory.produce(); //调用的是被代理类的produce()方法
            //⭐注意：这里被代理的对象factory，以及调用factory的哪个方法，都是写死的！
            System.out.println("代理工厂做一些后续工作或增强功能");
        }
    }
    ```
1. `ProxyDemo` ➡️ **Client(请求者)**

    ```Java
    public class ProxyDemo {
        public static void main(String[] args) {
            ConcreteFactory concreteFactory = new ConcreteFactory();
            ProxyFactory proxyFactory = new ProxyFactory(concreteFactory);
            proxyFactory.produce();
        }
    }
    ```
1. 数据结果

    ```Java
    代理工厂做一些准备工作或增强功能
    A工厂生产一批产品
    代理工厂做一些后续工作或增强功能
    ```

# 四、️⃣ JDK 动态代理 ⭐
> ❗ **动态代理的好处：**
>
> 1. **灵活性** ：
>
>     1. 不需要必须实现接口，可以直接代理实现类，并且可以不需要针对每个目标类都创建一个代理类
>
>     1. 另外，静态代理中，接口一旦新增加方法，目标对象和代理对象都要进行修改，这是非常麻烦的
>
>
> 1. **JVM 层面** ：
>
>     1. 静态代理在编译时就将接口、实现类、代理类这些都变成了一个个实际的 `.class` 字节码文件
>
>     1. 而动态代理是在运行时，使用 **反射（JDK）** 或 **动态生成类字节码（CgLib）**，来生成代理类的
>
## 1. 介绍
1. **==首先，JDK 动态代理的被代理类需要实现一个接口==**
    1. 因为根据依赖倒转原则，Java 程序设计时，尽量会去依赖一个抽象接口
    1. JDK 动态代理基于接口实现代理，这样的设计和系统设计的思想是吻合的
1. ==**利用 JDK 的 API**== `java.lang.reflect.Proxy` ==**构造一个实现同样接口的代理对象，再将目标对象作为代理对象中的一个成员变量，来实现通过代理类执行被代理类方法的效果**==

    ![[IMG-20260315213658125.png|Untitled 3 125.png]]

1. ==JDK 动态代理如何动态地代理目标对象 ❓==（[[Export-e8249ac1-4ae1-42cd-83d5-041a266841fe/1question/Java 开发/Java/反射|反射]]）
    1. ==**代理类如何拿到被代理类的方法及变量❓**==
        1. 在运行时，被代理类已经加载在了内存当中，但是此时想使用被代理类的方法
        1. 就通过 ==反射== 获得 被代理类的完整结构，并将这些结构，赋给 代理类对象
    1. **==拿到被代理类的方法后，如何执行❓==**
        1. 当代理类执行 被代理类的方法时，实际就是使用 `InvocationHandler` 中由反射获取的被代理的 `Method`
        1. 同时使用被代理类对象 `Object target`来调用方法

## 2. 代码示例
1. `Factory` ➡️ **Subject(主体)**

    ```Java
    interface Factory{
        void produce();
    }
    ```
1. `ConcreteFactory` ➡️ **RealSubject(实际的主体)**

    ```Java
    class ConcreteFactory implements Factory {
        @Override
        public void produce() {
            System.out.println("A工厂生产一批产品");
        }
    }
    ```
1. `ProxyFactory` ➡️ **Proxy(代理人)** ，动态代理中代理人无需和实际的主体实现相同的接口，而是“动态地”获取被代理对象的代理实例

    ```Java
    class ProxyFactory {
        //维护一个目标对象 , Object
        private Object target;
        //构造器 ， 对target 进行初始化
        public ProxyFactory(Object target) {
            this.target = target;
        }
        //给目标对象 生成一个代理对象
        public Object getProxyInstance(){
            return Proxy.newProxyInstance(target.getClass().getClassLoader(),
                    target.getClass().getInterfaces(),
                    new InvocationHandler() {
                        @Override
                        public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
                            System.out.println("代理工厂做一些准备工作或增强功能");
                            // 反射机制调用目标对象的方法
                            Object returnValue = method.invoke(target, args);
                            System.out.println("代理工厂做一些后续工作或增强功能");
                            return returnValue;
                        }
                    });
        }
    }
    ```
    > ❗
    >
    > `Proxy`代理类所在的包： `java.lang.reflect.Proxy`
    > ❗
    >
    > `public static Object` **`newProxyInstance(ClassLoader loader, Class<?>[] interfaces, InvocationHandler h)`** 参数说明
    >
    > 1. `ClassLoader loader` ： 指定当前目标对象使用的类加载器, 获取加载器的方法固定
    >
    > 1. `Class<?>[] interfaces` ：
    >
    >     1. ⭐**JDK 动态代理是基于**==**接口**==**来生产代理对象的**
    >
    >         1. 如果目标对象是一个接口实现类，那么要传入目标对象实现的接口类型
    >
    >         1. 如果目标对象是本身就是一个接口，那么要传入他自己即可[[方案 2：HTTP + JDK 动态代理 + SpringBoot]]
    >
    >
    >     1. 并且是一个`Class[]`，因此支持多个接口
    >
    >
    > 1. `InvocationHandler h` ：
    >
    >     1. 静态代理类中由于代理类的不同，需要有与之相对的代理类
    >
    >     1. 而 JDK 动态代理中==把调用委托类方法的这个动作抽取出来，==把它封装成一个通用性的代理类，就是 `InvocationHandler`
    >
    >     1. 也就是由 **`InvocationHandler.invoke()`** 来统一处理代理类调用被代理类方法这个操作
    >
    > ![[IMG-20260315213658610.png|Untitled 4 102.png]]
    >
    >
    >
    > 1. 通过 `Proxy` 类的 `newProxyInstance()` 创建代理对象，在代理对象调用方法的时候，实际会调用到实现了 `InvocationHandler` 接口的处理类的 `invoke()`方法
    >
    > 1. 可以在 `invoke()` 方法中自定义处理逻辑，比如在方法执行前后做什么事情
1. `ProxyDemo` ➡️ **Client(请求者)**

    ```Java
    public class ProxyDemo {
        public static void main(String[] args) {
            Factory concreteFactory = new ConcreteFactory();
            ProxyFactory proxyFactory = new ProxyFactory(concreteFactory);
            Factory proxyInstance = (Factory) proxyFactory.getProxyInstance();
            proxyInstance.produce();
        }
    }
    ```
1. 运行结果

    ```Java

    A工厂生产一批产品
    代理工厂做一些后续工作或增强功能
    ```

# 五、️⃣ Cglib 动态代理 ⭐
## 1. 介绍
1. **静态代理和 JDK 动态代理的局限**
    1. 都要求目标对象类实现一个接口，然后基于这个接口产生代理对象
    1. 但是有时候被代理的目标对象只是一个单独类的实例，并没有实现任何的接口
    1. 这个时候 JDK 动态代理就无法满足了，因为 `java.lang.reflect.Proxy.newProxyInstance()` 的第二个参数要求传入一个接口数组

    ![[IMG-20260315213659507.png|Untitled 6 72.png]]

1. **这种情况下，就需要使用 Cglib 代理：==也叫作子类代理==**
    1. ==**它是在内存中构建一个目标对象的子类对象**==，见下方示例代码：`enhancer.setSuperclass(target.getClass());`
    1. ==**子类重写父类的所有非**== **`final`** **`private`** ==**修饰的方法，在子类中拦截父类方法的调用，从而实现对目标对象功能扩展**==
    1. 底层实现方式：
        1. 类加载过程： `.class` 文件加载进 JVM 内存，生成 Class 对象，再进过准备、链接、初始化等操作，可以得到一个该类对应的实例（[[类的生命周期]]）
        1. 其实现基础就是利用 [[类的生命周期]]（重点看红色部分），**允许我们在运行时对字节码进行修改和动态生成**

            ![[IMG-20260315224944294.png|Untitled 7 61.png]]

        1. 而 CgLib 就是使用==**字节码技术**== **`ASM`** 生产一组新的 `.class` 文件，然后加载类，并实例化生产一个代理对象，所以对于没有实现接口的 Bean 也能生成代理对象

        ![[IMG-20260315224944667.png|Untitled 8 52.png]]

> ❗ **被代理类中的 `private` 属性和方法能被 cglib 代理对象包含吗❓**
>
> - 继承：子类拥有父类非 `private` 的属性、方法。如果父类中有 `private` 方法，生成的代理类中是看不到的
>
> - 所以，由 CGLIB 创建的代理类，不会包含父类中的私有方法
>
> - 也即使用 AOP 注解的 private 方法是不会生效的
> ❗ **在 编程中如何选择代理模式**
>
> ❓
>
> 1. 目标对象需要实现接口，用JDK代理
>
> 1. 如果目标对象本身就是一个接口，也可以使用 JDK 代理[[方案 2：HTTP + JDK 动态代理 + SpringBoot]]
>
> 1. 目标对象不需要实现接口，用 Cglib 代理
## 2. 代码示例
1. 引入依赖

    ```XML
        <groupId>cglib</groupId>
    ```

1. `ConcreteFactory` ➡️ **RealSubject(实际的主体)，被代理对象是一个单独的对象，并没有实现任何的接口**

    ```Java
    class ConcreteFactory {

            System.out.println("A工厂生产一批产品");
    ```

1. `ProxyFactory` ➡️ **Proxy(代理人)**

    ```Java
    class ProxyFactory implements MethodInterceptor {
        //维护一个目标对象 , Object
        private Object target;
        //构造器 ，对 target 进行初始化
        public ProxyFactory(Object target) {
            this.target = target;
        }
        //给目标对象 生成一个代理对象
        public Object getProxyInstance(){
            //1. 可以通过Enhancer对象的create()方法可以去生成一个类，用于生成代理对象
            Enhancer enhancer = new Enhancer();
            //2. 设置父类(将目标类作为代理类的父类)
            enhancer.setSuperclass(target.getClass());
            //3. 设置拦截器(回调对象为本身对象)
            enhancer.setCallback(this);
            //4. 生成一个代理类对象并返回给调用者
            return enhancer.create();
        }
        @Override
        public Object intercept(Object o, Method method, Object[] args, MethodProxy methodProxy) throws Throwable {
            //增强行为
            System.out.println("方法执行前的增强行为");
            //调用目标类中的方法
            Object returnValue=  methodProxy.invoke(target, args);
            //增强行为
            System.out.println("方法执行后的增强行为");
            return returnValue;
        }
    }
    ```
