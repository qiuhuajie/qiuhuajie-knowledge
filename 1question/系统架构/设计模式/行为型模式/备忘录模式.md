# 1. 介绍
![[IMG-20260315213656194.png|Untitled 272.png]]
**保存对象状态**
1. **介绍：**
    
    1. 忘录模式（Memento Pattern）保存一个对象的某个状态，以便在适当的时候恢复对象以使用户能够比较方便地回到某个历史的状态
    
    1. 备忘录模式属于行为型模式
    
1. **意图：==在不破坏封装性的前提下，捕获一个对象的内部状态，并在该对象之外保存这个状态==**
1. **主要解决：**所谓备忘录模式就是在==**不破坏封装的前提下，捕获一个对象的内部状态，并在该对象之外保存这个状态**==，这样可以在以后将对象恢复到原先保存的状态
1. **何时使用：**很多时候我们总是需要记录一个对象的内部状态，这样做的目的就是为了允许用户取消不确定或者错误的操作，能够恢复到他原先的状态，使得他有"后悔药"可吃
1. **如何解决：**通过一个备忘录类专门存储对象状态
1. **关键代码：**客户不与备忘录类耦合，与备忘录管理类耦合
1. **应用实例：**
    
    1. 游戏存档
    
    1. ctrl + z
    
    1. IE 中的后退
    
    1. 数据库的事务管理
    
1. **优点：**
    
    1. 给用户提供了一种可以恢复状态的机制，可以使用户能够比较方便地回到某个历史的状态
    
    1. 实现了信息的封装，使得用户不需要关心状态的保存细节
    
1. **缺点：**消耗资源。如果类的成员变量过多，势必会占用比较大的资源，而且每一次保存都会消耗一定的内存
1. **使用场景：**
    
    1. 需要保存/恢复数据的相关状态场景
    
    1. 提供一个可回滚的操作
    
1. **注意事项：**
    
    1. 为了符合迪米特原则，还要增加一个管理备忘录的类
    
    1. 为了节约内存，可使用原型模式+备忘录模式
    
# 2. 登场角色
## 2.1 Memento 模式中有以下登场角色
1. **Originator(生成者)**
    
    需要保存状态的对象
    
1. **Memento(忘录对象)**
    
    1. 忘录对象,负责保存好记录，即Originator内部状态
    
    1. Memento角色会将Originator 角色的内部信息整合在一起
    
    1. 在 Memento 角色中虽然保存了Originator角色的信息，但它不会向外部公开这些信息
    
1. **Caretaker(守护者对象)**
    
    1. 守护者对象，负责保存多个备忘录对象，使用集合管理，提高效率
    
    1. 如果希望保存多个 Originator 对象的**不同时间的状态**，可以使用 `HashMap <String, 集合>`

# 3. 代码示例
## 3.1 需求描述
游戏攻击力存档
## 3.2 传统方案
1. 每次存档就对当前状态 new 一个对象来保存，就对应一个保存对象状态的对象
1. 这样当我们游戏的对象很多时，不利于管理，开销也很大，且暴露了对象内部的细节
1. 解决方案 ➡️ **备忘录模式**
## 3.2 **备忘录模式解决**
1. `Originator`
    
    ```Java
    @Data
    public class Originator {
    
        // 状态信息
        private String state;
    
        // 编写一个方法，可以保存一个状态对象 Memento
        // 因此编写一个方法，返回 Memento
        public Memento saveStateMemento() {
            return new Memento(state);
        }
    
        // 通过备忘录对象，恢复状态
        public void getStateFromMemento(Memento memento) {
            state = memento.getState();
        }
    }
    ```
    
1. `Memento`
    
    ```Java
    @Data
    @AllArgsConstructor
    @NoArgsConstructor
    public class Memento {
        private String state;
    }
    ```
    
1. `Caretaker`
    
    ```Java
    public class Caretaker {
        //在List 集合中会有很多的备忘录对象
        private List<Memento> mementoList = new ArrayList<Memento>();
    
        public void add(Memento memento) {
            mementoList.add(memento);
        }
    
        //获取到第index个Originator 的 备忘录对象(即保存状态)
        public Memento get(int index) {
            return mementoList.get(index);
        }
    }
    ```
    
1. `MementoDemo`
    
    ```Java
    public class MementoDemo {
        public static void main(String[] args) {
    
            Originator originator = new Originator();
            Caretaker caretaker = new Caretaker();
    
            originator.setState(" 状态#1 攻击力 100 ");
            caretaker.add(originator.saveStateMemento());
    
            originator.setState(" 状态#2 攻击力 80 ");
            caretaker.add(originator.saveStateMemento());
    
            originator.setState(" 状态#3 攻击力 50 ");
            caretaker.add(originator.saveStateMemento());
    
            //测试
            System.out.println("当前的状态是 =" + originator.getState());
            System.out.println("恢复到状态1");
            originator.getStateFromMemento(caretaker.get(0));
            System.out.println("当前的状态是 =" + originator.getState());
        }
    }
    ```
    
1. 运行结果
    
    ```Java
    当前的状态是 = 状态#3 攻击力 50 
    恢复到状态1
    当前的状态是 = 状态#1 攻击力 100
    ```
