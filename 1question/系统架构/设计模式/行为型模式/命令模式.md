# 1. 介绍
![[IMG-20260315213656191.png|Untitled 267.png]]
1. **介绍：**
    
    1. 命令模式（Command Pattern）是一种数据驱动的设计模式，它属于行为型模式
    
    1. 通俗易懂的理解：将军发布命令，士兵去执行。其中有几个角色：将军（命令发布者）、士兵（命令的具体执行者）、命令(连接将军和士兵)
    
    1. **Invoker是调用者（将军）， Receiver是被调用者（士兵），MyCommand是命令，实现了Command接口，持有接收对象**
    
1. ==**意图：**====**在命名模式中，会将一个请求封装为一个对象， 以便使用不同参数来表示不同的请求(即命令)，同时命令模式也支持可撤销的操作**==
1. ==**主要解决：**==**==在软件系统中，行为请求者与行为实现者通常是一种紧耦合的关系，但某些场合，比如需要对行为进行记录、撤销或重做、事务等处理时，这种无法抵御变化的紧耦合的设计就不太合适==**
1. **何时使用：**在某些场合，比如要对行为进行"记录、撤销/重做、事务"等处理，这种无法抵御变化的紧耦合是不合适的。在这种情况下，如何将"行为请求者"与"行为实现者"解耦？将一组行为抽象为对象，可以实现二者之间的松耦合
1. **如何解决：**
    
    1. 通过调用者调用接受者执行命令，顺序：调用者→命令→接受者
    
1. **关键代码：**定义三个角色：
    
    1. eceived 真正的命令执行对象
    
    1. Command
    
    1. invoker 使用命令对象的入口
    
1. **应用实例：**struts 1 中的 action 核心控制器 ActionServlet 只有一个，相当于 Invoker，而模型层的类会随着不同的应用有不同的模型类，相当于具体的 Command
1. **优点：**
    
    1. 降低了系统耦合度
    
    1. 新的命令可以很容易添加到系统中去
    
1. **缺点：**使用命令模式可能会导致某些系统有过多的具体命令类
1. **使用场景：**认为是命令的地方都可以使用命令模式，比如： 1、GUI 中每一个按钮都是一条命令。 2、模拟 CMD
1. **注意事项：**
    
    1. 系统需要支持命令的撤销(Undo)操作和恢复(Redo)操作，也可以考虑使用命令模式，见命令模式的扩展
    
    1. 命令模式不足：可能导致某些系统有过多的具体命令类， 增加了系统的复杂度， 这点在在使用的时候要注意
    
# 2. 登场角色
## 2.1 Command 模式中有以下登场角色
1. **Command(命令)**
    
    Command角色负责定义命令的接口(API)
    
1. **ConcreteCommand(具体的命令)**
    
    1. ConcreteCommand角色负责实现在Command角色中定义的接口(API)
    
    1. ==并在其中组合一个Receiver(接收者)的对象，调用接受者相应的操作，实现execute==，命令对象起到了纽带桥梁的作用
    
1. **Receiver(接收者)**
    
    1. Receiver 角色是Command 角色执行命令时的对象，也可以称其为命令接收者
    
    1. 命名模式使得请求发送者与请求接收者消除彼此之间的耦合，让对象之间的调用关系更加灵活，实现解耦
    
1. **Client(请求者)**
    
    Client角色负责生成ConcreteCommand 角色并分配 Receiver 角色
    
1. **Invoker(发动者)**
    
    Invoker角色是开始执行命令的角色，它会调用在Command角色中定义的接口(API)

# 3. 代码示例
## 3.1 需求描述
编写程序， 使用命令模式 完成智能家电项目
1. 一套智能家电，有照明灯、风扇、冰箱、洗衣机，只要用一个集成的遥控器就可以控制对这些家电工作
1. 动作的请求者是遥控器，动作的执行者是每个厂商的一个家电产品
## 3.2 Command 模式解决
1. `Command` ➡️ **Command(命令)**
    
    ```Java
    //创建命令接口
    public interface Command {
        //执行动作(操作)
        public void execute();
        //撤销动作(操作)
        public void undo();
    }
    ```
    
1. `NoCommand` ➡️ **ConcreteCommand(具体的命令)**
    
    1. 空命令也是一种设计模式，它为我们省去了判空的操作
    
    1. 如果没有用空命令，我们每按下一个按键都要判空，这给我们编码带来一定的麻烦
    
    ```Java
    /**
     * 没有任何命令，即空执行: 用于初始化每个按钮, 当调用空命令时，对象什么都不做
     *  * 其实，这样是一种设计模式, 可以省掉对空判断
     */
    public class NoCommand implements Command {
        @Override
        public void execute() {}
    
        @Override
        public void undo() {}
    }
    ```
    
1. `LightOffCommand` ➡️ **ConcreteCommand(具体的命令)**
    
    ```Java
    public class LightOffCommand implements Command {
    
        // 聚合LightReceiver
        LightReceiver light;
    
        // 构造器
        public LightOffCommand(LightReceiver light) {
            this.light = light;
        }
    
        @Override
        public void execute() {
            // 调用接收者的方法
            light.off();
        }
    
        @Override
        public void undo() {
            // 调用接收者的方法
            light.on();
        }
    }
    ```
    
1. `LightOnCommand` ➡️ **ConcreteCommand(具体的命令)**
    
    ```Java
    public class LightOnCommand implements Command {
    
        //聚合LightReceiver
        LightReceiver light;
    
        //构造器
        public LightOnCommand(LightReceiver light) {
            this.light = light;
        }
    
        @Override
        public void execute() {
            //调用接收者的方法
            light.on();
        }
    
        @Override
        public void undo() {
            //调用接收者的方法
            light.off();
        }
    }
    ```
    
1. `LightReceiver` ➡️ **Receiver(接收者)**
    
    ```Java
    public class LightReceiver {
    
        public void on() {
            System.out.println(" 电灯打开了.. ");
        }
    
        public void off() {
            System.out.println(" 电灯关闭了.. ");
        }
    }
    ```
    
1. `RemoteController` ➡️ **Invoker(发动者)**
    
    ```Java
    public class RemoteController {
    
        // 模拟开关两列按钮的数组，数组元素是Command
        Command[] onCommands;
        Command[] offCommands;
    
        // 保存上一条执行的命令，用作撤销操作
        Command undoCommand;
    
        // 构造器，完成对按钮初始化
        public RemoteController() {
    
            onCommands = new Command[5];
            offCommands = new Command[5];
    
            for (int i = 0; i < 5; i++) {
                onCommands[i] = new NoCommand();
                offCommands[i] = new NoCommand();
            }
        }
    
        // 给我们的按钮设置你需要的命令
        public void setCommand(int no, Command onCommand, Command offCommand) {
            onCommands[no] = onCommand;
            offCommands[no] = offCommand;
        }
    
        // 按下开按钮
        public void onButtonWasPushed(int no) {
            // 找到你按下的开的按钮， 并调用对应方法
            onCommands[no].execute();
            // 记录这次的操作，用于撤销
            undoCommand = onCommands[no];
    
        }
    
        // 按下开按钮
        public void offButtonWasPushed(int no) {
            // 找到你按下的关的按钮， 并调用对应方法
            offCommands[no].execute();
            // 记录这次的操作，用于撤销
            undoCommand = offCommands[no];
    
        }
    
        // 按下撤销按钮
        public void undoButtonWasPushed() {
            undoCommand.undo();
        }
    }
    ```
    
1. `CommandDemo` ➡️ **Client(请求者)**
    
    ```Java
    public class CommandDemo{
        public static void main(String[] args) {
    
            //创建电灯的对象(接受者)
            LightReceiver lightReceiver = new LightReceiver();
    
            //创建电灯相关的开关命令
            LightOnCommand lightOnCommand = new LightOnCommand(lightReceiver);
            LightOffCommand lightOffCommand = new LightOffCommand(lightReceiver);
    
            //需要一个遥控器
            RemoteController remoteController = new RemoteController();
    
            //给我们的遥控器设置命令, 比如 no = 0 是电灯的开和关的操作
            remoteController.setCommand(0, lightOnCommand, lightOffCommand);
    
            System.out.println("--------按下灯的开按钮-----------");
            remoteController.onButtonWasPushed(0);
            System.out.println("--------按下灯的关按钮-----------");
            remoteController.offButtonWasPushed(0);
            System.out.println("--------按下撤销按钮-----------");
            remoteController.undoButtonWasPushed();
        }
    }
    ```
    
1. 运行结果
    
    ```Java
    --------按下灯的开按钮-----------
     电灯打开了.. 
    --------按下灯的关按钮-----------
     电灯关闭了.. 
    --------按下撤销按钮-----------
     电灯打开了..
    ```

# 4. 命令模式在Spring框架JdbcTemplate应用的源码分析
1. 在使用 `JDBCTemplate` 查询时，会调用 `jdbcTemplate.query()` 方法
    
    ```Java
    @Repository
    public class BookDaoImpl implements BookDao {
        //注入 JdbcTemplate
        @Autowired
        private JdbcTemplate jdbcTemplate;
    
        @Override
        public List<Book> findAll() {
            String sql = "select * from t_book";
            List<Book> list = jdbcTemplate.query(sql, new BeanPropertyRowMapper<Book>(Book.class));
            return list;
        }
    }
    ```
    
1. `jdbcTemplate.query()` 方法中有一个 `QueryStatementCallback` 的内部类，其实现了 `StatementCallback` 接口
![[IMG-20260315213658358.png|Untitled 4 104.png]]
1. 这个 `StatementCallback` 接口就可以看作是 ➡️ **Command(命令)，**`StatementCallback` 接口中定义了`doInStatemnt()`
    
    ![[IMG-20260315213658842.png|Untitled 5 85.png]]
    
1. 这个`QueryStatementCallback` 内部类可以看作是 ➡️ **ConcreteCommand(具体的命令**)；且看到实际操作数据库的代码也直接重写了在了`doInStatemnt()` 中，因此`QueryStatementCallback` 内部类可以看作是➡️ **Receiver(接收者)**
1. 在`jdbcTemplate.query()` 最后执行时会调用本类的 `execute()` ，并会将一个 `QueryStatementCallback` 内部类对象作为参数传入，而`execute()` 的参数是接受一个 `StatementCallback` 类型的对象
1. 其中 `execute(StatementCallback<T> action)` 方法中，调用`action.doInStatement` 方法。不同的 实现 StatementCallback 接口的对象，对应不同的`doInStatemnt` 实现逻辑
    
    ![[IMG-20260315213659311.png|Untitled 6 74.png]]
    
    1. 实现 `StatementCallback` 命令接口的子类还有：
        
        - `execute()` 中传入不同的`StatementCallback` 接口实现类，由于不同的实现类`doInStatemnt()` 实现逻辑不同，即可达到==**使用不同参数来表示不同的请求(即命令)**==的目的
        
        ![[IMG-20260315213659663.png|Untitled 7 63.png]]
        
    
    1. **虽然没有完全按标准的命令模式开发代码，但是确实是一种命令模式的思想，况且设计模式本身并没有标准**
