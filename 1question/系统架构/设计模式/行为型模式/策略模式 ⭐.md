# 1. 介绍
![[IMG-20260315213656207.png|Untitled 276.png]]
**整体地替换算法策略**
1. **介绍：**
    
    1. 在策略模式（Strategy Pattern）中，一个类的行为或其算法可以在运行时更改。这种类型的设计模式属于行为型模式
    
    1. 在策略模式中，我们创建表示各种策略的对象和一个行为随着策略对象改变而改变的 context 对象。策略对象改变 context 对象的执行算法。
    
1. **意图：**定义一系列的算法（策略），把它们一个个封装起来, 并且使它们可相互替换
1. ==**主要解决：**==
    
    1. 策略模式的核心思想是：==**多用组合/聚合 少用继承；用行为类组合，而不是行为的继承**==。更有弹性
    
    1. 提供了可以替换继承关系的办法： ==**策略模式将算法封装在独立的Strategy类中使得你可以独立于其Context改变它，使它易于切换、易于理解、易于扩展**==
    
1. **何时使用：**一个系统有许多许多类，而区分它们的只是他们直接的行为
1. **如何解决：**将这些算法封装成一个一个的类，任意地替换
1. **关键代码：**实现同一个接口
1. **应用实例：** 1、诸葛亮的锦囊妙计，每一个锦囊就是一个策略。 2、旅行的出游方式，选择骑自行车、坐汽车，每一种旅行方式都是一个策略。 3、JAVA AWT 中的 LayoutManager
1. **优点：** 1、算法可以自由切换。 2、避免使用多重条件判断。 3、扩展性良好
1. **缺点：** 1、策略类会增多。 2、所有策略类都需要对外暴露
1. **使用场景：** 1、如果在一个系统里面有许多类，它们之间的区别仅在于它们的行为，那么使用策略模式可以动态地让一个对象在许多行为中选择一种行为。 2、一个系统需要动态地在几种算法中选择一种。 3、如果一个对象有很多的行为，如果不用恰当的模式，这些行为就只好使用多重的条件选择语句来实现
1. **注意事项：**如果一个系统的策略多于四个，就需要考虑使用混合模式，解决策略类膨胀的问题。
# 2. 登场角色
## 2.1 Strategy 模式中有以下登场角色
1. **Strategy(策略)**
1. Strategy角色负责决定实现策略所必需的接口(API)。在示例序中，由strategy 接口扮演此角色
1. **ConcreteStrategy(具体的策略)**
1. ConcreteStrategy角色负责实现 Strategy 角色的接口(API)，即负责实现具体的策略(战略、方向、方法和算法
1. **Context(上下文)**
1. 负责使用Strategy角色。Context角色保存了 ConcreteStrategy角色的实例，并使用ConcreteStrategy角色去实现需求(总之，还是要调用Strategy 角色的接口(API))

# 3. 代码示例
## 3.1 需求描述
- 不同鸭子有不同的能力：野鸭会飞，北京鸭、玩具鸭不会飞
## 3.2 传统方案
1. 其它鸭子都继承Duck类，但是这样让所有子类都会飞了，这是不正确的
1. 上面说的问题，是继承带来的问题： 对类的局部改动，尤其超类的局部改动，会影响其他部分。会有溢出效应
1. 为了改进上面问题，我们可以通过覆盖 fly 方法来解决，但是问题又来了，如果我们有一个玩具鸭子ToyDuck（啥都不会），这样就需要ToyDuck去覆盖Duck的所有实现的方法
1. 解决方案 ➡️ **Strategy 模式**
## 3.3 Strategy 模式**解决**
1. `FlyBehavior` ➡️ **Strategy(策略)**
    
    ```Java
    public interface FlyBehavior {
        void fly(); // 子类具体实现
    }
    ```
    
    > [!important]
    
1. `CanFlyBehavior` ➡️ **ConcreteStrategy(具体的策略)**
    
    ```Java
    public class CanFlyBehavior implements FlyBehavior {
        @Override
        public void fly() {
            System.out.println("可以飞翔");
        }
    }
    ```
    
1. `NoFlyBehavior` ➡️ **ConcreteStrategy(具体的策略)**
    
    ```Java
    public class NoFlyBehavior implements FlyBehavior{
        @Override
        public void fly() {
            System.out.println("不可以飞翔");
        }
    }
    ```
    
1. `QuackBehavior` ➡️ **Strategy(策略)**
    
    ```Java
    public interface QuackBehavior {
        void quack();//子类实现
    }
    ```
    
1. `CanQuackBehavior` ➡️ **ConcreteStrategy(具体的策略)**
    
    ```Java
    public class CanQuackBehavior implements QuackBehavior{
        @Override
        public void quack() {
            
        }
    }
    ```
    
1. `Duck` ➡️ **Context(上下文)**
    
    ```Java
    public abstract class Duck {
    
        //飞翔属性 -- 策略接口
        FlyBehavior flyBehavior;
        //叫声属性 -- 略接口
        QuackBehavior quackBehavior;
    
        public Duck() {
    
        }
    
        public abstract void display();//显示鸭子信息
    
        public void fly() {
            display();
            // 调用当前所持有策略的具体方法实现
            if(flyBehavior != null) {
                flyBehavior.fly();
            }
    
            if(quackBehavior != null) {
                quackBehavior.quack();
            }
    
        }
    
        public void setFlyBehavior(FlyBehavior flyBehavior) {
            this.flyBehavior = flyBehavior;
        }
    
    
        public void setQuackBehavior(QuackBehavior quackBehavior) {
            this.quackBehavior = quackBehavior;
        }
    }
    ```
    
1. `WildDuck` ➡️ `Duck` 子类，==1️⃣====**可以在构造方法中直接指定写死该子类所使用的行为策略，**====2️⃣====**也可以将使用的策略作为构造方法的参数传入**==
    
    ```Java
    public class WildDuck extends Duck {
    
        // 野生鸭可以飞翔
        public WildDuck() {
            flyBehavior = new CanFlyBehavior();  // 直接指定写死
        }
    
        @Override
        public void display() {
            System.out.println("===野生鸭===");
        }
    }
    ```
    
1. `PekingDuck` ➡️ `Duck` 子类
    
    ```Java
    public class PekingDuck extends Duck{
        //假如北京鸭不可以飞翔
        public PekingDuck() {
            // TODO Auto-generated constructor stub
            flyBehavior = new NoFlyBehavior();
    
        }
    
        @Override
        public void display() {
            System.out.println("===北京鸭===");
        }
    }
    ```
    
1. `ToyDuck` ➡️ `Duck` 子类
    
    ```Java
    public class ToyDuck extends Duck{
    
        public ToyDuck() {
            flyBehavior = new NoFlyBehavior();
        }
    
        @Override
        public void display() {
            System.out.println("===玩具鸭===");
        }
    }
    ```
    
1. `StrategyDemo`
    
    ```Java
    public static void main(String[] args) {
            WildDuck wildDuck = new WildDuck();
            wildDuck.fly();
    
            ToyDuck toyDuck = new ToyDuck();
            toyDuck.fly();
    
            PekingDuck pekingDuck = new PekingDuck();
            pekingDuck.fly();
    
            //动态改变某个对象的行为, 北京鸭 不能飞
            System.out.println("更换北京鸭的飞翔策略");
            pekingDuck.setFlyBehavior(new CanFlyBehavior());
    
            pekingDuck.fly();
        }
    ```
    
1. 运行结果
    
    ```Java
    ===野生鸭===
    可以飞翔
    ===玩具鸭===
    不可以飞翔
    ===北京鸭===
    不可以飞翔
    更换北京鸭的飞翔策略
    ===北京鸭===
    可以飞翔
    ```
