## 介绍
![[IMG-20260315213656229.png|Untitled 269.png]]
**访问数据结构并处理数据**
1. **介绍：**
    
    1. 在访问者模式（Visitor Pattern）中，我们使用了一个访问者类，它改变了元素类的执行算法。通过这种方式，元素的执行算法可以随着访问者改变而改变。这种类型的设计模式属于行为型模式
    
    1. 根据模式，元素对象已接受访问者对象，这样访问者对象就可以处理元素对象上的操作
    
1. ==**意图：**==**==主要将数据结构与数据操作分离==**
1. ==**主要解决：**==**解决**==**数据结构**==**和**==**操作**== ==**耦合性**== **问题**
1. ==**何时使用：**==**==需要对一个对象结构中的对象进行很多不同操作（这些操作彼此没有关联），同时需要避免让这些操作"污染"这些对象的类，可以选用访问者模式解决==**
1. ==**如何解决：**==在被访问的类里面加一个对外提供接待访问者的接口
1. **关键代码：**在数据基础类里面有一个方法接受访问者，将自身引用传入访问者
1. **应用实例：**您在朋友家做客，您是访问者，朋友接受您的访问，您通过朋友的描述，然后对朋友的描述做出一个判断，这就是访问者模式
1. **优点：**
    
    1. 符合单一职责原则
    
    1. 优秀的扩展性
    
    1. 灵活性
    
1. **缺点：**
    
    1. 具体元素对访问者公布细节，违反了迪米特原则
    
    1. 具体元素变更比较困难
    
    1. 违反了依赖倒置原则，依赖了具体类，没有依赖抽象
    
1. **使用场景：**
    
    1. 对象结构中对象对应的类很少改变，但经常需要在此对象结构上定义新的操作
    
    1. 需要对一个对象结构中的对象进行很多不同的并且不相关的操作，而需要避免让这些操作"污染"这些对象的类，也不希望在增加新操作时修改这些类
    
1. **注意事项：访问者可以对功能进行统一，可以做报表、UI、拦截器与过滤器**
# 2. 登场角色
## 2.1 Visitor 模式中有以下登场角色
1. **Visitor(访问者)**
    
    1. Visitor角色负责对数据结构中每个具体的元素(ConcreteElement角色)**声明一个用于访问**`**XXXXX**`**的**`**visit(XXXXX)**`**方法**
    
    1. visit(XXXXX)是用于处理XXXXX的方法，负责实现该方法的是ConcreteVisitor角色
    
1. **ConcreteVisitor(具体的访问者)**
    
    1. ConcreteVisitor角色负责实现 Visitor角色所定义的接口(API)
    
    1. 它要**实现所有的**`**visit(XXXXX)**`**方法**，即实现如何处理每个ConcreteElement角色
    
1. **Element(元素)**
    
    1. Element角色表示 **Visitor角色要的访问对象**
    
    1. 定义一个accept 方法，接收一个访问者对象
    
1. **ConcreteElement**
    
    为具体元素， 实现了accept 方法
    
1. **ObiectStructure( 对象结构)**
    
    能枚举它的元素， 可以提供一个高层的接口，用来允许访问者访问元素

# 3. 代码示例
1. `Action` ➡️ **Visitor(访问者)**
    
    ```Java
    public abstract class Action {
        //得到男性 的测评
        public abstract void getManResult(Man man);
    
        //得到女的 测评
        public abstract void getWomanResult(Woman woman);
    }
    ```
    
1. `Success` ➡️ **ConcreteVisitor(具体的访问者)**
    
    ```Java
    public class Success extends Action {
    
        @Override
        public void getManResult(Man man) {
            System.out.println(" 男人给的评价该歌手很成功 !");
        }
    
        @Override
        public void getWomanResult(Woman woman) {
            System.out.println(" 女人给的评价该歌手很成功 !");
        }
    }
    ```
    
1. `Fail` ➡️ **ConcreteVisitor(具体的访问者)**
    
    ```Java
    public class Fail extends Action {
    
        @Override
        public void getManResult(Man man) {
            System.out.println(" 男人给的评价该歌手失败 !");
        }
    
        @Override
        public void getWomanResult(Woman woman) {
            System.out.println(" 女人给的评价该歌手失败 !");
        }
    }
    ```
    
1. `Person` ➡️ **Element(元素)**
    
    ```Java
    public abstract class Person {
        // 提供一个方法，让访问者可以访问
        public abstract void accept(Action action);
    }
    ```
    
1. `Man` ➡️ **ConcreteElement**
    
    ```Java
    public class Man extends Person {
    
        @Override
        public void accept(Action action) {
            action.getManResult(this);
        }
    }
    ```
    
1. `Woman` ➡️ **ConcreteElement**
    
    - 这里使用到了**双分派**，即首先在客户端程序中，将具体状态作为参数传递Woman中（**第一次分派**）
    
    - 然后 Woman 类调用作为参数的 "具体方法" 中方法`getWomanResult`， 同时将自己(`this`)作为参数传入，完成**第二次的分派**
    
    ```Java
    public class Woman extends Person{
    
        @Override
        public void accept(Action action) {
            action.getWomanResult(this);
        }
    }
    ```
    
1. `ObjectStructure` ➡️ **ObiectStructure( 对象结构)**
    
    ```Java
    public class ObjectStructure {
    
        //维护了一个集合
        private List<Person> persons = new LinkedList<>();
    
        //增加到list
        public void attach(Person p) {
            persons.add(p);
        }
        //移除
        public void detach(Person p) {
            persons.remove(p);
        }
    
        //显示测评情况
        public void display(Action action) {
            for(Person p: persons) {
                p.accept(action);
            }
        }
    }
    ```
    
1. `VisitorDemo`
    
    - **假设要添加一个`Wait`的状态类， 考察Man类和Woman类的反应，由于使用了双分派， 只需增加一个Action子类即可在客户端调用即可， 不需要改动任何其他类的代码**
    
    ```Java
    public class VisitorDemo {
        public static void main(String[] args) {
    
            //创建ObjectStructure
            ObjectStructure objectStructure = new ObjectStructure();
    
            objectStructure.attach(new Man());
            objectStructure.attach(new Woman());
    
            //成功
            Success success = new Success();
            objectStructure.display(success);
    
            System.out.println("===============");
    
            Fail fail = new Fail();
            objectStructure.display(fail);
        }
    }
    ```
    
1. 运行结果
    
    ```Java
    男人给的评价该歌手很成功 !
     女人给的评价该歌手很成功 !
    ===============
     男人给的评价该歌手失败 !
     女人给的评价该歌手失败 !
    ```
