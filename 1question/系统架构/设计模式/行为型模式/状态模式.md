# 一、介绍

![[IMG-20260315213656204.png|Untitled 274.png]]

**用类表示状态**

1. **介绍：**
    1. 在状态模式（State Pattern）中，类的行为是基于它的状态改变的。这种类型的设计模式属于行为型模式
    1. 在状态模式中，我们创建**表示各种状态的对象**和**一个行为随着状态对象改变而改变的 context 对象**
1. **意图：**允许对象在内部状态发生改变时改变它的行为，对象看起来好像修改了它的类
1. ==**主要解决：**==
    1. ==**对象的行为依赖于它的状态（属性），并且可以根据它的状态改变而改变它的相关行为**==
    1. 它主要用来解决对象在多种状态转换时，需要对外输出不同的行为的问题。状态和行为是一一对应的，状态之间可以相互转换
1. ==**何时使用：**====**当一个事件或者对象有很多种状态，且状态之间会相互转换，对不同的状态要求有不同的行为的时候，可以考虑使用状态模式**==
1. **如何解决：**将各种具体的状态类抽象出来
1. **关键代码：**通常命令模式的接口中只有一个方法。而状态模式的接口中有一个或者多个方法。而且，状态模式的实现类的方法，一般返回值，或者是改变实例变量的值。也就是说，状态模式一般和对象的状态有关。实现类的方法有不同的功能，覆盖接口中的方法。**状态模式和命令模式一样，也可以用于消除 `if...else` 等条件选择语句**
1. **应用实例：**
    1. 打篮球的时候运动员可以有正常状态、不正常状态和超常状态
    1. 曾侯乙编钟中，'钟是抽象接口','钟A'等是具体状态，'曾侯乙编钟'是具体环境（Context）
1. **优点：**
    1. 方便维护。**将容易产生问题的`if-else`语句删除了**，如果把每个状态的行为都放到一个类中，每次调用方法时都要判断当前是什么状态，不但会产出很多`if-else`语句，而且容易出错
    1. 封装了转换规则
    1. 枚举可能的状态，在枚举状态之前需要确定状态种类
    1. 将所有与某个状态有关的行为放到一个类中，并且可以方便地增加新的状态，只需要改变对象状态即可改变对象的行为
    1. 允许状态转换逻辑与状态对象合成一体，而不是某一个巨大的条件语句块
    1. 可以让多个环境对象共享一个状态对象，从而减少系统中对象的个数
1. **缺点：**
    1. 状态模式的使用必然会增加系统类和对象的个数
    1. 状态模式的结构与实现都较为复杂，如果使用不当将导致程序结构和代码的混乱
    1. 状态模式对"开闭原则"的支持并不太好，对于可以切换状态的状态模式，增加新的状态类需要修改那些负责状态转换的源代码，否则无法切换到新增状态，而且修改某个状态类的行为也需修改对应类的源代码
1. **使用场景：**
    1. 行为随状态改变而改变的场景
    1. 条件、分支语句的代替者
1. **注意事项：**在行为受状态约束的时候使用状态模式，而且状态不超过 5 个
# 二、登场角色
## 1. State 模式中有以下登场角色
1. **State(状态)**
    1. State 角色表示状态，定义了根据不同状态进行不同处理的接口(API)
    1. 是抽象状态角色,定义一个接口封装与Context 的一个特点接口相关行为
1. **ConcreteState(具体状态)**

    具体的状态角色，每个ConcreteState子类实现一个与Context 的一个状态相关行为

1. **Context(环境)**
    1. Context角色持有表示当前状态的ConcreteState角色
    1. 此外，维护State实例，这个实例定义当前状态

# 三、代码示例
## 1. 需求描述
- 借贷平台的订单，有审核-发布-抢单 等等 步骤，随着操作的不同，会改变订单的状态, 项目中的这个模块实现就会使用到状态模式
## 2. 传统方案
1. 通常通过if/else判断订单的状态，从而实现不同的逻辑，伪代码如下

    ```Java
    if(审核){
    //审核逻辑
    }elseif(发布){
    //发布逻辑
    }elseif(接单){
    //接单逻辑
    }
    ```
1. 问题分析 ：
    1. 这类代码难以应对变化，在添加一种状态时，我们**需要手动添加****`if/else`**，在添加一种功能时，要对所有的状态进行判断
    1. 因此代码会变得越来越臃肿，并且**一旦没有处理某个状态，便会发生极其严重的BUG**，难以维护
1. 解决方案 ➡️ **State 模式**
## 3. State 模式解决
1. `State` ➡️ **State(状态) ，定义了所有状态转移方法的接口**

    ```Java
    public interface State {
        /**
         * 电审
         */
        void checkEvent(Context context);
        /**
         * 电审失败
         */
        void checkFailEvent(Context context);
        /**
         * 定价发布
         */
        void makePriceEvent(Context context);
        /**
         * 接单
         */
        void acceptOrderEvent(Context context);
        /**
         * 无人接单失效
         */
        void notPeopleAcceptEvent(Context context);
        /**
         * 付款
         */
        void payOrderEvent(Context context);
        /**
         * 接单有人支付失效
         */
        void orderFailureEvent(Context context);
        /**
         * 反馈
         */
        void feedBackEvent(Context context);
        String getCurrentState();
    }
    ```
1. `AbstractState`
    - **`AbstractState` 的子类会选择性的重写 `AbstractState` 中的状态转移方法，因为某一状态并不能够调用任意方法，从而跳转到任一状态**
    - 因此 **`AbstractState` 会默认实现所有状态转移方法，里面的实现就是抛出不能调用的异常提示，提示用户不能进行当前操作**
    - ==**这样如果某一状态重写了状态转移方法，那么会走其自己重写的逻辑；如果没有重写，那么会走其父类异常提示的逻辑**==

    ```Java
    public abstract class AbstractState implements State {
        protected static final RuntimeException EXCEPTION = new RuntimeException("操作流程不允许");
        //抽象类，默认实现了 State 接口的所有方法
        //该类的所有方法，其子类(具体的状态类)，可以有选择的进行重写
        @Override
        public void checkEvent(Context context) {
            throw EXCEPTION;
        }
        @Override
        public void checkFailEvent(Context context) {
            throw EXCEPTION;
        }
        @Override
        public void makePriceEvent(Context context) {
            throw EXCEPTION;
        }
        @Override
        public void acceptOrderEvent(Context context) {
            throw EXCEPTION;
        }
        @Override
        public void notPeopleAcceptEvent(Context context) {
            throw EXCEPTION;
        }
        @Override
        public void payOrderEvent(Context context) {
            throw EXCEPTION;
        }
        @Override
        public void orderFailureEvent(Context context) {
            throw EXCEPTION;
        }
        @Override
        public void feedBackEvent(Context context) {
            throw EXCEPTION;
        }
    }
    ```
1. `FeedBackState` ➡️ **ConcreteState(具体状态)**

    ```Java
    class FeedBackState extends AbstractState {
        @Override
        public String getCurrentState() {
            return StateEnum.FEED_BACKED.getValue();
        }
    }
    ```
1. `NotPayState` ➡️ **ConcreteState(具体状态)**

    ```Java
    class NotPayState extends AbstractState {
        @Override
        public void payOrderEvent(Context context) {
            context.setState(new PaidState());
        }
        @Override
        public void feedBackEvent(Context context) {
            context.setState(new FeedBackState());
        }
        @Override
        public String getCurrentState() {
            return StateEnum.NOT_PAY.getValue();
        }
    }
    ```
1. `PaidState` ➡️ **ConcreteState(具体状态)**

    ```Java
    class PaidState extends AbstractState {
        @Override
        public void feedBackEvent(Context context) {
            context.setState(new FeedBackState());
        }
        @Override
        public String getCurrentState() {
            return StateEnum.PAID.getValue();
        }
    }
    ```
1. `PublishState` ➡️ **ConcreteState(具体状态)**

    ```Java
    class PublishState extends AbstractState {
        @Override
        public void acceptOrderEvent(Context context) {
            //把当前状态设置为  NotPayState。。。
            //至于应该变成哪个状态，有流程图来决定
            context.setState(new NotPayState());
        }
        @Override
        public void notPeopleAcceptEvent(Context context) {
            context.setState(new FeedBackState());
        }
        @Override
        public String getCurrentState() {
            return StateEnum.PUBLISHED.getValue();
        }
    }
    ```
1. `ReviewState` ➡️ **ConcreteState(具体状态)**

    ```Java
    class ReviewState extends AbstractState {
        @Override
        public void makePriceEvent(Context context) {
            context.setState(new PublishState());
        }
        @Override
        public String getCurrentState() {
            return StateEnum.REVIEWED.getValue();
        }
    }
    ```
1. `Context` ➡️ **Context(环境)，**==**聚合了状态对象**==

    ```Java
    public class Context extends AbstractState{
        //当前的状态 state, 根据我们的业务流程处理，不停的变化
        private State state;
        @Override
        public void checkEvent(Context context) {
            state.checkEvent(this);
            getCurrentState();
        }
        @Override
        public void checkFailEvent(Context context) {
            state.checkFailEvent(this);
            getCurrentState();
        }
        @Override
        public void makePriceEvent(Context context) {
            state.makePriceEvent(this);
            getCurrentState();
        }
        @Override
        public void acceptOrderEvent(Context context) {
            state.acceptOrderEvent(this);
            getCurrentState();
        }
        @Override
        public void notPeopleAcceptEvent(Context context) {
            state.notPeopleAcceptEvent(this);
            getCurrentState();
        }
        @Override
        public void payOrderEvent(Context context) {
            state.payOrderEvent(this);
            getCurrentState();
        }
        @Override
        public void orderFailureEvent(Context context) {
            state.orderFailureEvent(this);
            getCurrentState();
        }
        @Override
        public void feedBackEvent(Context context) {
            state.feedBackEvent(this);
            getCurrentState();
        }
        public State getState() {
            return state;
        }
        public void setState(State state) {
            this.state = state;
        }
        @Override
        public String getCurrentState() {
            System.out.println("当前状态 : " + state.getCurrentState());
            return state.getCurrentState();
        }
    }
    ```
1. `StateEnum`

    ```Java
    public enum StateEnum {
        //订单生成
        GENERATE(1, "GENERATE"),
        //已审核
        REVIEWED(2, "REVIEWED"),
        //已发布
        PUBLISHED(3, "PUBLISHED"),
        //待付款
        NOT_PAY(4, "NOT_PAY"),
        //已付款
        PAID(5, "PAID"),
        //已完结
        FEED_BACKED(6, "FEED_BACKED");
        private int key;
        private String value;
        StateEnum(int key, String value) {
            this.key = key;
            this.value = value;
        }
        public int getKey() {return key;}
        public String getValue() {return value;}
    }
    ```
1. `StateDemo`

    ```Java
    public class StateDemo {
        public static void main(String[] args) {
            //创建context 对象
            Context context = new Context();
            //将当前状态设置为 PublishState
            context.setState(new PublishState());
            System.out.println(context.getCurrentState());
    //        //publish --> not pay
            context.acceptOrderEvent(context);
    //        //not pay --> paid
            context.payOrderEvent(context);
    //        // 失败, 检测失败时，会抛出异常
            try {
    	        	context.checkFailEvent(context);
    	        	System.out.println("流程正常..");
    				} catch (Exception e) {
    					System.out.println(e.getMessage());
    				}
        }
    }
    ```
    - **当状态在 `PaidState` 时，调用 `checkFailEvent()` 但是 由于 `PaidState` 没有重写这个方法，故会调用其父类`AbstractState` 所默认实现的 `checkFailEvent()` ，提示用户不能进行当前操作**

    ```Java
    当前状态 : PUBLISHED
    PUBLISHED
    当前状态 : NOT_PAY
    当前状态 : PAID
    操作流程不允许
    ```
